## Overview

Phone testing validates your agent through the complete production telephony stack—from PSTN carriers through your SIP trunk to the voice infrastructure. This is essential for catching issues that only appear in real phone call scenarios.

**Why phone testing matters:**
- Tests actual carrier audio quality
- Validates DTMF/IVR functionality
- Exposes latency from phone routing
- Verifies echo cancellation
- Tests voicemail detection
- Confirms transfer functionality

---

## How to Test by Phone

<Steps>
  <Step title="Click 'Call test number'">
    In the agent editor, click **Call test number** button

    This provisions a temporary phone number for testing
  </Step>

  <Step title="Dial the test number">
    Use any phone to call the test number displayed

    The call routes through the same telephony stack as production
  </Step>

  <Step title="Test your scenarios">
    Speak naturally and test all conversation paths

    Try interruptions, DTMF inputs, and actions
  </Step>

  <Step title="Review the call log">
    After the call, find it in **Conversations**

    Review transcripts, actions, and audio quality
  </Step>
</Steps>

<Note>
Test phone numbers are temporary and may expire after a period of inactivity. Request a new one if needed.
</Note>

## Testing Checklist

<AccordionGroup>
  <Accordion title="Quiet environment test" icon="volume-off">
    **Setup:** Call from a quiet room with a headset

    **What to test:**
    - Clear audio in both directions
    - Accurate transcription
    - Natural response timing
    - All greeting variations
    - Basic conversation paths

    **Goal:** Establish baseline performance without environmental variables
  </Accordion>

  <Accordion title="Speaker phone test" icon="phone-volume">
    **Setup:** Call using speaker phone in a quiet room

    **What to test:**
    - Echo cancellation effectiveness
    - Audio feedback loops
    - Barge-in reliability
    - Volume levels

    **Goal:** Verify echo handling doesn't cause issues
  </Accordion>

  <Accordion title="Noisy environment test" icon="wind">
    **Setup:** Call from noisy locations (street, cafe, car)

    **What to test:**
    - Transcription accuracy with background noise
    - Voice activity detection
    - Whether agent speaks over background sounds

    **Goal:** Ensure robustness in real-world conditions
  </Accordion>

  <Accordion title="Mobile carrier test" icon="mobile">
    **Setup:** Call from different mobile carriers (if possible)

    **What to test:**
    - Audio quality variations by carrier
    - Latency differences
    - Compression artifacts
    - Connection stability

    **Goal:** Identify carrier-specific issues
  </Accordion>

  <Accordion title="DTMF input test" icon="phone-keypad">
    **Setup:** Configure DTMF actions in agent settings

    **What to test:**
    - Press keys during conversation
    - Verify agent captures and responds to input
    - Test all configured DTMF options
    - Test invalid key presses

    **Goal:** Validate IVR functionality
  </Accordion>

  <Accordion title="Interruption test" icon="hand">
    **Setup:** Call and let agent speak

    **What to test:**
    - Cut off agent mid-sentence
    - Speak over agent continuously
    - Rapid back-and-forth exchanges
    - Agent recovery after interruption

    **Goal:** Test barge-in behavior matches expectations
  </Accordion>

  <Accordion title="Transfer test" icon="right-left">
    **Setup:** Configure transfer actions

    **What to test:**
    - Trigger transfer through conversation
    - Verify transfer intent is logged
    - If transferred, test audio quality continues
    - Test fallback if transfer fails

    **Goal:** Confirm transfer routing works
  </Accordion>

  <Accordion title="Long conversation test" icon="message">
    **Setup:** Have an extended conversation (5-10 minutes)

    **What to test:**
    - Context retention throughout call
    - Audio quality stability
    - Latency consistency
    - Memory/context limits

    **Goal:** Verify performance over extended calls
  </Accordion>

  <Accordion title="Silence and timeout test" icon="clock-rotate-left">
    **Setup:** Call and remain silent at various points

    **What to test:**
    - Inactivity timeout triggers correctly
    - Agent handles silence gracefully
    - Timeout warnings (if configured)
    - End-of-call handling

    **Goal:** Validate timeout and silence handling
  </Accordion>

  <Accordion title="Edge case scenarios" icon="triangle-exclamation">
    **Setup:** Test unusual situations

    **What to test:**
    - Multiple people speaking simultaneously
    - Speaking in different accents
    - Speaking very quietly or loudly
    - Speaking very fast or slow
    - Switching languages mid-call
    - Background conversations

    **Goal:** Identify edge case failures
  </Accordion>
</AccordionGroup>

## Device Testing Matrix

Test with different device types to ensure compatibility:

| Device Type | Test Priority | Key Focus |
|-------------|--------------|-----------|
| **Mobile Phone** | High | Most common caller device |
| **Landline** | Medium | Audio quality, no echo |
| **VoIP Phone** | Medium | Codec compatibility |
| **Headset** | High | Best audio quality baseline |
| **Speaker Phone** | High | Echo cancellation |
| **Bluetooth Headset** | Low | Wireless audio quality |

<Tip>
Test with at least 2-3 different device types before going to production.
</Tip>

## After Each Test Call

<Steps>
  <Step title="Review the call log">
    Navigate to **Conversations**

    Filter by agent name to find your test call
  </Step>

  <Step title="Listen to the recording">
    Play back the full call audio

    Listen for quality issues you may have missed live
  </Step>

  <Step title="Check the transcript">
    Read the full transcript

    Verify transcription accuracy and agent responses
  </Step>

  <Step title="Inspect actions">
    Review any tool calls or actions triggered

    Check API payloads and responses
  </Step>

  <Step title="Document issues">
    Note any problems discovered

    Create a list of fixes needed before production
  </Step>
</Steps>

## Common Phone Testing Issues

<AccordionGroup>
  <Accordion title="Poor audio quality" icon="volume-xmark">
    **Symptoms:**
    - Distortion, static, or crackling
    - One-way audio
    - Volume too low/high

    **Causes:**
    - Carrier codec mismatch
    - SIP trunk configuration issue
    - Network bandwidth problems
    - Phone hardware quality

    **Solutions:**
    - Verify SIP trunk codec settings
    - Test from different locations
    - Try a different phone
    - Check network quality (if using VoIP)
    - Contact telephony provider
  </Accordion>

  <Accordion title="Echo or feedback" icon="repeat">
    **Symptoms:**
    - Hearing your own voice delayed
    - Agent's voice feeds back
    - Audio loops

    **Causes:**
    - Speaker phone without echo cancellation
    - Multiple devices on same call
    - Poor acoustic environment

    **Solutions:**
    - Use headset instead of speaker
    - Reduce volume
    - Test from different environment
    - Verify echo cancellation is enabled in settings
  </Accordion>

  <Accordion title="High latency" icon="clock">
    **Symptoms:**
    - Long delays between speech and response
    - Awkward conversation timing
    - Overlapping speech

    **Causes:**
    - Geographic distance to servers
    - Network congestion
    - Slow LLM inference
    - Carrier routing delays

    **Solutions:**
    - Test at different times of day
    - Try faster LLM model
    - Simplify agent instructions
    - Check carrier routing path
  </Accordion>

  <Accordion title="DTMF not captured" icon="phone-keypad">
    **Symptoms:**
    - Key presses not recognized
    - Agent doesn't respond to DTMF

    **Causes:**
    - DTMF not enabled in agent settings
    - Carrier not passing DTMF tones
    - In-band vs out-of-band DTMF mismatch

    **Solutions:**
    - Enable DTMF in **Abilities → Actions → Call Control**
    - Verify SIP trunk DTMF method (RFC2833 recommended)
    - Test with different phone/carrier
    - Check conversation logs for DTMF events
  </Accordion>

  <Accordion title="Call drops unexpectedly" icon="phone-slash">
    **Symptoms:**
    - Call disconnects mid-conversation
    - Sudden hangups

    **Causes:**
    - Max duration reached
    - Inactivity timeout triggered
    - Carrier disconnect
    - Error in conversation flow

    **Solutions:**
    - Check max duration setting
    - Review inactivity timeout configuration
    - Check logs for error messages
    - Test with longer conversations
  </Accordion>
</AccordionGroup>

## Comparing Test Results

Track test results to measure improvements:

**Create a test log:**
- Date and time of test
- Device and carrier used
- Environment (quiet, noisy, etc.)
- Issues discovered
- Audio quality rating (1-5)
- Transcription accuracy rating (1-5)
- Overall experience rating (1-5)

**Example test log:**
```
Date: 2025-01-15 14:30
Device: iPhone 13 (AT&T)
Environment: Quiet office, headset
Issues: None
Audio Quality: 5/5
Transcription: 5/5
Overall: 5/5
Notes: Perfect baseline test
---
Date: 2025-01-15 14:45
Device: Same as above
Environment: Busy street, no headset
Issues: Some transcription errors with traffic noise
Audio Quality: 4/5
Transcription: 3/5
Overall: 4/5
Notes: Acceptable in noisy conditions
```
