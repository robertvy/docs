## Overview

The web simulator is built into the agent editor and provides real-time testing using the same technology as production calls. This means you can hear timing, barge-in behavior, and interruptions exactly as your callers would experience them.

**Location:** Right-hand panel inside the agent editor

**Key benefit:** Test instantly without phone numbers or external tools

---

## How to Use the Web Simulator

<Steps>
  <Step title="Open your agent">
    Navigate to **AI Agents** and open the agent you want to test
  </Step>

  <Step title="Start the simulator">
    Click **Test Call** in the top-right corner of the agent editor
  </Step>

  <Step title="Grant microphone access">
    Allow your browser to access your microphone when prompted
  </Step>

  <Step title="Start speaking">
    The agent will greet you based on your configured greeting settings

    Speak naturally to test the conversation flow
  </Step>

  <Step title="Review transcript">
    Watch the real-time transcript appear as you speak

    See the agent's responses in text and audio
  </Step>
</Steps>

## Real-Time Transcript

The simulator shows a live transcript of the conversation:

**User input** (your speech):
- Shown in one color/style
- Transcribed in real-time by your selected transcriber (Deepgram, Azure, etc.)

**Agent responses** (AI output):
- Shown in a different color/style
- Includes both text and audio output
- Shows when actions are triggered

**System events**:
- Tool/action executions
- DTMF inputs (if configured)
- Transfers or call control events

## Testing Best Practices

<AccordionGroup>
  <Accordion title="Test in a quiet environment first" icon="volume-off">
    Start testing in a quiet room with a good microphone to eliminate audio quality variables.

    Once basic functionality works, test with background noise to verify robustness.
  </Accordion>

  <Accordion title="Test interruptions deliberately" icon="hand">
    Real callers will interrupt your agent:
    - Cut off the agent mid-sentence
    - Speak while the agent is still talking
    - Verify the agent handles it gracefully

    This tests your barge-in settings and response handling.
  </Accordion>

  <Accordion title="Test edge cases" icon="triangle-exclamation">
    Don't just test happy paths:
    - Ask questions the agent shouldn't know
    - Request actions that aren't configured
    - Stay silent to test inactivity handling
    - Provide unclear or ambiguous input
    - Ask off-topic questions
  </Accordion>

  <Accordion title="Test all configured actions" icon="bolt">
    Trigger every action at least once:
    - Custom API calls
    - Transfers (won't actually transfer, but will show the intent)
    - Bookings
    - Call control actions

    Verify the agent properly invokes each tool.
  </Accordion>

  <Accordion title="Reset between test runs" icon="rotate">
    Click **Reset** to clear conversation history and start fresh.

    This ensures you're testing from a clean state without cached context.
  </Accordion>

  <Accordion title="Test with different microphones" icon="microphone">
    Try different devices:
    - Built-in laptop mic
    - USB headset
    - Bluetooth headphones
    - Phone speaker (if using phone browser)

    Audio quality affects transcription accuracy.
  </Accordion>
</AccordionGroup>

## Reset and Restart

**Reset button** - Clears the conversation context and restarts the test session without reloading the page.

Use this to:
- Start a fresh conversation
- Test greeting behavior again
- Clear any cached conversation data
- Verify changes you just made to the agent

<Tip>
Reset frequently during testing to ensure you're testing from a clean state.
</Tip>

## Web Simulator vs Phone Testing

| Feature | Web Simulator | Phone Test Call |
|---------|---------------|-----------------|
| **Setup** | Instant - no configuration | Requires test number |
| **Audio Path** | Browser → Platform → Browser | PSTN → Phone Network → Platform |
| **Latency** | Lower (direct connection) | Higher (carrier routing) |
| **Echo/Noise** | Controlled environment | Real-world carrier conditions |
| **DTMF Testing** | Not available | Full DTMF support |
| **Carrier Testing** | No | Yes - tests actual telephony |
| **Use Case** | Quick iteration, prompt testing | Final validation, carrier testing |

<Note>
The web simulator uses the same voice technology as phone calls, so most behavior is identical. However, phone testing is essential for validating carrier audio quality, latency, and DTMF functionality.
</Note>

## Common Issues

<AccordionGroup>
  <Accordion title="Microphone not working" icon="microphone-slash">
    **Causes:**
    - Browser permissions denied
    - Another app using the microphone
    - Hardware issue

    **Solutions:**
    1. Check browser permissions (click lock icon in address bar)
    2. Close other apps that might use the microphone (Zoom, Teams, etc.)
    3. Refresh the page and grant permissions again
    4. Try a different browser
  </Accordion>

  <Accordion title="Agent doesn't respond" icon="comment-slash">
    **Causes:**
    - Microphone not picking up audio
    - Transcriber not configured
    - LLM API error
    - Instructions blocking responses

    **Solutions:**
    1. Check that your microphone is working (see visual input indicator)
    2. Verify transcriber is configured in **Models → Transcriber**
    3. Check conversation logs for errors
    4. Review your instructions for overly restrictive rules
  </Accordion>

  <Accordion title="Poor transcription accuracy" icon="closed-captioning">
    **Causes:**
    - Background noise
    - Poor microphone quality
    - Fast or unclear speech
    - Accent or language mismatch

    **Solutions:**
    1. Test in a quieter environment
    2. Use a better microphone (headset recommended)
    3. Speak clearly and at moderate pace
    4. Verify correct language/model in transcriber settings
    5. Try a different transcriber provider (Deepgram vs Azure)
  </Accordion>

  <Accordion title="High latency" icon="gauge">
    **Causes:**
    - Slow internet connection
    - LLM provider latency
    - Complex prompts causing slow inference
    - Large knowledge bases

    **Solutions:**
    1. Check your internet speed
    2. Try a faster LLM model (e.g., GPT-4o-mini instead of GPT-4)
    3. Simplify your instructions
    4. Reduce knowledge base size or use RAG mode
    5. Test with different provider regions
  </Accordion>
</AccordionGroup>
