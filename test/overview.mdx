## How Test Mode Works

Every agent in the dashboard ships with two integrated test surfaces:

- **Web Simulator** – the right-hand panel inside the agent editor. It uses the same voice technology that production calls use, so you can hear timing, barge-in, and interruptions exactly as your callers would.
- **Phone Test Call** – click **Call test number** to spin up a temporary PSTN line. The call is routed through the same phone infrastructure, which makes it ideal for verifying carrier audio, latency, and IVR navigation.

## Testing Checklist

<Steps>
  <Step title="Baseline web run">
    Use the simulator to confirm greetings, instructions, and initial actions. Fix prompt issues here before dialing a phone.
  </Step>
  <Step title="Phone testing">
    Call the test number twice: once with phone on ear, once on speakerphone. Listen for echo cancellation, barge-in reliability, and DTMF capture.
  </Step>
  <Step title="Test in real environments">
    Test in environments where your customers will call from (e.g., restaurant, car, office). This validates performance with background noise and real-world conditions.
  </Step>
  <Step title="Edge cases">
    Interrupt the agent mid-sentence, stay silent for inactivity handling, and trigger each configured action (transfer, booking, custom API) at least once.
  </Step>
  <Step title="Review call log">
    Open **Conversations**, filter by agent name, and review the transcript, goal analysis, and action details to ensure data is being captured correctly.

## Quick Tips

<CardGroup cols={2}>
  <Card title="Reset context fast" icon="repeat">
    Use **Reset test** in the simulator to drop any cached conversation data and restart the pipeline without reloading the page.
  </Card>
  <Card title="DTMF debugging" icon="hashtag">
    Check the log for DTMF input data. If it's empty, re-check the agent's DTMF settings in **Abilities → Actions → Call Control**.
  </Card>
  <Card title="Compare providers" icon="arrows-left-right">
    Switch between Deepgram and Azure in **Models → Transcriber**, run identical scripts, and save the best performer as the agent default.
  </Card>
  <Card title="Share results" icon="share-nodes">
    Forward the log link to teammates. Access is scoped by team permissions, so other editors can replay audio without exporting files.
  </Card>
</CardGroup>
