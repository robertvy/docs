## Overview

Every conversation your agent has—whether a test call or production call—creates a detailed log in the itellicoAI dashboard. These logs are essential for debugging, quality assurance, and understanding how your agent performs in real-world scenarios.

**Access logs:** Navigate to **Conversations → Logs** in the dashboard

---

Each conversation log contains:

## Reading the Transcript

The transcript shows the complete conversation with distinct formatting:

**User messages:**
- What the caller said (transcribed)
- Timestamp of when they spoke
- Confidence scores (if available)

**Agent messages:**
- What the agent said (generated text)
- Timestamp of response
- Audio duration

**System events:**
- Actions triggered
- DTMF captured
- Transfers initiated
- Errors occurred

### Example transcript format:

```
[00:00:00] System: Call started
[00:00:01] Agent: "Hello! Thanks for calling Acme Support. How can I help you today?"
[00:00:05] User: "I need help resetting my password"
[00:00:09] Agent: "I can help you with that. What's the email address on your account?"
[00:00:13] User: "It's john@example.com"
[00:00:17] System: Action triggered - send_password_reset
[00:00:18] Agent: "Perfect. I've sent a password reset email to john@example.com. You should receive it within a few minutes."
[00:00:25] User: "Great, thank you"
[00:00:27] Agent: "You're welcome! Is there anything else I can help you with?"
[00:00:30] User: "No, that's all"
[00:00:32] Agent: "Have a great day!"
[00:00:34] System: Call ended - Duration: 34 seconds
```

## Listening to Call Recordings

Most conversations include audio recordings for review:

<Steps>
  <Step title="Open conversation log">
    Navigate to the specific conversation
  </Step>

  <Step title="Find audio player">
    Look for the audio playback controls at the top of the log
  </Step>

  <Step title="Play recording">
    Click play to hear the entire call

    Use the timeline to skip to specific moments
  </Step>

  <Step title="Compare with transcript">
    Listen while reading transcript to identify transcription errors
  </Step>
</Steps>

**What to listen for:**
- Audio quality and clarity
- Transcription accuracy
- Voice naturalness
- Response timing and latency
- Interruption handling
- Background noise issues
- Echo or feedback

<Tip>
Listen at 1.5x or 2x speed to review calls faster.
</Tip>

## Action and Tool Execution Logs

When your agent executes actions or calls APIs, the logs show complete details:

**For each action, you'll see:**
- Action name
- Timestamp of execution
- Input parameters extracted from conversation
- Full request payload (JSON)
- API response (JSON)
- Response status code
- Execution time
- Success or failure status

### Example action log:

```json
{
  "action": "send_password_reset",
  "timestamp": "2025-01-15T10:17:15Z",
  "parameters": {
    "email": "john@example.com"
  },
  "request": {
    "method": "POST",
    "url": "https://api.example.com/auth/reset",
    "headers": {
      "Authorization": "Bearer ***",
      "Content-Type": "application/json"
    },
    "body": {
      "email": "john@example.com",
      "source": "voice_agent"
    }
  },
  "response": {
    "status": 200,
    "body": {
      "success": true,
      "reset_token_sent": true,
      "message": "Password reset email sent"
    }
  },
  "execution_time_ms": 245,
  "status": "success"
}
```

**Debugging with action logs:**
- Verify correct parameters were extracted
- Check API was called with correct data
- Review API response for errors
- Confirm agent handled response appropriately
- Identify timeout or connectivity issues

## DTMF Input Logs

If your agent captures DTMF (phone keypad) inputs, logs show:

- When keys were pressed
- Which keys were pressed
- How agent responded to each input
- Any DTMF errors or invalid inputs

**Example DTMF log:**
```
[00:00:12] System: DTMF detected - Key: 1
[00:00:12] Agent: "You selected option 1. Let me help you with that."
[00:00:18] System: DTMF detected - Key: 9
[00:00:18] Agent: "I'm sorry, 9 is not a valid option. Please press 1, 2, or 3."
```

## Goal Analytics

If you've configured conversation goals, logs show:

**Goal completion status:**
- ✅ Achieved
- ❌ Not achieved
- ⚠️ Partially achieved

**Goal details:**
- Goal name
- Evaluation criteria
- Reasoning for status
- Timestamp of evaluation

### Example goal analysis:

```json
{
  "goal": "Resolve customer issue",
  "status": "achieved",
  "reasoning": "Customer stated their password reset issue was resolved and thanked the agent. The agent successfully sent a reset email and confirmed the action.",
  "confidence": 0.95,
  "evaluated_at": "2025-01-15T10:18:00Z"
}
```

## Post-Call Analysis

Post-call analysis questions (if configured) appear with responses:

**Example:**
- **Question:** "Was the customer satisfied?"
  - **Answer:** "Yes - customer thanked the agent and confirmed their issue was resolved"

- **Question:** "Did the agent follow company procedures?"
  - **Answer:** "Yes - agent verified email address before sending reset"

- **Question:** "Were there any errors or issues?"
  - **Answer:** "No - call proceeded smoothly with no errors"

## Performance Metrics

Key metrics to track in conversation logs:

<CardGroup cols={2}>
  <Card title="Average Call Duration" icon="clock">
    Track typical conversation length

    Identify unusually short (failures?) or long (complex issues?) calls
  </Card>

  <Card title="Goal Completion Rate" icon="bullseye">
    Percentage of calls where goals were achieved

    Target: >80% for most use cases
  </Card>

  <Card title="Action Success Rate" icon="check">
    Percentage of actions that execute successfully

    Target: >95%
  </Card>

  <Card title="Transfer Rate" icon="right-left">
    Percentage of calls transferred to humans

    Lower is usually better (automation working)
  </Card>

  <Card title="Average Response Time" icon="gauge">
    Time between user speech and agent response

    Target: <2 seconds for good UX
  </Card>

  <Card title="Transcription Accuracy" icon="closed-captioning">
    Quality of speech-to-text conversion

    Review transcripts vs audio manually
  </Card>
</CardGroup>

## Identifying Patterns

Use logs to identify trends and issues:

<AccordionGroup>
  <Accordion title="Common failure points" icon="triangle-exclamation">
    **Look for:**
    - Specific questions agent struggles with
    - Actions that frequently fail
    - Topics that trigger transfers
    - Conversation paths that dead-end

    **Action:**
    - Add missing knowledge
    - Fix broken API integrations
    - Improve instructions for common patterns
  </Accordion>

  <Accordion title="Performance degradation" icon="arrow-trend-down">
    **Look for:**
    - Increasing call duration over time
    - Decreasing goal completion rates
    - More errors or failures
    - Slower response times

    **Action:**
    - Check for service issues
    - Review recent changes
    - Optimize prompts or knowledge base
  </Accordion>

  <Accordion title="Successful patterns" icon="star">
    **Look for:**
    - Conversations that go perfectly
    - High satisfaction indicators
    - Quick resolutions
    - Smooth action execution

    **Action:**
    - Replicate successful patterns
    - Use as training examples
    - Apply to other agents
  </Accordion>

  <Accordion title="Edge cases" icon="circle-exclamation">
    **Look for:**
    - Unusual requests or questions
    - Unexpected conversation flows
    - Novel user behaviors
    - System limitations discovered

    **Action:**
    - Document edge cases
    - Add test scenarios
    - Improve error handling
    - Update knowledge or instructions
  </Accordion>
</AccordionGroup>

## Regular Log Review Routine

Establish a regular review process:

<Steps>
  <Step title="Daily quick check (5-10 min)">
    - Check for any failed calls
    - Review newest conversations
    - Look for obvious errors
    - Respond to critical issues
  </Step>

  <Step title="Weekly deep dive (30-60 min)">
    - Review goal completion trends
    - Sample 10-20 random conversations
    - Identify recurring issues
    - Check action success rates
    - Export data for analysis
  </Step>

  <Step title="Monthly analysis (2-3 hours)">
    - Compare metrics month-over-month
    - Identify patterns and trends
    - Review transcripts for quality
    - Update knowledge or instructions
    - Plan optimizations
  </Step>

  <Step title="Quarterly audit (half day)">
    - Comprehensive review of all metrics
    - Listen to sample of calls
    - Stakeholder report
    - Strategic improvements
    - Roadmap updates
  </Step>
</Steps>

## Best Practices

<AccordionGroup>
  <Accordion title="Review test calls immediately" icon="vial">
    After each test session, review logs right away while context is fresh

    Document issues and fixes needed
  </Accordion>

  <Accordion title="Sample production calls regularly" icon="shuffle">
    Don't just review failures - sample successful calls too

    Understand what "good" looks like
  </Accordion>

  <Accordion title="Listen to audio, don't just read transcripts" icon="headphones">
    Transcripts miss audio quality issues, tone, pacing, etc.

    Listen to a representative sample
  </Accordion>

  <Accordion title="Track changes over time" icon="chart-line">
    Export metrics regularly to track trends

    Measure impact of changes you make
  </Accordion>

  <Accordion title="Share insights with team" icon="users">
    Don't silo log analysis - share learnings

    Build institutional knowledge
  </Accordion>
</AccordionGroup>
