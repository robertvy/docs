## Overview

This guide covers the most common issues encountered when testing and deploying voice agents, with step-by-step solutions for each.

**Before troubleshooting:**
1. Try to reproduce the issue consistently
2. Check conversation logs for error messages
3. Verify basic configuration is complete
4. Test in web simulator first (faster iteration)

---

## Speech Recognition Issues

### Transcription is Inaccurate

<AccordionGroup>
  <Accordion title="Solution 1: Verify transcriber configuration" icon="gear">
    <Steps>
      <Step title="Check transcriber settings">
        Navigate to **Models → Transcriber**

        Verify a transcriber is selected
      </Step>

      <Step title="Verify language matches">
        Ensure language setting matches the language being spoken

        Example: If speaking English, select "en-US" not "es-ES"
      </Step>

      <Step title="Try a different model">
        Switch between available models:
        - Deepgram Nova-2 → Nova-3
        - Azure → Deepgram
        - General model → specific use case model
      </Step>

      <Step title="Test again">
        Make a new test call and check transcription quality
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Solution 2: Improve audio quality" icon="microphone">
    **Common causes:**
    - Background noise
    - Poor microphone quality
    - Distance from microphone
    - Multiple people talking

    **Solutions:**
    - Test in quiet environment
    - Use headset instead of speaker phone
    - Speak directly into microphone
    - Speak clearly at normal pace
    - Reduce background music/TV/traffic

    **Test result:**
    If transcription improves dramatically, audio quality was the issue.
  </Accordion>

  <Accordion title="Solution 3: Check for accent/dialect mismatch" icon="globe">
    <Steps>
      <Step title="Verify language code">
        Example: "en-US" vs "en-GB" vs "en-AU"

        Different English dialects have different models
      </Step>

      <Step title="Try multilingual model">
        Some transcribers offer multilingual models that handle accents better

        Example: Deepgram "multilingual" vs "en-US"
      </Step>

      <Step title="Test with different speaker">
        Have someone with a different accent test

        Helps isolate whether it's accent-specific
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>

### Words or Phrases Consistently Misheard

<AccordionGroup>
  <Accordion title="Solution: Add custom pronunciations" icon="spell-check">
    <Steps>
      <Step title="Identify problematic words">
        Review logs to find consistently misheard words:
        - Company name
        - Product names
        - Technical terms
        - Acronyms
      </Step>

      <Step title="Add pronunciations">
        Navigate to **Voice & Speech → Custom Pronunciations**

        Add entry for each problem word
      </Step>

      <Step title="Specify phonetic spelling">
        Write how the word should sound phonetically

        Example: "itellico" → "eye-tell-ee-co"
      </Step>

      <Step title="Test pronunciation">
        Make test call using the word

        Verify it's now heard correctly
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Alternative: Modify agent responses" icon="comment">
    If the issue is with how the agent SAYS something (not hears):

    <Steps>
      <Step title="Add SSML or formatting">
        Use punctuation to improve pronunciation

        Example: "AI" becomes "A.I." for clarity
      </Step>

      <Step title="Spell out acronyms">
        Instead of "FAQ", say "F A Q" or "frequently asked questions"
      </Step>

      <Step title="Test different phrasing">
        Find alternative ways to say problematic words
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>

## LLM Response Issues

### Agent Gives Wrong or Off-Topic Answers

<AccordionGroup>
  <Accordion title="Solution 1: Review and improve instructions" icon="message">
    <Steps>
      <Step title="Check current instructions">
        Navigate to **Abilities → Instructions**

        Read through your prompt
      </Step>

      <Step title="Look for issues">
        - Are instructions clear and specific?
        - Do they contradict each other?
        - Are they too vague?
        - Are there missing boundaries?
      </Step>

      <Step title="Add specific guidance">
        Example improvements:
        ```
        BAD: "Be helpful"
        GOOD: "Answer questions about our products and services. If asked about competitors, politely decline."
        ```
      </Step>

      <Step title="Test the change">
        Ask the same question that previously got wrong answer

        Verify improvement
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Solution 2: Add missing knowledge" icon="book">
    <Steps>
      <Step title="Identify knowledge gap">
        Review logs to see what information agent is missing

        Example: "Agent doesn't know business hours"
      </Step>

      <Step title="Add knowledge item">
        Navigate to **Knowledge**

        Add the missing information in clear, structured format
      </Step>

      <Step title="Verify knowledge is indexed">
        Check status shows INDEXED (not just COMPLETED)

        Wait for indexing to complete if pending
      </Step>

      <Step title="Assign to agent">
        If not already assigned, link knowledge base to agent

        Save agent configuration
      </Step>

      <Step title="Test knowledge retrieval">
        Ask the question again

        Agent should now have correct information
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Solution 3: Adjust model or settings" icon="sliders">
    <Steps>
      <Step title="Try different model">
        Navigate to **Models → Model**

        Switch between:
        - Claude Haiku 4.5 ↔ GPT-4.1 mini
        - GPT-4 ↔ Claude Sonnet
        - Different providers

        Some models are better for specific tasks
      </Step>

      <Step title="Adjust temperature">
        Lower temperature (0.3-0.5) = more consistent, focused
        Higher temperature (0.7-0.9) = more creative, varied

        For factual Q&A, use lower temperature
      </Step>

      <Step title="Test again">
        Make new test call with adjusted settings
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>

### Agent Hallucinates or Makes Up Information

<AccordionGroup>
  <Accordion title="Solution: Add explicit boundaries" icon="ban">
    <Steps>
      <Step title="Update instructions">
        Add explicit rules about what agent should NOT do:
        ```
        DO NOT make up information you don't have.
        DO NOT guess at answers.
        If you don't know something, say "I don't have that information" and offer to help with something else.
        ```
      </Step>

      <Step title="Test with questions agent shouldn't know">
        Ask questions outside agent's knowledge

        Example: "What's the weather?" or "Tell me about [competitor]"
      </Step>

      <Step title="Verify graceful declining">
        Agent should admit it doesn't know instead of guessing
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>

## Voice and Audio Issues

### Voice Sounds Robotic or Unnatural

<AccordionGroup>
  <Accordion title="Solution 1: Try different voice" icon="waveform">
    <Steps>
      <Step title="Browse available voices">
        Navigate to **Models → Voice**

        Preview different voices from each provider
      </Step>

      <Step title="Try voice designed for conversations">
        Some voices are optimized for dialogue vs narration

        Look for "conversational" voices
      </Step>

      <Step title="Test multiple voices">
        Make short test calls with 3-4 different voices

        Pick the most natural sounding
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Solution 2: Adjust voice settings" icon="sliders">
    For ElevenLabs voices:

    <Steps>
      <Step title="Navigate to voice settings">
        **Models → Voice → Voice Settings**
      </Step>

      <Step title="Adjust stability">
        Lower stability (0.3-0.5) = more expressive
        Higher stability (0.7-0.9) = more consistent

        Try lower for more natural sound
      </Step>

      <Step title="Adjust similarity">
        Controls how close to original voice

        Usually keep at default (0.75)
      </Step>

      <Step title="Enable speaker boost">
        Can improve clarity and presence

        Toggle on if available
      </Step>

      <Step title="Test changes">
        Make test call with new settings
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>

### Specific Words Mispronounced

<AccordionGroup>
  <Accordion title="Solution: Add custom pronunciations" icon="spell-check">
    <Steps>
      <Step title="Identify mispronounced words">
        Listen to recordings and note problem words:
        - Company names
        - Product names
        - Technical terms
        - Acronyms
        - Foreign words
      </Step>

      <Step title="Add pronunciation rule">
        Navigate to **Voice & Speech → Custom Pronunciations**

        Add new pronunciation:
        - Word: "itellico"
        - Pronunciation: "eye-tell-ee-co"
      </Step>

      <Step title="Test pronunciation">
        Make test call where agent says the word

        Verify improvement
      </Step>

      <Step title="Iterate if needed">
        Adjust phonetic spelling until it sounds correct
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Alternative: Rewrite agent responses" icon="pen">
    Sometimes rewording is easier than pronunciation rules:

    **Examples:**
    - "AI" → "artificial intelligence" or "A I" (with periods)
    - "FAQ" → "frequently asked questions"
    - "SQL" → "S Q L" or "sequel"
    - Numbers: "2024" → "twenty twenty-four" vs "two thousand twenty-four"
  </Accordion>
</AccordionGroup>

### Echo or Feedback During Calls

<AccordionGroup>
  <Accordion title="Solution 1: Device and environment" icon="phone">
    **Most common cause: Speaker phone without proper echo cancellation**

    <Steps>
      <Step title="Test with headset">
        Have caller use headset or headphones

        This usually eliminates echo completely
      </Step>

      <Step title="Reduce volume">
        If using speaker phone, lower volume

        Prevents speaker output from being picked up by microphone
      </Step>

      <Step title="Test in different location">
        Hard surfaces (tile, concrete) cause more echo

        Try carpeted room or add acoustic dampening
      </Step>

      <Step title="Verify one device only">
        Make sure only one phone/computer is on the call

        Multiple devices cause feedback loops
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Solution 2: Check SIP trunk settings" icon="network-wired">
    <Steps>
      <Step title="Review SIP trunk configuration">
        Navigate to **Telephony → SIP Trunks**

        Check your SIP trunk settings
      </Step>

      <Step title="Verify echo cancellation enabled">
        Ensure echo cancellation is turned on in SIP settings

        Contact your SIP provider if uncertain
      </Step>

      <Step title="Check codec settings">
        Some codecs have better echo cancellation

        G.722 often performs well
      </Step>

      <Step title="Test with different carrier">
        Try calling from different phone/carrier

        Helps isolate if it's carrier-specific
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>

## Action and Integration Issues

### Actions Not Triggering

<AccordionGroup>
  <Accordion title="Solution 1: Verify action is enabled and configured" icon="toggle-on">
    <Steps>
      <Step title="Check action exists">
        Navigate to **Actions** in agent editor

        Verify the action is in the list
      </Step>

      <Step title="Verify action is enabled">
        Check that toggle is ON for the action
      </Step>

      <Step title="Review action trigger">
        Check the description tells agent WHEN to use it

        Example: "Use this action when user asks to book an appointment"
      </Step>

      <Step title="Test trigger explicitly">
        Say exactly what should trigger the action

        Example: "I want to book an appointment"
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Solution 2: Improve action description" icon="message">
    **The agent needs clear instructions about when to use actions**

    <Steps>
      <Step title="Review action description">
        Check description in action configuration

        Is it clear and specific?
      </Step>

      <Step title="Improve description">
        Make it explicit:

        ```
        BAD: "Booking system"
        GOOD: "Use this action when the user wants to schedule, book, or reserve an appointment. Call this after confirming the date and time with the user."
        ```
      </Step>

      <Step title="Add trigger keywords">
        Include likely phrases:
        - "schedule appointment"
        - "book a time"
        - "reserve a slot"
      </Step>

      <Step title="Test again">
        Use the specific trigger phrases in test call
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Solution 3: Check logs for errors" icon="file-lines">
    <Steps>
      <Step title="Open conversation logs">
        Navigate to **Conversations**

        Find the test call where action should have triggered
      </Step>

      <Step title="Search for action name">
        Look through the log for mentions of the action

        Check if it was attempted
      </Step>

      <Step title="Review error messages">
        If action was triggered but failed, check error details:
        - API endpoint unreachable
        - Authentication failure
        - Timeout
        - Invalid parameters
      </Step>

      <Step title="Fix identified issue">
        Based on error, fix:
        - API endpoint URL
        - API credentials
        - Parameter extraction
        - Network/firewall issues
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>

### Actions Trigger But Fail

<AccordionGroup>
  <Accordion title="Solution: Debug the API call" icon="code">
    <Steps>
      <Step title="Check action payload in logs">
        Open conversation log

        Find the action execution details

        Review the full JSON request payload
      </Step>

      <Step title="Verify parameters">
        Check that required parameters were extracted correctly:
        - Are all required fields present?
        - Are values in correct format?
        - Are there any null/empty values?
      </Step>

      <Step title="Test API endpoint directly">
        Use Postman or curl to test the API with the same payload

        Example:
        ```bash
        curl -X POST https://api.example.com/bookings \
          -H "Authorization: Bearer YOUR_KEY" \
          -H "Content-Type: application/json" \
          -d '{"date": "2025-01-15", "time": "14:00"}'
        ```
      </Step>

      <Step title="Check API response">
        Review what the API returned:
        - Status code (200 = success, 400/500 = error)
        - Error message in response body
        - Whether data was actually created/updated
      </Step>

      <Step title="Fix the issue">
        Based on findings:
        - Fix authentication (API key wrong?)
        - Fix endpoint URL (typo?)
        - Fix parameter format (date format wrong?)
        - Fix API endpoint (is it working?)
      </Step>

      <Step title="Test action again">
        Trigger action in new test call

        Verify it now succeeds
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>

## Knowledge Base Issues

### Agent Can't Find Information in Knowledge Base

<AccordionGroup>
  <Accordion title="Solution 1: Verify knowledge is indexed" icon="database">
    <Steps>
      <Step title="Check knowledge base status">
        Navigate to **Knowledge**

        Open the knowledge base assigned to agent
      </Step>

      <Step title="Check item status">
        Look at each knowledge item

        Status should be **INDEXED** (not just COMPLETED)
      </Step>

      <Step title="Wait for indexing">
        If status is PENDING or INDEXING, wait for it to complete

        Can take several minutes for large documents
      </Step>

      <Step title="Re-index if failed">
        If status is FAILED, try deleting and re-creating the item
      </Step>

      <Step title="Test again">
        Ask the question again once items are INDEXED
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Solution 2: Improve knowledge item titles" icon="heading">
    **RAG retrieval relies on good titles and descriptions**

    <Steps>
      <Step title="Review knowledge item titles">
        Are titles descriptive and specific?

        ```
        BAD: "Policy 1"
        GOOD: "Return Policy for Physical Products"
        ```
      </Step>

      <Step title="Add keywords to titles">
        Include terms users might search for:

        ```
        BETTER: "Return Policy - Refunds, Exchanges, Shipping for Physical Products"
        ```
      </Step>

      <Step title="Update titles">
        Edit knowledge items with better titles
      </Step>

      <Step title="Wait for re-indexing">
        Changes may require re-indexing

        Wait a few minutes
      </Step>

      <Step title="Test retrieval">
        Ask questions using keywords from new titles
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Solution 3: Try Context Mode instead of RAG" icon="brain">
    <Steps>
      <Step title="Navigate to knowledge assignment">
        In agent editor, find where knowledge base is assigned
      </Step>

      <Step title="Switch to Context Mode">
        Change from RAG to Context Mode

        This injects ALL knowledge into every conversation
      </Step>

      <Step title="Test again">
        Ask the same question

        Agent should now have the information
      </Step>

      <Step title="Consider trade-offs">
        Context Mode pros:
        - More reliable (all knowledge always available)

        Context Mode cons:
        - Uses more tokens
        - May include irrelevant information
        - Limited by context window size

        RAG pros:
        - Only retrieves relevant information
        - Scales to larger knowledge bases

        RAG cons:
        - May miss relevant items if titles/content unclear
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>

## Call Control Issues

### DTMF Not Working

<AccordionGroup>
  <Accordion title="Solution: Enable and configure DTMF" icon="phone-keypad">
    <Steps>
      <Step title="Enable DTMF in settings">
        Navigate to **Abilities → Actions → Call Control**

        Enable DTMF capture
      </Step>

      <Step title="Configure DTMF actions">
        Define what each key press should do:
        - 1 = Option A
        - 2 = Option B
        - etc.
      </Step>

      <Step title="Update agent instructions">
        Tell agent to prompt for DTMF input:

        ```
        "Press 1 for sales, press 2 for support, or press 3 for billing"
        ```
      </Step>

      <Step title="Verify SIP trunk supports DTMF">
        Check SIP trunk configuration

        Ensure DTMF method is set (RFC2833 recommended)
      </Step>

      <Step title="Test DTMF capture">
        Make phone test call

        Press keys when prompted

        Check logs to see if DTMF was captured
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>

### Transfers Not Working

<AccordionGroup>
  <Accordion title="Solution: Configure transfer destination" icon="right-left">
    <Steps>
      <Step title="Check transfer action exists">
        Navigate to **Actions**

        Verify transfer action is configured
      </Step>

      <Step title="Verify destination">
        Check transfer destination is correct:
        - Phone number format: +1234567890
        - SIP URI format: sip:support@pbx.example.com
        - Agent ID (if transferring to another agent)
      </Step>

      <Step title="Test destination manually">
        Try calling the destination directly

        Verify it's reachable
      </Step>

      <Step title="Review transfer logs">
        Check conversation logs for transfer attempts

        Look for error messages
      </Step>

      <Step title="Fix configuration">
        Update transfer destination based on findings

        Save agent configuration
      </Step>

      <Step title="Test transfer">
        Trigger transfer in test call

        Verify it connects (or at least attempts)
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>

## Performance Issues

### High Latency / Slow Responses

<AccordionGroup>
  <Accordion title="Solution 1: Use faster model" icon="gauge">
    <Steps>
      <Step title="Check current model">
        Navigate to **Models → Model**

        Note which model you're using
      </Step>

      <Step title="Switch to faster model">
        Try:
        - GPT-4 → GPT-4.1 mini
        - Claude Sonnet → Claude Haiku 4.5

        Smaller models are faster
      </Step>

      <Step title="Test latency">
        Make test call

        Time the response delay
      </Step>

      <Step title="Evaluate quality trade-off">
        Faster models may give slightly lower quality responses

        Find the right balance for your use case
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Solution 2: Simplify instructions" icon="scissors">
    **Longer prompts = slower inference**

    <Steps>
      <Step title="Review instructions length">
        Navigate to **Abilities → Instructions**

        Count words/characters
      </Step>

      <Step title="Remove unnecessary content">
        Cut:
        - Redundant explanations
        - Excessive examples
        - Overly detailed rules

        Keep only essential guidance
      </Step>

      <Step title="Move details to knowledge base">
        Put detailed information in knowledge base instead of instructions

        Instructions should be high-level guidance
      </Step>

      <Step title="Test response time">
        Make new test call

        Verify improvement in latency
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Solution 3: Optimize knowledge base" icon="database">
    <Steps>
      <Step title="Switch to RAG mode">
        If using Context Mode with large knowledge base, switch to RAG

        RAG only retrieves relevant items (faster)
      </Step>

      <Step title="Reduce knowledge size">
        Remove or unlink knowledge items that aren't frequently needed

        Break large documents into focused items
      </Step>

      <Step title="Test again">
        Make test call

        Check if latency improved
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>
