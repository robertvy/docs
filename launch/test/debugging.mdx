## Debugging Philosophy

When your agent behaves unexpectedly, systematic debugging saves time. Follow this process:

1. **Reproduce** - Can you make the issue happen again consistently?
2. **Isolate** - Is it instructions, integrations, model, or infrastructure?
3. **Inspect** - Review logs, traces, and metrics for clues
4. **Fix** - Apply smallest change that resolves the issue
5. **Validate** - Test to confirm fix works

## Common Debugging Scenarios

### Agent Gives Wrong Answers

<Steps>
  <Step title="Check instructions">
    Review agent → **Conversation → Instructions**:
    - Are instructions clear and specific?
    - Any conflicting rules?
    - Does agent have context for the question?
  </Step>

  <Step title="Verify knowledge base">
    - Is the correct answer in your KB?
    - Test KB search manually:
      ```bash
      curl -X POST https://api.itellico.ai/v1/knowledge-bases/kb_123/search \
           -H "Authorization: Bearer YOUR_API_KEY" \
           -d '{"query": "your test query"}'
      ```
    - Check retrieval in logs → **Knowledge Retrieved** section
  </Step>

  <Step title="Review conversation log">
    - What context did agent have when answering?
    - Was relevant KB content retrieved?
    - Did agent follow instruction templates?
  </Step>

  <Step title="Test in isolation">
    Create minimal test:
    - Simple instruction
    - Clear question
    - Expected KB content

    If works in isolation, issue is context/complexity.
  </Step>
</Steps>

### Actions Don't Trigger

<Accordion title="API action not firing">
  **Check in conversation logs**:
  1. Go to conversation → **Actions** tab
  2. Is action listed? If not, it never triggered
  3. If listed, check status: Success vs Failed

  **Why actions don't trigger**:
  - Agent instructions don't mention when to use it
  - User didn't provide required information
  - Action trigger condition not met

  **Fix**:
  - Add clear instruction: "When customer asks about order status, use getOrderStatus action"
  - Ensure agent collects required parameters first
  - Test trigger phrase in simulator
</Accordion>

<Accordion title="API returns error">
  **Check action details in logs**:
  - Request sent (payload format correct?)
  - Response received (error code? message?)
  - Auth headers (API key valid?)

  **Common errors**:
  - `401 Unauthorized` → Check API key
  - `404 Not Found` → Verify endpoint URL
  - `408 Timeout` → API too slow, increase timeout
  - `500 Internal Server Error` → External API issue

  **Test API manually**:
  ```bash
  # Copy request from logs and test with curl
  curl -X POST https://api.example.com/endpoint \
       -H "Authorization: Bearer YOUR_KEY" \
       -H "Content-Type: application/json" \
       -d '{"param": "value"}'
  ```

  If manual call works, issue is in action configuration.
</Accordion>

### Transcription Errors

<Accordion title="Agent mishears words">
  **Review transcript accuracy**:
  - Listen to call recording
  - Compare to transcript text
  - Identify misheard words (technical terms? names? accents?)

  **Solutions**:

  **1. Add Custom Vocabulary**:
  ```json
  // Agent settings → Transcriber → Custom Vocabulary
  {
    "custom_words": [
      {"word": "itellicoAI", "pronunciation": "eye-TELL-ih-koh AI"},
      {"word": "API", "pronunciation": "A P I"}
    ]
  }
  ```

  **2. Try Different Transcriber**:
  - Deepgram: Best for accents, noise
  - Azure: Best for technical terms
  - Whisper: Best for multilingual

  **3. Improve Audio Quality**:
  - Enable noise suppression
  - Use higher bitrate codec (Opus 64kbps)
  - Test in quieter environment
</Accordion>

### Agent Loops or Repeats

<Accordion title="Agent asks same question multiple times">
  **Diagnose**:
  1. Review transcript - where does loop start?
  2. What user response triggers repeat?
  3. Is transcription failing? (agent didn't "hear" answer)

  **Fixes**:

  **1. Add Loop Prevention Instructions**:
  ```markdown
  If you've asked the same question twice without getting a clear answer:
  1. Rephrase the question
  2. If still unclear, offer to transfer to a human

  Never ask the same question more than 3 times.
  ```

  **2. Use Structured Input (DTMF)**:
  For critical info, use keypad:
  ```markdown
  "Which department? Press 1 for Sales, 2 for Support."
  ```

  More reliable than speech recognition.

  **3. Check Transcription**:
  If agent didn't "hear" response, fix audio quality or transcriber settings.
</Accordion>

## Debugging with Logs

### Reading Conversation Logs

Each conversation log includes:

<Tabs>
  <Tab title="Transcript">
    **What to Look For**:
    - Transcription accuracy (compare to audio)
    - Agent response relevance
    - Conversation flow (natural vs awkward)
    - Goal achievement indicators

    **Timestamps**: Check if delays between turns indicate latency issues
  </Tab>

  <Tab title="Actions">
    **Details**:
    - Which actions triggered
    - Input parameters
    - API request/response
    - Execution time
    - Success/failure status

    **Debug Failures**: Error messages usually indicate root cause
  </Tab>

  <Tab title="Knowledge">
    **What Retrieved**:
    - Search query used
    - KB chunks retrieved
    - Similarity scores
    - Content used in response

    **Debug "I don't know"**: Check if relevant content retrieved but not used, or if retrieval failed entirely
  </Tab>

  <Tab title="Goals">
    **Outcomes**:
    - Which goals achieved
    - Confidence scores
    - Extracted entities (dates, names, etc.)

    **Debug Misclassification**: Review conversation - should goal have been achieved?
  </Tab>

  <Tab title="Quality">
    **Metrics**:
    - Audio quality (jitter, packet loss, MOS score)
    - Response latency
    - Call duration
    - Interruption frequency

    **Performance Issues**: Check latency and quality metrics
  </Tab>
</Tabs>

### Error Log Analysis

Filter errors by:
- Time range (last 24h, last week)
- Error type (API, transcription, system)
- Agent ID (specific agent)
- Severity (critical, warning, info)

**Common Patterns**:
- Same error repeating → Systemic issue (config, integration)
- Error spikes at specific time → Capacity or external service issue
- Errors on specific agent → Agent-specific config problem

## Advanced Debugging

### Model Behavior Debugging

<Accordion title="Compare model responses">
  Test same scenario with different models:

  **Steps**:
  1. Clone agent
  2. Change model (GPT-4 → Claude, etc.)
  3. Run identical test scenario
  4. Compare responses

  **When Useful**:
  - Agent giving unexpected responses
  - Testing if model upgrade improves quality
  - Debugging hallucinations or incorrect facts
</Accordion>

<Accordion title="Prompt engineering iteration">
  **Systematic Testing**:
  1. Create 5-10 test scenarios (saved in Test Scenarios)
  2. Run all scenarios with current instructions
  3. Document failures
  4. Modify instructions to address failures
  5. Re-run all scenarios
  6. Repeat until all pass

  **A/B Test Instructions**:
  - Create two agent versions with different instructions
  - Route 50% traffic to each
  - Compare goal achievement rates
  - Promote winner
</Accordion>

### Integration Debugging

<Accordion title="Test APIs in isolation">
  **Remove Agent from Equation**:
  1. Copy API request from logs
  2. Test with curl/Postman
  3. Verify response format matches expectations
  4. Confirm auth works

  If API works in isolation but fails from agent:
  - Check IP allowlisting
  - Verify timeout settings (agent may timeout before API responds)
  - Check request payload formatting
</Accordion>

<Accordion title="Mock API responses">
  For testing without hitting real APIs:

  **Use mock server**:
  ```javascript
  // mockapi.example.com/order-status
  {
    "order_id": "12345",
    "status": "shipped",
    "eta": "2024-01-15"
  }
  ```

  Point agent action to mock URL, test conversation flow without external dependencies.
</Accordion>

## Debugging Checklist

When debugging an issue:

- [ ] Can you reproduce the issue consistently?
- [ ] Reviewed conversation log for the failing call?
- [ ] Checked error logs for related errors?
- [ ] Verified agent instructions are clear?
- [ ] Tested in web simulator (faster iteration)?
- [ ] Isolated the issue (instructions vs integration vs model)?
- [ ] Applied minimal fix (changed one thing)?
- [ ] Re-tested to confirm fix works?
- [ ] Tested for regressions (didn't break other scenarios)?
- [ ] Documented the issue and fix?
