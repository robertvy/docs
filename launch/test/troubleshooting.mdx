## Troubleshooting Approach

When testing doesn't go as planned, follow this systematic approach:

1. **Identify the symptom** - What exactly is wrong?
2. **Isolate the cause** - Is it agent config, browser/phone issue, or platform?
3. **Check logs** - Review conversation logs for errors
4. **Test in isolation** - Remove variables to find root cause
5. **Apply fix** - Make smallest change that resolves issue
6. **Verify** - Re-test to confirm fix works

## Web Simulator Issues

### Microphone Not Working

**Symptom**: Agent can't hear you, no transcription appears.

<AccordionGroup>
  <Accordion title="Browser permissions">
    **Check Permissions**:
    - Chrome: `chrome://settings/content/microphone`
    - Firefox: Click lock icon in address bar → Permissions
    - Safari: Safari → Preferences → Websites → Microphone

    **Fix**: Allow microphone access for the site, reload page
  </Accordion>

  <Accordion title="Wrong microphone selected">
    **Diagnose**:
    - Click gear icon in simulator
    - Check which mic is selected
    - Verify it's your active microphone (not default system mic)

    **Fix**: Select correct microphone from dropdown, test audio levels
  </Accordion>

  <Accordion title="Microphone in use">
    **Symptom**: "Microphone busy" error

    **Cause**: Another app (Zoom, Meets, Discord) is using the mic

    **Fix**: Close other apps using microphone, restart browser if needed
  </Accordion>

  <Accordion title="Browser compatibility">
    **Requirements**:
    - Chrome 60+ (recommended)
    - Firefox 55+
    - Safari 14.1+
    - Edge 79+

    **Not Supported**: IE11, older mobile browsers

    **Fix**: Update browser or switch to supported browser
  </Accordion>
</AccordionGroup>

### No Audio from Agent

**Symptom**: You speak, transcript appears, but agent doesn't respond with voice.

<AccordionGroup>
  <Accordion title="Speaker/headphone issues">
    **Check**:
    - Volume not muted
    - Correct output device selected
    - Speakers/headphones plugged in and working

    **Fix**: Adjust volume, select correct speakers in simulator settings
  </Accordion>

  <Accordion title="Browser autoplay blocked">
    **Symptom**: First test works, subsequent tests have no audio

    **Cause**: Browser autoplay policy blocks audio

    **Fix**: Interact with page before starting test (click anywhere), or allow autoplay in browser settings
  </Accordion>

  <Accordion title="WebRTC connection failed">
    **Symptom**: "Connection failed" error in console

    **Cause**: Firewall or network blocking WebRTC

    **Fix**:
    - Check firewall allows WebRTC traffic
    - Try different network (mobile hotspot)
    - Enable TURN relay in agent settings (Enterprise)
  </Accordion>
</AccordionGroup>

### Simulator Stuck or Frozen

**Symptoms**: Simulator won't start, hangs during call, or won't end call.

<AccordionGroup>
  <Accordion title="Clear browser cache">
    **Steps**:
    1. Open browser dev tools (F12)
    2. Right-click refresh button → "Empty Cache and Hard Reload"
    3. Or clear site data: Settings → Privacy → Clear browsing data

    **Fix**: Clears stuck state from previous tests
  </Accordion>

  <Accordion title="Refresh page">
    **Simple Fix**: Reload page (Cmd+R or Ctrl+R)

    **Force reload**: Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows)

    Test again after reload
  </Accordion>

  <Accordion title="Check browser console">
    **Steps**:
    1. Open dev tools (F12)
    2. Go to Console tab
    3. Look for errors (red text)

    **Common Errors**:
    - `WebSocket connection failed` → Network issue
    - `Microphone permission denied` → Permissions issue
    - `Uncaught TypeError` → Platform bug, contact support

    Screenshot error and include in support ticket
  </Accordion>
</AccordionGroup>

## Phone Test Issues

### Test Call Doesn't Connect

**Symptom**: Click "Start Test" but phone never rings.

<AccordionGroup>
  <Accordion title="Verify phone number format">
    **Correct Format**: E.164 with country code

    ✅ Correct: `+12125551234`
    ❌ Wrong: `212-555-1234`, `(212) 555-1234`, `2125551234`

    **Fix**: Reformat to E.164, try again
  </Accordion>

  <Accordion title="Check 'From' number provisioned">
    **Steps**:
    1. Go to **Telephony → Numbers**
    2. Find your "From" number
    3. Verify status is "Active" not "Pending" or "Failed"

    **If Pending**: Wait 5-10 minutes for provisioning
    **If Failed**: Contact support with number details
  </Accordion>

  <Accordion title="Account capacity">
    **Check**: Are you at max concurrent calls?

    Navigate to **Dashboard → Usage** → Check "Concurrent Calls"

    **If at limit**: End active calls or upgrade plan
  </Accordion>

  <Accordion title="Carrier blocking">
    **Symptom**: Calls from other numbers work, but not to your phone

    **Cause**: Your carrier may be blocking calls (spam filter)

    **Fix**:
    - Try calling different phone
    - Add number to your contacts
    - Check carrier's spam/block list
    - Register CNAM (caller name) with carrier
  </Accordion>
</AccordionGroup>

### Poor Call Quality

**Symptoms**: Choppy audio, echo, delays, robotic voice.

<AccordionGroup>
  <Accordion title="Network quality">
    **Test Your Network**:
    - Run speed test: >5 Mbps upload/download
    - Check ping: &lt;100ms
    - Test from different network (WiFi vs cellular)

    **Fix**:
    - Move closer to WiFi router
    - Switch to wired connection
    - Test during off-peak hours (less congestion)
  </Accordion>

  <Accordion title="Cellular vs WiFi">
    **Test Both**:
    - Call on cellular network
    - Call on WiFi
    - Compare quality

    **Finding**: One much better than other? Use that network for future tests
  </Accordion>

  <Accordion title="Phone settings">
    **Check**:
    - Bluetooth disconnected (adds latency)
    - WiFi calling disabled (can cause issues)
    - VPN disabled (adds latency)
    - Call waiting disabled (no interruptions)

    **Fix**: Adjust phone settings, test again
  </Accordion>

  <Accordion title="Agent voice settings">
    **Try Adjusting**:
    - Voice speed (slower may be clearer)
    - Voice provider (ElevenLabs vs Azure vs Google)
    - Audio processing settings (echo cancellation, noise suppression)

    Navigate to agent → **Voice Settings** to adjust
  </Accordion>
</AccordionGroup>

### DTMF Not Detected

**Symptom**: Press phone keys but agent doesn't detect input.

<AccordionGroup>
  <Accordion title="DTMF enabled in agent">
    **Check**: Agent → **Advanced → DTMF Controls**

    **Verify**:
    - [ ] DTMF detection enabled
    - [ ] Detection mode set correctly (always on vs keyword-activated)
    - [ ] Key mappings configured

    **Fix**: Enable DTMF if disabled, save and re-test
  </Accordion>

  <Accordion title="DTMF type">
    **Two Types**:
    - **RFC 2833** (in-band): Carried with voice audio
    - **SIP INFO** (out-of-band): Separate SIP messages

    **Check Carrier**: Some carriers only support one type

    **Fix**: Try changing DTMF type in agent settings, or use different carrier/phone
  </Accordion>

  <Accordion title="Phone keypad">
    **Test**: Press keys during call - do you hear tones?

    **If No Tones**: Phone may be sending SIP INFO instead of tones (not compatible)

    **If Tones Heard**: Should work. Check agent logs for DTMF events. If not logged, may be carrier issue.
  </Accordion>
</AccordionGroup>

## Agent Behavior Issues

### Agent Not Following Instructions

**Symptom**: Agent does something different than what you specified in instructions.

<AccordionGroup>
  <Accordion title="Instructions too vague">
    **Example**:
    ```markdown
    ❌ Vague: "Be helpful and friendly."

    ✅ Specific: "Greet by saying: 'Welcome to Acme Corp. I'm here to help with orders and tracking. What can I do for you today?'"
    ```

    **Fix**: Make instructions more explicit and specific
  </Accordion>

  <Accordion title="Instructions conflicting">
    **Example**:
    ```markdown
    ❌ Conflict:
    "Be concise and brief."
    "Provide detailed explanations for every feature."

    ✅ Aligned:
    "Be concise unless customer asks for more details."
    ```

    **Fix**: Remove conflicting rules, prioritize one behavior
  </Accordion>

  <Accordion title="Model limitations">
    **Reality Check**:
    - Models don't follow instructions perfectly 100% of time
    - Complex instructions harder to follow than simple ones
    - Long instructions (>500 words) reduce compliance

    **Fix**:
    - Simplify instructions
    - Test with different model (GPT-4 vs Claude)
    - Add examples of desired behavior
  </Accordion>

  <Accordion title="Test in isolation">
    **Minimal Test**:
    1. Create new agent with minimal instructions
    2. Test single behavior in question
    3. If works → Issue is complexity/conflict in full instructions
    4. If doesn't work → May be model limitation or unrealistic expectation

    **Iterate**: Gradually add instructions back until issue reappears, then fix that specific instruction
  </Accordion>
</AccordionGroup>

### Transcription Errors

**Symptom**: Agent consistently mishears certain words or phrases.

<AccordionGroup>
  <Accordion title="Add custom vocabulary">
    **For**: Technical terms, product names, acronyms

    **Steps**:
    1. Go to agent → **Transcriber → Custom Vocabulary**
    2. Add misheard words with phonetic pronunciation
    3. Test again

    **Example**:
    ```json
    {
      "custom_words": [
        {"word": "itellicoAI", "pronunciation": "eye-TELL-ih-koh AI"},
        {"word": "Kubernetes", "pronunciation": "koo-ber-NET-eez"}
      ]
    }
    ```
  </Accordion>

  <Accordion title="Try different transcriber">
    **Options**:
    - **Deepgram**: Best for accents, noise, real-time
    - **Azure**: Best for technical terms, high accuracy
    - **Whisper**: Best for multilingual, offline

    **Test**: Switch transcriber in agent settings, run same test, compare accuracy
  </Accordion>

  <Accordion title="Improve audio quality">
    **Factors**:
    - Background noise → Use quiet environment
    - Poor mic → Use better mic/headset
    - Low volume → Speak louder or closer to mic
    - Mumbling → Speak clearly and slowly

    **Test**: Same test in ideal conditions (quiet room, good mic, clear speech)
  </Accordion>
</AccordionGroup>

### Actions Not Working

**Symptom**: Custom actions don't trigger or fail when triggered.

<AccordionGroup>
  <Accordion title="Action not triggering">
    **Check Instructions**: Do they mention when to use the action?

    **Example**:
    ```markdown
    ❌ Missing: No mention of action in instructions

    ✅ Explicit: "When customer asks about order status, use the getOrderStatus action with their order number."
    ```

    **Test**: Explicitly request action in test: "Check order status for order 12345"
  </Accordion>

  <Accordion title="Missing parameters">
    **Check**: Does action require parameters customer hasn't provided?

    **Example**: Action needs `order_id` but customer just said "check my order" without number

    **Fix in Instructions**:
    ```markdown
    "If customer asks about order but doesn't provide order number, ask: 'What's your order number?'"
    ```
  </Accordion>

  <Accordion title="API errors">
    **Check Logs**: Conversations → Logs → Actions tab

    **Common Errors**:
    - 401 Unauthorized → Check API key
    - 404 Not Found → Verify endpoint URL
    - Timeout → Increase timeout limit

    **Test API Manually**:
    ```bash
    curl -X POST https://api.example.com/endpoint \
         -H "Authorization: Bearer YOUR_KEY" \
         -d '{"param": "value"}'
    ```

    If manual call works, issue is in action configuration
  </Accordion>
</AccordionGroup>

## Platform Issues

### Dashboard Not Loading

**Symptoms**: Blank page, loading forever, errors.

<AccordionGroup>
  <Accordion title="Check platform status">
    Visit: `https://status.itellico.ai`

    **If Outage**: Wait for resolution, check status page for updates

    **If Operational**: Issue is likely local (browser, network, cache)
  </Accordion>

  <Accordion title="Clear browser cache">
    **Steps**:
    1. Open settings (Cmd+, or Ctrl+,)
    2. Privacy → Clear browsing data
    3. Select "Cached images and files"
    4. Clear data
    5. Reload dashboard

    **Force Reload**: Cmd+Shift+R or Ctrl+Shift+R
  </Accordion>

  <Accordion title="Try different browser">
    **Test in**:
    - Chrome (incognito mode)
    - Firefox (private window)
    - Safari (private browsing)

    **If works in incognito/private**: Issue is browser extension or cached data

    **Fix**: Disable extensions one by one to find conflict
  </Accordion>

  <Accordion title="Check browser console">
    **Steps**:
    1. F12 to open dev tools
    2. Console tab
    3. Look for errors

    **Common**:
    - CORS errors → Contact support
    - 401/403 errors → Session expired, logout and login again
    - JavaScript errors → Browser compatibility issue

    Screenshot error for support ticket
  </Accordion>
</AccordionGroup>

### Session Expired / Auth Issues

**Symptoms**: Logged out unexpectedly, "unauthorized" errors.

<AccordionGroup>
  <Accordion title="Re-login">
    **Simple Fix**: Logout and login again

    **If persists**: Clear cookies and cache, then login
  </Accordion>

  <Accordion title="API key issues">
    **If using API**: Check key is valid

    **Steps**:
    1. Go to **Accounts → API Keys**
    2. Verify key exists and not expired
    3. Regenerate if needed
    4. Update in your code/config

    **Security**: Never commit API keys to version control
  </Accordion>

  <Accordion title="Permissions">
    **Check**: Do you have correct role?

    **Roles**:
    - Admin: Full access
    - Developer: Can edit agents, can't manage billing
    - Viewer: Read-only

    **Fix**: Ask account owner to adjust your permissions
  </Accordion>
</AccordionGroup>

## Getting Help

When you can't resolve an issue:

<Steps>
  <Step title="Collect information">
    Gather before contacting support:
    - Agent ID
    - Conversation ID (if specific call)
    - Timestamp of issue
    - What you expected vs what happened
    - Steps to reproduce
    - Screenshots or screen recording
    - Browser/device info
  </Step>

  <Step title="Check documentation">
    Search docs for similar issues:
    - This troubleshooting guide
    - Agent configuration docs
    - API reference (for integration issues)
  </Step>

  <Step title="Community forum">
    Visit: `https://community.itellico.ai`

    - Search for similar issues
    - Post your question with details
    - Community and staff respond
  </Step>

  <Step title="Contact support">
    Email: `support@itellico.ai`

    Include:
    - Clear subject line (e.g., "Test call not connecting - Agent ID: agent_123")
    - Detailed description
    - Steps to reproduce
    - All information from Step 1
    - What you've already tried

    **Response Times**:
    - Critical (production down): &lt;1 hour
    - High (testing blocked): &lt;4 hours
    - Medium: &lt;24 hours
  </Step>
</Steps>
