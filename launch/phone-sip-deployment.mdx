## Deployment Overview

Phone deployment routes live PSTN calls through your tested agent. You can assign agents to:

- **Provisioned phone numbers** managed directly in the platform
- **SIP trunks** connecting your existing carrier infrastructure
- **IVR systems** for multi-agent routing scenarios

All configurations use the same LiveKit + FreeSWITCH telephony stack that Test mode uses, ensuring consistent behavior.

## Inbound Phone Deployment

### Step 1: Assign a Phone Number

<Steps>
  <Step title="Navigate to Telephony">
    Go to **Telephony → Numbers** in the dashboard. You'll see all provisioned numbers and their current assignments.
  </Step>

  <Step title="Select your number">
    Click on the phone number you want to assign. If you don't have a number yet, click **+ Purchase Number** to provision a new DID.
  </Step>

  <Step title="Assign the agent">
    In the number configuration:
    - **Inbound Agent**: Select your tested agent from the dropdown
    - **Business Hours**: Optionally restrict when the agent answers (default: 24/7)
    - **Fallback**: Configure what happens when agent is unavailable (voicemail, transfer, etc.)

    Click **Save** to activate the assignment.
  </Step>

  <Step title="Test the live number">
    Call the number from your phone. The agent should answer immediately using the greeting you configured in Test mode.

    Monitor the call in **Conversations → Live Monitor** to watch transcripts and actions in real-time.
  </Step>
</Steps>

### Step 2: Configure SIP Trunk (BYOC)

If you're bringing your own carrier, connect via SIP trunk:

  </div>
  <div className="hidden dark:block">
    <img src="/images/sip/create-sip-dark.png" alt="Create SIP Trunk dialog showing trunk name, inbound URI, and IP security configuration" />
  </div>
</Frame>

<Accordion title="SIP trunk configuration">
  1. Navigate to **Telephony → SIP Trunks** and click **+ Add Trunk**
  2. Enter your carrier's details:
     - **SIP Host**: Your carrier's SIP gateway (e.g., `sip.yourcarrier.com`)
     - **Username**: SIP authentication username
     - **Password**: SIP authentication password
     - **Port**: Usually 5060 (UDP) or 5061 (TLS)
  3. **Test Registration**: Click **Test Connection** to verify SIP registration succeeds
  4. **Assign Agent**: Select which agent handles calls from this trunk
  5. **Save** and monitor logs for SIP REGISTER success messages
</Accordion>

<Accordion title="Carrier compatibility">
  Tested carriers include:
  - Twilio Elastic SIP Trunking
  - Vonage (formerly Nexmo)
  - Bandwidth
  - Telnyx
  - Most standard SIP providers

  **Requirements**:
  - Support for Opus or PCMU/PCMA codecs
  - DTMF via RFC 2833 (for keypad input)
  - Reinvite support for transfers
</Accordion>

<Accordion title="Troubleshooting SIP registration">
  If registration fails:

  - **Check credentials**: Verify username/password with carrier
  - **Firewall rules**: Ensure UDP 5060 and RTP ports (10000-20000) are open
  - **IP allowlisting**: Some carriers require you to allowlist our signaling IPs (contact support)
  - **Logs**: Check `backend/telephony/logs` for SIP error codes (401, 403, etc.)

  Common error codes:
  - `401 Unauthorized`: Wrong username/password
  - `403 Forbidden`: IP not allowlisted with carrier
  - `503 Service Unavailable`: Carrier endpoint down
</Accordion>

### Step 3: IVR Integration

For multi-agent routing, integrate with your IVR:

<CodeGroup>
```xml IVR Transfer Example
<!-- Transfer to itellicoAI agent from Twilio IVR -->
<Response>
  <Dial>
    <Sip>sip:agent-123@your-tenant.itellico.ai</Sip>
  </Dial>
</Response>
```

```javascript API Transfer
// Programmatic transfer to agent via API
await itellicoClient.calls.transfer({
  call_id: "call_abc123",
  destination: "agent_uuid_xyz",
  transfer_type: "warm" // or "cold"
});
```
</CodeGroup>

SIP URI format: `sip:{agent_uuid}@{tenant}.itellico.ai`

## Outbound Phone Deployment

### API-Triggered Outbound Calls

Initiate calls programmatically via API:

<CodeGroup>
```bash cURL
curl -X POST https://api.itellico.ai/v1/calls \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "agent_123abc",
    "to_number": "+14155551234",
    "from_number": "+14155559999",
    "context_metadata": {
      "customer_name": "Jane Doe",
      "order_id": "ORD-456"
    }
  }'
```

```python Python SDK
from itellico import ItellicoClient

client = ItellicoClient(api_key="YOUR_API_KEY")

call = client.calls.create(
    agent_id="agent_123abc",
    to_number="+14155551234",
    from_number="+14155559999",
    context_metadata={
        "customer_name": "Jane Doe",
        "order_id": "ORD-456"
    }
)

print(f"Call initiated: {call.id}")
```

```typescript TypeScript SDK
import { ItellicoClient } from '@itellico/sdk';

const client = new ItellicoClient({ apiKey: 'YOUR_API_KEY' });

const call = await client.calls.create({
  agentId: 'agent_123abc',
  toNumber: '+14155551234',
  fromNumber: '+14155559999',
  contextMetadata: {
    customerName: 'Jane Doe',
    orderId: 'ORD-456'
  }
});

console.log(`Call initiated: ${call.id}`);
```
</CodeGroup>

<Note>
The `context_metadata` object is passed to your agent's dynamic context endpoint, allowing you to customize the conversation based on CRM data, order status, etc.
</Note>

## Deployment Configuration

### Concurrency & Scaling

Configure how many simultaneous calls your agent can handle:

<Steps>
  <Step title="Check account limits">
    Navigate to **Accounts → Usage** to see your concurrent call capacity. Plans range from 5 concurrent calls (Starter) to unlimited (Enterprise).
  </Step>

  <Step title="Set per-agent limits">
    In the agent settings, configure:
    - **Max Concurrent Calls**: Cap per-agent concurrency (useful for agents with expensive integrations)
    - **Queue Overflow**: Enable queueing when capacity is reached
    - **Overflow Behavior**: Route to voicemail, play busy signal, or transfer to human backup
  </Step>

  <Step title="Monitor capacity">
    Watch real-time concurrency in **Conversations → Live Monitor**. Set alerts when approaching 80% capacity.
  </Step>
</Steps>

### Business Hours & Availability

Restrict when your agent answers calls:

1. Go to agent settings → **Availability**
2. Configure timezone and business hours
3. Set **After-Hours Behavior**:
   - Play custom message and hangup
   - Send to voicemail
   - Transfer to emergency contact
   - Route to alternate agent

### Geographic Routing

For international deployments:

- Provision local numbers in target countries (**Telephony → Purchase Numbers**)
- Assign different agents per region for language/timezone optimization
- Configure SIP trunks with geo-redundant carriers for failover

## First Call Monitoring

<Warning>
**Critical**: Monitor your first 10-20 production calls closely. This catches configuration issues before they impact many customers.
</Warning>

### Real-Time Monitoring

1. Open **Conversations → Live Monitor** before routing live traffic
2. Watch for:
   - Agent greeting plays correctly
   - Transcription accuracy (check for misheard words)
   - Actions trigger at expected points
   - Call quality indicators (latency, packet loss)
3. Use **Whisper** or **Takeover** features if agent needs help

### Post-Call Review

After each early call:

1. Open the conversation log in **Conversations → Logs**
2. Review full transcript for:
   - Goal achievement
   - Unexpected agent responses
   - Failed action attempts
   - Customer satisfaction signals
3. Export and share with team for QA review

## Security Best Practices

<AccordionGroup>
  <Accordion title="Caller authentication">
    For sensitive operations (account lookups, payments):
    - Enable caller ID verification
    - Implement PIN/password challenges via DTMF
    - Use dynamic context to verify caller against CRM records
  </Accordion>

  <Accordion title="PCI compliance (if handling payments)">
    - Never log or record full credit card numbers
    - Use DTMF masking for payment input
    - Integrate with PCI-compliant payment gateways via custom actions
    - Disable transcript recording during payment flows
  </Accordion>

  <Accordion title="Call recording & consent">
    - Enable recording consent prompts if required by local laws (e.g., two-party consent states)
    - Configure recording settings in agent → **Abilities → Recording**
    - Ensure GDPR/CCPA compliance for stored recordings
  </Accordion>
</AccordionGroup>
