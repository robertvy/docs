## Web Widget Overview

The itellicoAI web widget adds voice calling to any website. Visitors can click a button and immediately talk to your AI agent—no phone required. The widget uses WebRTC for browser-based audio, connecting to the same LiveKit pipeline as phone calls.

**Key Features**:
- One-click voice conversations (no app download)
- Mobile and desktop browser support
- Customizable UI and positioning
- Works with same agent config as phone deployment
- Full transcript and analytics in dashboard

## Installation

### Step 1: Get Widget Code

<Steps>
  <Step title="Navigate to Web Widget settings">
    Go to **Integrations → Web Widget** in the dashboard.
  </Step>

  <Step title="Select your agent">
    Choose which agent the widget will connect to. You can create separate widgets for different pages/agents.
  </Step>

  <Step title="Copy installation code">
    Click **Generate Code** to get your unique snippet:

    ```html
    <!-- itellicoAI Widget -->
    <script src="https://widget.itellico.ai/v1/widget.js"></script>
    <script>
      ItellicoWidget.init({
        agentId: 'agent_abc123xyz',
        position: 'bottom-right',
        theme: 'light'
      });
    </script>
    ```
  </Step>

  <Step title="Add to your website">
    Paste the code snippet before the closing `</body>` tag on pages where you want the widget to appear.

    **Testing**: Load your page and verify the widget button appears in the configured position.
  </Step>
</Steps>

## Authentication & Security

### Public Widget (No Auth)

For general support or sales use cases where user identity doesn't matter:

- No authentication required
- Widget works immediately after installation
- User metadata is optional (can pass anonymous analytics IDs)
- Conversations are logged but not associated with user accounts

### Authenticated Widget (JWT)

For logged-in users accessing account-specific features:

<Steps>
  <Step title="Generate user tokens">
    Create a JWT on your backend signing endpoint:

    ```python Python Example
    import jwt
    import time

    def generate_widget_token(user_id, user_metadata):
        payload = {
            'userId': user_id,
            'userMetadata': user_metadata,
            'exp': int(time.time()) + 3600  # 1 hour expiry
        }

        token = jwt.encode(
            payload,
            'YOUR_WIDGET_SECRET',  # From dashboard
            algorithm='HS256'
        )

        return token
    ```

    ```javascript Node.js Example
    const jwt = require('jsonwebtoken');

    function generateWidgetToken(userId, userMetadata) {
      const payload = {
        userId: userId,
        userMetadata: userMetadata,
        exp: Math.floor(Date.now() / 1000) + 3600  // 1 hour
      };

      return jwt.sign(payload, process.env.WIDGET_SECRET, {
        algorithm: 'HS256'
      });
    }
    ```
  </Step>

  <Step title="Pass token to widget">
    On your frontend (after user logs in):

    ```javascript
    // Fetch token from your backend
    const response = await fetch('/api/widget-token');
    const { token, userId } = await response.json();

    // Initialize widget with auth
    ItellicoWidget.init({
      agentId: 'agent_abc123xyz',
      userId: userId,
      userToken: token,
      userMetadata: {
        name: currentUser.name,
        email: currentUser.email,
        plan: currentUser.plan
      }
    });
    ```
  </Step>

  <Step title="Access user data in agent">
    The agent receives user metadata via dynamic context:

    ```json Example Context Payload
    {
      "user_id": "user_12345",
      "user_metadata": {
        "name": "Jane Doe",
        "email": "jane@example.com",
        "plan": "premium",
        "accountId": "acct_789"
      },
      "call_source": "web_widget"
    }
    ```

    Reference in agent instructions: `"You're speaking with {{user_metadata.name}}, a {{user_metadata.plan}} customer..."`
  </Step>
</Steps>

<Warning>
**Security**: Never expose your `WIDGET_SECRET` client-side. Always generate tokens server-side.
</Warning>

### Mobile Responsiveness

The widget automatically adapts to mobile screens:

- **Desktop**: Fixed position button + expandable panel
- **Mobile**: Full-screen takeover when opened
- **Tablet**: Adaptive sizing based on viewport

**Manual Overrides**:
```javascript
ItellicoWidget.init({
  agentId: 'agent_abc123xyz',
  mobileBreakpoint: 768,        // Mobile mode threshold (px)
  mobileFullscreen: true,       // Force fullscreen on mobile
  hideOnMobile: false,          // Hide widget on small screens
  mobilePosition: 'bottom'      // bottom, top (mobile only)
});
```

## Traffic & Concurrency Controls

### Rate Limiting

Prevent abuse with built-in rate limits:

<Accordion title="Per-visitor limits">
  Configure in **Integrations → Web Widget → Rate Limits**:
  - **Max calls per hour**: Limit per IP address (default: 5)
  - **Max concurrent calls per user**: Prevent tab spam (default: 1)
  - **Cooldown period**: Seconds between calls (default: 30)
</Accordion>

<Accordion title="Global limits">
  Set platform-wide widget limits:
  - **Max concurrent widget calls**: Cap across all visitors (default: account plan limit)
  - **Queue overflow**: Enable queue or show "Agents busy" message
  - **Business hours**: Only allow calls during specific times
</Accordion>

### Capacity Management

When approaching concurrent call limits:

1. **Queue Mode**: Visitors wait in line, widget shows estimated wait time
2. **Busy Mode**: Widget shows "All agents busy" with callback form
3. **Fallback Mode**: Switch to text chat or email form when capacity reached

Configure in **Integrations → Web Widget → Overflow Behavior**.

## Testing Widget in Staging

Before production deployment:

<Steps>
  <Step title="Create staging widget">
    Generate a separate widget code with `stagingMode: true`:

    ```javascript
    ItellicoWidget.init({
      agentId: 'agent_abc123xyz_staging',
      stagingMode: true,  // Uses staging backend
      debugMode: true     // Verbose console logging
    });
    ```
  </Step>

  <Step title="Test on staging site">
    Deploy to your staging/dev environment first:
    - Test on multiple browsers (Chrome, Safari, Firefox)
    - Test on mobile devices (iOS Safari, Android Chrome)
    - Verify microphone permissions flow
    - Check WebRTC connectivity through corporate firewalls
  </Step>

  <Step title="Validate conversation flow">
    Run full test scenarios:
    - Start call, verify greeting plays
    - Test interruptions and barge-in
    - Trigger actions (transfers, API calls)
    - End call gracefully, check transcript in dashboard
  </Step>

  <Step title="Review logs">
    Check **Conversations → Logs** for staging calls:
    - Verify user metadata passed correctly (if using auth)
    - Confirm goal tracking works
    - Check for WebRTC errors or quality issues
  </Step>
</Steps>

## Production Deployment

### Deployment Checklist

- <Icon icon="square" /> Widget tested on staging with multiple browsers/devices
- <Icon icon="square" /> Authentication flow validated (if using JWT)
- <Icon icon="square" /> Rate limits configured to prevent abuse
- <Icon icon="square" /> Overflow behavior set (queue vs busy message)
- <Icon icon="square" /> Mobile responsiveness tested on iOS and Android
- <Icon icon="square" /> Firewall/VPN compatibility verified (WebRTC ports)
- <Icon icon="square" /> Analytics tracking confirmed in dashboard
- <Icon icon="square" /> Fallback content for browsers without WebRTC support

### Gradual Rollout

For high-traffic sites, roll out incrementally:

1. **Soft Launch**: Deploy to one low-traffic page (e.g., /pricing)
2. **Monitor**: Watch for errors, call quality issues, capacity problems
3. **Expand**: Add to more pages if stable
4. **A/B Test**: Compare conversion with/without widget
5. **Full Deploy**: Add to all relevant pages once validated

### CDN & Performance

The widget script (~50KB gzipped) loads from our global CDN:
- Cached at edge for <50ms load times
- Async loading doesn't block page render
- Fallback to primary if CDN unavailable

**Optimize Loading**:
```html
<!-- Async load for faster page speed -->
<script async src="https://widget.itellico.ai/v1/widget.js"></script>
<script>
  window.addEventListener('ItellicoWidgetLoaded', () => {
    ItellicoWidget.init({ agentId: 'agent_abc123xyz' });
  });
</script>
```

## Analytics & Observability

Widget calls appear in **Conversations → Logs** with:
- Source: `web_widget`
- User metadata (if provided)
- Full transcript and audio recording
- Goal outcomes and action logs
- Session duration and quality metrics

**Custom Events**:
```javascript
ItellicoWidget.on('callStarted', (data) => {
  // Track in your analytics (Google Analytics, Mixpanel, etc.)
  gtag('event', 'call_started', {
    agent_id: data.agentId,
    user_id: data.userId
  });
});

ItellicoWidget.on('callEnded', (data) => {
  gtag('event', 'call_ended', {
    duration: data.durationSeconds,
    goals_achieved: data.goalsAchieved
  });
});

ItellicoWidget.on('error', (error) => {
  console.error('Widget error:', error);
  // Send to error tracking (Sentry, Rollbar, etc.)
});
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Widget not appearing">
    - Check browser console for JavaScript errors
    - Verify `agentId` is correct
    - Ensure script loaded before `init()` called
    - Check if CSP (Content Security Policy) blocks widget domain
  </Accordion>

  <Accordion title="Microphone access denied">
    - Browsers require HTTPS for microphone access (except localhost)
    - User must explicitly grant permission (can't be bypassed)
    - Some browsers block mic on first visit (security)
    - Check browser mic settings: chrome://settings/content/microphone
  </Accordion>

  <Accordion title="No audio / poor quality">
    - WebRTC requires UDP ports 10000-60000 open (firewall issue)
    - Corporate VPNs often block WebRTC (try TURN relay)
    - Enable TURN servers in widget config for restrictive networks
    - Check browser WebRTC stats: chrome://webrtc-internals
  </Accordion>

  <Accordion title="Widget blocked by ad blockers">
    - Some ad blockers flag widget domains
    - Workaround: Self-host widget script (contact support)
    - Educate users to allowlist your site
  </Accordion>
</AccordionGroup>
