## Agent-Name

Der Agent-Name hilft dir, diesen Agent in deinem Dashboard, Kampagnen und Berichten zu identifizieren.

<Warning>
Der Agent-Name ist für Besucher in Demo-Links sichtbar. Verwende für kundenseitige Namen einen professionellen Namen wie "Acme Support" oder "Sarah von Acme Sales" anstelle interner Labels wie "Lead Qualifier v2".
</Warning>

### Best Practices

**Für kundenseitige Agents (sichtbar in Demos):**
- "Acme Support"
- "Alex von Acme Sales"
- "Buchungs-Assistent"

**Für interne Organisation (mehrere Agents):**
- "Kundensupport - Produktfragen"
- "Ausgehender Vertrieb - Lead-Qualifizierung"
- "Terminbuchung - Zahnklinik"

**Vermeiden:**
- "Agent 1"
- "Test"
- "Neuer Agent"

**Verwende konsistente Benennung**
Entwickle eine Namenskonvention:
- `[Funktion] - [Spezialität]`
- `[Abteilung] - [Agent-Typ]`
- `[Anwendungsfall] - [Sprache]`

<Note>
Der Avatar ist sichtbar, wenn du Demo-Links teilst. Wähle daher ein Bild, das deine Marke professionell repräsentiert.
</Note>

### Avatar-Richtlinien

**Bildspezifikationen**
- **Format:** PNG, JPG oder SVG
- **Größe:** Empfohlen mindestens 200x200px
- **Seitenverhältnis:** Quadratisch (1:1)
- **Dateigröße:** Unter 2MB

### Tagging-Strategien

**Nach Funktion**
```text wrap
support, sales, booking, billing, technical
```

**Nach Sprache**
```text wrap
deutsch, spanisch, französisch, englisch, mehrsprachig
```

**Nach Abteilung**
```text wrap
kundenservice, vertriebsteam, operationen, finanzen
```

**Nach Status**
```text wrap
produktion, testing, archiviert, beta
```

**Nach Region**
```text wrap
de, eu, us, latam
```

### Tags effektiv nutzen

<Steps>
  <Step title="Tagging-System erstellen">
    Definiere Kategorien und Konventionen, bevor du viele Agents erstellst
  </Step>
  <Step title="Konsistent anwenden">
    Verwende dieselben Tags für ähnliche Agents
  </Step>
  <Step title="Filtern und suchen">
    Verwende Tags, um schnell bestimmte Agent-Gruppen im Dashboard zu finden
  </Step>
  <Step title="Regelmäßig überprüfen">
    Räume ungenutzte Tags auf und aktualisiere sie nach Bedarf
  </Step>
</Steps>

## Interne Notizen

Verwende den Notizenbereich, um Konfigurationsdetails, Versionsverlauf und Teaminformationen über diesen Agent zu dokumentieren.

### Nützliche Notiztypen

**Konfigurationsverlauf**
```text wrap
2025-01-15: Auf GPT-4.1 Mini für schnellere Antworten umgestellt
2025-01-10: Cal.com-Integration für Buchung hinzugefügt
2025-01-05: Wissensdatenbank mit neuen Produktinformationen aktualisiert
```

**Bekannte Probleme**
```text wrap
- Hat gelegentlich Schwierigkeiten mit starken Akzenten (VAD-Einstellungen anpassen)
- Weiterleitung zum Vertrieb bricht manchmal freitags ab (wird untersucht)
```

**Team-Verantwortlichkeiten**
```text wrap
Verantwortlicher: Sarah (Produktteam)
Backup: Mike (Support-Leitung)
Reviews: Monatlich am ersten Montag
```

**Leistungsnotizen**
```text wrap
Durchschnittliche Anrufdauer: 3m 45s
Lösungsrate: 87%
Kundenzufriedenheit: 4.6/5
Zuletzt optimiert: 2025-01-10
```

## Organisations-Best-Practices

### Für kleine Teams (1-5 Agents)

- Verwende einfache, beschreibende Namen
- Minimales Tagging erforderlich
- Grundlegende Notizen für wichtige Änderungen

### Für mittlere Teams (5-20 Agents)

- Entwickle Namenskonventionen
- Verwende Tags für Funktion und Status
- Pflege regelmäßige Notizen zu Änderungen

### Für große Teams (20+ Agents)

- Strenge Namenskonventionen erforderlich
- Mehrstufiges Tagging (Funktion, Region, Sprache, Status)
- Detaillierte Change-Logs in Notizen
- Erwäge die Erstellung von Agent-Dokumentation außerhalb der Plattform
