## Übersicht

Die Identität Ihres Agents umfasst die grundlegenden Informationen, die Ihnen und Ihrem Team helfen, Agents in Ihrem Dashboard zu identifizieren, zu organisieren und zu verwalten. Einige Details wie Name und Avatar sind auch in Demo-Links sichtbar.

## Agent-Name

Der Agent-Name hilft Ihnen, diesen Agent in Ihrem Dashboard, Kampagnen und Berichten zu identifizieren.

<Warning>
Der Agent-Name ist für Besucher in Demo-Links sichtbar. Verwenden Sie für kundenseitige Namen einen professionellen Namen wie "Acme Support" oder "Sarah von Acme Sales" anstelle interner Labels wie "Lead Qualifier v2".
</Warning>

### Best Practices

**Für kundenseitige Agents (sichtbar in Demos):**
- "Acme Support"
- "Alex von Acme Sales"
- "Buchungs-Assistent"

**Für interne Organisation (mehrere Agents):**
- "Kundensupport - Produktfragen"
- "Ausgehender Vertrieb - Lead-Qualifizierung"
- "Terminbuchung - Zahnklinik"

**Vermeiden:**
- "Agent 1"
- "Test"
- "Neuer Agent"

**Verwenden Sie konsistente Benennung**
Entwickeln Sie eine Namenskonvention:
- `[Funktion] - [Spezialität]`
- `[Abteilung] - [Agent-Typ]`
- `[Anwendungsfall] - [Sprache]`

## Avatar

Laden Sie ein Avatar-Bild hoch, um Ihren Agent im Dashboard und in Demo-Links, die Sie mit anderen teilen, visuell zu identifizieren.

<Note>
Der Avatar ist sichtbar, wenn Sie Demo-Links teilen. Wählen Sie daher ein Bild, das Ihre Marke professionell repräsentiert.
</Note>

### Avatar-Richtlinien

**Bildspezifikationen**
- **Format:** PNG, JPG oder SVG
- **Größe:** Empfohlen mindestens 200x200px
- **Seitenverhältnis:** Quadratisch (1:1)
- **Dateigröße:** Unter 2MB

## Tags

Tags helfen, Agents in Ihrem Dashboard zu organisieren und zu filtern, besonders nützlich bei der Verwaltung vieler Agents.

### Tagging-Strategien

**Nach Funktion**
```
support, sales, booking, billing, technical
```

**Nach Sprache**
```
deutsch, spanisch, französisch, englisch, mehrsprachig
```

**Nach Abteilung**
```
kundenservice, vertriebsteam, operationen, finanzen
```

**Nach Status**
```
produktion, testing, archiviert, beta
```

**Nach Region**
```
de, eu, us, latam
```

### Tags effektiv nutzen

<Steps>
  <Step title="Tagging-System erstellen">
    Definieren Sie Kategorien und Konventionen, bevor Sie viele Agents erstellen
  </Step>
  <Step title="Konsistent anwenden">
    Verwenden Sie dieselben Tags für ähnliche Agents
  </Step>
  <Step title="Filtern und suchen">
    Verwenden Sie Tags, um schnell bestimmte Agent-Gruppen im Dashboard zu finden
  </Step>
  <Step title="Regelmäßig überprüfen">
    Räumen Sie ungenutzte Tags auf und aktualisieren Sie sie nach Bedarf
  </Step>
</Steps>

## Interne Notizen

Verwenden Sie den Notizenbereich, um Konfigurationsdetails, Versionsverlauf und Teaminformationen über diesen Agent zu dokumentieren.

### Nützliche Notiztypen

**Konfigurationsverlauf**
```
2025-01-15: Auf GPT-4.1 Mini für schnellere Antworten umgestellt
2025-01-10: Cal.com-Integration für Buchung hinzugefügt
2025-01-05: Wissensdatenbank mit neuen Produktinformationen aktualisiert
```

**Bekannte Probleme**
```
- Hat gelegentlich Schwierigkeiten mit starken Akzenten (VAD-Einstellungen anpassen)
- Weiterleitung zum Vertrieb bricht manchmal freitags ab (wird untersucht)
```

**Team-Verantwortlichkeiten**
```
Verantwortlicher: Sarah (Produktteam)
Backup: Mike (Support-Leitung)
Reviews: Monatlich am ersten Montag
```

**Leistungsnotizen**
```
Durchschnittliche Anrufdauer: 3m 45s
Lösungsrate: 87%
Kundenzufriedenheit: 4.6/5
Zuletzt optimiert: 2025-01-10
```

## Organisations-Best-Practices

### Für kleine Teams (1-5 Agents)

- Verwenden Sie einfache, beschreibende Namen
- Minimales Tagging erforderlich
- Grundlegende Notizen für wichtige Änderungen

### Für mittlere Teams (5-20 Agents)

- Entwickeln Sie Namenskonventionen
- Verwenden Sie Tags für Funktion und Status
- Pflegen Sie regelmäßige Notizen zu Änderungen

### Für große Teams (20+ Agents)

- Strenge Namenskonventionen erforderlich
- Mehrstufiges Tagging (Funktion, Region, Sprache, Status)
- Detaillierte Change-Logs in Notizen
- Erwägen Sie die Erstellung von Agent-Dokumentation außerhalb der Plattform
