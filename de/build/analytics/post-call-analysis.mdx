## Übersicht

Post-Call-Analyse verwendet KI, um strukturierte Informationen aus Gesprächstranskripten zu extrahieren. Definieren Sie, was die KI analysieren soll (Qualitätsscores, Sentiment, Kategorien, extrahierte Daten) und erhalten Sie konsistente, umsetzbare Einblicke aus jedem Anruf.

**Zugriff:** Agenteneinstellungen → **Anrufanalyse**

<Warning>
**Analyse vs. Ziele:** Analyse extrahiert Informationen und bewertet Qualität (z.B. "Wie zufrieden war der Kunde?"). Ziele verfolgen Conversions (z.B. "Wurde ein Termin gebucht?"). Verwenden Sie beides zusammen für vollständige Berichterstattung.
</Warning>

---

## Analysesprache

Legen Sie die Sprache für die KI-Analysebewertung fest:

**Zugriff:** Agenteneinstellungen → **Analytik → Sprache**

Wählen Sie die Sprache, in der Ziele und Analysen bewertet werden. Dies bestimmt, welche Sprache die KI verwendet, wenn sie Transkripte analysiert und Analysefragen beantwortet.

**Beispiel:** Wenn Sie die Analysesprache auf Spanisch setzen, bewertet die KI Gespräche auf Spanisch, auch wenn Ihr Agent mit Kunden auf Englisch spricht.

<Tip>
Diese Einstellung gilt sowohl für Anrufanalyse als auch für Zielbewertung.
</Tip>

---

## Standard- vs. benutzerdefinierte Analyse

**Standardanalysen** - Von itellicoAI vorkonfiguriert, immer verfügbar:
- Können nicht bearbeitet oder archiviert werden
- Mit dem Aktiv-Schalter ein-/ausschalten
- Für gängige Anwendungsfälle optimiert

**Benutzerdefinierte Analysen** - Erstellen Sie Ihre eigenen:
- Vollständige Kontrolle über Name, Beschreibung und Typ
- Können bearbeitet, archiviert und wiederhergestellt werden
- Auf Ihre spezifischen Geschäftsanforderungen zugeschnitten

<Info>
Standardanalysen bieten einen Ausgangspunkt. Erstellen Sie benutzerdefinierte Analysen für Ihre individuellen Anforderungen.
</Info>

## Einrichtung

<Steps>
  <Step title="Zugriff auf Anrufanalyse">
    Agenteneinstellungen → **Anrufanalyse** → **Analyse hinzufügen**
  </Step>

  <Step title="Frage konfigurieren">
    **Name:** Interner Bezeichner

    ```
    Beispiel: "Kundenzufriedenheitsscore"
    ```

    **Beschreibung:** Die Frage, die die KI beantworten wird (seien Sie so spezifisch wie möglich)

    ```
    Beispiel: "Bewerten Sie auf einer Skala von 1-5 Sternen die allgemeine
    Zufriedenheit des Kunden basierend auf seinem Ton, seiner Sprache
    und ob sein Problem während des Anrufs vollständig gelöst wurde."
    ```

    <Tip>
    Spezifischere Beschreibungen = genauere KI-Analyse
    </Tip>
  </Step>

  <Step title="Fragetyp wählen">
    **Ja/Nein** - Binäre Antwort
    ```
    Beispiel: "Wurde das Problem des Kunden vollständig gelöst?"
    ```

    **Offen** - Textantwort
    ```
    Beispiel: "Was war das Hauptthema der Diskussion?"
    ```

    **Bewertung** - 1-5 Sterne
    ```
    Beispiel: "Wie professionell war der Agent?"
    ```
  </Step>

  <Step title="Speichern">
    Klicken Sie auf **Analyse hinzufügen** zum Speichern
  </Step>
</Steps>

## Fragetypen

<Tabs>
  <Tab title="Ja/Nein">
    **Binäre Antwortfragen** für klare Metriken.

    **Am besten für:**
    - Compliance-Prüfungen
    - Qualitätsprüfung
    - Erkennung der Funktionsnutzung

    **Beispiele:**
    - ✅ "Wurden Zahlungsinformationen erfasst?"
    - ✅ "Hat der Agent dem Skript gefolgt?"
    - ✅ "Wurde der Anrufer weitergeleitet?"
    - ✅ "Hat der Kunde dem Angebot zugestimmt?"
  </Tab>

  <Tab title="Offen">
    **Textantwortfragen** für detaillierte Einblicke.

    **Am besten für:**
    - Problemkategorisierung
    - Informationsextraktion
    - Ursachenanalyse
    - Gesprächszusammenfassungen

    **Beispiele:**
    - 📝 "Was war das Hauptthema der Diskussion?"
    - 📝 "Welche Einwände hat der Kunde erhoben?"
    - 📝 "Welches Produkt wurde besprochen?"
    - 📝 "Fassen Sie das technische Problem des Kunden zusammen"
  </Tab>

  <Tab title="Bewertung">
    **1-5 Sterneskala** für qualitative Bewertung.

    **Standardskala:**
    - **5** = Außergewöhnliche Leistung
    - **4** = Überdurchschnittliche Leistung
    - **3** = Durchschnittliche/zufriedenstellende Leistung
    - **2** = Unterdurchschnittliche Leistung
    - **1** = Minimale/schlechte Leistung

    **Am besten für:**
    - Qualitätsbewertung
    - Sentiment-Analyse
    - Leistungsbewertung
    - Zufriedenheitsmessung

    **Beispiele:**
    - ⭐ "Bewerten Sie das Kunden-Sentiment (1=sehr negativ, 5=sehr positiv)"
    - ⭐ "Wie gut hat der Agent mit Einwänden umgegangen?"
    - ⭐ "Bewerten Sie die Anrufqualität (1=schlecht, 5=ausgezeichnet)"
    - ⭐ "Kundenzufriedenheitsscore"

    <Tip>
    Definieren Sie immer, was jedes Bewertungsniveau in Ihrer Beschreibung bedeutet, für genauere Analyse
    </Tip>
  </Tab>
</Tabs>

## Beispiele

<AccordionGroup>
  <Accordion title="Kundenzufriedenheit" icon="face-smile">
    **Name:** Kundenzufriedenheitsscore

    **Typ:** Bewertung (1-5 Sterne)

    **Beschreibung:**
    ```
    Bewerten Sie die Gesamtzufriedenheit des Kunden basierend auf
    seinem Ton, seiner Sprache und seinen Antworten (1-5 Sterne):

    5 Sterne = Sehr zufrieden, enthusiastisch, alle Bedürfnisse erfüllt
    4 Sterne = Zufrieden, positive Interaktion
    3 Sterne = Neutral, Grundbedürfnisse erfüllt
    2 Sterne = Etwas unzufrieden, etwas Frustration
    1 Stern = Sehr unzufrieden, wütend, ungelöste Probleme
    ```
  </Accordion>

  <Accordion title="Problemlösung" icon="check-circle">
    **Name:** Problem vollständig gelöst

    **Typ:** Ja/Nein

    **Beschreibung:**
    ```
    Wurde das Problem, die Frage oder die Anfrage des Kunden bis
    zum Ende des Anrufs vollständig gelöst? Antworten Sie mit Ja nur,
    wenn der Kunde die Zufriedenheit bestätigt oder ausdrücklich
    zugestimmt hat, dass das Problem gelöst wurde.
    ```
  </Accordion>

  <Accordion title="Anrufthemen-Klassifizierung" icon="tag">
    **Name:** Primäres Anrufthema

    **Typ:** Offen

    **Beschreibung:**
    ```
    Identifizieren Sie das Hauptthema oder den Grund für diesen Anruf. Wählen Sie aus:
    - Rechnungsanfrage
    - Technischer Support
    - Produktinformationen
    - Bestellstatus
    - Beschwerde
    - Kündigungsanfrage
    - Allgemeine Frage

    Geben Sie nur den Kategorienamen zurück. Wenn mehrere Themen
    besprochen wurden, geben Sie das zurück, das die meiste Gesprächszeit
    eingenommen hat.
    ```
  </Accordion>

  <Accordion title="Agenten-Performance" icon="user-gear">
    **Name:** Agenten-Professionalität

    **Typ:** Bewertung (1-5 Sterne)

    **Beschreibung:**
    ```
    Bewerten Sie die Professionalität des Agenten während des Anrufs (1-5 Sterne):

    5 = Außergewöhnlich professionell, höflich und hilfsbereit
    4 = Sehr professionell mit geringem Verbesserungspotenzial
    3 = Professionell und angemessen
    2 = Etwas unprofessionell (unterbrechend, ungeduldig)
    1 = Unprofessionell (unhöflich, abweisend, unangemessen)
    ```
  </Accordion>

  <Accordion title="Sentiment-Analyse" icon="chart-line">
    **Name:** Kunden-Sentiment

    **Typ:** Bewertung (1-5 Sterne)

    **Beschreibung:**
    ```
    Bewerten Sie den emotionalen Zustand des Kunden während des Anrufs:

    5 = Sehr positiv (glücklich, begeistert, dankbar, enthusiastisch)
    4 = Positiv (zufrieden, freundlich, erfreut)
    3 = Neutral (geschäftsmäßig, sachlich, ruhig)
    2 = Negativ (frustriert, enttäuscht, genervt)
    1 = Sehr negativ (wütend, feindselig, drohend)
    ```
  </Accordion>

  <Accordion title="Compliance-Überprüfung" icon="shield-check">
    **Name:** Ordnungsgemäße Identifikation

    **Typ:** Ja/Nein

    **Beschreibung:**
    ```
    Hat sich der Agent zu Beginn des Anrufs ordnungsgemäß mit Namen
    und Unternehmen identifiziert, wie es die Unternehmensrichtlinie
    vorschreibt?
    ```
  </Accordion>
</AccordionGroup>

## Analyse vs. Ziele

Verwenden Sie beide zusammen für vollständige Gesprächsverfolgung:

| Funktion | Ziele | Analyse |
|---------|-------|----------|
| **Zweck** | Conversions verfolgen | Einblicke extrahieren & Qualität beurteilen |
| **Beispiel** | "Termin gebucht" | "Wie zufrieden war der Kunde?" |
| **Antworttyp** | Erreicht/Nicht erreicht | Ja/Nein, Text oder 1-5 Bewertung |
| **Verwenden für** | Erfolgsmetriken | Qualitätsbewertung, Kategorisierung |
| **Am besten für** | Geschäftsergebnisse | Detaillierte Berichterstattung |

**Beispielkombination:**
- **Ziel:** "Termin gebucht" (Haben wir die Conversion erreicht?)
- **Analyse:** "Kundenzufriedenheit" (Wie gut haben wir es gemacht?)
- **Analyse:** "Primäres Anrufthema" (Was wollten sie?)

## Best Practices

<AccordionGroup>
  <Accordion title="Spezifische Fragen schreiben" icon="pen">
    Die KI verwendet Ihre Beschreibung zur Beantwortung. Seien Sie detailliert.

    ❌ Vage: "War der Anruf gut?"

    ✅ Spezifisch: "Hat der Agent genaue Informationen bereitgestellt, alle Kundenfragen beantwortet und während des gesamten Anrufs einen professionellen Ton beibehalten?"
  </Accordion>

  <Accordion title="Bewertungsskalen definieren" icon="star">
    Definieren Sie bei Bewertungsfragen immer, was jede Stufe bedeutet.

    ```
    Bewerten Sie die Agenten-Professionalität (1-5):
    5 = Außergewöhnlich professionell
    4 = Sehr professionell
    3 = Professionell
    2 = Etwas unprofessionell
    1 = Unprofessionell
    ```
  </Accordion>

  <Accordion title="Mit Schlüsselfragen beginnen" icon="list">
    Erstellen Sie nicht zu viele Analysen auf einmal. Beginnen Sie mit 3-5 kritischen Fragen:

    1. Problem gelöst (Ja/Nein)
    2. Kundenzufriedenheit (Bewertung)
    3. Anrufthema (Offen)
    4. Sentiment (Bewertung)

    Fügen Sie bei Bedarf weitere hinzu.
  </Accordion>

  <Accordion title="Alte Analysen archivieren" icon="folder">
    Archivieren Sie veraltete Analysefragen, um sie aus dem Weg zu räumen und dabei historische Daten zu bewahren.

    **Zum Archivieren:** Klicken Sie auf das Archivierungssymbol neben der Analyse

    **Zum Wiederherstellen:** Klicken Sie auf "Archivierte anzeigen" → Wiederherstellen
  </Accordion>
</AccordionGroup>

## Tipps

- **Mit Zielen kombinieren** - Verwenden Sie Ziele für Conversions, Analyse für Qualitätseinblicke
- **Testen und verfeinern** - Überprüfen Sie KI-Antworten bei einigen Anrufen und passen Sie Beschreibungen bei Bedarf an
- **Konsistent sein** - Verwenden Sie die gleichen Analysefragen über ähnliche Agenten hinweg für vergleichbare Daten
- **Nach Bedarf aktivieren/deaktivieren** - Schalten Sie Analysefragen mit dem Aktiv-Schalter ein/aus

## Fehlerbehebung

<AccordionGroup>
  <Accordion title="Ungenaue Antworten" icon="triangle-exclamation">
    **Ursachen:**
    - Fragebeschreibung zu vage
    - Mehrere Interpretationen möglich
    - Fehlender Kontext in der Beschreibung

    **Lösungen:**
    - Machen Sie die Beschreibung spezifischer und detaillierter
    - Geben Sie Beispiele in der Beschreibung an
    - Definieren Sie bei Bewertungen jede Stufe explizit
    - Testen Sie mit Beispielanrufen und verfeinern Sie
  </Accordion>

  <Accordion title="Inkonsistente Bewertungen" icon="arrows-left-right">
    **Ursachen:**
    - Bewertungskriterien nicht klar definiert
    - Subjektive Interpretation durch KI

    **Lösungen:**
    - Definieren Sie genaue Kriterien für jede Bewertungsstufe
    - Geben Sie spezifische Beispiele für jede Sternebewertung an
    - Verwenden Sie Ja/Nein-Fragen für objektivere Antworten
  </Accordion>

  <Accordion title="Analyse wird nicht angezeigt" icon="eye-slash">
    **Ursachen:**
    - Analysefrage deaktiviert
    - Nach Anrufen erstellt
    - Transkription nicht aktiviert

    **Lösungen:**
    - Überprüfen Sie, ob die Analyse Aktiv ist (Schalter einschalten)
    - Analyse läuft nur bei neuen Anrufen nach der Erstellung
    - Stellen Sie sicher, dass die Transkription für den Agenten aktiviert ist
  </Accordion>
</AccordionGroup>
