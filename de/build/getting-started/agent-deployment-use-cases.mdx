## Wie Agents funktionieren

Ein itellicoAI-Agent ist eine einzelne AI-Konfiguration, die gleichzeitig über mehrere Kanäle bereitgestellt werden kann. Derselbe Agent kann:
- Eingehende Telefonanrufe bearbeiten
- Ausgehende Anrufe in Kampagnen tätigen
- Als Widget auf Ihrer Website eingebettet sein
- Unterschiedliche Begrüßungen für eingehende vs. ausgehende Szenarien verwenden

Sie müssen nicht unbedingt separate "eingehende" oder "ausgehende" Agents erstellen - derselbe Agent kann über mehrere Kanäle bereitgestellt werden.

## Telefonanruf-Bereitstellung

Stellen Sie Ihren Agent bereit, um Telefongespräche zu führen.

### Eingehende Anrufe

<Steps>
  <Step title="Telefonnummer verbinden">
    Verbinden Sie eine Telefonnummer über SIP-Trunk mit Ihrem Agent
  </Step>
  <Step title="Begrüßung konfigurieren">
    Richten Sie Ihre eingehende Begrüßung für eingehende Anrufe ein
  </Step>
  <Step title="Live schalten">
    Ihr Agent ist bereit zu antworten, wenn Kunden anrufen
  </Step>
</Steps>

**Am besten für:**
- Kundensupport-Hotlines
- Bestellstatus-Anfragen
- Technische Support-Leitungen
- Allgemeine Informationsleitungen
- Terminplanung

**Beispiel-Begrüßung:**
```
Vielen Dank, dass Sie Acme Support anrufen. Wie kann ich Ihnen heute helfen?
```

### Ausgehende Anrufe (Kampagnen)

<Steps>
  <Step title="Kampagne erstellen">
    Erstellen Sie eine neue Kampagne und weisen Sie Ihren Agent zu
  </Step>
  <Step title="Kontakte hinzufügen">
    Laden Sie Ihre Kontaktliste hoch und fügen Sie Kontakte zur Kampagne hinzu
  </Step>
  <Step title="Ausgehende Begrüßung konfigurieren">
    Richten Sie die Überschreibung der ausgehenden Begrüßung ein
  </Step>
  <Step title="Kampagne starten">
    Starten Sie Ihre Kampagne und überwachen Sie die Ergebnisse
  </Step>
</Steps>

**Am besten für:**
- Lead-Qualifizierung
- Termin-Erinnerungen
- Kundenfeedback-Umfragen
- Zahlungs-Erinnerungen
- Follow-up-Anrufe

**Beispiel-Begrüßung:**
```
Hallo, hier ist Alex von Acme Software. Sie haben kürzlich Informationen über unseren Enterprise-Plan angefordert. Haben Sie einen Moment Zeit zum Chatten?
```

<Note>
Derselbe Agent kann sowohl eingehende als auch ausgehende Anrufe bearbeiten. Verwenden Sie die Funktion **Überschreibung der ausgehenden Begrüßung**, um anzupassen, wie sich der Agent bei ausgehenden Anrufen vorstellt.
</Note>

## Web-Widget-Bereitstellung

Betten Sie Ihren Agent auf Ihrer Website für browserbasierte Interaktionen ein.

<Steps>
  <Step title="Web-Widget erstellen">
    Erstellen Sie ein neues Web-Widget im Dashboard
  </Step>
  <Step title="Agent zuweisen">
    Wählen Sie aus, welcher Agent für dieses Widget verwendet werden soll
  </Step>
  <Step title="Aussehen anpassen">
    Passen Sie Farben, Position und Branding an
  </Step>
  <Step title="Auf Ihrer Website einbetten">
    Kopieren Sie den Einbettungscode und fügen Sie ihn Ihrer Website hinzu
  </Step>
  <Step title="Testen und live schalten">
    Besucher können jetzt Sprach- oder Text-Chat verwenden
  </Step>
</Steps>

**Am besten für:**
- Website-Live-Support
- Pre-Sales-Fragen
- Lead-Erfassung
- Interaktive Produktdemos
- 24/7-webbasierte Unterstützung

**Einzigartige Funktionen:**
- Sprach- UND Text-Chat in einer Oberfläche
- Keine Telefonnummer erforderlich
- Bleibt über Seitennavigation hinweg bestehen
- Kann während des Gesprächs zwischen Sprache und Text wechseln

## Begrüßungskonfiguration

Da Agents sowohl für eingehende als auch ausgehende Anrufe verwendet werden können, können Sie Begrüßungen separat konfigurieren:

### Eingehende Begrüßung
Wird verwendet, wenn Kunden Ihre Telefonnummer anrufen oder einen Web-Chat starten.

```
Hallo! Danke, dass Sie Acme Support anrufen. Wie kann ich Ihnen heute helfen?
```

### Ausgehende Begrüßung (Überschreibung)
Wird verwendet, wenn der Agent ausgehende Anrufe in Kampagnen tätigt.

```
Hallo, hier ist Alex von Acme. Sie haben kürzlich unseren Leitfaden heruntergeladen. Haben Sie einen kurzen Moment, um zu besprechen, wie wir helfen können?
```

<Tip>
Konfigurieren Sie die Überschreibung der ausgehenden Begrüßung in **Abilities → Greeting → Outbound Override**. Wenn nicht konfiguriert, verwendet der Agent die eingehende Begrüßung für alle Anrufe.
</Tip>
