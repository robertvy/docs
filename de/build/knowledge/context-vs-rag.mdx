## Überblick

Wenn Sie Wissen Ihrem Agent zuweisen, müssen Sie wählen, wie der Agent auf diese Informationen zugreift. itellicoAI bietet zwei Zugriffsmethoden: **Context Mode** (Prompt-Injektion) und **RAG Mode** (Retrieval-Augmented Generation).

Das Verständnis des Unterschieds zwischen diesen Ansätzen hilft Ihnen, die Leistung und Genauigkeit Ihres Agenten zu optimieren.

---

## Die zwei Zugriffsmethoden

<CardGroup cols={2}>
  <Card title="Context Mode" icon="file-import">
    **Prompt-Injektion**

    Alles Wissen wird direkt zu Beginn jeder Interaktion in den Konversationskontext des Agenten eingefügt.

    **Am besten für:** Kleine Wissensdatenbanken mit kritischen Informationen, die der Agent für jedes Gespräch benötigt.
  </Card>
  <Card title="RAG Mode" icon="magnifying-glass">
    **Retrieval-Augmented Generation**

    Agent sucht und ruft dynamisch relevantes Wissen basierend auf dem Gesprächsthema ab.

    **Am besten für:** Große Wissensdatenbanken, bei denen nur bestimmte Abschnitte pro Gespräch benötigt werden.
  </Card>
</CardGroup>

## Context Mode (Prompt-Injektion)

### Wie es funktioniert

Im Context Mode wird der Inhalt Ihrer Wissensdatenbank zu Beginn jedes Gesprächs direkt in den System-Prompt des Agenten geladen:

```text wrap
System Prompt:
Sie sind ein hilfreicher Kundenservice-Agent.

[Ihre Agentenanweisungen...]

==== WISSENSDATENBANK: Kunden-Support-FAQ ====

### Abrechnung & Zahlungen
- Frage: Wie aktualisiere ich meine Zahlungsmethode?
  Antwort: Sie können Ihre Zahlungsmethode aktualisieren, indem...

- Frage: Wann werde ich abgerechnet?
  Antwort: Die Abrechnung erfolgt am selben Tag jeden Monat...

### Rückgaberichtlinie
[Vollständiger Rückgaberichtlinieninhalt]

### Versandinformationen
[Vollständige Versandinformationen]

==== ENDE WISSENSDATENBANK ====

Unterstützen Sie nun den Kunden bei seiner Frage.
```

Der Agent sieht ALLE Wissensinhalte von Anfang an und kann sie während des gesamten Gesprächs referenzieren.

### Formatierte Ausgabe

Standardmäßig wird Wissen mit strukturierter Formatierung eingefügt:

```text wrap
========================================
# Wissensdatenbank-Name
Wissensdatenbank-Beschreibung

## Ordnername
Ordnerbeschreibung

### Elementtitel
----------------------------------------
Elementinhalt hier
----------------------------------------
========================================
```

Diese Formatierung hilft dem Agent, die Struktur und Organisation des Wissens zu verstehen.

### Wann Context Mode verwendet werden sollte

<AccordionGroup>
  <Accordion title="Kleine Wissensdatenbanken (< 1.000 Wörter)" icon="file">
    Wenn Ihre Wissensdatenbank kompakt ist, stellt Context Mode sicher, dass der Agent immer den vollen Kontext hat.

    **Beispiel-Anwendungsfall:**
    Ein Support-Agent mit einer Wissensdatenbank, die enthält:
    - 10 häufige FAQs
    - Rückgaberichtlinie (500 Wörter)
    - Kontaktinformationen
    - Versandoptionen

    Insgesamt: ~3.000 Wörter - passt problemlos in den Kontext.
  </Accordion>

  <Accordion title="Kritische Informationen, die jedes Mal benötigt werden" icon="circle-exclamation">
    Informationen, die der Agent bei den meisten oder allen Anrufen referenzieren muss.

    **Beispiel-Anwendungsfall:**
    Ein Buchungsagent, der benötigt:
    - Unternehmensrichtlinien (immer)
    - Verfügbare Dienste (immer)
    - Preisstruktur (immer)
    - Buchungsverfahren (immer)
  </Accordion>

  <Accordion title="Hochstrukturierte Informationen" icon="table">
    Wenn Wissenselemente aufeinander verweisen oder ein zusammenhängendes Ganzes bilden.

    **Beispiel-Anwendungsfall:**
    Produktkonfigurations-Agent mit:
    - Optionsabhängigkeiten ("Wenn sie X wählen, biete Y an")
    - Kompatibilitätsmatrix
    - Paketbündel
    - Preise, die von Kombinationen abhängen
  </Accordion>

</AccordionGroup>

### Vorteile des Context Mode

<CardGroup cols={2}>
  <Card title="Immer verfügbar" icon="check">
    Agent hat sofortigen Zugriff auf alles Wissen ohne Suchverzögerung
  </Card>
  <Card title="Besser für kleine Mengen" icon="gauge-high">
    Effizient, wenn Wissen bequem ins Kontextfenster passt
  </Card>
  <Card title="Deterministisch" icon="equals">
    Agent sieht jedes Mal genau dasselbe Wissen
  </Card>
  <Card title="Funktioniert mit Variablen" icon="brackets-curly">
    Wissen kann Jinja-Variablen enthalten, die im Kontext aufgelöst werden
  </Card>
</CardGroup>

### Einschränkungen des Context Mode

<Warning>
Context Mode ist **auf insgesamt 10.000 Token** über alles zugewiesene Wissen begrenzt.
</Warning>

**Einschränkungen:**
- Halten Sie Wissensdatenbanken klein (< 1.000 Wörter)
- Alles Wissen wird mit jeder Anfrage gesendet (höhere Kosten)
- Alles Wissen wird jedes Mal verarbeitet (höhere Latenz)
- Die gesamte Wissensdatenbank ist eingeschlossen, auch wenn nur ein kleiner Teil relevant ist

## RAG Mode (Retrieval-Augmented Generation)

### Wie es funktioniert

Im RAG Mode wird Ihre Wissensdatenbank in einer Vektordatenbank gespeichert. Wenn der Agent Informationen benötigt:

1. **Benutzer stellt eine Frage:** "Was ist Ihre Rückgaberichtlinie?"
2. **Agent identifiziert Bedarf:** Agent bestimmt, dass er Wissen über Rücksendungen benötigt
3. **System sucht:** RAG-System durchsucht Wissensdatenbank nach relevantem Inhalt
4. **Relevanter Inhalt abgerufen:** Nur der Rückgaberichtlinienabschnitt wird abgerufen
5. **Agent antwortet:** Agent verwendet abgerufenes Wissen, um zu antworten

Der Agent sieht nur das Wissen, das er benötigt, wenn er es benötigt.

### Intelligenter Abruf

RAG verwendet semantische Suche mit Vektoreinbettungen, um relevantes Wissen zu finden:

```text wrap
Benutzer: "Ich muss die Schuhe zurückschicken, die ich letzte Woche gekauft habe"

RAG-System denkt:
- Schlüsselwörter: "zurückschicken", "Schuhe", "gekauft"
- Semantische Bedeutung: Rücksendungen, möglicherweise Umtausch
- Suche Wissen nach: Rückgaberichtlinie, Versand, Umtausch

Abgerufenes Wissen:
- Rückgaberichtlinien-Übersicht
- Rücksendeversandanleitung
- Umtauschverfahren

NICHT abgerufen:
- Abrechnungs-FAQ
- Produktspezifikationen
- Kontoverwaltung
```

### Vorteile des RAG Mode

<CardGroup cols={2}>
  <Card title="Skaliert unendlich" icon="up-right-and-down-left-from-center">
    Unterstützt massive Wissensdatenbanken ohne Kontextlimits
  </Card>
  <Card title="Schneller für großes Wissen" icon="clock-rotate-left">
    Reduziert System-Prompt-Token für große Wissensdatenbanken
  </Card>
  <Card title="Effizient" icon="gauge-high">
    Ruft nur ab, was für das aktuelle Thema benötigt wird
  </Card>
  <Card title="Besser für Vielfalt" icon="shapes">
    Handhabt große Vielfalt unzusammenhängender Themen gut
  </Card>
</CardGroup>

### Überlegungen zum RAG Mode

<Note>
RAG basiert auf der Qualität der Vektoreinbettung. Wenn Ihre Wissenselemente nicht klar geschrieben sind, kann der Abruf relevante Informationen übersehen.
</Note>

**Abrufqualität hängt ab von:**
- Klaren, beschreibenden Wissenselement-Titeln
- Gut strukturiertem Inhalt
- Ordnungsgemäßer Kategorisierung in Ordnern
- Vermeidung von doppelten oder widersprüchlichen Informationen

## Indikator für verbleibende Kontext-Token

Wenn Sie Context Mode verwenden, überwachen Sie Ihre verbleibenden Kontext-Token, um sicherzustellen, dass Sie genug Platz für Gespräche haben.

### Den Indikator verstehen

Das Dashboard zeigt Ihre Kontextnutzung:

```text wrap

 Verwendet durch Wissen (Context Mode): 51.000 Token
 Verbleibend für Gespräch: 74.500 Token


```

## Den richtigen Modus wählen

Verwenden Sie diesen Entscheidungsleitfaden, um die beste Zugriffsmethode auszuwählen:

### Hybrid-Ansatz

Sie können beide Modi für verschiedene Wissensdatenbanken beim selben Agent verwenden:

**Beispielkonfiguration:**
- **Context Mode:** Kleine "Kernrichtlinien"-Wissensdatenbank (immer benötigt)
- **RAG Mode:** Große "Produktkatalog"-Wissensdatenbank (bei Bedarf abrufen)

Dies gibt Ihnen das Beste aus beiden Welten.

## Ihre Konfiguration testen

<Steps>
  <Step title="Wissensdatenbank zuweisen">
    Konfigurieren Sie Ihren Agent mit einer Wissensdatenbank im Context- oder RAG-Modus.
  </Step>

  <Step title="Testanruf starten">
    Verwenden Sie die Testanruf-Funktion im Dashboard.
  </Step>

  <Step title="Nach Wissensinhalten fragen">
    Stellen Sie Fragen, die aus Ihrer Wissensdatenbank beantwortet werden sollten.

    **Beispielfragen:**
    - "Was ist Ihre Rückgaberichtlinie?"
    - "Wie viel kostet der Pro-Plan?"
    - "Was sind Ihre Geschäftszeiten?"
  </Step>

  <Step title="Antworten überprüfen">
    Bestätigen Sie, dass der Agent Wissensinhalte korrekt in Antworten verwendet.
  </Step>

  <Step title="Grenzfälle testen">
    Fragen Sie nach Themen NICHT in Ihrer Wissensdatenbank, um sicherzustellen, dass der Agent angemessen antwortet ("Ich habe diese Information nicht").
  </Step>
</Steps>

## Best Practices

<AccordionGroup>
  <Accordion title="Beginnen Sie mit RAG für große Datenbanken" icon="magnifying-glass">
    Im Zweifelsfall verwenden Sie RAG Mode. Er ist sicherer für große Wissensdatenbanken und Sie können bei Bedarf immer zu Context Mode wechseln.
  </Accordion>

  <Accordion title="Schreiben Sie klare, vollständige Inhalte" icon="file-text">
    RAG verwendet semantische Suche, um relevantes Wissen zu finden. Schreiben Sie vollständige, gut geschriebene Inhalte, die natürlich die Begriffe und Konzepte enthalten, nach denen Benutzer fragen werden.

    **Gut:** "Unsere Rückgaberichtlinie erlaubt Rücksendungen innerhalb von 30 Tagen nach dem Kauf für physische Produkte. Digitale Produkte können nicht zurückgegeben werden, sobald sie heruntergeladen wurden."

    **Schlecht:** "Siehe Richtliniendokument" oder unvollständige Satzfragmente
  </Accordion>

  <Accordion title="Testen Sie beide Modi" icon="vial">
    Probieren Sie sowohl Context- als auch RAG-Modus mit Ihrer Wissensdatenbank aus und sehen Sie, welcher für Ihren Anwendungsfall besser funktioniert.
  </Accordion>

  <Accordion title="Strategisch kombinieren" icon="layer-group">
    Verwenden Sie Context Mode für kritische, häufig benötigte Informationen und RAG Mode für umfangreiches Referenzmaterial.
  </Accordion>
</AccordionGroup>

## Fehlerbehebung

<AccordionGroup>
  <Accordion title="Agent verwendet Wissen nicht (Context Mode)">
    **Prüfen:**
    - Sind alle Wissenselemente im COMPLETED-Status?
    - Ist der verbleibende Kontext ausreichend (nicht abgeschnitten)?

    **Lösung:**
    - Fehlgeschlagene Elemente beheben
    - Wissensgröße reduzieren oder zu RAG wechseln
  </Accordion>

  <Accordion title="Agent findet Wissen nicht (RAG Mode)">
    **Prüfen:**
    - Ist der Inhalt gut organisiert?
    - Stellen Sie Fragen, die zum Wissen passen?

    **Lösung:**
    - Detaillierteren Inhalt hinzufügen
    - Mit verschiedenen Formulierungen testen
    - Erwägen Sie, Schlüsselwörter zum Inhalt hinzuzufügen
  </Accordion>

  <Accordion title="Agent liefert falsche oder veraltete Informationen">
    **Prüfen:**
    - Ist der Wissensinhalt korrekt und aktuell?
    - Haben Sie widersprüchliche Informationen in mehreren Elementen?

    **Lösung:**
    - Wissensinhalt aktualisieren
    - Duplikate und Widersprüche entfernen
  </Accordion>

  <Accordion title="Kontextnutzung zu hoch">
    **Lösungen:**
    - Zu RAG Mode für große Wissensdatenbanken wechseln
    - Wissen in kleinere, fokussierte Datenbanken aufteilen
    - Länge der Agentenanweisungen reduzieren
    - Ausführlichen oder redundanten Inhalt entfernen
  </Accordion>
</AccordionGroup>
