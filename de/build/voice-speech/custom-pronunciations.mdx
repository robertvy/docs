## Übersicht

Benutzerdefinierte Aussprachen ermöglichen es dir, zu definieren, wie bestimmte Wörter klingen sollen, um sicherzustellen, dass dein Agent Markennamen, technische Begriffe und Akronyme immer korrekt ausspricht.

**Wie es funktioniert:**
1. Agent generiert Antwort: "Rufen du uns bei itellicoAI für Hilfe an"
2. System wendet Ausspracheregeln an: "Rufen du uns bei eye-tell-ee-co A I für Hilfe an"
3. Stimmsynthesizer spricht den geänderten Text
4. Kunde hört korrekte Aussprache

<Note>
Aussprache-Ersetzungen sind verfügbar unter **Stimme > Aussprachen** in deiner Agentenkonfiguration.
</Note>

---

## Hinzufügen von Aussprachen

### Einzelne Ersetzung

1. Klicke auf **Ersetzung hinzufügen**
2. Gib das **Zielwort** ein (das zu ersetzende Wort)
3. Gib die **Aussprache** ein (wie es klingen soll)
4. Klicke auf **Hinzufügen**

**Beispiele:**
- **Ziel:** Nike → **Aussprache:** Nai-key
- **Ziel:** API → **Aussprache:** A-P-I
- **Ziel:** 24/7 → **Aussprache:** vierundzwanzig sieben
- **Ziel:** EST → **Aussprache:** Eastern Time

<Warning>
**Zielwörter müssen einzelne Wörter sein** - Leerzeichen sind nicht erlaubt. Verwende vollständige Wörter wie `Nike` oder `API`, nicht Phrasen wie `Kundendienst`.
</Warning>

### Massenimport via CSV

Für den Import vieler Aussprachen auf einmal:

1. Klicke auf **CSV importieren**
2. Laden du die Vorlage herunter, falls erforderlich
3. Bereiten du deine CSV mit zwei Spalten vor: `target`, `replacement`
4. Ziehen du deine CSV-Datei per Drag & Drop oder klicke zum Auswählen
5. Überprüfe die Vorschau und klicke auf **Importieren**

**CSV-Format:**
```csv
target,replacement
Nike,Nai-key
EST,Eastern Time
24/7,vierundzwanzig sieben
API,A-P-I
```

**Limits:**
- Maximum 500 Aussprachen pro Agent
- CSV-Dateigröße max: 5MB

## Tipps zur phonetischen Schreibweise

### Einfache phonetische Schreibweise

Schreibe Ersetzungen als einfachen, lesbaren phonetischen Text:

**Gute Beispiele:**
- Nike → Nai-key
- SQL → sequel (oder) S-Q-L
- Wi-Fi → why-figh
- iOS → eye-oh-ess

**Tipps:**
- Verwende Bindestriche zur Trennung von Silben: Nai-key, A-P-I
- Buchstabieren du es so, wie du jemandem die Aussprache erklären würden
- Teste mit deiner Stimme, um zu hören, wie es klingt

### Häufige Muster

**Akronyme (buchstabieren du jeden Buchstaben):**
- API → A-P-I
- URL → U-R-L
- HTTP → H-T-T-P
- AWS → A-W-S

**Markennamen (phonetische Schreibweise):**
- Nike → Nai-key
- Adidas → ah-dee-dahs
- Chipotle → chih-poht-lay

**Zahlen und Zeit:**
- 24/7 → vierundzwanzig sieben
- 9am → neun A-M
- EST → Eastern Time
- 5pm → fünf P-M

**Technische Begriffe:**
- SQL → sequel (oder) S-Q-L
- API → A-P-I
- URL → you-are-ell
- WiFi → why-figh

## Erweitert: IPA-Phoneme

Für präzise Aussprachekontrolle können du die Notation des Internationalen Phonetischen Alphabets (IPA) mit dem `<< >>`-Format verwenden:

**Format:** `<<phoneme|phoneme|phoneme>>`

**Beispiel:**
```
espresso → <<ɛ|s|p|r|ɛ|s|oʊ>>
```

<Note>
IPA-Phoneme bieten präzise Kontrolle, erfordern aber Kenntnisse der phonetischen Notation. Einfache phonetische Schreibweise (Nai-key) funktioniert gut für die meisten Fälle.
</Note>

## Verwalten von Aussprachen

### Bearbeiten vorhandener Ersetzungen

1. Klicke auf einen beliebigen Ersetzungs-Chip in der Liste
2. Ändern du das Ziel oder die Aussprache
3. Klicke auf **Speichern**

### Löschen von Ersetzungen

- Klicke auf das **Papierkorb-Symbol** bei einem beliebigen Ersetzungs-Chip, um ihn zu löschen
- Oder klicke auf **Alle löschen**, um alle Aussprachen zu entfernen (erfordert Bestätigung)

### Exportieren von Aussprachen

1. Klicke auf **CSV exportieren**
2. CSV-Datei wird mit allen deinen aktuellen Aussprachen heruntergeladen
3. Dateinamenformat: `{agent_name}_pronunciations.csv`

Verwende exportierte CSV-Dateien, um deine Aussprachen zu sichern oder sie auf andere Agenten zu kopieren.

## Testen von Aussprachen

Nach dem Hinzufügen von Aussprachen:

1. Tätigen du einen Testanruf zu deinem Agenten
2. Lösen du Antworten aus, welche die Zielwörter enthalten
3. Hören du zu, um zu überprüfen, ob die Aussprache korrekt klingt
4. Passe die phonetische Schreibweise bei Bedarf an

**Test-Tipps:**
- Teste im Kontext, nicht isoliert
- Lassen du Team-Mitglieder zuhören
- Passe Silbentrennung oder Schreibweise an, wenn es nicht richtig klingt

## Best Practices

<AccordionGroup>
  <Accordion title="Beginne mit häufigen Problemen" icon="bullseye">
    Füge Aussprachen für Wörter hinzu, von denen du wissen, dass sie in echten Anrufen falsch ausgesprochen werden. Überprüfe Anruftranskripte, um wiederkehrende Ausspracheprobleme zu identifizieren.
  </Accordion>

  <Accordion title="Halte es einfach" icon="sparkles">
    Verwende phonetische Schreibweise (Nai-key) anstelle von komplexem IPA, es sei denn, es ist notwendig. Einfacher phonetischer Text ist einfacher zu pflegen und funktioniert gut für die meisten Fälle.
  </Accordion>

  <Accordion title="Vor der Bereitstellung testen" icon="vial">
    Teste Aussprachen immer mit echten Anrufen, bevor du live gehen. Tätigen du einen Testanruf und lösen du Antworten aus, welche die Zielwörter enthalten, um zu überprüfen, ob sie korrekt klingen.
  </Accordion>

  <Accordion title="Dokumentieren du deine Regeln" icon="file-arrow-down">
    Exportieren du deine CSV regelmäßig als Backup. Verwende die Schaltfläche CSV exportieren, um deine Aussprachen zu speichern und versionieren du sie zusammen mit deiner Agentenkonfiguration.
  </Accordion>

  <Accordion title="Verwende konsistente Schreibweise" icon="check">
    Wenn du "API" als "A-P-I" buchstabiert haben, verwenden du dieses Format konsistent für alle deine Aussprachen. Konsistenz verbessert die Wartbarkeit.
  </Accordion>

  <Accordion title="Nur einzelne Wörter" icon="letter-a">
    Denken du daran, dass Zielwörter keine Leerzeichen enthalten können - nur vollständige einzelne Wörter funktionieren. Verwende `Nike` oder `API`, nicht Phrasen wie `Kundendienst`.
  </Accordion>
</AccordionGroup>
