Das Web-Widget ermöglicht es Besuchern, mit Ihrem KI-Agenten über Sprache, Chat oder beides gleichzeitig zu interagieren. Konfigurieren Sie alles über den visuellen Editor – keine Programmierung erforderlich – und kopieren Sie dann ein einziges Einbettungsskript.

<Steps>
  <Step title="Widget-Manager öffnen">
    Navigieren Sie zu **Web → Widgets** im Dashboard
  </Step>
  <Step title="Neues Widget erstellen">
    Klicken Sie auf **Widget erstellen** und geben Sie an:
    - **Widget-Name** - Interne Bezeichnung für Ihre Referenz
    - **Agent** - Wählen Sie aus, welcher Agent dieses Widget antreibt
  </Step>
  <Step title="Widget konfigurieren">
    Verwenden Sie die visuellen Editor-Tabs, um Ihr Widget anzupassen (siehe Abschnitte unten)
  </Step>
  <Step title="Änderungen speichern">
    Klicken Sie auf **Änderungen speichern** unten rechts, um alle Updates zu übernehmen
  </Step>
  <Step title="Einbettungscode abrufen">
    Gehen Sie zum Tab **Export** und kopieren Sie Ihr einzigartiges Einbettungsskript
  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="Setup" icon="gear">
    - **Widget-Name** - Interne Bezeichnung in Ihrer Widget-Liste
    - **Agent** - Jederzeit zu einem anderen Agenten wechseln
    - **Anzeigename** - Kundenseitiger Name im Widget angezeigt
    - **Erlaubte Domains** - Beschränken Sie, wo das Widget geladen werden kann (leer lassen, um alle Domains zu erlauben)
  </Accordion>

  <Accordion title="Erscheinungsbild" icon="palette">
    **Themenvorlagen**
    - Von kuratierten Farbpaletten starten
    - Durch Vorlagenoptionen scrollen

    **Layout**
    - Position: unten links oder unten rechts
    - Widget-Größe und Abstände von Rändern
    - Anfangszustand: eingeklappt, erweitert oder immer offen

    **Avatar**
    - Animierter "AI Pulse"-Effekt
    - Benutzerdefiniertes Bild hochladen
    - Mit externer Bild-URL verknüpfen

    **Farben & Verläufe**
    - Hintergründe und CTA-Farben
    - Transkript-Blasen-Styling
    - Statusfarben und Orb-Verläufe

    **Randradius**
    - Launcher-Button-Rundung
    - Chat-Fenster-Ecken
    - Nachrichtenblasen
    - Eingabefeld und CTA-Button

    **Animationen**
    - Optionaler Wackel- oder Welleneffekt auf Anruf-Button
  </Accordion>

  <Accordion title="Inhalt" icon="language">
    **Sprachvorlagen**
    - Vorkonfigurierte englische und deutsche Pakete
    - Setzt alle Widget-Beschriftungen und rechtlichen Texte

    **Willkommensnachrichten**
    - Willkommens-Prompt-Text
    - Platzhaltertext
    - Statusnachrichten (zuhören, sprechen, denken, verbinden)

    **Feedback-Texte**
    - Feedback-Aufforderung
    - Absenden-Button-Text
    - Danke-Nachricht

    **Beschriftungsüberschreibungen**
    - Einzelne UI-Strings anpassen
    - Gruppen: Chat, Steuerelemente, Transkript, Audio, Barrierefreiheit
  </Accordion>

  <Accordion title="Funktionen" icon="sliders">
    - **Nur-Chat-Modus** - Sprache deaktivieren, nur Chat
    - **Text während Anruf erlauben** - Texteingabe während Sprachanrufen aktiv halten
    - **Transkript-Umschaltung** - Live-Transkript ein-/ausblenden
    - **Stummschaltung** - Stummschalt-Button anzeigen, damit Benutzer ihr Mikrofon stummschalten können
    - **Anruf-Timer** - Verstrichene Zeit anzeigen
    - **Branding** - "Powered by itellicoAI" ein-/ausblenden
    - **Begrüßung überspringen** - Begrüßung vermeiden, wenn Benutzer zuerst tippt
    - **Feedback-Sammlung** - Nachgesprächs-Feedback aktivieren (noch nicht gespeichert, demnächst verfügbar)
  </Accordion>

  <Accordion title="Aktionen" icon="link">
    Konfigurieren Sie clientseitige Tools, die Ihr Agent während Gesprächen aufrufen kann, um Links anzuzeigen, Seiten zu öffnen oder Telefonanrufe im Browser des Benutzers auszulösen.

    <Note>
    **Client-Tools**: Im Gegensatz zu API-Aktionen, die auf Ihrem Server laufen, werden Client-Tools direkt im Browser des Benutzers ausgeführt. Das LLM entscheidet basierend auf dem Gesprächskontext und der von Ihnen bereitgestellten Trigger-Beschreibung, wann diese Tools aufgerufen werden.
    </Note>

    Fügen Sie bis zu fünf Client-Tools hinzu:

    - **Button-Text** - Dem Benutzer angezeigte Beschriftung (z.B. "Preise ansehen", "Support anrufen")
    - **URL** - Weblink (`https://`) oder Telefonnummer (`tel:`)
    - **Trigger** - Beschreiben Sie, wann der Agent dieses Tool aufrufen soll (z.B. "Wenn Benutzer nach Preisen fragt", "Wenn Benutzer mit einem Menschen sprechen möchte")
    - **Verhalten**:
      - **Auto-Klick** - Link automatisch öffnen, wenn Tool aufgerufen wird
      - **Auf Antwort warten** - Button anzeigen und auf Benutzerklick warten (mit optionalem Timeout)
      - **Anruf bei Klick beenden** - Gespräch automatisch beenden, nachdem Benutzer geklickt hat

    Telefonnummer-Links (`tel:`) beenden den Anruf automatisch, nachdem der Benutzer geklickt hat.
  </Accordion>

  <Accordion title="Datenschutz" icon="shield-check">
    **AGB & Datenschutz**
    - Markdown-Editor für benutzerdefinierten rechtlichen Text
    - Standardtext für neue Widgets bereitgestellt

    **Einwilligungseinstellungen**
    - Akzeptieren/Ablehnen-Button-Beschriftungen
    - Lokaler Speicherschlüssel zum Merken der Einwilligung
    - Einwilligung vor Aktivierung erforderlich
  </Accordion>

  <Accordion title="Export" icon="code">
    **Einbettungsskript**
    - Einzigartige Loader-URL für Ihr Widget
    - Vor `</body>`-Tag kopieren und einfügen

    **Installationsanleitungen**
    - Schritt-für-Schritt-Anweisungen
    - Plattformspezifische Tipps:

      - Webflow
      - Shopify
      - Wix
      - Squarespace

## Ihr Widget bereitstellen

<Steps>
  <Step title="In der Vorschau testen">
    Verwenden Sie die Live-Vorschau auf der rechten Seite des Konfigurators, um Ihr Widget zu testen. Klicken Sie bei Bedarf auf das Aktualisierungssymbol.
  </Step>
  <Step title="Einbettungsskript kopieren">
    Gehen Sie zum Tab **Export** und kopieren Sie Ihr einzigartiges Einbettungsskript.
  </Step>
  <Step title="Zu Ihrer Website hinzufügen">
    Fügen Sie das Skript direkt vor dem schließenden `</body>`-Tag auf Ihrer Website ein.
  </Step>
  <Step title="Live testen">
    Laden Sie Ihre Website, klicken Sie auf das Widget und testen Sie ein Sprach- oder Chat-Gespräch. Prüfen Sie **Gespräche**, um zu verifizieren, dass die Sitzung aufgezeichnet wurde.
  </Step>
</Steps>

<Note>
Konfigurationsänderungen, die Sie im Dashboard vornehmen, werden automatisch auf Ihr eingebettetes Widget angewendet. Das Einbettungsskript muss nicht aktualisiert werden. Wenn Änderungen nicht sofort erscheinen, führen Sie eine harte Aktualisierung durch (Strg+Umschalt+R oder Cmd+Umschalt+R).
</Note>

**Hinweise:**
- **Domain-Beschränkungen** gelten sofort. Wenn konfiguriert, lädt das Widget nur auf erlaubten Domains.
- **HTTPS erforderlich** für Mikrofonzugriff (außer localhost).
- **Mobilgeräte-responsiv** - Das Widget passt sich automatisch an mobile Bildschirme an.
- Erstellen Sie separate Widgets für verschiedene Websites oder Branding-Anforderungen.

## Fehlerbehebung

<AccordionGroup>
  <Accordion title="Widget wird nicht angezeigt" icon="eye-slash">
    - Browser-Konsole auf JavaScript-Fehler prüfen
    - Überprüfen Sie, ob erlaubte Domains Ihre Website einschließen
    - Stellen Sie sicher, dass Skript vor `</body>`-Tag platziert ist
    - Content Security Policy-Einstellungen prüfen
  </Accordion>

  <Accordion title="Mikrofonzugriff verweigert" icon="microphone-slash">
    - HTTPS erforderlich (außer localhost)
    - Benutzer muss Berechtigung manuell erteilen
    - Browser-Mikrofoneinstellungen prüfen
    - Einige Browser blockieren beim ersten Besuch
  </Accordion>

  <Accordion title="Kein Audio oder schlechte Qualität" icon="volume-xmark">
    - WebRTC erfordert offene UDP-Ports
    - Unternehmens-VPNs können WebRTC blockieren
    - Von anderem Netzwerk testen
    - Browser-WebRTC-Statistiken unter `chrome://webrtc-internals` prüfen
  </Accordion>

  <Accordion title="Änderungen werden nicht aktualisiert" icon="arrows-rotate">
    - Auf **Änderungen speichern** im Konfigurator klicken
    - Ihre Website hart aktualisieren (Strg+Umschalt+R oder Cmd+Umschalt+R)
    - Browser-Cache bei Bedarf leeren
    - Überprüfen Sie, dass Sie das richtige Widget bearbeiten
  </Accordion>
</AccordionGroup>
