## Übersicht

Jeder Testanruf generiert umfassende Logs, die Ihnen helfen, das Verhalten des Agenten zu validieren, Probleme zu debuggen und Qualitätsmetriken zu verfolgen. Lernen Sie, Logs effektiv zu analysieren, um sicherzustellen, dass Ihr Agent produktionsreif ist.

## Zugriff auf Gesprächsprotokolle

<Steps>
  <Step title="Zu den Logs navigieren">
    Gehen Sie zu **Gespräche → Logs** im Dashboard. Sie sehen eine Liste aller aktuellen Gespräche (Test und Produktion).
  </Step>

  <Step title="Testanrufe filtern">
    Verwenden Sie Filter, um Ihre Testanrufe zu finden:
    - **Agent**: Wählen Sie einen bestimmten Agenten aus
    - **Zeitraum**: Letzte Stunde, letzte 24 Stunden, benutzerdefinierter Bereich
    - **Status**: Erfolgreich, fehlgeschlagen, in Bearbeitung
    - **Tags**: Filtern nach "Test"-Tag
    - **Richtung**: Eingehend vs. ausgehend
  </Step>

  <Step title="Gesprächsdetails öffnen">
    Klicken Sie auf ein beliebiges Gespräch, um alle Details anzuzeigen:
    - Audio-Wiedergabe
    - Vollständiges Transkript
    - Action-Ausführungslogs
    - Wissensdatenbank-Abrufe
    - Zielergebnisse
    - Qualitätsmetriken
  </Step>
</Steps>

## Verstehen der Log-Komponenten

### Transkript-Ansicht

**Was Sie sehen**:
```
[00:03] Agent: Willkommen bei itellicoAI. Wie kann ich Ihnen heute helfen?
[00:05] Kunde: Was sind Ihre Geschäftszeiten?
[00:07] Agent: Wir sind Montag bis Freitag von 9 bis 17 Uhr Eastern Time geöffnet.
[00:10] Kunde: Vielen Dank.
[00:11] Agent: Gerne! Kann ich Ihnen sonst noch weiterhelfen?
[00:13] Kunde: Nein, das war alles.
[00:14] Agent: Ich wünsche Ihnen einen schönen Tag!
[Anruf beendet - Dauer: 00:11]
```

**Wichtige Elemente**:
- **Zeitstempel**: Verfolgen Sie die Antwortlatenz (Zeit zwischen den Turns)
- **Sprecherbezeichnungen**: Agent vs. Kunde sind klar gekennzeichnet
- **Konfidenzwerte**: Bewegen Sie den Mauszeiger über das Transkript, um die Transkriptions-Konfidenz zu sehen
- **Bearbeitungsoption**: Korrigieren Sie Transkriptionsfehler manuell für das Retraining

<Accordion title="Analyse der Antwortzeiten">
  **Ziel-Latenz**:
  - Antwort des Agenten: &lt;1 Sekunde (hervorragend), &lt;2 Sekunden (akzeptabel)
  - API-Aufrufe: &lt;3 Sekunden gesamt (einschließlich Agentenverarbeitung)

  **Warnsignale**:
  - >3 Sekunden Verzögerung deuten auf Latenzprobleme hin
  - Überprüfen Sie den **Qualitäts**-Tab für Netzwerkmetriken
  - Überprüfen Sie die Modellauswahl (GPT-4 ist langsamer als GPT-3.5)
</Accordion>

### Actions-Tab

**Anzeige von Action-Ausführungsdetails**:
<CardGroup cols={2}>
  <Card title="Anfrage" icon="arrow-right">
    - Action-Name (z.B. "getOrderStatus")
    - Auslösezeitstempel
    - Extrahierte Eingabeparameter
    - Aufgerufener API-Endpunkt
    - Anfrage-Header & Body
  </Card>

  <Card title="Antwort" icon="arrow-left">
    - HTTP-Statuscode (200, 404, 500, etc.)
    - Antwort-Body (JSON, XML, Text)
    - Verarbeitungszeit (ms)
    - Erfolg-/Fehlerstatus
    - Fehlermeldungen (falls fehlgeschlagen)
  </Card>
</CardGroup>

**Beispiel Action-Log**:
```json
{
  "action": "getOrderStatus",
  "timestamp": "2024-01-12T15:23:45Z",
  "input": {
    "order_id": "12345",
    "customer_phone": "+12125551234"
  },
  "request": {
    "method": "POST",
    "url": "https://api.example.com/orders/status",
    "headers": {"Authorization": "Bearer ***"},
    "body": {"order_id": "12345"}
  },
  "response": {
    "status": 200,
    "body": {
      "status": "shipped",
      "tracking": "1Z999AA10123456784",
      "eta": "2024-01-15"
    },
    "duration_ms": 245
  },
  "result": "success"
}
```

**Debugging von Actions**:
- Fehlgeschlagene Actions zeigen Fehlerdetails an
- Testen Sie dieselbe Anfrage mit curl, um das Problem zu isolieren
- Überprüfen Sie die Gültigkeit und Berechtigungen des API-Schlüssels
- Überprüfen Sie, ob das Format der Anfrage-Payload mit der API-Spezifikation übereinstimmt

### Wissensdatenbank-Tab

**Anzeige von KB-Abrufen**:
```json
{
  "query": "was sind ihre Geschäftszeiten",
  "chunks_retrieved": 3,
  "results": [
    {
      "content": "Unser Support-Team ist Montag-Freitag, 9-17 Uhr EST verfügbar...",
      "similarity_score": 0.92,
      "source": "kb_article_123",
      "chunk_id": "chunk_456"
    },
    {
      "content": "Feiertagsöffnungszeiten variieren. Wir sind an wichtigen US-Feiertagen geschlossen...",
      "similarity_score": 0.78,
      "source": "kb_article_789"
    }
  ],
  "retrieval_time_ms": 123
}
```

**Analyse**:
- **Hohe Ähnlichkeit (>0.85)**: Gute Übereinstimmung, Agent sollte diesen Inhalt verwenden
- **Mittlere Ähnlichkeit (0.7-0.85)**: Relevant, aber möglicherweise besserer Inhalt oder Abfrage erforderlich
- **Niedrige Ähnlichkeit (&lt;0.7)**: Schlechte Übereinstimmung, Agent hat möglicherweise keine Antwort in der KB

**Wenn der Agent "Ich weiß es nicht" sagt**:
- Überprüfen Sie, ob relevanter Inhalt abgerufen wurde (Ähnlichkeit >0.8)
- Wenn ja → Problem mit Agentenanweisungen (abgerufener Inhalt wurde nicht verwendet)
- Wenn nein → KB-Inhalt fehlt oder Abfrage stimmt nicht überein

### Ziele-Tab

**Verfolgung von Gesprächszielen**:
<Tabs>
  <Tab title="Zielergebnisse">
    ```json
    {
      "goals": [
        {
          "name": "appointment_scheduled",
          "achieved": true,
          "confidence": 0.94,
          "entities": {
            "date": "2024-01-15",
            "time": "14:00",
            "service": "consultation"
          }
        },
        {
          "name": "contact_info_collected",
          "achieved": true,
          "confidence": 0.88,
          "entities": {
            "email": "customer@example.com",
            "phone": "+12125551234"
          }
        }
      ]
    }
    ```

    **Interpretation der Konfidenz**:
    - >0.9: Sehr sicher, zuverlässig
    - 0.7-0.9: Sicher, wahrscheinlich genau
    - &lt;0.7: Geringe Konfidenz, Transkript überprüfen
  </Tab>

  <Tab title="Ziel-Debugging">
    **Warum Ziele nicht erreicht werden**:
    1. **Zieldefinition zu strikt**: Auslösephrasen anpassen
    2. **Gespräch vorzeitig beendet**: Kunde hat vor Abschluss aufgelegt
    3. **Agent folgte dem Ablauf nicht**: Problem mit Anweisungen
    4. **Entitäten nicht extrahiert**: Klarere Kundenantworten erforderlich

    **Beispiel-Fix**:
    ```markdown
    # Vorher (zu strikt)
    Ziel: "appointment_scheduled"
    Auslöser: Kunde sagt "ja, buchen Sie mich für 14 Uhr"

    # Nachher (flexibler)
    Ziel: "appointment_scheduled"
    Auslöser: Jede bestätigte Datum/Uhrzeit-Buchung
    ```
  </Tab>
</Tabs>

### Qualitäts-Tab

**Anrufqualitätsmetriken**:

| Metrik | Zielwert | Bedeutung |
|--------|----------|-----------|
| **MOS (Mean Opinion Score)** | >3.5 | Gesamtanrufqualität (Skala 1-5) |
| **Jitter** | &lt;30ms | Netzwerkstabilität (Variation im Paket-Timing) |
| **Paketverlust** | &lt;1% | Netzwerkzuverlässigkeit |
| **RTT (Round Trip Time)** | &lt;200ms | Netzwerklatenz |
| **Bitrate** | 64kbps+ | Audioqualität (höher = besser) |

**Diagnose von Audioproblemen**:
- Hoher Jitter + Paketverlust → Netzwerküberlastung
- Hohe RTT → Geografische Entfernung oder Routing-Problem
- Niedriger MOS-Score → Insgesamt schlechte Qualität, mehrere Faktoren untersuchen

## Monitoring-Tools

### Live-Monitor

**Echtzeit-Anrufüberwachung**:
<Steps>
  <Step title="Live-Monitor öffnen">
    Navigieren Sie zu **Gespräche → Live-Monitor**, um aktive Anrufe in Echtzeit zu sehen.
  </Step>

  <Step title="Aktive Anrufe beobachten">
    Zeigen Sie eine Liste laufender Anrufe mit:
    - Anrufer-ID
    - Zugewiesener Agent
    - Anrufdauer (Live-Timer)
    - Aktuelles Transkript (aktualisiert alle 2-3 Sekunden)
    - Ausgeführte Aktionen
  </Step>

  <Step title="Bei Bedarf eingreifen">
    **Interventionsoptionen**:
    - **Flüstern**: Sprechen Sie mit dem Agenten, ohne dass der Kunde es hört (Agent anleiten)
    - **Einschalten**: Dem Anruf als 3-Wege-Teilnehmer beitreten
    - **Übernehmen**: Agent trennen, Anruf selbst übernehmen
    - **Anruf beenden**: Anruf bei Bedarf zwangsweise beenden

    **Anwendungsfälle**:
    - Agent gibt falsche Informationen
    - Testen eines Grenzfalls erfordert Anleitung
    - Notfallsituation
  </Step>
</Steps>

## Analyse von Testergebnissen

### Qualitätscheckliste

Nach dem Durchführen der Testsuite überprüfen:

<Accordion title="Transkript-Qualität">
  - [ ] Transkriptionsgenauigkeit >90%
  - [ ] Keine größeren falsch gehörten Wörter
  - [ ] Agentenantworten relevant zur Kundeneingabe
  - [ ] Gespräch verläuft natürlich
  - [ ] Keine unangenehmen Pausen oder Verzögerungen
</Accordion>

<Accordion title="Action-Ausführung">
  - [ ] Alle Actions werden zum richtigen Zeitpunkt ausgelöst
  - [ ] Actions erfolgreich abgeschlossen (keine Fehler)
  - [ ] API-Antworten werden vom Agenten korrekt verwendet
  - [ ] Action-Latenz &lt;3 Sekunden
  - [ ] Fehlerbehandlung funktioniert (Test-Fehler werden angemessen behandelt)
</Accordion>

<Accordion title="Wissensdatenbank-Genauigkeit">
  - [ ] KB-Abfragen rufen relevanten Inhalt ab
  - [ ] Agent verwendet abgerufenen Inhalt in Antworten
  - [ ] Agent halluziniert keine Fakten, die nicht in der KB sind
  - [ ] Agent sagt "Ich weiß es nicht", wenn angemessen
  - [ ] Abrufzeit &lt;500ms
</Accordion>

<Accordion title="Anrufqualität">
  - [ ] MOS-Score >3.5
  - [ ] Jitter &lt;30ms
  - [ ] Paketverlust &lt;1%
  - [ ] Keine Audioausfälle oder Verzerrungen
  - [ ] Stimme klingt natürlich und klar
</Accordion>

### Vergleich von Testläufen

**Metriken im Zeitverlauf verfolgen**:
```markdown
| Datum | Testanrufe | Erfolgsrate | Durchschn. Dauer | Zielerreichung |
|-------|-----------|-------------|------------------|----------------|
| 10. Jan | 25 | 96% | 3:45 | 68% |
| 11. Jan | 30 | 94% | 3:52 | 71% |
| 12. Jan | 28 | 98% | 3:38 | 75% |
```

**Trendanalyse**:
- Erfolgsrate verbessert sich → Guter Fortschritt
- Dauer nimmt ab → Agent wird effizienter
- Zielerreichung steigt → Anweisungen optimiert

**Regressionserkennung**:
- Plötzlicher Rückgang der Erfolgsrate → Kürzliche Änderung hat etwas kaputt gemacht
- Dauer-Spitze → Neues Latenzproblem eingeführt
- Zielerreichung sinkt → Anweisungen oder KB negativ geändert

## Logs exportieren

### Für Team-Review

<Steps>
  <Step title="Gesprächsprotokolle exportieren">
    Von Gespräche → Logs wählen Sie Gespräche aus und klicken auf **Exportieren**:
    - Format: JSON, CSV
    - Einschließen: Transkript, Actions, Ziele, Metadaten
    - Als Datei herunterladen oder per E-Mail senden
  </Step>

  <Step title="Mit Team teilen">
    **Anwendungsfälle**:
    - QA-Team überprüft zur Qualitätssicherung
    - Produktteam analysiert Gesprächsmuster
    - Engineering debuggt spezifische Probleme
    - Compliance-Team prüft für regulatorische Anforderungen
  </Step>
</Steps>

### Programmatischer Zugriff

**API-Export**:
```bash
# Alle Testgespräche der letzten 24 Stunden exportieren
curl -H "Authorization: Bearer YOUR_API_KEY" \
     "https://api.itellico.ai/v1/conversations?start_date=2024-01-12&tag=test" \
     > test_conversations.json

# Mit jq analysieren
cat test_conversations.json | jq '.[] | select(.status == "failed")'
```

**Verwendung für**:
- Automatisierte Testberichte
- Integration mit Test-Frameworks
- CI/CD-Pipeline-Validierung
- Benutzerdefinierte Analytics-Dashboards
