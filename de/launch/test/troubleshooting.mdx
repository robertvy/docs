## Ansatz zur Fehlerbehebung

Wenn Tests nicht wie geplant verlaufen, folgen Sie diesem systematischen Ansatz:

1. **Identifizieren Sie das Symptom** - Was ist genau falsch?
2. **Isolieren Sie die Ursache** - Liegt es an der Agentenkonfiguration, dem Browser/Telefon oder der Plattform?
3. **Protokolle prüfen** - Überprüfen Sie Konversationsprotokolle auf Fehler
4. **Isoliert testen** - Entfernen Sie Variablen, um die Grundursache zu finden
5. **Lösung anwenden** - Nehmen Sie die kleinste Änderung vor, die das Problem löst
6. **Überprüfen** - Erneut testen, um zu bestätigen, dass die Lösung funktioniert

## Web-Simulator-Probleme

### Mikrofon funktioniert nicht

**Symptom**: Agent kann Sie nicht hören, keine Transkription erscheint.

<AccordionGroup>
  <Accordion title="Browser-Berechtigungen">
    **Berechtigungen prüfen**:
    - Chrome: `chrome://settings/content/microphone`
    - Firefox: Klicken Sie auf das Schloss-Symbol in der Adressleiste → Berechtigungen
    - Safari: Safari → Einstellungen → Websites → Mikrofon

    **Lösung**: Erlauben Sie Mikrofonzugriff für die Website, laden Sie die Seite neu
  </Accordion>

  <Accordion title="Falsches Mikrofon ausgewählt">
    **Diagnose**:
    - Klicken Sie auf das Zahnrad-Symbol im Simulator
    - Prüfen Sie, welches Mikrofon ausgewählt ist
    - Überprüfen Sie, ob es Ihr aktives Mikrofon ist (nicht das Standard-Systemmikrofon)

    **Lösung**: Wählen Sie das richtige Mikrofon aus dem Dropdown-Menü, testen Sie die Audiopegel
  </Accordion>

  <Accordion title="Mikrofon wird verwendet">
    **Symptom**: Fehler "Mikrofon belegt"

    **Ursache**: Eine andere App (Zoom, Meets, Discord) verwendet das Mikrofon

    **Lösung**: Schließen Sie andere Apps, die das Mikrofon verwenden, starten Sie den Browser bei Bedarf neu
  </Accordion>

  <Accordion title="Browser-Kompatibilität">
    **Anforderungen**:
    - Chrome 60+ (empfohlen)
    - Firefox 55+
    - Safari 14.1+
    - Edge 79+

    **Nicht unterstützt**: IE11, ältere mobile Browser

    **Lösung**: Aktualisieren Sie den Browser oder wechseln Sie zu einem unterstützten Browser
  </Accordion>
</AccordionGroup>

### Kein Audio vom Agenten

**Symptom**: Sie sprechen, Transkript erscheint, aber der Agent antwortet nicht mit Stimme.

<AccordionGroup>
  <Accordion title="Lautsprecher-/Kopfhörerprobleme">
    **Prüfen**:
    - Lautstärke nicht stumm geschaltet
    - Korrektes Ausgabegerät ausgewählt
    - Lautsprecher/Kopfhörer eingesteckt und funktionsfähig

    **Lösung**: Passen Sie die Lautstärke an, wählen Sie die richtigen Lautsprecher in den Simulator-Einstellungen
  </Accordion>

  <Accordion title="Browser-Autoplay blockiert">
    **Symptom**: Erster Test funktioniert, nachfolgende Tests haben kein Audio

    **Ursache**: Browser-Autoplay-Richtlinie blockiert Audio

    **Lösung**: Interagieren Sie mit der Seite, bevor Sie den Test starten (klicken Sie irgendwo hin), oder erlauben Sie Autoplay in den Browsereinstellungen
  </Accordion>

  <Accordion title="WebRTC-Verbindung fehlgeschlagen">
    **Symptom**: Fehler "Verbindung fehlgeschlagen" in der Konsole

    **Ursache**: Firewall oder Netzwerk blockiert WebRTC

    **Lösung**:
    - Prüfen Sie, ob die Firewall WebRTC-Verkehr zulässt
    - Versuchen Sie ein anderes Netzwerk (mobiler Hotspot)
    - Aktivieren Sie TURN-Relay in den Agenteneinstellungen (Enterprise)
  </Accordion>
</AccordionGroup>

### Simulator hängt oder ist eingefroren

**Symptome**: Simulator startet nicht, hängt während des Anrufs oder beendet den Anruf nicht.

<AccordionGroup>
  <Accordion title="Browser-Cache leeren">
    **Schritte**:
    1. Öffnen Sie die Browser-Entwicklertools (F12)
    2. Rechtsklick auf Aktualisieren-Schaltfläche → "Cache leeren und erneut laden"
    3. Oder Site-Daten löschen: Einstellungen → Datenschutz → Browserdaten löschen

    **Lösung**: Löscht festgefahrenen Zustand von vorherigen Tests
  </Accordion>

  <Accordion title="Seite aktualisieren">
    **Einfache Lösung**: Seite neu laden (Cmd+R oder Ctrl+R)

    **Erzwungenes Neuladen**: Cmd+Shift+R (Mac) oder Ctrl+Shift+R (Windows)

    Nach dem Neuladen erneut testen
  </Accordion>

  <Accordion title="Browser-Konsole prüfen">
    **Schritte**:
    1. Öffnen Sie die Entwicklertools (F12)
    2. Gehen Sie zum Konsole-Tab
    3. Suchen Sie nach Fehlern (roter Text)

    **Häufige Fehler**:
    - `WebSocket connection failed` → Netzwerkproblem
    - `Microphone permission denied` → Berechtigungsproblem
    - `Uncaught TypeError` → Plattform-Bug, kontaktieren Sie den Support

    Erstellen Sie einen Screenshot des Fehlers und fügen Sie ihn dem Support-Ticket hinzu
  </Accordion>
</AccordionGroup>

## Telefontest-Probleme

### Testanruf verbindet nicht

**Symptom**: Klicken Sie auf "Test starten", aber das Telefon klingelt nie.

<AccordionGroup>
  <Accordion title="Telefonnummernformat überprüfen">
    **Korrektes Format**: E.164 mit Ländercode

    ✅ Korrekt: `+12125551234`
    ❌ Falsch: `212-555-1234`, `(212) 555-1234`, `2125551234`

    **Lösung**: Formatieren Sie auf E.164 um, versuchen Sie es erneut
  </Accordion>

  <Accordion title="'Von'-Nummer bereitgestellt prüfen">
    **Schritte**:
    1. Gehen Sie zu **Telefonie → Nummern**
    2. Finden Sie Ihre "Von"-Nummer
    3. Überprüfen Sie, ob der Status "Aktiv" ist, nicht "Ausstehend" oder "Fehlgeschlagen"

    **Wenn Ausstehend**: Warten Sie 5-10 Minuten auf die Bereitstellung
    **Wenn Fehlgeschlagen**: Kontaktieren Sie den Support mit Nummerndetails
  </Accordion>

  <Accordion title="Kontokapazität">
    **Prüfen**: Sind Sie bei maximalen gleichzeitigen Anrufen?

    Navigieren Sie zu **Dashboard → Nutzung** → Prüfen Sie "Gleichzeitige Anrufe"

    **Wenn am Limit**: Beenden Sie aktive Anrufe oder upgraden Sie den Plan
  </Accordion>

  <Accordion title="Carrier-Blockierung">
    **Symptom**: Anrufe von anderen Nummern funktionieren, aber nicht zu Ihrem Telefon

    **Ursache**: Ihr Carrier blockiert möglicherweise Anrufe (Spam-Filter)

    **Lösung**:
    - Versuchen Sie, ein anderes Telefon anzurufen
    - Fügen Sie die Nummer zu Ihren Kontakten hinzu
    - Überprüfen Sie die Spam-/Sperrliste des Carriers
    - Registrieren Sie CNAM (Anrufername) beim Carrier
  </Accordion>
</AccordionGroup>

### Schlechte Anrufqualität

**Symptome**: Abgehacktes Audio, Echo, Verzögerungen, robotische Stimme.

<AccordionGroup>
  <Accordion title="Netzwerkqualität">
    **Testen Sie Ihr Netzwerk**:
    - Führen Sie einen Geschwindigkeitstest durch: >5 Mbps Upload/Download
    - Prüfen Sie Ping: &lt;100ms
    - Testen Sie aus einem anderen Netzwerk (WLAN vs. Mobilfunk)

    **Lösung**:
    - Gehen Sie näher zum WLAN-Router
    - Wechseln Sie zu einer kabelgebundenen Verbindung
    - Testen Sie außerhalb der Stoßzeiten (weniger Überlastung)
  </Accordion>

  <Accordion title="Mobilfunk vs. WLAN">
    **Testen Sie beide**:
    - Anruf über Mobilfunknetz
    - Anruf über WLAN
    - Qualität vergleichen

    **Erkenntnis**: Eines viel besser als das andere? Verwenden Sie dieses Netzwerk für zukünftige Tests
  </Accordion>

  <Accordion title="Telefoneinstellungen">
    **Prüfen**:
    - Bluetooth getrennt (fügt Latenz hinzu)
    - WLAN-Anrufe deaktiviert (kann Probleme verursachen)
    - VPN deaktiviert (fügt Latenz hinzu)
    - Anklopfen deaktiviert (keine Unterbrechungen)

    **Lösung**: Passen Sie die Telefoneinstellungen an, testen Sie erneut
  </Accordion>

  <Accordion title="Agenten-Spracheinstellungen">
    **Versuchen Sie anzupassen**:
    - Sprachgeschwindigkeit (langsamer kann klarer sein)
    - Sprachangebot (ElevenLabs vs. Azure vs. Google)
    - Audioverarbeitungseinstellungen (Echounterdrückung, Geräuschunterdrückung)

    Navigieren Sie zu Agent → **Spracheinstellungen** zum Anpassen
  </Accordion>
</AccordionGroup>

### DTMF nicht erkannt

**Symptom**: Drücken Sie Telefontasten, aber der Agent erkennt die Eingabe nicht.

<AccordionGroup>
  <Accordion title="DTMF im Agenten aktiviert">
    **Prüfen**: Agent → **Erweitert → DTMF-Steuerung**

    **Überprüfen**:
    - [ ] DTMF-Erkennung aktiviert
    - [ ] Erkennungsmodus korrekt eingestellt (immer an vs. schlüsselwortaktiviert)
    - [ ] Tastenzuordnungen konfiguriert

    **Lösung**: Aktivieren Sie DTMF, falls deaktiviert, speichern und erneut testen
  </Accordion>

  <Accordion title="DTMF-Typ">
    **Zwei Typen**:
    - **RFC 2833** (in-band): Mit Sprachaudio übertragen
    - **SIP INFO** (out-of-band): Separate SIP-Nachrichten

    **Carrier prüfen**: Einige Carrier unterstützen nur einen Typ

    **Lösung**: Versuchen Sie, den DTMF-Typ in den Agenteneinstellungen zu ändern, oder verwenden Sie einen anderen Carrier/Telefon
  </Accordion>

  <Accordion title="Telefontastatur">
    **Test**: Drücken Sie Tasten während des Anrufs - hören Sie Töne?

    **Wenn keine Töne**: Telefon sendet möglicherweise SIP INFO anstelle von Tönen (nicht kompatibel)

    **Wenn Töne gehört werden**: Sollte funktionieren. Prüfen Sie Agentenprotokolle auf DTMF-Ereignisse. Wenn nicht protokolliert, kann es ein Carrier-Problem sein.
  </Accordion>
</AccordionGroup>

## Agentenverhalten-Probleme

### Agent folgt Anweisungen nicht

**Symptom**: Agent macht etwas anderes als das, was Sie in den Anweisungen angegeben haben.

<AccordionGroup>
  <Accordion title="Anweisungen zu vage">
    **Beispiel**:
    ```markdown
    ❌ Vage: "Seien Sie hilfsbereit und freundlich."

    ✅ Spezifisch: "Begrüßen Sie mit: 'Willkommen bei Acme Corp. Ich bin hier, um bei Bestellungen und Sendungsverfolgung zu helfen. Was kann ich heute für Sie tun?'"
    ```

    **Lösung**: Machen Sie Anweisungen expliziter und spezifischer
  </Accordion>

  <Accordion title="Anweisungen widersprüchlich">
    **Beispiel**:
    ```markdown
    ❌ Konflikt:
    "Seien Sie prägnant und kurz."
    "Geben Sie detaillierte Erklärungen für jede Funktion."

    ✅ Abgestimmt:
    "Seien Sie prägnant, es sei denn, der Kunde fragt nach mehr Details."
    ```

    **Lösung**: Entfernen Sie widersprüchliche Regeln, priorisieren Sie ein Verhalten
  </Accordion>

  <Accordion title="Modellbeschränkungen">
    **Realitätscheck**:
    - Modelle befolgen Anweisungen nicht zu 100 % perfekt
    - Komplexe Anweisungen sind schwerer zu befolgen als einfache
    - Lange Anweisungen (>500 Wörter) verringern die Einhaltung

    **Lösung**:
    - Vereinfachen Sie Anweisungen
    - Testen Sie mit einem anderen Modell (GPT-4 vs. Claude)
    - Fügen Sie Beispiele für gewünschtes Verhalten hinzu
  </Accordion>

  <Accordion title="Isoliert testen">
    **Minimaltest**:
    1. Erstellen Sie einen neuen Agenten mit minimalen Anweisungen
    2. Testen Sie ein einzelnes fragliches Verhalten
    3. Wenn es funktioniert → Problem ist Komplexität/Konflikt in vollständigen Anweisungen
    4. Wenn es nicht funktioniert → Kann Modellbeschränkung oder unrealistische Erwartung sein

    **Iterieren**: Fügen Sie schrittweise Anweisungen hinzu, bis das Problem wieder auftritt, beheben Sie dann diese spezifische Anweisung
  </Accordion>
</AccordionGroup>

### Transkriptionsfehler

**Symptom**: Agent verhört sich konsequent bei bestimmten Wörtern oder Phrasen.

<AccordionGroup>
  <Accordion title="Benutzerdefiniertes Vokabular hinzufügen">
    **Für**: Technische Begriffe, Produktnamen, Akronyme

    **Schritte**:
    1. Gehen Sie zu Agent → **Transcriber → Benutzerdefiniertes Vokabular**
    2. Fügen Sie falsch gehörte Wörter mit phonetischer Aussprache hinzu
    3. Testen Sie erneut

    **Beispiel**:
    ```json
    {
      "custom_words": [
        {"word": "itellicoAI", "pronunciation": "eye-TELL-ih-koh AI"},
        {"word": "Kubernetes", "pronunciation": "koo-ber-NET-eez"}
      ]
    }
    ```
  </Accordion>

  <Accordion title="Versuchen Sie einen anderen Transcriber">
    **Optionen**:
    - **Deepgram**: Am besten für Akzente, Geräusche, Echtzeit
    - **Azure**: Am besten für technische Begriffe, hohe Genauigkeit
    - **Whisper**: Am besten für mehrsprachig, offline

    **Test**: Wechseln Sie den Transcriber in den Agenteneinstellungen, führen Sie denselben Test durch, vergleichen Sie die Genauigkeit
  </Accordion>

  <Accordion title="Audioqualität verbessern">
    **Faktoren**:
    - Hintergrundgeräusche → Verwenden Sie eine ruhige Umgebung
    - Schlechtes Mikrofon → Verwenden Sie ein besseres Mikrofon/Headset
    - Niedrige Lautstärke → Sprechen Sie lauter oder näher am Mikrofon
    - Nuscheln → Sprechen Sie klar und langsam

    **Test**: Gleicher Test unter idealen Bedingungen (ruhiger Raum, gutes Mikrofon, klare Sprache)
  </Accordion>
</AccordionGroup>

### Aktionen funktionieren nicht

**Symptom**: Benutzerdefinierte Aktionen werden nicht ausgelöst oder schlagen fehl, wenn sie ausgelöst werden.

<AccordionGroup>
  <Accordion title="Aktion wird nicht ausgelöst">
    **Anweisungen prüfen**: Erwähnen sie, wann die Aktion verwendet werden soll?

    **Beispiel**:
    ```markdown
    ❌ Fehlend: Keine Erwähnung der Aktion in den Anweisungen

    ✅ Explizit: "Wenn der Kunde nach dem Bestellstatus fragt, verwenden Sie die getOrderStatus-Aktion mit der Bestellnummer."
    ```

    **Test**: Fordern Sie die Aktion explizit im Test an: "Überprüfen Sie den Bestellstatus für Bestellung 12345"
  </Accordion>

  <Accordion title="Fehlende Parameter">
    **Prüfen**: Benötigt die Aktion Parameter, die der Kunde nicht bereitgestellt hat?

    **Beispiel**: Aktion benötigt `order_id`, aber der Kunde sagte nur "überprüfen Sie meine Bestellung" ohne Nummer

    **Lösung in Anweisungen**:
    ```markdown
    "Wenn der Kunde nach einer Bestellung fragt, aber keine Bestellnummer angibt, fragen Sie: 'Was ist Ihre Bestellnummer?'"
    ```
  </Accordion>

  <Accordion title="API-Fehler">
    **Protokolle prüfen**: Konversationen → Protokolle → Aktionen-Tab

    **Häufige Fehler**:
    - 401 Unauthorized → Prüfen Sie den API-Schlüssel
    - 404 Not Found → Überprüfen Sie die Endpunkt-URL
    - Timeout → Erhöhen Sie die Timeout-Grenze

    **API manuell testen**:
    ```bash
    curl -X POST https://api.example.com/endpoint \
         -H "Authorization: Bearer YOUR_KEY" \
         -d '{"param": "value"}'
    ```

    Wenn der manuelle Aufruf funktioniert, liegt das Problem in der Aktionskonfiguration
  </Accordion>
</AccordionGroup>

## Plattformprobleme

### Dashboard lädt nicht

**Symptome**: Leere Seite, lädt ewig, Fehler.

<AccordionGroup>
  <Accordion title="Plattformstatus prüfen">
    Besuchen Sie: `https://status.itellico.ai`

    **Bei Ausfall**: Warten Sie auf Lösung, prüfen Sie Statusseite auf Updates

    **Wenn betriebsbereit**: Problem ist wahrscheinlich lokal (Browser, Netzwerk, Cache)
  </Accordion>

  <Accordion title="Browser-Cache leeren">
    **Schritte**:
    1. Öffnen Sie Einstellungen (Cmd+, oder Ctrl+,)
    2. Datenschutz → Browserdaten löschen
    3. Wählen Sie "Zwischengespeicherte Bilder und Dateien"
    4. Daten löschen
    5. Dashboard neu laden

    **Erzwungenes Neuladen**: Cmd+Shift+R oder Ctrl+Shift+R
  </Accordion>

  <Accordion title="Versuchen Sie einen anderen Browser">
    **Testen Sie in**:
    - Chrome (Inkognito-Modus)
    - Firefox (privates Fenster)
    - Safari (privates Surfen)

    **Wenn es im Inkognito-/privaten Modus funktioniert**: Problem ist Browser-Erweiterung oder zwischengespeicherte Daten

    **Lösung**: Deaktivieren Sie Erweiterungen nacheinander, um den Konflikt zu finden
  </Accordion>

  <Accordion title="Browser-Konsole prüfen">
    **Schritte**:
    1. F12 zum Öffnen der Entwicklertools
    2. Konsole-Tab
    3. Suchen Sie nach Fehlern

    **Häufig**:
    - CORS-Fehler → Kontaktieren Sie den Support
    - 401/403-Fehler → Sitzung abgelaufen, melden Sie sich ab und wieder an
    - JavaScript-Fehler → Browser-Kompatibilitätsproblem

    Screenshot-Fehler für Support-Ticket
  </Accordion>
</AccordionGroup>

### Sitzung abgelaufen / Auth-Probleme

**Symptome**: Unerwartet abgemeldet, "nicht autorisiert"-Fehler.

<AccordionGroup>
  <Accordion title="Erneut anmelden">
    **Einfache Lösung**: Abmelden und wieder anmelden

    **Wenn es weiterhin besteht**: Cookies und Cache löschen, dann anmelden
  </Accordion>

  <Accordion title="API-Schlüsselprobleme">
    **Wenn Sie die API verwenden**: Prüfen Sie, ob der Schlüssel gültig ist

    **Schritte**:
    1. Gehen Sie zu **Konten → API-Schlüssel**
    2. Überprüfen Sie, ob der Schlüssel existiert und nicht abgelaufen ist
    3. Generieren Sie bei Bedarf neu
    4. Aktualisieren Sie in Ihrem Code/Ihrer Konfiguration

    **Sicherheit**: Commiten Sie niemals API-Schlüssel in die Versionskontrolle
  </Accordion>

  <Accordion title="Berechtigungen">
    **Prüfen**: Haben Sie die richtige Rolle?

    **Rollen**:
    - Admin: Vollzugriff
    - Developer: Kann Agenten bearbeiten, kann keine Abrechnung verwalten
    - Viewer: Nur-Lesen

    **Lösung**: Bitten Sie den Kontoinhaber, Ihre Berechtigungen anzupassen
  </Accordion>
</AccordionGroup>

## Hilfe erhalten

Wenn Sie ein Problem nicht lösen können:

<Steps>
  <Step title="Informationen sammeln">
    Sammeln Sie vor der Kontaktaufnahme mit dem Support:
    - Agenten-ID
    - Konversations-ID (bei spezifischem Anruf)
    - Zeitstempel des Problems
    - Was Sie erwartet haben vs. was passiert ist
    - Schritte zur Reproduktion
    - Screenshots oder Bildschirmaufzeichnung
    - Browser-/Geräteinformationen
  </Step>

  <Step title="Dokumentation prüfen">
    Suchen Sie in den Dokumenten nach ähnlichen Problemen:
    - Dieser Fehlerbehebungsleitfaden
    - Agentenkonfigurationsdokumente
    - API-Referenz (für Integrationsprobleme)
  </Step>

  <Step title="Community-Forum">
    Besuchen Sie: `https://community.itellico.ai`

    - Suchen Sie nach ähnlichen Problemen
    - Stellen Sie Ihre Frage mit Details
    - Community und Mitarbeiter antworten
  </Step>

  <Step title="Support kontaktieren">
    E-Mail: `support@itellico.ai`

    Einschließen:
    - Klare Betreffzeile (z.B. "Testanruf verbindet nicht - Agenten-ID: agent_123")
    - Detaillierte Beschreibung
    - Schritte zur Reproduktion
    - Alle Informationen aus Schritt 1
    - Was Sie bereits versucht haben

    **Antwortzeiten**:
    - Kritisch (Produktion ausgefallen): &lt;1 Stunde
    - Hoch (Tests blockiert): &lt;4 Stunden
    - Mittel: &lt;24 Stunden
  </Step>
</Steps>
