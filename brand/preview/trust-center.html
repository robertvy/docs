Data Protection & GDPR

Roles, rights, and processing agreements.

▼
  • ✓ Role Separation: We act strictly as a Processor (Art. 28 GDPR). You retain full Controller status.
  • ✓ Data Minimization: We only process what is necessary for voice transmission and transcription.
  • ✓ Deletion Routines: Automatic deletion of PII after configurable retention periods (default 30 days).
  • ✓ No Third-Party Training: Customer data is contractually excluded from AI model training sets.
  • ✓ Subject Rights: Built-in tools to export or delete end-user data upon request (Art. 15-17 GDPR).
🔒

Technical Security (TOMs)

Encryption, access control, and resilience.

▼
  • ✓ Encryption Everywhere: TLS 1.3 for transit, AES-256 for storage (S3/DB).
  • ✓ Network Isolation: VPCs with strict firewall rules; public access only via secured API gateways.
  • ✓ Access Control: Role-Based Access Control (RBAC) and MFA enforcement for all admin access.
  • ✓ Resilience: Multi-AZ deployment (min 3 zones) for high availability and disaster recovery.
  • ✓ Logging: Comprehensive audit trails for system access and data modification events.
🤖

AI Transparency & Ethics

EU AI Act compliance and disclosure.

▼

In accordance with the EU AI Act, transparency is mandatory. Our system supports:

  • ✓ Automated Disclosure: The assistant can proactively state "I am an AI assistant" at the start of calls.
  • ✓ Human Handoff: Clear escalation paths to human agents when requested or when confidence is low.
  • ✓ Content Tagging: Generated content (audio/text) is metadata-tagged as AI-generated.
