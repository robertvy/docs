## Tests dans votre navigateur

Le simulateur web vous permet de tester votre agent rapidement sans passer d'appels téléphoniques. Cliquez sur **Tester l'agent** dans l'en-tête de l'agent pour ouvrir le panneau de test sur le côté droit. Vous pouvez basculer entre les onglets **Appel web** et **Appel téléphonique** pour tester différents scénarios.

## Comment effectuer un test web

<Steps>
  <Step title="Ouvrir votre agent">
    Allez dans **Agents IA** et sélectionnez l'agent que vous souhaitez tester. Assurez-vous que vos dernières instructions sont enregistrées.
  </Step>
  <Step title="Ouvrir Tester l'agent">
    Cliquez sur **Tester l'agent** en haut à droite. Le panneau de test s'ouvre sur le côté droit de l'écran.
  </Step>
  <Step title="Choisir Appel web">
    Assurez-vous que **Appel web** est sélectionné en haut du panneau de test.
  </Step>
  <Step title="Commencer à tester">
    Cliquez sur **Démarrer le test**. Un widget d'appel apparaît pour que vous puissiez parler avec votre agent en utilisant votre microphone ou en tapant des messages.
  </Step>
</Steps>

## Utilisation des variables de test

Les variables de test vous permettent de simuler différents scénarios sans modifier les instructions de votre agent :

- Toutes les variables que vous avez définies dans votre salutation et vos instructions apparaissent dans le panneau de test
- Mettez à jour les valeurs pour tester différentes situations (par ex., "Client VIP", "Compte d'essai")
- Cliquez sur **Enregistrer les modifications** pour conserver ces valeurs pour les tests futurs

## Pendant l'appel de test

Après avoir cliqué sur **Démarrer le test**, un widget d'appel apparaît avec ces fonctionnalités :

- **État de connexion** - Voyez si l'appel est en cours de connexion, d'écoute ou de parole
- **Transcription en direct** - Lisez ce que vous et l'agent dites en temps réel
- **Suivi des actions** - Voyez quand l'agent déclenche des actions comme des transferts ou des réservations
- **Contrôles d'appel** - Réduisez, agrandissez ou terminez l'appel
- **Reste actif** - Le widget reste ouvert même si vous naviguez vers d'autres pages

Vous pouvez continuer à travailler dans le tableau de bord pendant que l'appel de test est actif.

## Examen des résultats de test

Après avoir terminé un appel de test :

1. Allez dans l'onglet **Conversations** dans la navigation principale
2. Trouvez votre appel de test (le plus récent en haut)
3. Examinez la transcription, les objectifs atteints et toutes les actions déclenchées
4. Écoutez la lecture audio si nécessaire
5. Vérifiez que tout a fonctionné comme prévu

<Tip>
Vous pouvez supprimer les conversations de test pour garder votre historique de conversation axé sur les interactions clients réelles.
</Tip>

## Quand passer aux tests téléphoniques

Utilisez le simulateur web pour une itération rapide, mais passez aux tests téléphoniques lorsque vous devez :

- Tester la qualité vocale et le timing de parole naturel
- Vérifier les paramètres de détection de répondeur (AMD)
- Tester les transferts d'appels vers de vrais numéros de téléphone
- Confirmer que l'expérience correspond à ce que les clients entendront

<Note>
**Différences de performance :** Les tests web utilisent WebRTC avec des codecs de haute qualité et n'ont pas d'infrastructure PSTN impliquée, donc la latence est plus rapide et la qualité audio est supérieure aux appels téléphoniques. Testez toujours avec de vrais appels téléphoniques avant le lancement pour vivre ce que vos clients entendront réellement.
</Note>

Consultez le [guide de tests téléphoniques](/fr/test/phone-testing) pour des instructions détaillées sur les tests avec de vrais appels téléphoniques.
