## Aperçu

Les prononciations personnalisées vous permettent de définir comment des mots spécifiques doivent sonner, en vous assurant que votre agent prononce toujours correctement les noms de marque, les termes techniques et les acronymes.

**Comment ça fonctionne :**
1. L'agent génère une réponse : "Appelez-nous chez itellicoAI pour de l'aide"
2. Le système applique les règles de prononciation : "Appelez-nous chez eye-tell-ee-co A I pour de l'aide"
3. Le synthétiseur vocal lit le texte modifié
4. Le client entend la prononciation correcte

<Note>
Les remplacements de prononciation sont disponibles sous **Voix > Prononciations** dans la configuration de votre agent.
</Note>

## Ajouter des prononciations

### Remplacement unique

1. Cliquez sur **Ajouter un remplacement**
2. Entrez le **mot cible** (le mot à remplacer)
3. Entrez la **prononciation** (comment il devrait sonner)
4. Cliquez sur **Ajouter**

**Exemples :**
- **Cible :** Nike → **Prononciation :** Nai-key
- **Cible :** API → **Prononciation :** A-P-I
- **Cible :** 24/7 → **Prononciation :** vingt-quatre sept
- **Cible :** EST → **Prononciation :** Heure de l'Est

<Warning>
**Les mots cibles doivent être des mots uniques seulement** - les espaces ne sont pas autorisés. Utilisez des mots complets comme `Nike` ou `API`, pas des phrases comme `service client`.
</Warning>

### Importation en masse via CSV

Pour importer plusieurs prononciations à la fois :

1. Cliquez sur **Importer CSV**
2. Téléchargez le modèle si nécessaire
3. Préparez votre CSV avec deux colonnes : `target`, `replacement`
4. Glissez-déposez votre fichier CSV ou cliquez pour sélectionner
5. Examinez l'aperçu et cliquez sur **Importer**

**Format CSV :**
```csv
target,replacement
Nike,Nai-key
EST,Heure de l'Est
24/7,vingt-quatre sept
API,A-P-I
```

**Limites :**
- Maximum 500 prononciations par agent
- Taille max du fichier CSV : 5MB

## Conseils d'orthographe phonétique

### Orthographe phonétique simple

Écrivez les remplacements sous forme de texte phonétique simple et lisible :

**Bons exemples :**
- Nike → Nai-key
- SQL → sequel (ou) S-Q-L
- Wi-Fi → why-figh
- iOS → eye-oh-ess

**Conseils :**
- Utilisez des tirets pour séparer les syllabes : Nai-key, A-P-I
- Épelez-le comme vous expliqueriez la prononciation à quelqu'un
- Testez avec votre voix pour entendre comment ça sonne

### Modèles courants

**Acronymes (épeler chaque lettre) :**
- API → A-P-I
- URL → U-R-L
- HTTP → H-T-T-P
- AWS → A-W-S

**Noms de marque (orthographe phonétique) :**
- Nike → Nai-key
- Adidas → ah-dee-dahs
- Chipotle → chih-poht-lay

**Nombres et heures :**
- 24/7 → vingt-quatre sept
- 9am → neuf A-M
- EST → Heure de l'Est
- 5pm → cinq P-M

**Termes techniques :**
- SQL → sequel (ou) S-Q-L
- API → A-P-I
- URL → you-are-ell
- WiFi → why-figh

## Avancé : Phonèmes IPA

Pour un contrôle précis de la prononciation, vous pouvez utiliser la notation de l'Alphabet Phonétique International (IPA) avec le format `<< >>` :

**Format :** `<<phonème|phonème|phonème>>`

**Exemple :**
```
espresso → <<ɛ|s|p|r|ɛ|s|oʊ>>
```

<Note>
Les phonèmes IPA offrent un contrôle précis mais nécessitent une connaissance de la notation phonétique. L'orthographe phonétique simple (Nai-key) fonctionne bien pour la plupart des cas.
</Note>

## Gérer les prononciations

### Modifier les remplacements existants

1. Cliquez sur n'importe quelle puce de remplacement dans la liste
2. Modifiez la cible ou la prononciation
3. Cliquez sur **Enregistrer**

### Supprimer les remplacements

- Cliquez sur l'**icône corbeille** sur n'importe quelle puce de remplacement pour la supprimer
- Ou cliquez sur **Tout effacer** pour supprimer toutes les prononciations (nécessite une confirmation)

### Exporter les prononciations

1. Cliquez sur **Exporter CSV**
2. Le fichier CSV se télécharge avec toutes vos prononciations actuelles
3. Format du nom de fichier : `{nom_agent}_pronunciations.csv`

Utilisez les fichiers CSV exportés pour sauvegarder vos prononciations ou les copier vers d'autres agents.

## Tester les prononciations

Après avoir ajouté des prononciations :

1. Faites un appel test à votre agent
2. Déclenchez des réponses contenant les mots cibles
3. Écoutez pour vérifier que la prononciation sonne correcte
4. Ajustez l'orthographe phonétique si nécessaire

**Conseils de test :**
- Testez en contexte, pas isolément
- Faites écouter aux membres de l'équipe
- Ajustez la césure ou l'orthographe si ça ne sonne pas bien

## Bonnes pratiques

<AccordionGroup>
  <Accordion title="Commencez par les problèmes courants" icon="bullseye">
    Ajoutez des prononciations pour les mots que vous savez mal prononcés dans les appels réels. Examinez les transcriptions d'appels pour identifier les problèmes de prononciation récurrents.
  </Accordion>

  <Accordion title="Restez simple" icon="sparkles">
    Utilisez l'orthographe phonétique (Nai-key) plutôt que l'IPA complexe sauf si nécessaire. Le texte phonétique simple est plus facile à maintenir et fonctionne bien pour la plupart des cas.
  </Accordion>

  <Accordion title="Testez avant de déployer" icon="vial">
    Testez toujours les prononciations avec de vrais appels avant de passer en production. Faites un appel test et déclenchez des réponses contenant les mots cibles pour vérifier qu'ils sonnent correctement.
  </Accordion>

  <Accordion title="Documentez vos règles" icon="file-arrow-down">
    Exportez régulièrement votre CSV en guise de sauvegarde. Utilisez le bouton Exporter CSV pour enregistrer vos prononciations et les garder versionnées avec la configuration de votre agent.
  </Accordion>

  <Accordion title="Utilisez une orthographe cohérente" icon="check">
    Si vous avez "API" épelé comme "A-P-I", utilisez ce format de manière cohérente dans toutes vos prononciations. La cohérence améliore la maintenabilité.
  </Accordion>

  <Accordion title="Mots uniques seulement" icon="letter-a">
    N'oubliez pas que les mots cibles ne peuvent pas contenir d'espaces - seuls les mots uniques complets fonctionnent. Utilisez `Nike` ou `API`, pas des phrases comme `service client`.
  </Accordion>
</AccordionGroup>
