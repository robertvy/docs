## Aperçu

Les prononciations personnalisées te permettent de définir comment des mots spécifiques doivent sonner, en vous assurant que ton agent prononce toujours correctement les noms de marque, les termes techniques et les acronymes.

**Comment ça fonctionne :**
1. L'agent génère une réponse : "Appelez-nous chez itellicoAI pour de l'aide"
2. Le système applique les règles de prononciation : "Appelez-nous chez eye-tell-ee-co A I pour de l'aide"
3. Le synthétiseur vocal lit le texte modifié
4. Le client entend la prononciation correcte

<Note>
Les remplacements de prononciation sont disponibles sous **Voix > Prononciations** dans la configuration de ton agent.
</Note>

## Ajouter des prononciations

### Remplacement unique

1. Clique sur **Ajouter un remplacement**
2. Entre le **mot cible** (le mot à remplacer)
3. Entre la **prononciation** (comment il devrait sonner)
4. Clique sur **Ajouter**

**Exemples :**
- **Cible :** Nike → **Prononciation :** Nai-key
- **Cible :** API → **Prononciation :** A-P-I
- **Cible :** 24/7 → **Prononciation :** vingt-quatre sept
- **Cible :** EST → **Prononciation :** Heure de l'Est

<Warning>
**Les mots cibles doivent être des mots uniques seulement** - les espaces ne sont pas autorisés. Utilise des mots complets comme `Nike` ou `API`, pas des phrases comme `service client`.
</Warning>

### Importation en masse via CSV

Pour importer plusieurs prononciations à la fois :

1. Clique sur **Importer CSV**
2. Télécharge le modèle si nécessaire
3. Prépare ton CSV avec deux colonnes : `target`, `replacement`
4. Glisse-Dépose ton fichier CSV ou Clique pour sélectionner
5. Examine l'aperçu et Clique sur **Importer**

**Format CSV :**
```csv
target,replacement
Nike,Nai-key
EST,Heure de l'Est
24/7,vingt-quatre sept
API,A-P-I
```

**Limites :**
- Maximum 500 prononciations par agent
- Taille max du fichier CSV : 5MB

## Conseils d'orthographe phonétique

### Orthographe phonétique simple

Écris les remplacements sous forme de texte phonétique simple et lisible :

**Bons exemples :**
- Nike → Nai-key
- SQL → sequel (ou) S-Q-L
- Wi-Fi → why-figh
- iOS → eye-oh-ess

**Conseils :**
- Utilise des tirets pour séparer les syllabes : Nai-key, A-P-I
- Épelez-le comme vous expliqueriez la prononciation à quelqu'un
- Teste avec ton voix pour entendre comment ça sonne

### Modèles courants

**Acronymes (épeler chaque lettre) :**
- API → A-P-I
- URL → U-R-L
- HTTP → H-T-T-P
- AWS → A-W-S

**Noms de marque (orthographe phonétique) :**
- Nike → Nai-key
- Adidas → ah-dee-dahs
- Chipotle → chih-poht-lay

**Nombres et heures :**
- 24/7 → vingt-quatre sept
- 9am → neuf A-M
- EST → Heure de l'Est
- 5pm → cinq P-M

**Termes techniques :**
- SQL → sequel (ou) S-Q-L
- API → A-P-I
- URL → you-are-ell
- WiFi → why-figh

## Avancé : Phonèmes IPA

Pour un contrôle précis de la prononciation, tu peux utiliser la notation de l'Alphabet Phonétique International (IPA) avec le format `<< >>` :

**Format :** `<<phonème|phonème|phonème>>`

**Exemple :**
```
espresso → <<ɛ|s|p|r|ɛ|s|oʊ>>
```

<Note>
Les phonèmes IPA offrent un contrôle précis mais nécessitent une connaissance de la notation phonétique. L'orthographe phonétique simple (Nai-key) fonctionne bien pour la plupart des cas.
</Note>

## Gérer les prononciations

### Modifier les remplacements existants

1. Clique sur n'importe quelle puce de remplacement dans la liste
2. Modifie la cible ou la prononciation
3. Clique sur **Enregistrer**

### Supprimer les remplacements

- Clique sur l'**icône corbeille** sur n'importe quelle puce de remplacement pour la supprimer
- Ou Clique sur **Tout effacer** pour supprimer toutes les prononciations (nécessite une confirmation)

### Exporter les prononciations

1. Clique sur **Exporter CSV**
2. Le fichier CSV se télécharge avec toutes tes prononciations actuelles
3. Format du nom de fichier : `{nom_agent}_pronunciations.csv`

Utilise les fichiers CSV exportés pour sauvegarder tes prononciations ou les copier vers d'autres agents.

## Tester les prononciations

Après avoir ajouté des prononciations :

1. Fais un appel test à ton agent
2. Déclenchez des réponses contenant les mots cibles
3. Écoutez pour vérifier que la prononciation sonne correcte
4. Ajuste l'orthographe phonétique si nécessaire

**Conseils de test :**
- Teste en contexte, pas isolément
- Fais écouter aux membres de l'équipe
- Ajuste la césure ou l'orthographe si ça ne sonne pas bien

## Bonnes pratiques

<AccordionGroup>
  <Accordion title="Commence par les problèmes courants" icon="bullseye">
    Ajoute des prononciations pour les mots que tu sais mal prononcés dans les appels réels. Examine les transcriptions d'appels pour identifier les problèmes de prononciation récurrents.
  </Accordion>

  <Accordion title="Restez simple" icon="sparkles">
    Utilise l'orthographe phonétique (Nai-key) plutôt que l'IPA complexe sauf si nécessaire. Le texte phonétique simple est plus facile à maintenir et fonctionne bien pour la plupart des cas.
  </Accordion>

  <Accordion title="Teste avant de déployer" icon="vial">
    Teste toujours les prononciations avec de vrais appels avant de passer en production. Fais un appel test et déclenchez des réponses contenant les mots cibles pour vérifier qu'ils sonnent correctement.
  </Accordion>

  <Accordion title="Documentez tes règles" icon="file-arrow-down">
    Exporte régulièrement ton CSV en guise de sauvegarde. Utilise le bouton Exporter CSV pour enregistrer tes prononciations et les garder versionnées avec la configuration de ton agent.
  </Accordion>

  <Accordion title="Utilise une orthographe cohérente" icon="check">
    Si tu as "API" épelé comme "A-P-I", Utilise ce format de manière cohérente dans toutes tes prononciations. La cohérence améliore la maintenabilité.
  </Accordion>

  <Accordion title="Mots uniques seulement" icon="letter-a">
    N'oubliez pas que les mots cibles ne peuvent pas contenir d'espaces - seuls les mots uniques complets fonctionnent. Utilise `Nike` ou `API`, pas des phrases comme `service client`.
  </Accordion>
</AccordionGroup>
