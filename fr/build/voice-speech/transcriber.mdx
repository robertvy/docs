## Aperçu

Le transcripteur est la première étape dans le pipeline de traitement de votre agent. Il convertit la parole du client en texte que le modèle IA peut comprendre et auquel répondre. Une transcription précise est critique—les erreurs à cette étape se répercutent sur l'ensemble de la conversation.

<Note>
Configurez le transcripteur sous **Modèles > Transcripteur** dans les paramètres de votre agent. Les modifications s'appliquent immédiatement.
</Note>

    - Utilisez le mode multilingue Nova-3 lorsque vous servez plusieurs langues de cette liste

    **Langue pas dans Deepgram ?** → Azure Speech (150+ langues)

  </Accordion>

    - Azure Speech (~500-700ms)

    **Couverture linguistique plus importante que la vitesse ?**
    - Azure Speech (150+ langues, sacrifie la vitesse)

    - Azure Speech (régions UE : Europe Ouest, Europe Nord)
    - Deepgram utilise des endpoints UE mais les données peuvent être traitées sur des serveurs US

## Configuration de Langue

### Configuration Langue Unique

Pour les agents servant une langue :

1. Ouvrez **Modèles > Transcripteur**
2. Utilisez le **Sélecteur de Langue** pour filtrer les transcripteurs
3. Sélectionnez par nom de langue (ex., "Français") ou code de locale (ex., "fr-FR")
4. Choisissez le transcripteur avec la meilleure latence/précision pour vos besoins

**Variantes linguistiques courantes :**
- **en-US :** Anglais américain
- **en-GB :** Anglais britannique
- **en-AU :** Anglais australien
- **en-IN :** Anglais indien
- **de-DE :** Allemand
- **es-ES :** Espagnol (Espagne)
- **fr-FR :** Français
- **zh-CN :** Chinois (Simplifié)
- **zh-TW :** Chinois (Traditionnel)

Sélectionnez la variante correspondant à votre base de clients pour une meilleure précision.

### Support Multilingue

**Deepgram Nova-3 Multilingue :**
- Modèle unique supportant anglais, espagnol, français, allemand, hindi, russe, portugais, japonais, italien, néerlandais
- Pas de sélection de langue nécessaire—gère automatiquement les 10 langues

**Azure Speech Multilingue :**
- Sélectionnez plusieurs langues (2-10) pour la détection automatique
- Azure détecte la langue depuis les premiers mots et transcrit en conséquence

## Mots-clés

Aidez votre transcripteur à reconnaître avec précision les noms de marque et les termes techniques.

<Note>
Les mots-clés sont supportés sur les modèles Deepgram Nova-2 et Azure Speech. Nova-3 ne supporte actuellement pas les mots-clés.
</Note>

### Quoi Booster

Les transcripteurs peuvent avoir du mal avec :
- Noms de marque (entreprise, produits, concurrents)
- Jargon de l'industrie (termes techniques, acronymes)
- Noms propres (noms de personnes, lieux)

### Comment Ajouter des Mots-clés

1. Ouvrez **Modèles > Transcripteur**
2. Sélectionnez un transcripteur Deepgram Nova-2 ou Azure Speech
3. Dans la section **Mots-clés de Reconnaissance** en bas, tapez les mots-clés et appuyez sur Entrée après chacun
4. Les mots-clés sont sauvegardés automatiquement

Ajoutez des mots-clés au besoin pendant les tests. Examinez les transcriptions d'appels pour les mots mal entendus et ajoutez-les comme mots-clés pour améliorer la reconnaissance.

## Tester la Transcription

1. Passez un appel test et parlez de scénarios que les clients utilisent (noms de marque, adresses)
2. Ouvrez le journal de conversation et examinez la transcription
3. Notez les mots mal entendus
4. Ajoutez les erreurs récurrentes aux mots-clés (si vous utilisez Nova-2 ou Azure)
5. Changez de transcripteurs si les problèmes persistent

**Ce qu'il faut tester :**
- Noms de marque et de produits
- Terminologie technique
- Numéros et adresses
- Différents accents et schémas de parole
- Conditions de bruit de fond
