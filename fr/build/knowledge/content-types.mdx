## Éléments Texte

### Que sont les Éléments Texte ?

Les éléments texte sont du contenu que vous saisissez directement dans l'éditeur de base de connaissances itellicoAI. C'est le type de contenu le plus simple et le plus fiable.

### Quand Utiliser les Éléments Texte

<AccordionGroup>
  <Accordion title="Rédiger des FAQ" icon="circle-question">
    Créer des paires question-réponse directement dans le système.

    **Exemple :**
    ```text wrap
    Titre : Comment réinitialiser mon mot de passe ?

    Contenu :
    Pour réinitialiser votre mot de passe :

    1. Allez sur la page de connexion
    2. Cliquez sur "Mot de passe oublié"
    3. Entrez votre adresse e-mail
    4. Vérifiez votre e-mail pour un lien de réinitialisation
    5. Cliquez sur le lien et créez un nouveau mot de passe

    Exigences du mot de passe :
    - Minimum 8 caractères
    - Au moins une lettre majuscule
    - Au moins un chiffre
    - Au moins un caractère spécial

    Si vous ne recevez pas l'e-mail dans les 5 minutes, vérifiez votre dossier
    spam ou contactez support@company.com.
    ```
  </Accordion>

  <Accordion title="Créer des résumés de politique" icon="book">
    Rédiger des déclarations de politique claires et concises.

    **Exemple :**
    ```text wrap
    Titre : Politique de Retour - Produits Numériques

    Contenu :
    Les produits numériques peuvent être remboursés dans les 30 jours suivant l'achat si :

    Éligible au remboursement :
    - Le produit présente un défaut technique empêchant son utilisation
    - La description du produit était matériellement inexacte
    - Le client n'a pas accédé ni téléchargé le produit

    Non éligible au remboursement :
    - Changement d'avis après avoir accédé au produit
    - Problèmes de compatibilité divulgués dans la description du produit
    - Erreur utilisateur ou mauvaise compréhension des fonctionnalités

    Pour demander un remboursement :
    Envoyer un e-mail à support@company.com avec votre numéro de commande et la raison
    de la demande de remboursement.

    Délai de traitement : 5-7 jours ouvrables
    Méthode de remboursement : Mode de paiement d'origine
    ```
  </Accordion>

  <Accordion title="Documenter les procédures" icon="list-check">
    Instructions étape par étape pour les processus.

    **Exemple :**
    ```
    Titre : Processus de Modification de Commande

    Contenu :
    Les clients peuvent modifier les commandes dans les 24 heures suivant le placement.

    Ce qui peut être modifié :
    - Adresse de livraison
    - Vitesse de livraison
    - Quantités d'articles (si inventaire disponible)

    Ce qui ne peut pas être modifié :
    - Mode de paiement (doit annuler et repasser commande)
    - Articles après le début du traitement
    - Commandes passées il y a plus de 24 heures

    Processus de modification :
    1. Le client contacte le support par téléphone ou e-mail
    2. L'agent vérifie que la commande est dans la fenêtre de modification
    3. L'agent vérifie l'inventaire pour les changements demandés
    4. L'agent met à jour la commande dans le système
    5. Le client reçoit un e-mail de confirmation

    Si la commande a déjà été expédiée, le client doit utiliser le processus de retour.
    ```
  </Accordion>

  <Accordion title="Informations de référence rapide" icon="clock">
    Informations brèves et fréquemment référencées.

    **Exemple :**
    ```
    Titre : Horaires d'Ouverture & Coordonnées

    Contenu :
    Support Client :
    - Téléphone : 1-800-555-0123
    - E-mail : support@company.com
    - Horaires : Lundi-Vendredi, 9h-18h EST
    - Après-heures : emergency@company.com (problèmes urgents uniquement)

    Ventes :
    - Téléphone : 1-800-555-0124
    - E-mail : sales@company.com
    - Horaires : Lundi-Vendredi, 8h-20h EST

    Facturation :
    - Téléphone : 1-800-555-0125
    - E-mail : billing@company.com
    - Horaires : Lundi-Vendredi, 9h-17h EST
    ```
  </Accordion>
</AccordionGroup>

### Créer des Éléments Texte

<Steps>
  <Step title="Navigue vers le dossier">
    Ouvre le dossier où tu souhaites ajouter l'élément.
  </Step>

  <Step title="Cliquer sur 'Ajouter un Élément de Connaissance'">
    Clique sur le bouton pour créer un nouvel élément.
  </Step>

  <Step title="Sélectionner la catégorie 'TEXT'">
    Choisis TEXT parmi les options de type de contenu.
  </Step>

  <Step title="Entrer le titre">
    Donne à ton élément un titre clair et descriptif.
  </Step>

  <Step title="Rédiger le contenu">
    Entre ton contenu dans l'éditeur. Utilise le formatage pour la clarté :
    - Titres pour les sections
    - Points de liste pour les listes
    - Numéros pour les étapes
    - Gras pour l'emphase
  </Step>

  <Step title="Cliquer sur 'Créer'">
    Enregistre ton élément texte. Le statut sera immédiatement COMPLETED.
  </Step>
</Steps>

### Exemple JSON d'Élément Texte

```json wrap
{
  "category": "TEXT",
  "title": "Informations sur les Prix des Produits",
  "content": "Nos niveaux de tarification incluent Basic (29 $/mois), Pro (99 $/mois) et Enterprise (tarification personnalisée).",
  "status": "COMPLETED"
}
```

## Éléments de Téléchargement de Fichiers

### Que sont les Éléments de Téléchargement de Fichiers ?

Les éléments de téléchargement de fichiers te permettent de télécharger des documents existants dans divers formats. Le système extrait le contenu textuel et le rend disponible pour tes agents.

### Quand Utiliser les Téléchargements de Fichiers

<AccordionGroup>
  <Accordion title="Documentation existante" icon="file">
    Tu as déjà du contenu au format document.

    **Exemples :**
    - Manuels utilisateur
    - Spécifications produit
    - Documents légaux
    - Supports de formation
  </Accordion>

  <Accordion title="Documents formatés" icon="file-pdf">
    Documents avec des mises en page spécifiques plus faciles à maintenir sous forme de fichiers.

    **Exemples :**
    - Diagrammes techniques
    - Tableaux et graphiques
    - Mises en page multi-colonnes
    - Modèles de marque
  </Accordion>

  <Accordion title="Documents tiers" icon="building">
    Documentation que tu reçois de fournisseurs ou partenaires.

    **Exemples :**
    - Catalogues fournisseurs
    - Documents de conformité
    - Matériels de certification
  </Accordion>
</AccordionGroup>

### Exigences de Fichier

**Spécifications de fichier :**
- **Formats :** PDF, DOC, DOCX, TXT et autres formats de documents
- **Limite de taille :** 10 Mo maximum
- **Contenu :** Documents textuels et images numérisées (l'analyse avancée gère la plupart des numérisations)
- **Protection :** Pas de protection par mot de passe

<Note>
Le système utilise une analyse avancée de documents pour extraire le texte des images numérisées et des PDF. La plupart des documents numérisés seront traités correctement, bien que les numérisations de très mauvaise qualité puissent nécessiter une saisie manuelle du texte.
</Note>

### Créer des Éléments de Fichier

<Steps>
  <Step title="Navigue vers le dossier">
    Ouvre le dossier où tu souhaites ajouter le fichier.
  </Step>

  <Step title="Cliquer sur 'Ajouter un Élément de Connaissance'">
    Clique sur le bouton pour créer un nouvel élément.
  </Step>

  <Step title="Sélectionner la catégorie 'FILE'">
    Choisis FILE parmi les options de type de contenu.
  </Step>

  <Step title="Entrer le titre">
    Donne à ton fichier un titre descriptif (distinct du nom de fichier).
  </Step>

  <Step title="Télécharger le fichier">
    Clique sur **Télécharger** et Sélectionne ton document depuis ton ordinateur.
  </Step>

  <Step title="Cliquer sur 'Créer'">
    Le système téléchargera et commencera à traiter ton fichier.

    Progression du statut :
    - PENDING → Fichier téléchargé, en file d'attente pour traitement
    - PROCESSING → Extraction du contenu textuel
    - COMPLETED → Prêt à l'utilisation
    - FAILED → Erreur de traitement survenue
  </Step>
</Steps>

### Exemple JSON d'Élément de Fichier

```json wrap
{
  "category": "FILE",
  "title": "Manuel Utilisateur - Produit X",
  "file": "/uploads/knowledge/user-manual-product-x.pdf",
  "status": "COMPLETED"
}
```

### Temps de Traitement

Le temps de traitement des fichiers varie en fonction de :
- **Taille du fichier :** Les fichiers plus volumineux prennent plus de temps
- **Nombre de pages :** Plus de pages = traitement plus long
- **Complexité :** Tableaux, images et mises en page complexes ralentissent le traitement
- **Qualité du texte :** Le texte propre et simple s'extrait plus rapidement

**Temps de traitement typiques :**
- Petits fichiers (< 1 Mo, 10 pages) : 10-30 secondes
- Fichiers moyens (1-5 Mo, 10-50 pages) : 30-90 secondes
- Gros fichiers (5-10 Mo, 50+ pages) : 2-5 minutes

### Problèmes Courants de Fichiers

<AccordionGroup>
  <Accordion title="Échec du traitement" icon="circle-xmark">
    **Causes :**
    - Le fichier dépasse 10 Mo
    - Le fichier est protégé par mot de passe
    - Le fichier est corrompu
    - Images numérisées de très mauvaise qualité

    **Solutions :**
    - Compresser le fichier ou le diviser en fichiers plus petits
    - Supprimer la protection par mot de passe
    - Ré-exporter le fichier depuis la source
    - Pour les numérisations de très mauvaise qualité, Copie le contenu dans un élément texte
  </Accordion>

  <Accordion title="Contenu extrait incorrectement" icon="text-slash">
    **Causes :**
    - Mises en page complexes (multi-colonnes, tableaux)
    - Images numérisées de très mauvaise qualité
    - Polices ou encodage spéciaux
    - Champs de formulaire et éléments interactifs

    **Solutions :**
    - Vérifier le contenu extrait en mode édition
    - Recréer en tant qu'élément texte avec formatage approprié
    - Simplifier la mise en page du document avant téléchargement
    - Exporter en tant que document texte brut
  </Accordion>

  <Accordion title="Le traitement prend trop de temps" icon="clock">
    **Que faire :**
    - Attendre 5-10 minutes avant de supposer un échec
    - Vérifier la taille du fichier et le nombre de pages
    - Pour les gros fichiers, envisager de diviser en plusieurs fichiers
    - Convertir en texte et télécharger en tant qu'éléments TEXT
  </Accordion>
</AccordionGroup>

### Meilleures Pratiques de Téléchargement de Fichiers

<CardGroup cols={2}>
  <Card title="Optimiser avant téléchargement" icon="compress">
    - Compresser les gros fichiers
    - Supprimer les images inutiles
    - Utiliser des documents textuels
    - Rester sous 5 Mo si possible
  </Card>
  <Card title="Tester l'extraction" icon="vial">
    - Réviser le contenu extrait après traitement
    - Vérifier les problèmes de formatage
    - Vérifier que les informations critiques sont exactes
    - Retélécharger si l'extraction est mauvaise
  </Card>
</CardGroup>

## Éléments URL

### Que sont les Éléments URL ?

Les éléments URL extraient le contenu de pages web et le stockent dans ton base de connaissances. C'est utile pour référencer la documentation en ligne, les centres d'aide ou les articles de blog.

### Quand Utiliser les Éléments URL

<AccordionGroup>
  <Accordion title="Documentation publique" icon="globe">
    Référencer la documentation externe que vous ne Maintiens pas.

    **Exemples :**
    - Documentation API (la vôtre ou tierce)
    - Bases de connaissances publiques
    - Articles de centre d'aide
    - Pages produit
  </Accordion>

  <Accordion title="Contenu fréquemment mis à jour" icon="arrows-rotate">
    Contenu qui change régulièrement et que tu souhaites maintenir à jour en le ré-extrayant.

    **Exemples :**
    - Pages de tarification
    - Disponibilité produit
    - Promotions actuelles
    - Pages de statut
  </Accordion>

  <Accordion title="Articles de blog ou articles" icon="newspaper">
    Contenu éducatif ou annonces.

    **Exemples :**
    - Guides pratiques
    - Articles de meilleures pratiques
    - Annonces de produits
    - Tutoriels de fonctionnalités
  </Accordion>
</AccordionGroup>

### Créer des Éléments URL

<Steps>
  <Step title="Navigue vers le dossier">
    Ouvre le dossier où tu souhaites ajouter l'URL.
  </Step>

  <Step title="Cliquer sur 'Ajouter un Élément de Connaissance'">
    Clique sur le bouton pour créer un nouvel élément.
  </Step>

  <Step title="Sélectionner la catégorie 'URL'">
    Choisis URL parmi les options de type de contenu.
  </Step>

  <Step title="Entrer le titre">
    Donne au contenu un titre descriptif.
  </Step>

  <Step title="Entrer l'URL source">
    Colle l'URL complète incluant https://

    **Exemple :**
    ```
    https://docs.company.com/api/authentication
    ```
  </Step>

  <Step title="Cliquer sur 'Créer'">
    Le système récupérera et traitera la page web.

    Progression du statut :
    - PENDING → URL en file d'attente pour extraction
    - PROCESSING → Récupération et extraction du contenu
    - COMPLETED → Contenu disponible
    - FAILED → Erreur d'extraction survenue
  </Step>
</Steps>

### Exemple JSON d'Élément URL

```json wrap
{
  "category": "URL",
  "title": "Documentation Authentification API",
  "source_url": "https://docs.company.com/api/authentication",
  "status": "COMPLETED"
}
```

### Exigences URL

**URLs fonctionnelles :**
- Accessibles publiquement (pas de connexion requise)
- Pages de contenu HTML simples
- Sites de documentation
- Articles de blog
- Pages de contenu statique

**URLs problématiques :**
- Pages nécessitant une authentification
- Applications riches en JavaScript (SPA)
- Contenu payant
- Contenu chargé dynamiquement
- Applications interactives

<Note>
L'extraction URL fonctionne mieux avec des pages web simples basées sur du texte. Les applications web complexes peuvent ne pas s'extraire avec succès.
</Note>

### Problèmes Courants d'URL

<AccordionGroup>
  <Accordion title="Échec de l'extraction" icon="circle-xmark">
    **Causes :**
    - La page nécessite une connexion/authentification
    - L'URL est incorrecte ou rompue
    - Le contenu se charge via JavaScript
    - Le site web bloque l'extraction (robots.txt)
    - La page n'existe pas (404)

    **Solutions :**
    - Vérifier que l'URL est accessible publiquement
    - Tester l'URL dans une fenêtre de navigateur privée
    - Vérifier que l'URL est complète et correcte
    - Copier le contenu manuellement dans un élément texte
    - Utiliser l'export PDF de la page à la place
  </Accordion>

  <Accordion title="Contenu incomplet ou incorrect" icon="puzzle-piece">
    **Causes :**
    - Contenu rendu JavaScript non capturé
    - Chargement de contenu dynamique
    - Plusieurs onglets/sections sur la page
    - Commentaires ou barre latérale extraits au lieu du contenu principal

    **Solutions :**
    - Inspecter le contenu extrait en mode édition
    - Utiliser l'URL directe vers le contenu spécifique
    - Copier le contenu désiré dans un élément texte
    - Exporter la page en PDF et télécharger à la place
  </Accordion>

  <Accordion title="Le contenu devient obsolète" icon="calendar">
    **Solution :**
    Le contenu URL est extrait une fois au moment de la création. Pour mettre à jour :
    - Supprimer et recréer l'élément URL
    - Ou copier le contenu actuel dans un élément texte pour des mises à jour manuelles

    Pour le contenu changeant fréquemment, envisager :
    - Des éléments texte manuels que tu mets à jour régulièrement
    - Des exports PDF que vous Rafraîchis périodiquement
  </Accordion>
</AccordionGroup>

### Meilleures Pratiques URL

<CardGroup cols={2}>
  <Card title="Tester l'accessibilité" icon="unlock">
    - Ouvrir l'URL dans une fenêtre privée
    - Vérifier qu'aucune connexion n'est requise
    - Vérifier que le contenu est visible
    - S'assurer que la page se charge rapidement
  </Card>
  <Card title="Réviser le contenu extrait" icon="eye">
    - Vérifier le contenu après extraction
    - Vérifier que le bon contenu a été capturé
    - Rechercher des problèmes de formatage
    - Confirmer qu'il n'y a pas de contenu supplémentaire (publicités, barres latérales)
  </Card>
</CardGroup>

## Flux de Statut de Traitement

Les éléments de connaissance passent par deux pipelines de traitement distincts :

1. **Traitement du Contenu** - Extraction du texte depuis les fichiers/URLs
2. **Indexation Vectorielle** - Préparation du contenu pour RAG (recherche sémantique)

### Statut de Traitement du Contenu

Cela suit l'extraction du contenu textuel depuis ton source.

<Steps>
  <Step title="PENDING">
    **Signification :** Élément créé, en file d'attente pour l'extraction de contenu

    **Ce qui se passe :**
    - L'élément a été enregistré dans la base de données
    - En attente qu'un worker de traitement le prenne en charge
    - Généralement très bref (secondes)

    **Action nécessaire :** Aucune - attendre que le traitement commence
  </Step>

  <Step title="PROCESSING">
    **Signification :** Le contenu de l'élément est en cours d'extraction

    **Ce qui se passe :**
    - Pour les FICHIERS : Extraction de texte depuis PDF, Word, etc.
    - Pour les URLs : Récupération et extraction de la page web spécifique
    - Pour le TEXTE : N/A (passe directement à COMPLETED)

    **Action nécessaire :** Attendre la fin du traitement
  </Step>

  <Step title="COMPLETED">
    **Signification :** Extraction du contenu terminée avec succès

    **Ce qui se passe :**
    - Le contenu a été extrait et stocké
    - L'indexation vectorielle commencera automatiquement
    - L'élément sera disponible une fois l'indexation terminée

    **Action nécessaire :** Aucune - attendre l'indexation vectorielle
  </Step>

  <Step title="FAILED">
    **Signification :** L'extraction du contenu a rencontré une erreur

    **Ce qui se passe :**
    - L'extraction du fichier a échoué (PDF corrompu, format non pris en charge)
    - L'extraction d'URL a échoué (page introuvable, bloquée, nécessite authentification)
    - Le contenu n'est pas disponible pour les agents

    **Action nécessaire :** Vérifier le message d'erreur et corriger le problème
  </Step>
</Steps>

**Flux de Traitement du Contenu :**
CREATED → PENDING → PROCESSING → COMPLETED ou FAILED

---

### Statut d'Indexation Vectorielle

Après l'extraction du contenu, il doit être indexé pour RAG (recherche sémantique). Cela permet aux agents de trouver les connaissances pertinentes en fonction du sens, pas seulement des mots-clés.

<Steps>
  <Step title="PENDING">
    **Signification :** En attente du début de l'indexation vectorielle

    **Ce qui se passe :**
    - Le traitement du contenu s'est terminé avec succès
    - En file d'attente pour la génération d'embeddings
    - Généralement bref (secondes à minutes)

    **Action nécessaire :** Aucune
  </Step>

  <Step title="INDEXING">
    **Signification :** Création d'embeddings vectoriels pour RAG

    **Ce qui se passe :**
    - Le contenu est divisé en morceaux
    - Des embeddings IA sont générés pour chaque morceau
    - Les vecteurs sont stockés dans la base de connaissances

    **Action nécessaire :** Attendre - cela peut prendre plusieurs minutes pour les gros documents
  </Step>

  <Step title="INDEXED">
    **Signification :** L'élément est entièrement prêt pour la récupération RAG

    **Ce qui se passe :**
    - Les embeddings vectoriels sont stockés avec succès
    - L'élément peut être récupéré via recherche sémantique
    - Les agents peuvent maintenant utiliser ces connaissances

    **Action nécessaire :** Aucune - l'élément est entièrement prêt
  </Step>

  <Step title="FAILED">
    **Signification :** L'indexation vectorielle a échoué

    **Ce qui se passe :**
    - La génération d'embeddings a rencontré une erreur
    - L'élément n'apparaîtra pas dans les résultats RAG
    - Peut être disponible uniquement pour l'injection de contexte

    **Action nécessaire :** Essayer de réindexer ou contacter le support
  </Step>

  <Step title="REMOVED">
    **Signification :** Les vecteurs ont été supprimés

    **Ce qui se passe :**
    - L'élément a été supprimé ou explicitement retiré de l'index
    - N'est plus disponible pour RAG

    **Action nécessaire :** Réindexer si nécessaire
  </Step>
</Steps>

**Flux d'Indexation Vectorielle :**
PENDING → INDEXING → INDEXED ou FAILED

<Note>
**Les deux statuts doivent réussir pour une fonctionnalité complète :**
- Statut Contenu : COMPLETED
- Statut Vecteur : INDEXED

Seulement alors l'élément peut être utilisé à la fois dans l'injection de contexte et la récupération RAG.
</Note>

## Gestion des Erreurs

### Quand les Éléments Échouent

Si un élément de connaissance affiche le statut FAILED :

<Steps>
  <Step title="Vérifier le message d'erreur">
    Clique sur l'élément échoué pour voir les détails de l'erreur.
  </Step>

  <Step title="Identifier la cause">
    Causes courantes :
    - **Fichiers :** Fichier trop volumineux, corrompu, protégé par mot de passe, image numérisée
    - **URL :** Authentification requise, lien rompu, contenu non accessible
  </Step>

  <Step title="Essayer des solutions">
    - **Pour les Fichiers :** Compresser, supprimer la protection, ajouter une couche de texte, ou convertir en texte
    - **Pour les URLs :** Vérifier l'accessibilité, essayer une URL différente, ou copier le contenu dans un élément texte
  </Step>

  <Step title="Recréer ou convertir">
    - Supprimer l'élément échoué
    - Créer un nouvel élément avec les corrections appliquées
    - Ou convertir en élément texte en copiant le contenu manuellement
  </Step>
</Steps>

### Prévenir les Erreurs

## Surveillance du Traitement

### Indicateurs du Tableau de Bord

Dans ton tableau de bord de base de connaissances, tu peux voir le statut de traitement d'un coup d'œil pour chaque élément dans tes dossiers :

**Indicateurs de statut :**
- Coche verte = COMPLETED
- Sablier = PROCESSING
- Symbole pause = PENDING
- X rouge = FAILED

**Exemple de vue de dossier :**
- Manuel Utilisateur.pdf - COMPLETED
- Guide Démarrage Rapide.pdf - PROCESSING
- Documentation API - PENDING
- Manuel Hérité.pdf - FAILED

### Traitement par Lots

Lors du téléchargement de plusieurs éléments :

1. Les éléments se traitent séquentiellement ou en parallèle (dépend du système)
2. Revenir après 5-10 minutes pour les gros lots
3. Réviser le statut de chaque élément
4. Corriger les échecs individuellement

<Tip>
Télécharge un élément test d'abord pour vérifier le format et le traitement avant un téléchargement en masse.
</Tip>

## Choisir le Bon Type de Contenu

Utilise cet arbre de décision pour sélectionner le meilleur type de contenu :

**As-tu du contenu existant ?**
- Non → Utilise TEXT (écrire directement)
- Oui, c'est un document →
  - Moins de 10 Mo → Utilise FILE
  - Plus de 10 Mo → Extraire le texte, utiliser TEXT
- Oui, c'est une page web →
  - Accessible publiquement → Utilise URL (si l'extraction échoue, Copie vers TEXT)
  - Non accessible → Copie le contenu vers TEXT

### Recommandations Rapides

| Ton Situation | Meilleur Type de Contenu |
|---------------|-------------------|
| Rédiger des FAQ depuis zéro | TEXT |
| Avoir des documents Word/PDF existants | Télécharger comme FILE |
| Avoir des documents sous 10 Mo | FILE |
| Avoir des documents sur 10 Mo | Diviser en fichiers plus petits ou extraire vers TEXT |
| Documentation web publique | URL (avec TEXT comme solution de secours) |
| Contenu privé/authentifié | Copier vers TEXT |
| Besoin de disponibilité immédiate | TEXT (pas de délai de traitement) |
| Le formatage complexe compte | FILE |
