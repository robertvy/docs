## Vue d'ensemble

Créer une base de connaissances est votre première étape pour donner à votre agent IA une connaissance de niveau expert de votre entreprise. Ce guide vous accompagne tout au long du processus complet, de la création de la base de connaissances à l'organisation du contenu et au respect des meilleures pratiques.

<Tip>
Planifiez votre structure de connaissances avant de commencer. Une base de connaissances bien organisée est plus facile à maintenir et plus efficace pour vos agents.
</Tip>

---

## Étape 1 : Créer une Base de Connaissances

### Naviguer vers les Bases de Connaissances

1. Connectez-vous à votre [tableau de bord itellicoAI](https://app.itellico.ai)
2. Cliquez sur **Knowledge Bases** dans le menu de navigation de gauche
3. Cliquez sur le bouton **+ Créer une Base de Connaissances**

### Remplir les Informations de Base

<Frame>
  <div className="block dark:hidden">
    <img src="/images/knowledgebase/create-light.png" alt="Dialogue Créer une Base de Connaissances avec les champs nom et description" />
  </div>
  <div className="hidden dark:block">
    <img src="/images/knowledgebase/create-dark.png" alt="Dialogue Créer une Base de Connaissances avec les champs nom et description" />
  </div>
</Frame>

<Steps>
  <Step title="Entrer le Nom de la Base de Connaissances">
    Choisissez un nom clair et descriptif qui reflète la catégorie de contenu.

    **Règles de nommage :**
    - Lettres, chiffres, espaces, traits d'union et underscores uniquement
    - Pas de caractères spéciaux (@, #, $, etc.)
    - Maximum 100 caractères

    **Bons exemples :**
    - "FAQ Support Client"
    - "Documentation Produit"
    - "Politiques d'Entreprise"
    - "Dépannage Technique"

    **À éviter :**
    - "KB_001" (pas descriptif)
    - "Docs Divers" (trop vague)
    - "Tout!!!" (pas professionnel)
  </Step>

  <Step title="Ajouter une Description (Optionnel)">
    Fournissez un contexte sur ce que cette base de connaissances contient et quand l'utiliser.

    **Exemples de descriptions :**
    - "Documentation de support orientée client incluant FAQ, politiques et guides de dépannage"
    - "Spécifications produit internes, documentation des fonctionnalités et détails techniques"
    - "Politiques RH d'entreprise, documents de conformité et procédures internes"
  </Step>

  <Step title="Cliquer sur Créer">
    Votre base de connaissances est maintenant créée et prête pour le contenu !
  </Step>
</Steps>

## Étape 2 : Organiser avec des Dossiers

Les dossiers organisent vos éléments de connaissance en groupes logiques.

<Note>
Un dossier par défaut est automatiquement créé lorsque vous créez une nouvelle base de connaissances. Vous pouvez le renommer, ajouter d'autres dossiers ou commencer à ajouter des éléments immédiatement.
</Note>

### Ajouter des Dossiers Supplémentaires (Optionnel)

1. Ouvrez votre base de connaissances
2. Cliquez sur le bouton **Créer un Dossier**
3. Entrez un nom de dossier
4. Ajoutez une description optionnelle
5. Cliquez sur **Créer**

### Meilleures Pratiques de Nommage des Dossiers

<CardGroup cols={2}>
  <Card title="Clair & Spécifique" icon="check">
    - "Facturation & Paiements"
    - "Politiques de Retour"
    - "Spécifications Produit"
    - "Problèmes Techniques"
  </Card>
  <Card title="Vague ou Générique" icon="xmark">
    - "Trucs"
    - "Dossier 1"
    - "Docs"
    - "Divers"
  </Card>
</CardGroup>

### Combien de Dossiers ?

<Accordion title="Petite Base de Connaissances (< 50 éléments)">
  **3-5 dossiers** sont généralement suffisants

  **Exemple de structure :**
  - Questions Clients
  - Politiques d'Entreprise
  - Informations Produit
</Accordion>

<Accordion title="Base de Connaissances Moyenne (50-200 éléments)">
  **5-10 dossiers** offrent une bonne organisation

  **Exemple de structure :**
  - Facturation & Paiements
  - Support Technique
  - Fonctionnalités Produit
  - Gestion de Compte
  - Expédition & Retours
  - FAQ Générale
</Accordion>

<Accordion title="Grande Base de Connaissances (200+ éléments)">
  **10-15 dossiers** gardent les choses gérables

  Envisagez de diviser en plusieurs bases de connaissances si vous dépassez 15 dossiers.

  **Exemple de structure :**
  - Configuration Compte
  - Problèmes Facturation
  - Modes de Paiement
  - Remboursements & Crédits
  - Catégories Produit A-F
  - Catégories Produit G-M
  - Catégories Produit N-Z
  - Dépannage Technique
  - Guides Intégration
  - Documentation API
  - Retours & Échanges
  - Options Expédition
  - Suivi Commande
</Accordion>

## Étape 3 : Ajouter des Éléments de Connaissance

Les éléments de connaissance sont le contenu réel - les documents, FAQ, politiques et informations que vous souhaitez que votre agent connaisse.

<Frame>
  <div className="block dark:hidden">
    <img src="/images/knowledgebase/add-items-light.png" alt="Dialogue Créer un Nouvel Élément montrant les options Contenu Texte, Télécharger un Fichier et Page Web" />
  </div>
  <div className="hidden dark:block">
    <img src="/images/knowledgebase/add-items-dark.png" alt="Dialogue Créer un Nouvel Élément montrant les options Contenu Texte, Télécharger un Fichier et Page Web" />
  </div>
</Frame>

Vous pouvez ajouter des éléments de connaissance de trois manières différentes. Choisissez la méthode qui convient le mieux à votre contenu :

<AccordionGroup>
  <Accordion title="Saisie Texte - Rédiger directement le contenu" icon="text">
    **Idéal pour :** FAQ, politiques, procédures, informations de référence rapide

    Saisie directe de texte à l'aide de l'éditeur intégré. Parfait pour le contenu que vous créez spécifiquement pour votre base de connaissances.

    ### Comment Ajouter un Élément Texte

<Steps>
  <Step title="Cliquer sur 'Ajouter un Élément de Connaissance'">
    Dans votre dossier, cliquez sur le bouton ajouter pour créer un nouvel élément.
  </Step>

  <Step title="Sélectionner 'TEXT' comme catégorie">
    Choisissez texte si vous saisissez le contenu directement.
  </Step>

  <Step title="Entrer un titre descriptif">
    **Bons titres :**
    - "Politique de Retour pour Produits Numériques"
    - "Instructions Réinitialisation Mot de Passe"
    - "Niveaux de Tarification Expliqués"

    **À éviter :**
    - "Doc 1"
    - "Politique"
    - "Info"
  </Step>

  <Step title="Rédiger ou coller votre contenu">
    Entrez les informations que vous souhaitez que votre agent connaisse. Soyez clair et complet.

    **Exemple :**
    ```
    Politique de Retour pour Produits Numériques

    Les produits numériques peuvent être remboursés dans les 30 jours suivant l'achat si :
    1. Le produit présente un défaut technique empêchant son utilisation
    2. La description du produit était matériellement inexacte
    3. Le client n'a pas accédé ni téléchargé le produit

    Les remboursements ne sont pas disponibles pour :
    - Changement d'avis après avoir accédé au produit
    - Problèmes de compatibilité divulgués dans la description du produit
    - Erreur utilisateur ou mauvaise compréhension des fonctionnalités

    Pour demander un remboursement, les clients doivent envoyer un e-mail à support@company.com
    avec leur numéro de commande et la raison de la demande de remboursement.

    Délai de traitement : 5-7 jours ouvrables
    ```
  </Step>

  <Step title="Cliquer sur 'Créer'">
    Votre élément de connaissance est enregistré et sera traité immédiatement.
  </Step>
</Steps>
  </Accordion>

  <Accordion title="Téléchargement de Fichier - Télécharger des documents existants" icon="file-arrow-up">
    **Idéal pour :** Manuels utilisateur, spécifications produit, documents légaux, supports de formation

    Téléchargez des documents dans divers formats jusqu'à 10 Mo. Le système extrait et traite le contenu textuel.

    **Formats pris en charge :** PDF, Word (.doc, .docx), Texte (.txt, .log), Markdown (.md), CSV/TSV, JSON, YAML, XML

    ### Comment Ajouter un Fichier

<Steps>
  <Step title="Cliquer sur 'Ajouter un Élément de Connaissance'">
    Dans votre dossier, cliquez sur le bouton ajouter.
  </Step>

  <Step title="Sélectionner 'FILE' comme catégorie">
    Choisissez fichier pour les téléchargements de documents.
  </Step>

  <Step title="Entrer un titre">
    Donnez à votre fichier un titre descriptif (le nom de fichier sera stocké séparément).
  </Step>

  <Step title="Télécharger votre fichier">
    Cliquez sur **Télécharger** et sélectionnez votre document.

    **Exigences :**
    - Formats pris en charge : PDF, Word (.doc, .docx), Texte (.txt, .log), Markdown (.md), CSV/TSV, JSON, YAML, XML
    - Taille maximale : 10 Mo
    - Documents textuels et images numérisées (l'analyse avancée gère la plupart des numérisations)
  </Step>

  <Step title="Cliquer sur 'Créer'">
    Le système téléchargera et commencera à traiter votre fichier. Le statut changera de PENDING → PROCESSING → COMPLETED.
  </Step>
</Steps>

<Warning>
Le temps de traitement des fichiers varie en fonction de la taille et de la complexité du fichier. Les gros fichiers (5-10 Mo) peuvent prendre plusieurs minutes à traiter.
</Warning>
  </Accordion>

  <Accordion title="Extraction URL - Extraire le contenu de pages web" icon="link">
    **Idéal pour :** Pages de documentation, articles de blog, articles d'aide, ressources externes

    Extraire le contenu d'une seule page web (pas de sites web entiers).

    ### Comment Ajouter une URL

<Steps>
  <Step title="Cliquer sur 'Ajouter un Élément de Connaissance'">
    Dans votre dossier, cliquez sur le bouton ajouter.
  </Step>

  <Step title="Sélectionner 'URL' comme catégorie">
    Choisissez URL pour extraire du contenu web.
  </Step>

  <Step title="Entrer un titre">
    Donnez au contenu un titre descriptif.
  </Step>

  <Step title="Entrer l'URL source">
    Collez l'URL complète incluant https://

    **Exemple :**
    ```
    https://docs.company.com/api/authentication
    ```
  </Step>

  <Step title="Cliquer sur 'Créer'">
    Le système récupérera et traitera le contenu de la page web. Le statut changera de PENDING → PROCESSING → COMPLETED.
  </Step>
</Steps>

<Note>
L'extraction URL fonctionne mieux avec des pages web simples basées sur du texte. Les applications JavaScript complexes ou les pages nécessitant une authentification peuvent ne pas s'extraire avec succès.
</Note>
  </Accordion>
</AccordionGroup>

## Comprendre les Statuts de Traitement

Après avoir ajouté un élément de connaissance, il passe par deux pipelines de traitement distincts. Vous verrez les deux statuts dans l'interface :

### Statut de Traitement du Contenu

Cela suit si le contenu a été extrait avec succès depuis votre source.

<AccordionGroup>
  <Accordion title="PENDING" icon="clock">
    Élément créé et en file d'attente pour traitement. Prend généralement quelques secondes.
  </Accordion>

  <Accordion title="PROCESSING" icon="spinner">
    Le contenu est en cours de traitement maintenant :
    - **Fichiers** : Analyse de documents à l'aide de nos analyseurs de documents IA avancés
    - **URLs** : Récupération et extraction de la page web
    - **Texte** : Passe cette étape (va directement à COMPLETED)
  </Accordion>

  <Accordion title="COMPLETED" icon="check">
    Contenu extrait et stocké avec succès. L'indexation vectorielle commencera automatiquement.
  </Accordion>

  <Accordion title="FAILED" icon="xmark">
    L'extraction du contenu a échoué. Causes courantes :
    - Le fichier est corrompu ou protégé par mot de passe
    - L'URL nécessite une authentification ou utilise du JavaScript complexe
    - Le fichier dépasse la limite de 10 Mo

    **Corrections rapides :**
    - Essayez de télécharger le contenu en tant que TEXT à la place
    - Supprimez la protection par mot de passe du fichier
    - Divisez les gros fichiers en documents plus petits

    <Warning>
    **Besoin d'aide ?**

    Si le problème persiste, contactez notre équipe support à **support@itellico.ai** avec :
    - Le titre de l'élément de connaissance
    - Le type de fichier ou l'URL que vous essayez d'ajouter
    - Tout message d'erreur affiché
    </Warning>
  </Accordion>
</AccordionGroup>

### Statut d'Indexation Vectorielle

Après l'extraction du contenu, il doit être indexé pour RAG (Génération Augmentée par Récupération). Cela permet à votre agent de trouver les connaissances pertinentes en fonction du sens.

<AccordionGroup>
  <Accordion title="PENDING" icon="clock">
    En attente du début de l'indexation vectorielle. Le traitement du contenu s'est terminé avec succès.
  </Accordion>

  <Accordion title="INDEXING" icon="spinner">
    Création d'embeddings vectoriels pour RAG :
    - Le contenu est divisé en morceaux
    - Des embeddings IA sont générés
    - Les vecteurs sont stockés dans la base de données

    Cela peut prendre plusieurs minutes pour les gros documents.
  </Accordion>

  <Accordion title="INDEXED" icon="check">
    L'élément est entièrement prêt ! L'agent peut maintenant trouver et utiliser ces connaissances via la recherche sémantique.
  </Accordion>

  <Accordion title="FAILED" icon="xmark">
    L'indexation vectorielle a échoué. Le contenu est extrait mais n'apparaîtra pas dans les résultats de recherche RAG.

    **Corrections rapides :**
    - Cliquez sur le bouton **Réindexer** sur l'élément de connaissance pour réessayer l'indexation
    - Si la réindexation échoue, essayez de supprimer et recréer l'élément
    - Assurez-vous que le contenu n'est pas trop court (nécessite au moins quelques phrases)

    <Warning>
    **Besoin d'aide ?**

    Si la réindexation ne fonctionne pas, contactez notre équipe support à **support@itellico.ai** avec :
    - Le titre de l'élément de connaissance
    - Quand vous avez créé l'élément
    - Type de contenu (TEXT, FILE, ou URL)
    </Warning>
  </Accordion>
</AccordionGroup>

<Note>
**Les deux statuts doivent réussir pour une fonctionnalité complète :**
- Statut Contenu : **COMPLETED**
- Statut Vecteur : **INDEXED**

Seulement alors votre agent peut utiliser les connaissances dans les conversations.
</Note>

## Directives de Qualité du Contenu

### Rédiger un Contenu Clair et Complet

<AccordionGroup>
  <Accordion title="Être spécifique et détaillé" icon="magnifying-glass">
    Ne présumez pas de connaissances. Expliquez clairement.

    **Au lieu de :**
    "Les clients peuvent retourner les articles dans notre fenêtre standard."

    **Écrivez :**
    "Les clients peuvent retourner les articles dans les 30 jours suivant l'achat. Les articles doivent être inutilisés et dans l'emballage d'origine avec toutes les étiquettes attachées."
  </Accordion>

  <Accordion title="Utiliser un langage simple" icon="comment">
    Évitez le jargon à moins qu'il ne soit standard dans l'industrie et que votre public le connaisse.

    **Au lieu de :**
    "Utilisez le point de terminaison d'authentification pour générer un token porteur."

    **Écrivez :**
    "Pour vous authentifier, appelez le point de terminaison /auth. Vous recevrez un token d'accès à inclure dans les requêtes suivantes."
  </Accordion>

  <Accordion title="Inclure des exemples" icon="list-check">
    Les exemples rendent les concepts abstraits concrets.

    **Exemple :**
    "Nos niveaux de tarification incluent :
    - Basic : 29 $/mois - Jusqu'à 1 000 contacts
    - Pro : 99 $/mois - Jusqu'à 10 000 contacts
    - Enterprise : Tarification personnalisée - Contacts illimités"
  </Accordion>

  <Accordion title="Structurer avec des titres" icon="heading">
    Utilisez des titres et des points de liste pour un contenu scannable.

    **Bonne structure :**
    ```
    Politique de Retour

    Articles Éligibles :
    - Produits non ouverts dans l'emballage d'origine
    - Articles achetés dans les 30 jours
    - Produits sans signes d'utilisation

    Articles Non Éligibles :
    - Téléchargements numériques
    - Produits personnalisés
    - Articles en solde marqués comme vente finale

    Processus :
    1. Envoyer un e-mail à support@company.com
    2. Fournir le numéro de commande
    3. Recevoir l'autorisation de retour
    4. Expédier l'article dans les 7 jours
    ```
  </Accordion>

  <Accordion title="Garder le contenu concentré" icon="bullseye">
    Chaque élément devrait couvrir un sujet en profondeur. Diviser les grands sujets en plusieurs éléments.

    **Au lieu de :**
    Un "Manuel Complet de l'Entreprise" de 20 pages

    **Créer :**
    - Politique de Retour
    - Options d'Expédition
    - Modes de Paiement
    - Garantie Produit
    - Politique de Confidentialité
    - Coordonnées
  </Accordion>
</AccordionGroup>
