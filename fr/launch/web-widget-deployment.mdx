Le widget web permet aux visiteurs d'interagir avec votre agent IA par voix, chat, ou les deux simultanément. Configurez tout via l'éditeur visuel—aucun codage requis—puis copiez et collez un simple script d'intégration.

## Créer Votre Widget

  <div className="block dark:hidden">
    <img src="/images/widget/create-widget-light.png" alt="Dialogue Créer un Widget montrant le champ Nom du Widget avec texte de placeholder 'Entrez le nom du widget', menu déroulant Agent pour sélectionner quel agent alimente le widget, boutons Annuler et Créer un Widget" />
  </div>
  <div className="hidden dark:block">
    <img src="/images/widget/create-widget-dark.png" alt="Dialogue Créer un Widget montrant le champ Nom du Widget avec texte de placeholder 'Entrez le nom du widget', menu déroulant Agent pour sélectionner quel agent alimente le widget, boutons Annuler et Créer un Widget" />
  </div>
</Frame>

<Steps>
  <Step title="Ouvrez le gestionnaire de widgets">
    Naviguez vers **Web → Widgets** dans le tableau de bord
  </Step>
  <Step title="Créez un nouveau widget">
    Cliquez sur **Créer un Widget** et fournissez :
    - **Nom du widget** - Étiquette interne pour votre référence
    - **Agent** - Sélectionnez quel agent alimente ce widget
  </Step>
  <Step title="Configurez le widget">
    Utilisez les onglets de l'éditeur visuel pour personnaliser votre widget (voir sections ci-dessous)
  </Step>
  <Step title="Enregistrez vos modifications">
    Cliquez sur **Enregistrer les Modifications** en bas à droite pour conserver toutes les mises à jour
  </Step>
  <Step title="Obtenez le code d'intégration">
    Allez dans l'onglet **Exporter** et copiez votre script d'intégration unique
  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="Configuration" icon="gear">
    - **Nom du widget** - Étiquette interne affichée dans votre liste de widgets
    - **Agent** - Basculez vers un agent différent à tout moment
    - **Nom d'affichage** - Nom visible par le client affiché dans le widget
    - **Domaines autorisés** - Restreignez où le widget peut se charger (laissez vide pour autoriser tous les domaines)
  </Accordion>

  <Accordion title="Apparence" icon="palette">
    **Préréglages de Thème**
    - Commencez à partir de palettes de couleurs sélectionnées
    - Parcourez les options de préréglages

    **Disposition**
    - Position : en bas à gauche ou en bas à droite
    - Taille du widget et décalages par rapport aux bords
    - État initial : réduit, étendu ou toujours ouvert

    **Avatar**
    - Effet "Pulsation IA" animé
    - Téléchargez une image personnalisée
    - Lien vers une URL d'image externe

    **Couleurs & Dégradés**
    - Couleurs d'arrière-plan et de CTA
    - Style des bulles de transcription
    - Couleurs de statut et dégradés d'orbe

    **Rayon de Bordure**
    - Arrondi du bouton de lancement
    - Coins de la fenêtre de chat
    - Bulles de messages
    - Champ de saisie et bouton CTA

    **Animations**
    - Effet de oscillation ou ondulation optionnel sur le bouton d'appel
  </Accordion>

  <Accordion title="Contenu" icon="language">
    **Préréglages de Langue**
    - Packs Anglais et Allemand pré-configurés
    - Définit toutes les étiquettes du widget et le texte légal

    **Messages de Bienvenue**
    - Texte du message de bienvenue
    - Texte de placeholder
    - Messages de statut (écoute, parle, réfléchit, connexion)

    **Copie de Retour d'Information**
    - Invitation aux retours
    - Texte du bouton de soumission
    - Message de remerciement

    **Remplacements d'Étiquettes**
    - Personnalisez les chaînes UI individuelles
    - Groupes : Chat, Contrôles, Transcription, Audio, Accessibilité
  </Accordion>

  <Accordion title="Fonctionnalités" icon="sliders">
    - **Mode chat uniquement** - Désactivez la voix, chat uniquement
    - **Autoriser le texte pendant l'appel** - Gardez la saisie de texte active pendant les appels vocaux
    - **Bascule de transcription** - Afficher/masquer la transcription en direct
    - **Contrôle de mise en sourdine** - Afficher le bouton de sourdine pour permettre à l'utilisateur de couper son microphone
    - **Minuteur d'appel** - Afficher le temps écoulé
    - **Branding** - Afficher/masquer "Propulsé par itellicoAI"
    - **Passer le message d'accueil** - Éviter le message d'accueil si l'utilisateur tape en premier
    - **Collecte de retours** - Activer les retours post-appel (non stockés pour le moment, bientôt disponible)
  </Accordion>

  <Accordion title="Actions" icon="link">
    Configurez des outils côté client que votre agent peut invoquer pendant les conversations pour afficher des liens, ouvrir des pages ou déclencher des appels téléphoniques dans le navigateur de l'utilisateur.

    <Note>
    **Outils Client** : Contrairement aux actions API qui s'exécutent sur votre serveur, les outils client s'exécutent directement dans le navigateur de l'utilisateur. Le LLM décide quand appeler ces outils en fonction du contexte de la conversation et de la description du déclencheur que vous fournissez.
    </Note>

    Ajoutez jusqu'à cinq outils client :

    - **Texte du bouton** - Étiquette affichée à l'utilisateur (ex : "Voir les tarifs", "Appeler le support")
    - **URL** - Lien web (`https://`) ou numéro de téléphone (`tel:`)
    - **Déclencheur** - Décrivez quand l'agent devrait invoquer cet outil (ex : "Quand l'utilisateur demande les tarifs", "Quand l'utilisateur veut parler à un humain")
    - **Comportement** :
      - **Clic automatique** - Ouvre automatiquement le lien lorsque l'outil est invoqué
      - **Attendre la réponse** - Affiche le bouton et attend que l'utilisateur clique (avec délai optionnel)
      - **Terminer l'appel au clic** - Termine automatiquement la conversation après que l'utilisateur a cliqué

    Les liens de numéros de téléphone (`tel:`) terminent automatiquement l'appel après que l'utilisateur a cliqué.
  </Accordion>

  <Accordion title="Confidentialité" icon="shield-check">
    **Conditions & Confidentialité**
    - Éditeur Markdown pour texte légal personnalisé
    - Copie par défaut fournie pour les nouveaux widgets

    **Paramètres de Consentement**
    - Étiquettes des boutons accepter/rejeter
    - Clé de stockage local pour mémoriser le consentement
    - Exiger le consentement avant l'activation
  </Accordion>

  <Accordion title="Exporter" icon="code">
    **Script d'Intégration**
    - URL de chargement unique pour votre widget
    - Copier et coller avant la balise `</body>`

    **Guides d'Installation**
    - Instructions étape par étape
    - Conseils spécifiques à la plateforme :
      - Google Tag Manager
      - WordPress
      - Webflow
      - Shopify
      - Wix
      - Squarespace
  </Accordion>
</AccordionGroup>

## Déployez Votre Widget

<Steps>
  <Step title="Testez dans l'aperçu">
    Utilisez l'aperçu en direct sur le côté droit du configurateur pour tester votre widget. Cliquez sur l'icône de rafraîchissement si nécessaire.
  </Step>
  <Step title="Copiez le script d'intégration">
    Allez dans l'onglet **Exporter** et copiez votre script d'intégration unique.
  </Step>
  <Step title="Ajoutez à votre site web">
    Collez le script juste avant la balise de fermeture `</body>` sur votre site web.
  </Step>
  <Step title="Testez-le en direct">
    Chargez votre site, cliquez sur le widget et testez une conversation vocale ou par chat. Consultez **Conversations** pour vérifier que la session a été enregistrée.
  </Step>
</Steps>

<Note>
Les modifications de configuration que vous effectuez dans le tableau de bord s'appliquent automatiquement à votre widget intégré. Pas besoin de mettre à jour le script d'intégration. Si les modifications n'apparaissent pas immédiatement, effectuez un rafraîchissement complet (Ctrl+Shift+R ou Cmd+Shift+R).
</Note>

**Remarques :**
- **Les restrictions de domaine** s'appliquent immédiatement. Si configurées, le widget ne se charge que sur les domaines autorisés.
- **HTTPS requis** pour l'accès au microphone (sauf localhost).
- **Responsive mobile** - Le widget s'adapte automatiquement aux écrans mobiles.
- Créez des widgets séparés pour différents sites ou besoins de branding.

## Dépannage

<AccordionGroup>
  <Accordion title="Le widget n'apparaît pas" icon="eye-slash">
    - Vérifiez la console du navigateur pour les erreurs JavaScript
    - Vérifiez que les domaines autorisés incluent votre site
    - Assurez-vous que le script est placé avant la balise `</body>`
    - Vérifiez les paramètres de Content Security Policy
  </Accordion>

  <Accordion title="Accès au microphone refusé" icon="microphone-slash">
    - HTTPS requis (sauf localhost)
    - L'utilisateur doit accorder l'autorisation manuellement
    - Vérifiez les paramètres du microphone du navigateur
    - Certains navigateurs bloquent lors de la première visite
  </Accordion>

  <Accordion title="Pas d'audio ou mauvaise qualité" icon="volume-xmark">
    - WebRTC nécessite des ports UDP ouverts
    - Les VPN d'entreprise peuvent bloquer WebRTC
    - Testez depuis un réseau différent
    - Vérifiez les stats WebRTC du navigateur sur `chrome://webrtc-internals`
  </Accordion>

  <Accordion title="Les modifications ne se mettent pas à jour" icon="arrows-rotate">
    - Cliquez sur **Enregistrer les Modifications** dans le configurateur
    - Rafraîchissement complet de votre site (Ctrl+Shift+R ou Cmd+Shift+R)
    - Videz le cache du navigateur si nécessaire
    - Vérifiez que vous modifiez le bon widget
  </Accordion>
</AccordionGroup>
