## Inviter des membres d'équipe

Ajoutez de nouveaux utilisateurs à votre compte en envoyant des invitations par e-mail.

<Steps>
  <Step title="Ouvrir les paramètres d'équipe">
    Naviguez vers **Paramètres → Équipe** dans votre compte
  </Step>

  <Step title="Cliquer sur Inviter des membres">
    Cliquez sur le bouton **Inviter des membres**
  </Step>

  <Step title="Entrer les adresses e-mail">
    Ajoutez une ou plusieurs adresses e-mail (une par ligne ou séparées par des virgules)
  </Step>

  <Step title="Envoyer les invitations">
    Cliquez sur **Envoyer les invitations**

    Chaque personne reçoit un e-mail avec un lien sécurisé pour rejoindre votre équipe
  </Step>
</Steps>

<Note>
Les invitations expirent après **72 heures (3 jours)**. Vous pouvez renvoyer les invitations expirées depuis la page d'équipe.
</Note>

### Détails de l'invitation

**Expiration :** Les invitations expirent après **72 heures (3 jours)**

**L'e-mail contient :**
- Qui les a invités
- Nom du compte
- Lien d'acceptation sécurisé
- Date d'expiration

## Accepter les invitations

Lorsque quelqu'un reçoit une invitation :

1. Il clique sur le lien dans l'e-mail
2. **S'il n'a pas de compte :** Il s'inscrit avec l'e-mail invité
3. **S'il a déjà un compte :** Il se connecte et accepte l'invitation
4. Il est ajouté à votre équipe en tant que membre
5. Le compte apparaît dans son sélecteur de compte

## Gérer les invitations en attente

Consultez et gérez les invitations que vous avez envoyées :

1. Allez dans **Paramètres → Équipe**
2. Cliquez sur l'onglet **Invitations en attente**

**Vous pouvez :**
- **Renvoyer** - Envoyer un nouvel e-mail d'invitation avec une nouvelle expiration de 72 heures
- **Annuler** - Supprimer l'invitation (le lien devient invalide)

<Tip>
Si quelqu'un ne reçoit pas l'invitation, vérifiez son dossier spam ou renvoyez-la.
</Tip>

## Voir les membres de l'équipe

La page d'équipe affiche tous les membres avec :
- Nom et e-mail
- Statut (Actif ou Invitation en attente)
- Date de dernière connexion
- Date d'adhésion

**Rechercher et filtrer :**
- Rechercher par nom ou e-mail
- Filtrer par statut (Actif vs En attente)

## Supprimer des membres d'équipe

<Warning>
Les membres supprimés perdent l'accès immédiatement.
</Warning>

<Steps>
  <Step title="Aller aux paramètres d'équipe">
    Naviguez vers **Paramètres → Équipe**
  </Step>

  <Step title="Trouver le membre">
    Localisez le membre d'équipe que vous souhaitez supprimer
  </Step>

  <Step title="Supprimer">
    Cliquez sur le menu (⋯) à côté de son nom → **Retirer de l'équipe**
  </Step>

  <Step title="Confirmer">
    Confirmez la suppression

    Il ne peut plus accéder à ce compte
  </Step>
</Steps>

**Ce qui se passe :**
- Le membre perd l'accès immédiatement
- Le compte disparaît de son sélecteur de compte
- Leur contenu créé reste (agents, configurations)
- Ils peuvent être réinvités si nécessaire

## Meilleures pratiques

<AccordionGroup>
  <Accordion title="Intégration de nouveaux membres" icon="user-plus">
    **Quand quelqu'un rejoint :**
    1. Donnez-lui une visite de vos agents
    2. Partagez la documentation pertinente
    3. Établissez des attentes claires pour leur rôle
    4. Faites le point après leur première semaine
  </Accordion>

  <Accordion title="Gestion des invitations" icon="envelope">
    **Conseils :**
    - Vérifiez les adresses e-mail avant d'envoyer
    - Faites un suivi si pas de réponse après 2 jours
    - Renvoyez si l'invitation expire
    - Annulez les invitations envoyées aux mauvaises adresses
    - Nettoyez les invitations expirées mensuellement
  </Accordion>

  <Accordion title="Départ" icon="user-minus">
    **Quand quelqu'un part :**
    1. Retirez son accès immédiatement
    2. Révoquez toutes les clés API qu'il a créées
    3. Mettez à jour les identifiants partagés auxquels il avait accès
    4. Documentez son travail pour la transition
  </Accordion>

  <Accordion title="Collaborateurs externes" icon="user-tie">
    **Pour les sous-traitants ou partenaires :**
    - Envisagez d'utiliser un sous-compte séparé pour les clients
    - Définissez des rappels de calendrier pour réviser l'accès
    - Retirez l'accès quand l'engagement se termine
  </Accordion>
</AccordionGroup>

## Dépannage

<AccordionGroup>
  <Accordion title="Invitation non reçue" icon="envelope">
    **Causes courantes :**
    - E-mail dans le dossier spam
    - Faute de frappe dans l'adresse e-mail
    - E-mail bloqué par le serveur du destinataire

    **Solutions :**
    1. Demandez-leur de vérifier les spams
    2. Vérifiez que l'adresse e-mail est correcte
    3. Renvoyez l'invitation
  </Accordion>

  <Accordion title="Invitation expirée" icon="clock">
    **Solution :**
    1. Allez dans **Paramètres → Équipe → Invitations en attente**
    2. Trouvez l'invitation expirée
    3. Cliquez sur **Renvoyer**
    4. Une nouvelle invitation de 72 heures est envoyée
  </Accordion>

  <Accordion title="Impossible de supprimer un membre" icon="ban">
    **Causes possibles :**
    - Vous n'avez pas la permission
    - Le membre ne peut pas être supprimé

    **Solution :**
    Contactez votre administrateur de compte si vous avez besoin d'aide.
  </Accordion>

  <Accordion title="Le membre ne peut pas voir le compte après acceptation" icon="eye-slash">
    **Causes courantes :**
    - L'utilisateur s'est connecté avec un e-mail différent
    - Compte désactivé après l'invitation

    **Solutions :**
    - Vérifiez qu'il a utilisé l'adresse e-mail invitée
    - Demandez-lui de vérifier tous les comptes dans le sélecteur de compte
    - Vérifiez si le compte est toujours actif
  </Accordion>
</AccordionGroup>

## Conseils de sécurité

<AccordionGroup>
  <Accordion title="Vérifier les adresses e-mail" icon="shield-check">
    Avant d'inviter :
    - Vérifiez l'orthographe
    - Vérifiez que le domaine est correct (attention à .co vs .com)
    - Utilisez des e-mails professionnels pour les comptes d'entreprise
  </Accordion>

  <Accordion title="Révisions d'accès" icon="calendar-check">
    Révisez les membres de l'équipe trimestriellement :
    - Vérifiez que chaque personne a toujours besoin d'accès
    - Supprimez les utilisateurs inactifs
    - Documentez la révision
  </Accordion>

  <Accordion title="Liste de contrôle de départ" icon="list-check">
    Quand quelqu'un part :
    - [ ] Retirer de tous les comptes immédiatement
    - [ ] Révoquer toutes les clés API créées
    - [ ] Changer les mots de passe partagés qu'il connaissait
    - [ ] Examiner son activité récente
  </Accordion>
</AccordionGroup>

## FAQ

<AccordionGroup>

  <Accordion title="Combien de temps les invitations sont-elles valides ?">
    Les invitations expirent après **72 heures (3 jours)**. Vous pouvez renvoyer les invitations expirées pour générer un nouveau lien.
  </Accordion>

  <Accordion title="Puis-je envoyer un message personnel avec l'invitation ?">
    Pas actuellement. Les invitations sont envoyées avec un modèle d'e-mail standard.
  </Accordion>

  <Accordion title="Combien de personnes puis-je inviter ?">
    La taille de l'équipe dépend de votre plan d'abonnement. Vérifiez vos limites actuelles dans **Paramètres → Compte**.
  </Accordion>

  <Accordion title="Quelqu'un peut-il accepter une invitation après expiration ?">
    Non. Les liens d'invitation expirés ne fonctionnent pas. Renvoyez l'invitation pour générer un nouveau lien valide.
  </Accordion>
</AccordionGroup>
