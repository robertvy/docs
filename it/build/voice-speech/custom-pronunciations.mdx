## Panoramica

Le pronunce personalizzate ti permettono di definire come parole specifiche dovrebbero suonare, assicurando che il tuo agente pronunci sempre correttamente nomi di brand, termini tecnici e acronimi.

**Come funziona:**
1. L'agente genera risposta: "Chiamaci a itellicoAI per aiuto"
2. Il sistema applica regole di pronuncia: "Chiamaci a eye-tell-ee-co A I per aiuto"
3. Il sintetizzatore vocale legge il testo modificato
4. Il cliente sente la pronuncia corretta

<Note>
Le sostituzioni di pronuncia sono disponibili sotto **Voce > Pronunce** nella configurazione del tuo agente.
</Note>

---

## Aggiungere Pronunce

### Sostituzione Singola

1. Fai clic su **Aggiungi Sostituzione**
2. Inserisci la **parola target** (la parola da sostituire)
3. Inserisci la **pronuncia** (come dovrebbe suonare)
4. Fai clic su **Aggiungi**

**Esempi:**
- **Target:** Nike → **Pronuncia:** Nai-key
- **Target:** API → **Pronuncia:** A-P-I
- **Target:** 24/7 → **Pronuncia:** ventiquattro sette
- **Target:** EST → **Pronuncia:** Ora Orientale

<Warning>
**Le parole target devono essere solo parole singole** - gli spazi non sono consentiti. Usa parole complete come `Nike` o `API`, non frasi come `servizio clienti`.
</Warning>

### Importazione di Massa via CSV

Per importare molte pronunce in una volta:

1. Fai clic su **Importa CSV**
2. Scarica il modello se necessario
3. Prepara il tuo CSV con due colonne: `target`, `replacement`
4. Trascina e rilascia il tuo file CSV o fai clic per selezionare
5. Rivedi l'anteprima e fai clic su **Importa**

**Formato CSV:**
```csv
target,replacement
Nike,Nai-key
EST,Ora Orientale
24/7,ventiquattro sette
API,A-P-I
```

**Limiti:**
- Massimo 500 pronunce per agente
- Dimensione massima file CSV: 5MB

## Suggerimenti per l'Ortografia Fonetica

### Ortografia Fonetica Semplice

Scrivi le sostituzioni come testo fonetico semplice e leggibile:

**Buoni esempi:**
- Nike → Nai-key
- SQL → sequel (o) S-Q-L
- Wi-Fi → why-figh
- iOS → eye-oh-ess

**Suggerimenti:**
- Usa trattini per separare le sillabe: Nai-key, A-P-I
- Scrivilo come spiegheresti la pronuncia a qualcuno
- Testa con la tua voce per sentire come suona

### Schemi Comuni

**Acronimi (scrivi ogni lettera):**
- API → A-P-I
- URL → U-R-L
- HTTP → H-T-T-P
- AWS → A-W-S

**Nomi di brand (ortografia fonetica):**
- Nike → Nai-key
- Adidas → ah-dee-dahs
- Chipotle → chih-poht-lay

**Numeri e ora:**
- 24/7 → ventiquattro sette
- 9am → nove A-M
- EST → Ora Orientale
- 5pm → cinque P-M

**Termini tecnici:**
- SQL → sequel (o) S-Q-L
- API → A-P-I
- URL → you-are-ell
- WiFi → why-figh

## Avanzato: Fonemi IPA

Per un controllo preciso della pronuncia, puoi usare la notazione dell'Alfabeto Fonetico Internazionale (IPA) con il formato `<< >>`:

**Formato:** `<<fonema|fonema|fonema>>`

**Esempio:**
```
espresso → <<ɛ|s|p|r|ɛ|s|oʊ>>
```

<Note>
I fonemi IPA forniscono un controllo preciso ma richiedono conoscenza della notazione fonetica. L'ortografia fonetica semplice (Nai-key) funziona bene per la maggior parte dei casi.
</Note>

## Gestire le Pronunce

### Modificare Sostituzioni Esistenti

1. Fai clic su qualsiasi chip di sostituzione nell'elenco
2. Modifica il target o la pronuncia
3. Fai clic su **Salva**

### Eliminare Sostituzioni

- Fai clic sull'**icona cestino** su qualsiasi chip di sostituzione per eliminarlo
- Oppure fai clic su **Cancella Tutto** per rimuovere tutte le pronunce (richiede conferma)

### Esportare Pronunce

1. Fai clic su **Esporta CSV**
2. Il file CSV viene scaricato con tutte le tue pronunce attuali
3. Formato nome file: `{nome_agente}_pronunciations.csv`

Usa i file CSV esportati per fare il backup delle tue pronunce o copiarle su altri agenti.

## Testare le Pronunce

Dopo aver aggiunto le pronunce:

1. Fai una chiamata di test al tuo agente
2. Attiva risposte contenenti le parole target
3. Ascolta per verificare che la pronuncia suoni corretta
4. Regola l'ortografia fonetica se necessario

**Suggerimenti per i test:**
- Testa nel contesto, non in isolamento
- Fai ascoltare ai membri del team
- Regola la sillabazione o l'ortografia se non suona bene

## Migliori Pratiche

<AccordionGroup>
  <Accordion title="Inizia con Problemi Comuni" icon="bullseye">
    Aggiungi pronunce per parole che sai essere pronunciate male nelle chiamate reali. Rivedi le trascrizioni delle chiamate per identificare problemi di pronuncia ricorrenti.
  </Accordion>

  <Accordion title="Mantienilo Semplice" icon="sparkles">
    Usa ortografia fonetica (Nai-key) piuttosto che IPA complesso a meno che non sia necessario. Il testo fonetico semplice è più facile da mantenere e funziona bene per la maggior parte dei casi.
  </Accordion>

  <Accordion title="Testa Prima di Implementare" icon="vial">
    Testa sempre le pronunce con chiamate reali prima di andare in produzione. Fai una chiamata di test e attiva risposte contenenti le parole target per verificare che suonino correttamente.
  </Accordion>

  <Accordion title="Documenta le tue Regole" icon="file-arrow-down">
    Esporta regolarmente il tuo CSV come backup. Usa il pulsante Esporta CSV per salvare le tue pronunce e mantenerle versionate insieme alla configurazione del tuo agente.
  </Accordion>

  <Accordion title="Usa Ortografia Coerente" icon="check">
    Se hai "API" scritto come "A-P-I", usa quel formato in modo coerente in tutte le tue pronunce. La coerenza migliora la manutenibilità.
  </Accordion>

  <Accordion title="Solo Parole Singole" icon="letter-a">
    Ricorda che le parole target non possono contenere spazi - funzionano solo parole singole complete. Usa `Nike` o `API`, non frasi come `servizio clienti`.
  </Accordion>
</AccordionGroup>
