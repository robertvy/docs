## Elementi di Testo

### Cosa Sono gli Elementi di Testo?

Gli elementi di testo sono contenuti che inserisci direttamente nell'editor della base di conoscenza di itellicoAI. Sono il tipo di contenuto più diretto e affidabile.

### Quando Usare gli Elementi di Testo

<AccordionGroup>
  <Accordion title="Scrivere FAQ" icon="circle-question">
    Crea coppie domanda-risposta direttamente nel sistema.

    **Esempio:**
    ```text wrap
    Titolo: Come posso reimpostare la mia password?

    Contenuto:
    Per reimpostare la tua password:

    1. Vai alla pagina di accesso
    2. Clicca "Password Dimenticata"
    3. Inserisci il tuo indirizzo email
    4. Controlla la tua email per un link di reimpostazione
    5. Clicca il link e crea una nuova password

    Requisiti della password:
    - Minimo 8 caratteri
    - Almeno una lettera maiuscola
    - Almeno un numero
    - Almeno un carattere speciale

    Se non ricevi l'email entro 5 minuti, controlla la cartella spam
    o contatta support@company.com.
    ```
  </Accordion>

  <Accordion title="Creare riassunti di politiche" icon="book">
    Scrivi dichiarazioni di politica chiare e concise.

    **Esempio:**
    ```text wrap
    Titolo: Politica di Reso - Prodotti Digitali

    Contenuto:
    I prodotti digitali possono essere rimborsati entro 30 giorni dall'acquisto se:

    Idoneo per rimborso:
    - Il prodotto ha un difetto tecnico che impedisce l'utilizzo
    - La descrizione del prodotto era materialmente imprecisa
    - Il cliente non ha acceduto o scaricato il prodotto

    Non idoneo per rimborso:
    - Cambio di idea dopo aver acceduto al prodotto
    - Problemi di compatibilità rivelati nella descrizione del prodotto
    - Errore dell'utente o incomprensione delle funzionalità

    Per richiedere un rimborso:
    Invia un'email a support@company.com con il tuo numero d'ordine e il motivo
    della richiesta di rimborso.

    Tempo di elaborazione: 5-7 giorni lavorativi
    Metodo di rimborso: Metodo di pagamento originale
    ```
  </Accordion>

  <Accordion title="Documentare procedure" icon="list-check">
    Istruzioni passo-passo per i processi.

    **Esempio:**
    ```
    Titolo: Processo di Modifica Ordine

    Contenuto:
    I clienti possono modificare gli ordini entro 24 ore dal loro posizionamento.

    Cosa può essere modificato:
    - Indirizzo di spedizione
    - Velocità di consegna
    - Quantità degli articoli (se l'inventario è disponibile)

    Cosa non può essere modificato:
    - Metodo di pagamento (deve annullare e riordinare)
    - Articoli dopo l'inizio dell'elaborazione
    - Ordini effettuati più di 24 ore fa

    Processo di modifica:
    1. Il cliente contatta il supporto via telefono o email
    2. L'agente verifica che l'ordine sia nella finestra di modifica
    3. L'agente controlla l'inventario per le modifiche richieste
    4. L'agente aggiorna l'ordine nel sistema
    5. Il cliente riceve un'email di conferma

    Se l'ordine è già stato spedito, il cliente deve usare il processo di reso.
    ```
  </Accordion>

  <Accordion title="Informazioni di riferimento rapido" icon="clock">
    Informazioni brevi, a cui si fa riferimento frequentemente.

    **Esempio:**
    ```
    Titolo: Orari di Apertura & Informazioni di Contatto

    Contenuto:
    Assistenza Clienti:
    - Telefono: 1-800-555-0123
    - Email: support@company.com
    - Orari: Lunedì-Venerdì, 9:00 - 18:00 CET
    - Fuori orario: emergency@company.com (solo problemi urgenti)

    Vendite:
    - Telefono: 1-800-555-0124
    - Email: sales@company.com
    - Orari: Lunedì-Venerdì, 8:00 - 20:00 CET

    Fatturazione:
    - Telefono: 1-800-555-0125
    - Email: billing@company.com
    - Orari: Lunedì-Venerdì, 9:00 - 17:00 CET
    ```
  </Accordion>
</AccordionGroup>

### Creare Elementi di Testo

<Steps>
  <Step title="Naviga alla cartella">
    Apri la cartella in cui vuoi aggiungere l'elemento.
  </Step>

  <Step title="Clicca 'Aggiungi Elemento di Conoscenza'">
    Clicca il pulsante per creare un nuovo elemento.
  </Step>

  <Step title="Seleziona la categoria 'TEXT'">
    Scegli TEXT dalle opzioni del tipo di contenuto.
  </Step>

  <Step title="Inserisci il titolo">
    Dai al tuo elemento un titolo chiaro e descrittivo.
  </Step>

  <Step title="Scrivi il contenuto">
    Inserisci il tuo contenuto nell'editor. Usa la formattazione per chiarezza:
    - Intestazioni per le sezioni
    - Elenchi puntati per le liste
    - Numeri per i passaggi
    - Grassetto per enfasi
  </Step>

  <Step title="Clicca 'Crea'">
    Salva il tuo elemento di testo. Lo stato mostrerà immediatamente COMPLETATO.
  </Step>
</Steps>


## Elementi di Caricamento File

### Cosa Sono gli Elementi di Caricamento File?

Gli elementi di caricamento file ti permettono di caricare documenti esistenti in vari formati. Il sistema estrae il contenuto testuale e lo rende disponibile ai tuoi agenti.

### Quando Usare i Caricamenti di File

<AccordionGroup>
  <Accordion title="Documentazione esistente" icon="file">
    Hai già contenuto in formato documento.

    **Esempi:**
    - Manuali utente
    - Specifiche di prodotto
    - Documenti legali
    - Materiali di formazione
  </Accordion>

  <Accordion title="Documenti formattati" icon="file-pdf">
    Documenti con layout specifici che sono più facili da mantenere come file.

    **Esempi:**
    - Diagrammi tecnici
    - Tabelle e grafici
    - Layout multi-colonna
    - Template brandizzati
  </Accordion>

  <Accordion title="Documenti di terze parti" icon="building">
    Documentazione che ricevi da fornitori o partner.

    **Esempi:**
    - Cataloghi fornitori
    - Documenti di conformità
    - Materiali di certificazione
  </Accordion>
</AccordionGroup>

### Requisiti dei File

**Specifiche dei file:**
- **Formati:** PDF, DOC, DOCX, TXT e altri formati di documenti
- **Limite di dimensione:** Massimo 10MB
- **Contenuto:** Documenti basati su testo e immagini scansionate (il parsing avanzato gestisce la maggior parte delle scansioni)
- **Protezione:** Nessuna protezione con password

<Note>
Il sistema utilizza il parsing avanzato dei documenti per estrarre testo da immagini scansionate e PDF. La maggior parte dei documenti scansionati verrà elaborata correttamente, anche se scansioni di qualità molto scarsa potrebbero richiedere l'inserimento manuale del testo.
</Note>

### Creare Elementi di File

<Steps>
  <Step title="Naviga alla cartella">
    Apri la cartella in cui vuoi aggiungere il file.
  </Step>

  <Step title="Clicca 'Aggiungi Elemento di Conoscenza'">
    Clicca il pulsante per creare un nuovo elemento.
  </Step>

  <Step title="Seleziona la categoria 'FILE'">
    Scegli FILE dalle opzioni del tipo di contenuto.
  </Step>

  <Step title="Inserisci il titolo">
    Dai al tuo file un titolo descrittivo (separato dal nome del file).
  </Step>

  <Step title="Carica il file">
    Clicca **Carica** e seleziona il tuo documento dal tuo computer.
  </Step>

  <Step title="Clicca 'Crea'">
    Il sistema caricherà e inizierà a elaborare il tuo file.

    Progressione dello stato:
    - PENDING → File caricato, in coda per l'elaborazione
    - PROCESSING → Estrazione del contenuto testuale
    - COMPLETED → Pronto per l'uso
    - FAILED → Si è verificato un errore di elaborazione
  </Step>
</Steps>

### Esempio JSON di Elemento File

```json wrap
{
  "category": "FILE",
  "title": "Manuale Utente - Prodotto X",
  "file": "/uploads/knowledge/manuale-utente-prodotto-x.pdf",
  "status": "COMPLETED"
}
```

### Tempo di Elaborazione

Il tempo di elaborazione dei file varia in base a:
- **Dimensione del file:** File più grandi richiedono più tempo
- **Numero di pagine:** Più pagine = elaborazione più lunga
- **Complessità:** Tabelle, immagini e layout complessi rallentano l'elaborazione
- **Qualità del testo:** Testo pulito e semplice si estrae più velocemente

**Tempi di elaborazione tipici:**
- File piccoli (< 1MB, 10 pagine): 10-30 secondi
- File medi (1-5MB, 10-50 pagine): 30-90 secondi
- File grandi (5-10MB, 50+ pagine): 2-5 minuti

### Problemi Comuni con i File

<AccordionGroup>
  <Accordion title="Elaborazione fallita" icon="circle-xmark">
    **Cause:**
    - Il file supera i 10MB
    - Il file è protetto da password
    - Il file è corrotto
    - Immagini scansionate di qualità molto scarsa

    **Soluzioni:**
    - Comprimi il file o dividilo in file più piccoli
    - Rimuovi la protezione con password
    - Riesporta il file dalla fonte
    - Per scansioni di qualità molto scarsa, copia il contenuto in un elemento di testo
  </Accordion>

  <Accordion title="Contenuto estratto in modo errato" icon="text-slash">
    **Cause:**
    - Layout complessi (multi-colonna, tabelle)
    - Immagini scansionate di qualità molto scarsa
    - Font speciali o codifica
    - Campi modulo ed elementi interattivi

    **Soluzioni:**
    - Controlla il contenuto estratto in modalità modifica
    - Ricrea come elemento di testo con formattazione corretta
    - Semplifica il layout del documento prima di caricarlo
    - Esporta come documento di testo semplice
  </Accordion>

  <Accordion title="L'elaborazione richiede troppo tempo" icon="clock">
    **Cosa fare:**
    - Aspetta 5-10 minuti prima di presumere un fallimento
    - Controlla la dimensione del file e il numero di pagine
    - Per file grandi, considera di dividerli in più file
    - Converti in testo e carica come elementi TEXT
  </Accordion>
</AccordionGroup>

### Best Practice per il Caricamento di File

<CardGroup cols={2}>
  <Card title="Ottimizza prima del caricamento" icon="compress">
    - Comprimi file grandi
    - Rimuovi immagini non necessarie
    - Usa documenti basati su testo
    - Mantieni sotto i 5MB quando possibile
  </Card>
  <Card title="Testa l'estrazione" icon="vial">
    - Rivedi il contenuto estratto dopo l'elaborazione
    - Controlla i problemi di formattazione
    - Verifica che le informazioni critiche siano accurate
    - Ricarica se l'estrazione è scarsa
  </Card>
</CardGroup>

## Elementi URL

### Cosa Sono gli Elementi URL?

Gli elementi URL estraggono contenuto da pagine web e lo memorizzano nella tua base di conoscenza. Questo è utile per fare riferimento a documentazione online, centri di assistenza o post di blog.

### Quando Usare gli Elementi URL

<AccordionGroup>
  <Accordion title="Documentazione pubblica" icon="globe">
    Fai riferimento a documentazione esterna che non mantieni.

    **Esempi:**
    - Documentazione API (tua o di terze parti)
    - Basi di conoscenza pubbliche
    - Articoli del centro assistenza
    - Pagine di prodotto
  </Accordion>

  <Accordion title="Contenuto aggiornato frequentemente" icon="arrows-rotate">
    Contenuto che cambia regolarmente e che vuoi mantenere aggiornato ri-estraendo.

    **Esempi:**
    - Pagine dei prezzi
    - Disponibilità prodotti
    - Promozioni attuali
    - Pagine di stato
  </Accordion>

  <Accordion title="Post di blog o articoli" icon="newspaper">
    Contenuto educativo o annunci.

    **Esempi:**
    - Guide how-to
    - Articoli di best practice
    - Annunci di prodotto
    - Tutorial sulle funzionalità
  </Accordion>
</AccordionGroup>

### Creare Elementi URL

<Steps>
  <Step title="Naviga alla cartella">
    Apri la cartella in cui vuoi aggiungere l'URL.
  </Step>

  <Step title="Clicca 'Aggiungi Elemento di Conoscenza'">
    Clicca il pulsante per creare un nuovo elemento.
  </Step>

  <Step title="Seleziona la categoria 'URL'">
    Scegli URL dalle opzioni del tipo di contenuto.
  </Step>

  <Step title="Inserisci il titolo">
    Dai al contenuto un titolo descrittivo.
  </Step>

  <Step title="Inserisci l'URL di origine">
    Incolla l'URL completo incluso https://

    **Esempio:**
    ```
    https://docs.company.com/api/authentication
    ```
  </Step>

  <Step title="Clicca 'Crea'">
    Il sistema recupererà ed elaborerà la pagina web.

    Progressione dello stato:
    - PENDING → URL in coda per lo scraping
    - PROCESSING → Recupero ed estrazione del contenuto
    - COMPLETED → Contenuto disponibile
    - FAILED → Si è verificato un errore di scraping
  </Step>
</Steps>

### Esempio JSON di Elemento URL

```json wrap
{
  "category": "URL",
  "title": "Documentazione Autenticazione API",
  "source_url": "https://docs.company.com/api/authentication",
  "status": "COMPLETED"
}
```

### Requisiti URL

**URL funzionanti:**
- Accessibili pubblicamente (nessun login richiesto)
- Pagine di contenuto HTML semplici
- Siti di documentazione
- Post di blog e articoli
- Pagine di contenuto statico

**URL problematici:**
- Pagine che richiedono autenticazione
- Applicazioni pesanti in JavaScript (SPA)
- Contenuto a pagamento
- Contenuto caricato dinamicamente
- Applicazioni interattive

<Note>
Lo scraping di URL funziona meglio con pagine web semplici basate su testo. Applicazioni web complesse potrebbero non essere estratte con successo.
</Note>

### Problemi Comuni con gli URL

<AccordionGroup>
  <Accordion title="Scraping fallito" icon="circle-xmark">
    **Cause:**
    - La pagina richiede login/autenticazione
    - L'URL è errato o rotto
    - Il contenuto si carica tramite JavaScript
    - Il sito web blocca lo scraping (robots.txt)
    - La pagina non esiste (404)

    **Soluzioni:**
    - Verifica che l'URL sia accessibile pubblicamente
    - Testa l'URL in una finestra del browser in incognito
    - Controlla che l'URL sia completo e corretto
    - Copia il contenuto manualmente in un elemento di testo
    - Usa l'esportazione PDF della pagina
  </Accordion>

  <Accordion title="Contenuto incompleto o errato" icon="puzzle-piece">
    **Cause:**
    - Contenuto renderizzato con JavaScript non catturato
    - Caricamento dinamico del contenuto
    - Più schede/sezioni sulla pagina
    - Commenti o barra laterale estratti invece del contenuto principale

    **Soluzioni:**
    - Ispeziona il contenuto estratto in modalità modifica
    - Usa URL diretto al contenuto specifico
    - Copia il contenuto desiderato in un elemento di testo
    - Esporta la pagina come PDF e carica invece
  </Accordion>

  <Accordion title="Il contenuto diventa obsoleto" icon="calendar">
    **Soluzione:**
    Il contenuto URL viene estratto una volta al momento della creazione. Per aggiornare:
    - Elimina e ricrea l'elemento URL
    - Oppure copia il contenuto attuale in un elemento di testo per aggiornamenti manuali

    Per contenuto che cambia frequentemente, considera:
    - Elementi di testo manuali che aggiorni regolarmente
    - Esportazioni PDF che aggiorni periodicamente
  </Accordion>
</AccordionGroup>

### Best Practice per gli URL

<CardGroup cols={2}>
  <Card title="Testa l'accessibilità" icon="unlock">
    - Apri l'URL in una finestra in incognito
    - Verifica che non sia richiesto il login
    - Controlla che il contenuto sia visibile
    - Assicurati che la pagina si carichi rapidamente
  </Card>
  <Card title="Rivedi il contenuto estratto" icon="eye">
    - Controlla il contenuto dopo lo scraping
    - Verifica che sia stato catturato il contenuto corretto
    - Cerca problemi di formattazione
    - Conferma che non ci sia contenuto extra (annunci, barre laterali)
  </Card>
</CardGroup>

## Flusso dello Stato di Elaborazione

Gli elementi di conoscenza attraversano due pipeline di elaborazione separate:

1. **Elaborazione del Contenuto** - Estrazione del testo da file/URL
2. **Indicizzazione Vettoriale** - Preparazione del contenuto per RAG (ricerca semantica)

### Stato Elaborazione Contenuto

Questo traccia l'estrazione del contenuto testuale dalla tua fonte.

<Steps>
  <Step title="PENDING">
    **Significato:** Elemento creato, in coda per l'estrazione del contenuto

    **Cosa sta succedendo:**
    - L'elemento è stato salvato nel database
    - In attesa che il worker di elaborazione lo prelevi
    - Solitamente molto breve (secondi)

    **Azione necessaria:** Nessuna - aspetta che inizi l'elaborazione
  </Step>

  <Step title="PROCESSING">
    **Significato:** Il contenuto dell'elemento è in fase di estrazione

    **Cosa sta succedendo:**
    - Per FILE: Estrazione del testo da PDF, Word, ecc.
    - Per URL: Recupero e scraping della pagina web specifica
    - Per TEXT: N/A (passa direttamente a COMPLETED)

    **Azione necessaria:** Aspetta che l'elaborazione sia completata
  </Step>

  <Step title="COMPLETED">
    **Significato:** Estrazione del contenuto completata con successo

    **Cosa sta succedendo:**
    - Il contenuto è stato estratto e memorizzato
    - L'indicizzazione vettoriale inizierà automaticamente
    - L'elemento sarà disponibile una volta completata l'indicizzazione

    **Azione necessaria:** Nessuna - aspetta l'indicizzazione vettoriale
  </Step>

  <Step title="FAILED">
    **Significato:** L'estrazione del contenuto ha riscontrato un errore

    **Cosa sta succedendo:**
    - L'estrazione del file è fallita (PDF corrotto, formato non supportato)
    - Lo scraping dell'URL è fallito (pagina non trovata, bloccata, richiede autenticazione)
    - Il contenuto non è disponibile per gli agenti

    **Azione necessaria:** Controlla il messaggio di errore e risolvi il problema
  </Step>
</Steps>

**Flusso Elaborazione Contenuto:**
CREATED → PENDING → PROCESSING → COMPLETED o FAILED

---

### Stato Indicizzazione Vettoriale

Dopo che il contenuto è estratto, deve essere indicizzato per RAG (ricerca semantica). Questo permette agli agenti di trovare conoscenza rilevante basata sul significato, non solo sulle parole chiave.

<Steps>
  <Step title="PENDING">
    **Significato:** In attesa che inizi l'indicizzazione vettoriale

    **Cosa sta succedendo:**
    - Elaborazione del contenuto completata con successo
    - In coda per la generazione di embedding
    - Solitamente breve (da secondi a minuti)

    **Azione necessaria:** Nessuna
  </Step>

  <Step title="INDEXING">
    **Significato:** Creazione di embedding vettoriali per RAG

    **Cosa sta succedendo:**
    - Il contenuto viene diviso in blocchi
    - Vengono generati embedding AI per ogni blocco
    - I vettori vengono memorizzati nella base di conoscenza

    **Azione necessaria:** Aspetta - può richiedere diversi minuti per documenti grandi
  </Step>

  <Step title="INDEXED">
    **Significato:** L'elemento è completamente pronto per il recupero RAG

    **Cosa sta succedendo:**
    - Gli embedding vettoriali sono stati memorizzati con successo
    - L'elemento può essere recuperato tramite ricerca semantica
    - Gli agenti possono ora usare questa conoscenza

    **Azione necessaria:** Nessuna - l'elemento è completamente pronto
  </Step>

  <Step title="FAILED">
    **Significato:** L'indicizzazione vettoriale è fallita

    **Cosa sta succedendo:**
    - La generazione di embedding ha riscontrato un errore
    - L'elemento non apparirà nei risultati RAG
    - Potrebbe essere disponibile solo per l'iniezione nel contesto

    **Azione necessaria:** Prova a reindicizzare o contatta il supporto
  </Step>

  <Step title="REMOVED">
    **Significato:** I vettori sono stati eliminati

    **Cosa sta succedendo:**
    - L'elemento è stato eliminato o esplicitamente rimosso dall'indice
    - Non più disponibile per RAG

    **Azione necessaria:** Reindicizza se necessario
  </Step>
</Steps>

**Flusso Indicizzazione Vettoriale:**
PENDING → INDEXING → INDEXED o FAILED

<Note>
**Entrambi gli stati devono avere successo per la piena funzionalità:**
- Stato Contenuto: COMPLETED
- Stato Vettoriale: INDEXED

Solo allora l'elemento può essere utilizzato sia nell'iniezione del contesto che nel recupero RAG.
</Note>

## Gestione degli Errori

### Quando gli Elementi Falliscono

Se un elemento di conoscenza mostra lo stato FAILED:

<Steps>
  <Step title="Controlla il messaggio di errore">
    Clicca sull'elemento fallito per vedere i dettagli dell'errore.
  </Step>

  <Step title="Identifica la causa">
    Cause comuni:
    - **File:** File troppo grande, corrotto, protetto da password, immagine scansionata
    - **URL:** Autenticazione richiesta, link rotto, contenuto non accessibile
  </Step>

  <Step title="Prova le soluzioni">
    - **Per File:** Comprimi, rimuovi protezione, aggiungi livello di testo, o converti in testo
    - **Per URL:** Verifica accessibilità, prova URL diverso, o copia contenuto in elemento di testo
  </Step>

  <Step title="Ricrea o converti">
    - Elimina l'elemento fallito
    - Crea un nuovo elemento con le correzioni applicate
    - Oppure converti in elemento di testo copiando il contenuto manualmente
  </Step>
</Steps>

## Monitoraggio dell'Elaborazione

### Indicatori del Dashboard

Nel dashboard della tua base di conoscenza, puoi vedere lo stato di elaborazione a colpo d'occhio per ogni elemento nelle tue cartelle:

**Indicatori di stato:**
- Segno di spunta verde = COMPLETED
- Clessidra = PROCESSING
- Simbolo di pausa = PENDING
- X rossa = FAILED

**Esempio vista cartella:**
- Manuale Utente.pdf - COMPLETED
- Guida Avvio Rapido.pdf - PROCESSING
- Documentazione API - PENDING
- Manuale Legacy.pdf - FAILED

### Elaborazione in Blocco

Quando carichi più elementi:

1. Gli elementi vengono elaborati in sequenza o in parallelo (dipende dal sistema)
2. Controlla dopo 5-10 minuti per batch grandi
3. Rivedi lo stato di ogni elemento
4. Risolvi eventuali fallimenti individualmente

<Tip>
Carica prima un elemento di test per verificare il formato e l'elaborazione prima del caricamento in blocco.
</Tip>

## Scegliere il Tipo di Contenuto Giusto

Usa questo albero decisionale per selezionare il miglior tipo di contenuto:

**Hai contenuto esistente?**
- No → Usa TEXT (scrivi direttamente)
- Sì, è un documento →
  - Sotto 10MB → Usa FILE
  - Oltre 10MB → Estrai testo, usa TEXT
- Sì, è una pagina web →
  - Accessibile pubblicamente → Usa URL (se lo scraping fallisce, copia in TEXT)
  - Non accessibile → Copia contenuto in TEXT

### Raccomandazioni Rapide

| La Tua Situazione | Miglior Tipo di Contenuto |
|-------------------|---------------------------|
| Scrivere FAQ da zero | TEXT |
| Hai documenti Word/PDF esistenti | Carica come FILE |
| Hai documenti sotto 10MB | FILE |
| Hai documenti oltre 10MB | Dividi in file più piccoli o estrai in TEXT |
| Documentazione web pubblica | URL (con TEXT come backup) |
| Contenuto privato/autenticato | Copia in TEXT |
| Necessità di disponibilità immediata | TEXT (nessun ritardo di elaborazione) |
| La formattazione complessa è importante | FILE |
