Il widget web consente ai visitatori di interagire con il tuo agente AI tramite voce, chat o entrambi contemporaneamente. Configura tutto tramite l'editor visuale—nessuna codifica richiesta—poi copia e incolla un singolo script di incorporamento.

<Steps>
  <Step title="Apri il gestore widget">
    Vai su **Web → Widget** nella dashboard
  </Step>
  <Step title="Crea un nuovo widget">
    Clicca **Crea Widget** e fornisci:
    - **Nome widget** - Etichetta interna per riferimento
    - **Agente** - Seleziona quale agente alimenta questo widget
  </Step>
  <Step title="Configura il widget">
    Usa le schede dell'editor visuale per personalizzare il tuo widget (vedi sezioni sotto)
  </Step>
  <Step title="Salva le modifiche">
    Clicca **Salva Modifiche** in basso a destra per rendere persistenti tutti gli aggiornamenti
  </Step>
  <Step title="Ottieni il codice di incorporamento">
    Vai alla scheda **Esporta** e copia il tuo script di incorporamento unico
  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="Setup" icon="gear">
    - **Nome widget** - Etichetta interna mostrata nell'elenco widget
    - **Agente** - Passa a un agente diverso in qualsiasi momento
    - **Nome visualizzato** - Nome rivolto al cliente mostrato nel widget
    - **Domini consentiti** - Limita dove il widget può caricarsi (lascia vuoto per consentire tutti i domini)
  </Accordion>

  <Accordion title="Aspetto" icon="palette">
    **Preset Temi**
    - Parti da palette di colori curate
    - Scorri le opzioni preset

    **Layout**
    - Posizione: in basso a sinistra o in basso a destra
    - Dimensione widget e offset dai bordi
    - Stato iniziale: compresso, espanso o sempre aperto

    **Avatar**
    - Effetto animato "AI Pulse"
    - Carica immagine personalizzata
    - Collega a URL immagine esterna

    **Colori e Gradienti**
    - Sfondi e colori CTA
    - Stile bolle trascrizione
    - Colori stato e gradienti orb

    **Raggio Bordo**
    - Arrotondamento pulsante launcher
    - Angoli finestra chat
    - Bolle messaggio
    - Campo input e pulsante CTA

    **Animazioni**
    - Effetto wiggle o ripple opzionale sul pulsante chiamata
  </Accordion>

  <Accordion title="Contenuto" icon="language">
    **Preset Lingua**
    - Bundle inglese e tedesco preconfigurati
    - Imposta tutte le etichette widget e testo legale

    **Messaggi di Benvenuto**
    - Testo prompt di benvenuto
    - Testo placeholder
    - Messaggi di stato (ascolto, parlando, pensando, connessione)

    **Testo Feedback**
    - Prompt feedback
    - Testo pulsante invio
    - Messaggio di ringraziamento

    **Override Etichette**
    - Personalizza singole stringhe UI
    - Gruppi: Chat, Controlli, Trascrizione, Audio, Accessibilità
  </Accordion>

  <Accordion title="Funzionalità" icon="sliders">
    - **Modalità solo chat** - Disabilita voce, solo chat
    - **Consenti testo durante la chiamata** - Mantieni l'input di testo attivo durante le chiamate vocali
    - **Toggle trascrizione** - Mostra/nascondi trascrizione live
    - **Controllo mute** - Mostra pulsante mute per consentire all'utente di disattivare il microfono
    - **Timer chiamata** - Visualizza tempo trascorso
    - **Branding** - Mostra/nascondi "Powered by itellicoAI"
    - **Salta saluto** - Evita il saluto se l'utente scrive per primo
    - **Raccolta feedback** - Abilita feedback post-chiamata (non ancora memorizzato, in arrivo)
  </Accordion>

  <Accordion title="Azioni" icon="link">
    Configura strumenti lato client che il tuo agente può invocare durante le conversazioni per mostrare link, aprire pagine o attivare chiamate telefoniche nel browser dell'utente.

    <Note>
    **Strumenti Client**: A differenza delle azioni API che vengono eseguite sul tuo server, gli strumenti client vengono eseguiti direttamente nel browser dell'utente. L'LLM decide quando chiamare questi strumenti in base al contesto della conversazione e alla descrizione del trigger che fornisci.
    </Note>

    Aggiungi fino a cinque strumenti client:

    - **Testo pulsante** - Etichetta mostrata all'utente (es. "Visualizza prezzi", "Chiama supporto")
    - **URL** - Link web (`https://`) o numero di telefono (`tel:`)
    - **Trigger** - Descrivi quando l'agente dovrebbe invocare questo strumento (es. "Quando l'utente chiede informazioni sui prezzi", "Quando l'utente vuole parlare con un umano")
    - **Comportamento**:
      - **Auto-click** - Apri automaticamente il link quando viene invocato lo strumento
      - **Attendi risposta** - Mostra pulsante e attendi che l'utente clicchi (con timeout opzionale)
      - **Termina chiamata al click** - Termina automaticamente la conversazione dopo che l'utente clicca

    I link dei numeri di telefono (`tel:`) terminano automaticamente la chiamata dopo che l'utente clicca.
  </Accordion>

  <Accordion title="Privacy" icon="shield-check">
    **Termini e Privacy**
    - Editor Markdown per testo legale personalizzato
    - Testo predefinito fornito per nuovi widget

    **Impostazioni Consenso**
    - Etichette pulsante accetta/rifiuta
    - Chiave local storage per ricordare il consenso
    - Richiedi consenso prima dell'attivazione
  </Accordion>

  <Accordion title="Esporta" icon="code">
    **Script di Incorporamento**
    - URL loader unico per il tuo widget
    - Copia e incolla prima del tag `</body>`

    **Guide di Installazione**
    - Istruzioni passo-passo
    - Suggerimenti specifici per piattaforma:
      - Google Tag Manager
      - WordPress
      - Webflow
      - Shopify
      - Wix
      - Squarespace
  </Accordion>

## Distribuisci il Tuo Widget

<Steps>
  <Step title="Testa nell'anteprima">
    Usa l'anteprima live sul lato destro del configuratore per testare il tuo widget. Clicca l'icona di aggiornamento se necessario.
  </Step>
  <Step title="Copia lo script di incorporamento">
    Vai alla scheda **Esporta** e copia il tuo script di incorporamento unico.
  </Step>
  <Step title="Aggiungi al tuo sito web">
    Incolla lo script appena prima del tag di chiusura `</body>` sul tuo sito web.
  </Step>
  <Step title="Testalo live">
    Carica il tuo sito, clicca sul widget e testa una conversazione vocale o chat. Controlla **Conversazioni** per verificare che la sessione sia stata registrata.
  </Step>
</Steps>

<Note>
Le modifiche di configurazione che apporti nella dashboard si applicano automaticamente al tuo widget incorporato. Non è necessario aggiornare lo script di incorporamento. Se le modifiche non appaiono immediatamente, fai un hard refresh (Ctrl+Shift+R o Cmd+Shift+R).
</Note>

**Note:**
- **Restrizioni dominio** si applicano immediatamente. Se configurate, il widget si carica solo sui domini consentiti.
- **HTTPS richiesto** per l'accesso al microfono (tranne localhost).
- **Responsive mobile** - Il widget si adatta automaticamente agli schermi mobili.
- Crea widget separati per siti diversi o esigenze di branding.

## Risoluzione Problemi

<AccordionGroup>
  <Accordion title="Widget non appare" icon="eye-slash">
    - Controlla la console del browser per errori JavaScript
    - Verifica che i domini consentiti includano il tuo sito
    - Assicurati che lo script sia posizionato prima del tag `</body>`
    - Controlla le impostazioni Content Security Policy
  </Accordion>

  <Accordion title="Accesso microfono negato" icon="microphone-slash">
    - HTTPS richiesto (tranne localhost)
    - L'utente deve concedere il permesso manualmente
    - Controlla le impostazioni del microfono del browser
    - Alcuni browser bloccano alla prima visita
  </Accordion>

  <Accordion title="Nessun audio o qualità scarsa" icon="volume-xmark">
    - WebRTC richiede porte UDP aperte
    - Le VPN aziendali potrebbero bloccare WebRTC
    - Testa da una rete diversa
    - Controlla le statistiche WebRTC del browser su `chrome://webrtc-internals`
  </Accordion>

  <Accordion title="Le modifiche non si aggiornano" icon="arrows-rotate">
    - Clicca **Salva Modifiche** nel configuratore
    - Fai un hard refresh del tuo sito (Ctrl+Shift+R o Cmd+Shift+R)
    - Cancella la cache del browser se necessario
    - Verifica che stai modificando il widget corretto
  </Accordion>
</AccordionGroup>
