## Panoramica

Ogni chiamata di test genera log completi che ti aiutano a validare il comportamento dell'agente, debug dei problemi e tracciamento delle metriche di qualità. Impara ad analizzare efficacemente i log per assicurarti che il tuo agente sia pronto per la produzione.

## Accesso ai Log delle Conversazioni

<Steps>
  <Step title="Naviga ai log">
    Vai a **Conversazioni → Registri** nella dashboard. Vedrai un elenco di tutte le conversazioni recenti (test e produzione).
  </Step>

  <Step title="Filtra le chiamate di test">
    Usa i filtri per trovare le tue chiamate di test:
    - **Agent**: Seleziona agente specifico
    - **Date Range**: Ultima ora, ultime 24 ore, intervallo personalizzato
    - **Status**: Successo, fallito, in corso
    - **Tags**: Filtra per tag "test"
    - **Direction**: In entrata vs in uscita
  </Step>

  <Step title="Apri i dettagli della conversazione">
    Clicca su qualsiasi conversazione per visualizzare i dettagli completi:
    - Riproduzione audio
    - Trascrizione completa
    - Log esecuzione azioni
    - Recuperi knowledge base
    - Risultati obiettivi
    - Metriche di qualità
  </Step>
</Steps>

## Comprensione dei Componenti dei Log

### Vista Trascrizione

**Cosa Vedi**:
```
[00:03] Agent: Benvenuto in itellicoAI. Come posso aiutarti oggi?
[00:05] Customer: Quali sono i vostri orari?
[00:07] Agent: Siamo aperti dal lunedì al venerdì, dalle 9 alle 17 ora dell'Est.
[00:10] Customer: Grazie.
[00:11] Agent: Prego! C'è altro con cui posso aiutarti?
[00:13] Customer: No, è tutto.
[00:14] Agent: Buona giornata!
[Chiamata terminata - Durata: 00:11]
```

**Elementi Chiave**:
- **Timestamp**: Traccia la latenza di risposta (tempo tra i turni)
- **Etichette Speaker**: Agent vs Customer chiaramente marcati
- **Punteggi di Confidenza**: Passa sopra la trascrizione per vedere la confidenza della trascrizione
- **Opzione Modifica**: Correggi manualmente gli errori di trascrizione per il riaddestramento

<Accordion title="Analisi dei tempi di risposta">
  **Latenza Obiettivo**:
  - Risposta agente: <1 secondo (eccellente), <2 secondi (accettabile)
  - Chiamate API: <3 secondi totali (inclusa elaborazione agente)

  **Segnali d'Allarme**:
  - Gap >3 secondi indicano problemi di latenza
  - Controlla scheda **Qualità** per metriche di rete
  - Rivedi selezione modello (GPT-4 più lento di GPT-3.5)
</Accordion>

**Esempio Log Azione**:
```json
{
  "action": "getOrderStatus",
  "timestamp": "2024-01-12T15:23:45Z",
  "input": {
    "order_id": "12345",
    "customer_phone": "+12125551234"
  },
  "request": {
    "method": "POST",
    "url": "https://api.example.com/orders/status",
    "headers": {"Authorization": "Bearer ***"},
    "body": {"order_id": "12345"}
  },
  "response": {
    "status": 200,
    "body": {
      "status": "shipped",
      "tracking": "1Z999AA10123456784",
      "eta": "2024-01-15"
    },
    "duration_ms": 245
  },
  "result": "success"
}
```

**Debugging Azioni**:
- Le azioni fallite mostrano dettagli dell'errore
- Testa la stessa richiesta con curl per isolare il problema
- Controlla validità e permessi della chiave API
- Verifica che il formato del payload della richiesta corrisponda alle specifiche API

### Scheda Knowledge

**Visualizza Recupero KB**:
```json
{
  "query": "quali sono i vostri orari",
  "chunks_retrieved": 3,
  "results": [
    {
      "content": "Il nostro team di supporto è disponibile dal lunedì al venerdì, dalle 9 alle 17 EST...",
      "similarity_score": 0.92,
      "source": "kb_article_123",
      "chunk_id": "chunk_456"
    },
    {
      "content": "Gli orari festivi variano. Siamo chiusi nelle principali festività USA...",
      "similarity_score": 0.78,
      "source": "kb_article_789"
    }
  ],
  "retrieval_time_ms": 123
}
```

**Analisi**:
- **Alta Similarità (>0.85)**: Buona corrispondenza, l'agente dovrebbe usare questo contenuto
- **Media Similarità (0.7-0.85)**: Rilevante ma potrebbe necessitare di contenuto o query migliori
- **Bassa Similarità (<0.7)**: Scarsa corrispondenza, l'agente potrebbe non avere la risposta nella KB

**Se l'Agente Dice "Non lo so"**:
- Controlla se è stato recuperato contenuto rilevante (similarità >0.8)
- Se sì → Problema di istruzioni agente (non ha usato il contenuto recuperato)
- Se no → Contenuto KB mancante o mismatch query

### Scheda Obiettivi

**Tracciamento Obiettivi Conversazione**:
<Tabs>
  <Tab title="Risultati Obiettivi">
    ```json
    {
      "goals": [
        {
          "name": "appointment_scheduled",
          "achieved": true,
          "confidence": 0.94,
          "entities": {
            "date": "2024-01-15",
            "time": "14:00",
            "service": "consultation"
          }
        },
        {
          "name": "contact_info_collected",
          "achieved": true,
          "confidence": 0.88,
          "entities": {
            "email": "customer@example.com",
            "phone": "+12125551234"
          }
        }
      ]
    }
    ```

    **Interpretazione Confidenza**:
    - >0.9: Molto confidente, affidabile
    - 0.7-0.9: Confidente, probabilmente accurato
    - <0.7: Bassa confidenza, rivedi trascrizione
  </Tab>

  <Tab title="Debug Obiettivi">
    **Perché gli Obiettivi Non Vengono Raggiunti**:
    1. **Definizione Obiettivo Troppo Rigida**: Regola le frasi di attivazione
    2. **Conversazione Interrotta**: Il cliente ha riattaccato prima del completamento
    3. **L'Agente Non Ha Seguito il Flusso**: Problema di istruzioni
    4. **Entità Non Estratte**: Necessitano risposte cliente più chiare

    **Esempio di Correzione**:
    ```markdown
    # Prima (troppo rigido)
    Goal: "appointment_scheduled"
    Trigger: Il cliente dice "sì, prenotami per le 14"

    # Dopo (più flessibile)
    Goal: "appointment_scheduled"
    Trigger: Qualsiasi prenotazione data/ora confermata
    ```
  </Tab>
</Tabs>

### Scheda Qualità

**Metriche Qualità Chiamata**:

| Metrica | Obiettivo | Indica |
|--------|--------|-----------|
| **MOS (Mean Opinion Score)** | >3.5 | Qualità complessiva chiamata (scala 1-5) |
| **Jitter** | <30ms | Stabilità rete (variazione timing pacchetti) |
| **Packet Loss** | <1% | Affidabilità rete |
| **RTT (Round Trip Time)** | <200ms | Latenza rete |
| **Bitrate** | 64kbps+ | Qualità audio (più alto = migliore) |

**Diagnosi Problemi Audio**:
- Alto jitter + perdita pacchetti → Congestione rete
- Alto RTT → Distanza geografica o problema di routing
- Basso punteggio MOS → Scarsa qualità complessiva, indaga fattori multipli

## Strumenti di Monitoraggio

### Monitor Live

**Monitoraggio Chiamate in Tempo Reale**:
<Steps>
  <Step title="Apri monitor live">
    Naviga a **Conversazioni → Monitor Live** per vedere le chiamate attive in tempo reale.
  </Step>

  <Step title="Guarda le chiamate attive">
    Visualizza l'elenco delle chiamate in corso con:
    - ID chiamante
    - Agente assegnato
    - Durata chiamata (timer live)
    - Trascrizione corrente (aggiornamenti ogni 2-3 secondi)
    - Azioni in esecuzione
  </Step>

  <Step title="Intervieni se necessario">
    **Opzioni di Intervento**:
    - **Whisper**: Parla all'agente senza che il cliente senta (guida l'agente)
    - **Barge In**: Unisciti alla chiamata come partecipante a 3 vie
    - **Takeover**: Disconnetti l'agente, prendi il controllo della chiamata
    - **End Call**: Termina forzatamente la chiamata se necessario

    **Casi d'Uso**:
    - L'agente fornisce informazioni sbagliate
    - Il test di casi limite richiede guida
    - Situazione di emergenza
  </Step>
</Steps>

## Analisi Risultati Test

### Checklist Qualità

Dopo aver eseguito la suite di test, rivedi:

<Accordion title="Qualità trascrizione">
  - [ ] Accuratezza trascrizione >90%
  - [ ] Nessuna parola importante sentita male
  - [ ] Risposte agente rilevanti all'input cliente
  - [ ] La conversazione fluisce naturalmente
  - [ ] Nessuna pausa imbarazzante o ritardo
</Accordion>

<Accordion title="Esecuzione azioni">
  - [ ] Tutte le azioni attivate nei momenti corretti
  - [ ] Azioni completate con successo (nessun errore)
  - [ ] Risposte API usate correttamente dall'agente
  - [ ] Latenza azioni <3 secondi
  - [ ] Gestione errori funzionante (fallimenti gestiti con grazia)
</Accordion>

<Accordion title="Accuratezza knowledge">
  - [ ] Query KB recuperano contenuto rilevante
  - [ ] L'agente usa il contenuto recuperato nelle risposte
  - [ ] L'agente non allucina fatti non nella KB
  - [ ] L'agente dice "Non lo so" quando appropriato
  - [ ] Tempo di recupero <500ms
</Accordion>

<Accordion title="Qualità chiamata">
  - [ ] Punteggio MOS >3.5
  - [ ] Jitter <30ms
  - [ ] Perdita pacchetti <1%
  - [ ] Nessun dropout audio o distorsione
  - [ ] Voce suona naturale e chiara
</Accordion>

### Confronto Esecuzioni Test

**Traccia Metriche nel Tempo**:
```markdown
| Data | Chiamate Test | Tasso Successo | Durata Media | Raggiungimento Obiettivi |
|------|-----------|--------------|--------------|------------------|
| 10 Gen | 25 | 96% | 3:45 | 68% |
| 11 Gen | 30 | 94% | 3:52 | 71% |
| 12 Gen | 28 | 98% | 3:38 | 75% |
```

**Analisi Tendenze**:
- Tasso successo in miglioramento → Buon progresso
- Durata in diminuzione → Agente sta diventando più efficiente
- Raggiungimento obiettivi in aumento → Istruzioni ottimizzate

**Rilevamento Regressione**:
- Calo improvviso tasso successo → Cambio recente ha rotto qualcosa
- Picco durata → Nuovo problema di latenza introdotto
- Calo raggiungimento obiettivi → Istruzioni o KB cambiati negativamente

## Esportazione Log

### Per Revisione Team

<Steps>
  <Step title="Esporta log conversazioni">
    Da Conversazioni → Registri, seleziona conversazioni e clicca **Esporta**:
    - Formato: JSON, CSV
    - Include: Trascrizione, azioni, obiettivi, metadati
    - Scarica come file o invia via email
  </Step>

  <Step title="Condividi con il team">
    **Casi d'Uso**:
    - Team QA rivede per assicurazione qualità
    - Team prodotto analizza pattern conversazioni
    - Ingegneria esegue debug di problemi specifici
    - Team compliance controlla per requisiti normativi
  </Step>
</Steps>

### Accesso Programmatico

**Esportazione API**:
```bash
# Esporta tutte le conversazioni test dalle ultime 24 ore
curl -H "Authorization: Bearer YOUR_API_KEY" \
     "https://api.itellico.ai/v1/conversations?start_date=2024-01-12&tag=test" \
     > test_conversations.json

# Analizza con jq
cat test_conversations.json | jq '.[] | select(.status == "failed")'
```

**Usa per**:
- Report test automatizzati
- Integrazione con framework di test
- Validazione pipeline CI/CD
- Dashboard analytics personalizzate
