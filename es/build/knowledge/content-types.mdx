## Elementos de Texto

### ¿Qué son los Elementos de Texto?

Los elementos de texto son contenido que ingresa directamente en el editor de base de conocimiento de itellicoAI. Son el tipo de contenido más directo y confiable.

### Cuándo Usar Elementos de Texto

<AccordionGroup>
  <Accordion title="Escribir preguntas frecuentes" icon="circle-question">
    Cree pares de preguntas y respuestas directamente en el sistema.

    **Ejemplo:**
    ```text wrap
    Título: ¿Cómo restablezco mi contraseña?

    Contenido:
    Para restablecer su contraseña:

    1. Vaya a la página de inicio de sesión
    2. Haga clic en "Olvidé mi Contraseña"
    3. Ingrese su dirección de correo electrónico
    4. Revise su correo electrónico para un enlace de restablecimiento
    5. Haga clic en el enlace y cree una nueva contraseña

    Requisitos de contraseña:
    - Mínimo 8 caracteres
    - Al menos una letra mayúscula
    - Al menos un número
    - Al menos un carácter especial

    Si no recibe el correo electrónico en 5 minutos, revise su carpeta
    de spam o contacte a support@company.com.
    ```
  </Accordion>

  <Accordion title="Crear resúmenes de políticas" icon="book">
    Escriba declaraciones de políticas claras y concisas.

    **Ejemplo:**
    ```text wrap
    Título: Política de Devolución - Productos Digitales

    Contenido:
    Los productos digitales pueden ser reembolsados dentro de 30 días de compra si:

    Elegible para reembolso:
    - El producto tiene un defecto técnico que impide su uso
    - La descripción del producto fue materialmente inexacta
    - El cliente no ha accedido o descargado el producto

    No elegible para reembolso:
    - Cambio de opinión después de acceder al producto
    - Problemas de compatibilidad revelados en la descripción del producto
    - Error del usuario o malentendido de características

    Para solicitar un reembolso:
    Envíe un correo electrónico a support@company.com con su número de pedido y razón
    de la solicitud de reembolso.

    Tiempo de procesamiento: 5-7 días hábiles
    Método de reembolso: Método de pago original
    ```
  </Accordion>

  <Accordion title="Documentar procedimientos" icon="list-check">
    Instrucciones paso a paso para procesos.

    **Ejemplo:**
    ```
    Título: Proceso de Modificación de Pedidos

    Contenido:
    Los clientes pueden modificar pedidos dentro de 24 horas de realizado.

    Lo que se puede modificar:
    - Dirección de envío
    - Velocidad de entrega
    - Cantidades de artículos (si hay inventario disponible)

    Lo que no se puede modificar:
    - Método de pago (debe cancelar y volver a pedir)
    - Artículos después de que el procesamiento ha comenzado
    - Pedidos realizados hace más de 24 horas

    Proceso de modificación:
    1. El cliente contacta a soporte por teléfono o correo electrónico
    2. El agente verifica que el pedido está dentro de la ventana de modificación
    3. El agente verifica inventario para cambios solicitados
    4. El agente actualiza el pedido en el sistema
    5. El cliente recibe correo electrónico de confirmación

    Si el pedido ya ha sido enviado, el cliente debe usar el proceso de devolución en su lugar.
    ```
  </Accordion>

  <Accordion title="Información de referencia rápida" icon="clock">
    Información breve y frecuentemente referenciada.

    **Ejemplo:**
    ```
    Título: Horario Comercial e Información de Contacto

    Contenido:
    Soporte al Cliente:
    - Teléfono: 1-800-555-0123
    - Correo electrónico: support@company.com
    - Horario: Lunes-Viernes, 9 AM - 6 PM EST
    - Fuera de horario: emergency@company.com (solo problemas urgentes)

    Ventas:
    - Teléfono: 1-800-555-0124
    - Correo electrónico: sales@company.com
    - Horario: Lunes-Viernes, 8 AM - 8 PM EST

    Facturación:
    - Teléfono: 1-800-555-0125
    - Correo electrónico: billing@company.com
    - Horario: Lunes-Viernes, 9 AM - 5 PM EST
    ```
  </Accordion>
</AccordionGroup>

### Crear Elementos de Texto

<Steps>
  <Step title="Navegar a la carpeta">
    Abra la carpeta donde desea agregar el elemento.
  </Step>

  <Step title="Haga clic en 'Agregar Elemento de Conocimiento'">
    Haga clic en el botón para crear un nuevo elemento.
  </Step>

  <Step title="Seleccione la categoría 'TEXTO'">
    Elija TEXTO de las opciones de tipo de contenido.
  </Step>

  <Step title="Ingrese el título">
    Dé a su elemento un título claro y descriptivo.
  </Step>

  <Step title="Escriba el contenido">
    Ingrese su contenido en el editor. Use formato para claridad:
    - Encabezados para secciones
    - Viñetas para listas
    - Números para pasos
    - Negrita para énfasis
  </Step>

  <Step title="Haga clic en 'Crear'">
    Guarde su elemento de texto. El estado mostrará inmediatamente COMPLETADO.
  </Step>
</Steps>

### Ejemplo JSON de Elemento de Texto

```json wrap
{
  "category": "TEXT",
  "title": "Información de Precios del Producto",
  "content": "Nuestros niveles de precios incluyen Básico ($29/mes), Pro ($99/mes) y Enterprise (precios personalizados).",
  "status": "COMPLETED"
}
```

## Elementos de Carga de Archivos

### ¿Qué son los Elementos de Carga de Archivos?

Los elementos de carga de archivos le permiten cargar documentos existentes en varios formatos. El sistema extrae el contenido de texto y lo hace disponible para sus agentes.

### Cuándo Usar Cargas de Archivos

<AccordionGroup>
  <Accordion title="Documentación existente" icon="file">
    Ya tiene contenido en formato de documento.

    **Ejemplos:**
    - Manuales de usuario
    - Especificaciones de productos
    - Documentos legales
    - Materiales de capacitación
  </Accordion>

  <Accordion title="Documentos formateados" icon="file-pdf">
    Documentos con diseños específicos que son más fáciles de mantener como archivos.

    **Ejemplos:**
    - Diagramas técnicos
    - Tablas y gráficos
    - Diseños de múltiples columnas
    - Plantillas con marca
  </Accordion>

  <Accordion title="Documentos de terceros" icon="building">
    Documentación que recibe de proveedores o socios.

    **Ejemplos:**
    - Catálogos de proveedores
    - Documentos de cumplimiento
    - Materiales de certificación
  </Accordion>
</AccordionGroup>

### Requisitos de Archivos

**Especificaciones de archivos:**
- **Formatos:** PDF, DOC, DOCX, TXT y otros formatos de documento
- **Límite de tamaño:** Máximo 10MB
- **Contenido:** Documentos basados en texto e imágenes escaneadas (el análisis avanzado maneja la mayoría de los escaneos)
- **Protección:** Sin protección de contraseña

<Note>
El sistema usa análisis avanzado de documentos para extraer texto de imágenes escaneadas y PDFs. La mayoría de los documentos escaneados se procesarán correctamente, aunque escaneos de muy mala calidad pueden requerir entrada manual de texto.
</Note>

### Crear Elementos de Archivo

<Steps>
  <Step title="Navegar a la carpeta">
    Abra la carpeta donde desea agregar el archivo.
  </Step>

  <Step title="Haga clic en 'Agregar Elemento de Conocimiento'">
    Haga clic en el botón para crear un nuevo elemento.
  </Step>

  <Step title="Seleccione la categoría 'ARCHIVO'">
    Elija ARCHIVO de las opciones de tipo de contenido.
  </Step>

  <Step title="Ingrese el título">
    Dé a su archivo un título descriptivo (esto es separado del nombre del archivo).
  </Step>

  <Step title="Cargue el archivo">
    Haga clic en **Cargar** y seleccione su documento de su computadora.
  </Step>

  <Step title="Haga clic en 'Crear'">
    El sistema cargará y comenzará a procesar su archivo. El estado cambiará de PENDIENTE → PROCESANDO → COMPLETADO.
  </Step>
</Steps>

<Warning>
El tiempo de procesamiento de archivos varía según el tamaño y complejidad del archivo. Archivos grandes (5-10MB) pueden tomar varios minutos para procesar.
</Warning>

### Ejemplo JSON de Elemento de Archivo

```json wrap
{
  "category": "FILE",
  "title": "Manual de Usuario - Producto X",
  "file": "/uploads/knowledge/user-manual-product-x.pdf",
  "status": "COMPLETED"
}
```

### Tiempo de Procesamiento

El tiempo de procesamiento de archivos varía según:
- **Tamaño del archivo:** Archivos más grandes toman más tiempo
- **Conteo de páginas:** Más páginas = procesamiento más largo
- **Complejidad:** Tablas, imágenes y diseños complejos ralentizan el procesamiento
- **Calidad del texto:** Texto limpio y simple se extrae más rápido

**Tiempos de procesamiento típicos:**
- Archivos pequeños (< 1MB, 10 páginas): 10-30 segundos
- Archivos medianos (1-5MB, 10-50 páginas): 30-90 segundos
- Archivos grandes (5-10MB, 50+ páginas): 2-5 minutos

### Problemas Comunes con Archivos

<AccordionGroup>
  <Accordion title="Procesamiento fallido" icon="circle-xmark">
    **Causas:**
    - El archivo excede 10MB
    - El archivo está protegido con contraseña
    - El archivo está corrupto
    - Imágenes escaneadas de muy mala calidad

    **Soluciones:**
    - Comprima el archivo o divídalo en archivos más pequeños
    - Elimine la protección de contraseña
    - Re-exporte el archivo desde la fuente
    - Para escaneos de muy mala calidad, copie el contenido en un elemento de texto en su lugar
  </Accordion>

  <Accordion title="Contenido extraído incorrectamente" icon="text-slash">
    **Causas:**
    - Diseños complejos (múltiples columnas, tablas)
    - Imágenes escaneadas de muy mala calidad
    - Fuentes especiales o codificación
    - Campos de formulario y elementos interactivos

    **Soluciones:**
    - Verifique el contenido extraído en modo de edición
    - Recree como elemento de texto con formato apropiado
    - Simplifique el diseño del documento antes de cargar
    - Exporte como documento de texto plano
  </Accordion>

  <Accordion title="El procesamiento toma demasiado tiempo" icon="clock">
    **Qué hacer:**
    - Espere 5-10 minutos antes de asumir fallo
    - Verifique el tamaño del archivo y conteo de páginas
    - Para archivos grandes, considere dividir en múltiples archivos
    - Convierta a texto y cargue como elementos de TEXTO en su lugar
  </Accordion>
</AccordionGroup>

### Mejores Prácticas de Carga de Archivos

<CardGroup cols={2}>
  <Card title="Optimice antes de cargar" icon="compress">
    - Comprima archivos grandes
    - Elimine imágenes innecesarias
    - Use documentos basados en texto
    - Mantenga bajo 5MB cuando sea posible
  </Card>
  <Card title="Pruebe la extracción" icon="vial">
    - Revise el contenido extraído después del procesamiento
    - Verifique problemas de formato
    - Verifique que la información crítica sea precisa
    - Vuelva a cargar si la extracción es mala
  </Card>
</CardGroup>

## Elementos de URL

### ¿Qué son los Elementos de URL?

Los elementos de URL raspan contenido de páginas web y lo almacenan en su base de conocimiento. Esto es útil para referenciar documentación en línea, centros de ayuda o publicaciones de blog.

### Cuándo Usar Elementos de URL

<AccordionGroup>
  <Accordion title="Documentación pública" icon="globe">
    Referencie documentación externa que no mantiene.

    **Ejemplos:**
    - Documentación de API (propia o de terceros)
    - Bases de conocimiento públicas
    - Artículos del centro de ayuda
    - Páginas de productos
  </Accordion>

  <Accordion title="Contenido actualizado frecuentemente" icon="arrows-rotate">
    Contenido que cambia regularmente y desea mantener actualizado raspándolo nuevamente.

    **Ejemplos:**
    - Páginas de precios
    - Disponibilidad de productos
    - Promociones actuales
    - Páginas de estado
  </Accordion>

  <Accordion title="Publicaciones de blog o artículos" icon="newspaper">
    Contenido educativo o anuncios.

    **Ejemplos:**
    - Guías de cómo hacer
    - Artículos de mejores prácticas
    - Anuncios de productos
    - Tutoriales de características
  </Accordion>
</AccordionGroup>

### Crear Elementos de URL

<Steps>
  <Step title="Navegar a la carpeta">
    Abra la carpeta donde desea agregar la URL.
  </Step>

  <Step title="Haga clic en 'Agregar Elemento de Conocimiento'">
    Haga clic en el botón para crear un nuevo elemento.
  </Step>

  <Step title="Seleccione la categoría 'URL'">
    Elija URL de las opciones de tipo de contenido.
  </Step>

  <Step title="Ingrese el título">
    Dé al contenido un título descriptivo.
  </Step>

  <Step title="Ingrese la URL de origen">
    Pegue la URL completa incluyendo https://

    **Ejemplo:**
    ```
    https://docs.company.com/api/authentication
    ```
  </Step>

  <Step title="Haga clic en 'Crear'">
    El sistema obtendrá y procesará la página web.

    Progresión del estado:
    - PENDIENTE → URL en cola para raspado
    - PROCESANDO → Obteniendo y extrayendo contenido
    - COMPLETADO → Contenido disponible
    - FALLIDO → Error de raspado ocurrió
  </Step>
</Steps>

### Ejemplo JSON de Elemento de URL

```json wrap
{
  "category": "URL",
  "title": "Documentación de Autenticación de API",
  "source_url": "https://docs.company.com/api/authentication",
  "status": "COMPLETED"
}
```

### Requisitos de URL

**URLs que funcionan:**
- Públicamente accesibles (sin inicio de sesión requerido)
- Páginas de contenido HTML simple
- Sitios de documentación
- Publicaciones de blog y artículos
- Páginas de contenido estático

**URLs problemáticas:**
- Páginas que requieren autenticación
- Aplicaciones con mucho JavaScript (SPAs)
- Contenido de pago
- Contenido cargado dinámicamente
- Aplicaciones interactivas

<Note>
El raspado de URL funciona mejor con páginas web simples basadas en texto. Las aplicaciones web complejas pueden no rasparse exitosamente.
</Note>

### Problemas Comunes con URL

<AccordionGroup>
  <Accordion title="Raspado fallido" icon="circle-xmark">
    **Causas:**
    - La página requiere inicio de sesión/autenticación
    - La URL es incorrecta o está rota
    - El contenido se carga vía JavaScript
    - El sitio web bloquea el raspado (robots.txt)
    - La página no existe (404)

    **Soluciones:**
    - Verifique que la URL sea públicamente accesible
    - Pruebe la URL en ventana de navegador incógnito
    - Verifique que la URL esté completa y correcta
    - Copie el contenido manualmente en elemento de texto
    - Use exportación PDF de la página en su lugar
  </Accordion>

  <Accordion title="Contenido incompleto o incorrecto" icon="puzzle-piece">
    **Causas:**
    - Contenido renderizado por JavaScript no capturado
    - Carga de contenido dinámico
    - Múltiples pestañas/secciones en la página
    - Comentarios o barra lateral raspados en lugar del contenido principal

    **Soluciones:**
    - Inspeccione el contenido raspado en modo de edición
    - Use URL directa a contenido específico
    - Copie el contenido deseado en elemento de texto
    - Exporte la página como PDF y cargue en su lugar
  </Accordion>

  <Accordion title="El contenido queda obsoleto" icon="calendar">
    **Solución:**
    El contenido de URL se raspa una vez en el momento de creación. Para actualizar:
    - Elimine y recree el elemento de URL
    - O copie el contenido actual en un elemento de texto para actualizaciones manuales

    Para contenido que cambia frecuentemente, considere:
    - Elementos de texto manuales que actualice regularmente
    - Exportaciones PDF que actualice periódicamente
  </Accordion>
</AccordionGroup>

### Mejores Prácticas de URL

<CardGroup cols={2}>
  <Card title="Pruebe la accesibilidad" icon="unlock">
    - Abra la URL en ventana incógnito
    - Verifique que no se requiera inicio de sesión
    - Verifique que el contenido sea visible
    - Asegúrese de que la página cargue rápidamente
  </Card>
  <Card title="Revise el contenido raspado" icon="eye">
    - Verifique el contenido después del raspado
    - Verifique que se capturó el contenido correcto
    - Busque problemas de formato
    - Confirme que no hay contenido extra (anuncios, barras laterales)
  </Card>
</CardGroup>

## Flujo de Estado de Procesamiento

Los elementos de conocimiento pasan por dos pipelines de procesamiento separados:

1. **Procesamiento de Contenido** - Extracción de texto de archivos/URLs
2. **Indexación Vectorial** - Preparación de contenido para RAG (búsqueda semántica)

### Estado de Procesamiento de Contenido

Esto rastrea la extracción de contenido de texto de su fuente.

<Steps>
  <Step title="PENDIENTE">
    **Significado:** Elemento creado, en cola para extracción de contenido

    **Qué está sucediendo:**
    - El elemento ha sido guardado en la base de datos
    - Esperando que el trabajador de procesamiento lo recoja
    - Usualmente muy breve (segundos)

    **Acción necesaria:** Ninguna - espere a que el procesamiento comience
  </Step>

  <Step title="PROCESANDO">
    **Significado:** El contenido del elemento se está extrayendo ahora mismo

    **Qué está sucediendo:**
    - Para ARCHIVOS: Extrayendo texto de PDF, Word, etc.
    - Para URLs: Obteniendo y raspando la página web específica
    - Para TEXTO: N/A (salta directamente a COMPLETADO)

    **Acción necesaria:** Espere a que el procesamiento se complete
  </Step>

  <Step title="COMPLETADO">
    **Significado:** La extracción de contenido finalizó exitosamente

    **Qué está sucediendo:**
    - El contenido ha sido extraído y almacenado
    - La indexación vectorial comenzará automáticamente
    - El elemento estará disponible una vez que la indexación se complete

    **Acción necesaria:** Ninguna - espere la indexación vectorial
  </Step>

  <Step title="FALLIDO">
    **Significado:** La extracción de contenido encontró un error

    **Qué está sucediendo:**
    - La extracción del archivo falló (PDF corrupto, formato no soportado)
    - El raspado de URL falló (página no encontrada, bloqueada, requiere autenticación)
    - El contenido no está disponible para los agentes

    **Acción necesaria:** Verifique el mensaje de error y corrija el problema
  </Step>
</Steps>

**Flujo de Procesamiento de Contenido:**
CREADO → PENDIENTE → PROCESANDO → COMPLETADO o FALLIDO

---

### Estado de Indexación Vectorial

Después de que se extrae el contenido, debe ser indexado para RAG (búsqueda semántica). Esto permite a los agentes encontrar conocimiento relevante basado en significado, no solo palabras clave.

<Steps>
  <Step title="PENDIENTE">
    **Significado:** Esperando que la indexación vectorial comience

    **Qué está sucediendo:**
    - El procesamiento de contenido se completó exitosamente
    - En cola para generación de incrustaciones
    - Usualmente breve (segundos a minutos)

    **Acción necesaria:** Ninguna
  </Step>

  <Step title="INDEXANDO">
    **Significado:** Creando incrustaciones vectoriales para RAG

    **Qué está sucediendo:**
    - El contenido se está dividiendo en fragmentos
    - Se están generando incrustaciones de IA para cada fragmento
    - Los vectores se están almacenando en la base de conocimiento

    **Acción necesaria:** Espere - esto puede tomar varios minutos para documentos grandes
  </Step>

  <Step title="INDEXADO">
    **Significado:** El elemento está completamente listo para recuperación RAG

    **Qué está sucediendo:**
    - Las incrustaciones vectoriales se almacenaron exitosamente
    - El elemento puede ser recuperado vía búsqueda semántica
    - Los agentes ahora pueden usar este conocimiento

    **Acción necesaria:** Ninguna - el elemento está completamente listo
  </Step>

  <Step title="FALLIDO">
    **Significado:** La indexación vectorial falló

    **Qué está sucediendo:**
    - La generación de incrustaciones encontró un error
    - El elemento no aparecerá en resultados RAG
    - Puede estar disponible solo para inyección de contexto

    **Acción necesaria:** Intente reindexar o contacte a soporte
  </Step>

  <Step title="ELIMINADO">
    **Significado:** Los vectores fueron eliminados

    **Qué está sucediendo:**
    - El elemento fue eliminado o explícitamente removido del índice
    - Ya no está disponible para RAG

    **Acción necesaria:** Reindexe si es necesario
  </Step>
</Steps>

**Flujo de Indexación Vectorial:**
PENDIENTE → INDEXANDO → INDEXADO o FALLIDO

<Note>
**Ambos estados deben tener éxito para funcionalidad completa:**
- Estado de Contenido: COMPLETADO
- Estado Vectorial: INDEXADO

Solo entonces el elemento puede usarse tanto en inyección de contexto como en recuperación RAG.
</Note>

## Manejo de Errores

### Cuando los Elementos Fallan

Si un elemento de conocimiento muestra estado FALLIDO:

<Steps>
  <Step title="Verifique el mensaje de error">
    Haga clic en el elemento fallido para ver detalles del error.
  </Step>

  <Step title="Identifique la causa">
    Causas comunes:
    - **Archivos:** Archivo demasiado grande, corrupto, protegido con contraseña, imagen escaneada
    - **URL:** Autenticación requerida, enlace roto, contenido no accesible
  </Step>

  <Step title="Intente soluciones">
    - **Para Archivos:** Comprimir, eliminar protección, agregar capa de texto, o convertir a texto
    - **Para URLs:** Verifique accesibilidad, intente URL diferente, o copie contenido a elemento de texto
  </Step>

  <Step title="Recree o convierta">
    - Elimine el elemento fallido
    - Cree nuevo elemento con correcciones aplicadas
    - O convierta a elemento de texto copiando contenido manualmente
  </Step>
</Steps>

## Monitoreo del Procesamiento

### Indicadores del Panel

En su panel de base de conocimiento, puede ver el estado de procesamiento de un vistazo para cada elemento en sus carpetas:

**Indicadores de estado:**
- Marca verde = COMPLETADO
- Reloj de arena = PROCESANDO
- Símbolo de pausa = PENDIENTE
- X roja = FALLIDO

**Vista de carpeta de ejemplo:**
- Manual de Usuario.pdf - COMPLETADO
- Guía de Inicio Rápido.pdf - PROCESANDO
- Documentación de API - PENDIENTE
- Manual Antiguo.pdf - FALLIDO

### Procesamiento Masivo

Al cargar múltiples elementos:

1. Los elementos se procesan secuencialmente o en paralelo (dependiente del sistema)
2. Vuelva a verificar después de 5-10 minutos para lotes grandes
3. Revise el estado de cada elemento
4. Corrija cualquier falla individualmente

<Tip>
Cargue un elemento de prueba primero para verificar formato y procesamiento antes de cargar masivamente.
</Tip>

## Elegir el Tipo de Contenido Correcto

Use este árbol de decisión para seleccionar el mejor tipo de contenido:

**¿Tiene contenido existente?**
- No → Use TEXTO (escriba directamente)
- Sí, es un documento →
  - Bajo 10MB → Use ARCHIVO
  - Sobre 10MB → Extraiga texto, use TEXTO
- Sí, es una página web →
  - Públicamente accesible → Use URL (si el raspado falla, copie a TEXTO)
  - No accesible → Copie contenido a TEXTO

### Recomendaciones Rápidas

| Su Situación | Mejor Tipo de Contenido |
|---------------|-------------------|
| Escribir preguntas frecuentes desde cero | TEXTO |
| Tiene documentos Word/PDF existentes | Cargar como ARCHIVO |
| Tiene documentos bajo 10MB | ARCHIVO |
| Tiene documentos sobre 10MB | Dividir en archivos más pequeños o extraer a TEXTO |
| Documentación web pública | URL (con TEXTO como respaldo) |
| Contenido privado/autenticado | Copiar a TEXTO |
| Necesita disponibilidad inmediata | TEXTO (sin retraso de procesamiento) |
| El formato complejo importa | ARCHIVO |
