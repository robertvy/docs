## Descripción General

Las pronunciaciones personalizadas te permiten definir cómo deben sonar palabras específicas, asegurando que tu agente siempre pronuncie correctamente nombres de marca, términos técnicos y acrónimos.

**Cómo funciona:**
1. El agente genera respuesta: "Llámanos en itellicoAI para ayuda"
2. El sistema aplica reglas de pronunciación: "Llámanos en eye-tell-ee-co A I para ayuda"
3. El sintetizador de voz habla el texto modificado
4. El cliente escucha la pronunciación correcta

<Note>
Los reemplazos de pronunciación están disponibles en **Voz > Pronunciaciones** en la configuración de tu agente.
</Note>

---

## Añadir Pronunciaciones

### Reemplazo Único

1. Haz clic en **Añadir Reemplazo**
2. Ingresa la **palabra objetivo** (la palabra a reemplazar)
3. Ingresa la **pronunciación** (cómo debe sonar)
4. Haz clic en **Añadir**

**Ejemplos:**
- **Objetivo:** Nike → **Pronunciación:** Nai-key
- **Objetivo:** API → **Pronunciación:** A-P-I
- **Objetivo:** 24/7 → **Pronunciación:** veinticuatro siete
- **Objetivo:** EST → **Pronunciación:** Hora del Este

<Warning>
**Las palabras objetivo deben ser solo palabras individuales** - no se permiten espacios. Usa palabras completas como `Nike` o `API`, no frases como `servicio al cliente`.
</Warning>

### Importación Masiva vía CSV

Para importar muchas pronunciaciones a la vez:

1. Haz clic en **Importar CSV**
2. Descarga la plantilla si es necesario
3. Prepara tu CSV con dos columnas: `target`, `replacement`
4. Arrastra y suelta tu archivo CSV o haz clic para seleccionar
5. Revisa la vista previa y haz clic en **Importar**

**Formato CSV:**
```csv
target,replacement
Nike,Nai-key
EST,Hora del Este
24/7,veinticuatro siete
API,A-P-I
```

**Límites:**
- Máximo 500 pronunciaciones por agente
- Tamaño máximo del archivo CSV: 5MB

## Consejos de Ortografía Fonética

### Ortografía Fonética Simple

Escribe reemplazos como texto fonético simple y legible:

**Buenos ejemplos:**
- Nike → Nai-key
- SQL → sequel (o) S-Q-L
- Wi-Fi → why-figh
- iOS → eye-oh-ess

**Consejos:**
- Usa guiones para separar sílabas: Nai-key, A-P-I
- Escríbelo como explicarías la pronunciación a alguien
- Prueba con tu voz para escuchar cómo suena

### Patrones Comunes

**Acrónimos (deletrear cada letra):**
- API → A-P-I
- URL → U-R-L
- HTTP → H-T-T-P
- AWS → A-W-S

**Nombres de marca (ortografía fonética):**
- Nike → Nai-key
- Adidas → ah-dee-dahs
- Chipotle → chih-poht-lay

**Números y hora:**
- 24/7 → veinticuatro siete
- 9am → nueve A-M
- EST → Hora del Este
- 5pm → cinco P-M

**Términos técnicos:**
- SQL → sequel (o) S-Q-L
- API → A-P-I
- URL → you-are-ell
- WiFi → why-figh

## Avanzado: Fonemas IPA

Para control preciso de pronunciación, puedes usar la notación del Alfabeto Fonético Internacional (IPA) con el formato `<< >>`:

**Formato:** `<<fonema|fonema|fonema>>`

**Ejemplo:**
```
espresso → <<ɛ|s|p|r|ɛ|s|oʊ>>
```

<Note>
Los fonemas IPA proporcionan control preciso pero requieren conocimiento de notación fonética. La ortografía fonética simple (Nai-key) funciona bien para la mayoría de los casos.
</Note>

## Gestionar Pronunciaciones

### Editar Reemplazos Existentes

1. Haz clic en cualquier chip de reemplazo en la lista
2. Modifica el objetivo o la pronunciación
3. Haz clic en **Guardar**

### Eliminar Reemplazos

- Haz clic en el **icono de papelera** en cualquier chip de reemplazo para eliminarlo
- O haz clic en **Limpiar Todo** para eliminar todas las pronunciaciones (requiere confirmación)

### Exportar Pronunciaciones

1. Haz clic en **Exportar CSV**
2. El archivo CSV se descarga con todas tus pronunciaciones actuales
3. Formato del nombre de archivo: `{nombre_agente}_pronunciations.csv`

Usa archivos CSV exportados para hacer copias de seguridad de tus pronunciaciones o copiarlas a otros agentes.

## Probar Pronunciaciones

Después de añadir pronunciaciones:

1. Haz una llamada de prueba a tu agente
2. Activa respuestas que contengan las palabras objetivo
3. Escucha para verificar que la pronunciación suena correcta
4. Ajusta la ortografía fonética si es necesario

**Consejos de prueba:**
- Prueba en contexto, no aislado
- Haz que los miembros del equipo escuchen
- Ajusta la separación por guiones u ortografía si no suena bien

## Mejores Prácticas

<AccordionGroup>
  <Accordion title="Comienza con Problemas Comunes" icon="bullseye">
    Añade pronunciaciones para palabras que sabes que se pronuncian mal en llamadas reales. Revisa las transcripciones de llamadas para identificar problemas de pronunciación recurrentes.
  </Accordion>

  <Accordion title="Manténlo Simple" icon="sparkles">
    Usa ortografía fonética (Nai-key) en lugar de IPA complejo a menos que sea necesario. El texto fonético simple es más fácil de mantener y funciona bien para la mayoría de los casos.
  </Accordion>

  <Accordion title="Prueba Antes de Implementar" icon="vial">
    Siempre prueba pronunciaciones con llamadas reales antes de ponerlas en producción. Haz una llamada de prueba y activa respuestas que contengan las palabras objetivo para verificar que suenan correctamente.
  </Accordion>

  <Accordion title="Documenta tus Reglas" icon="file-arrow-down">
    Exporta tu CSV regularmente como respaldo. Usa el botón Exportar CSV para guardar tus pronunciaciones y mantenerlas versionadas junto con la configuración de tu agente.
  </Accordion>

  <Accordion title="Usa Ortografía Consistente" icon="check">
    Si tienes "API" deletreado como "A-P-I", usa ese formato consistentemente en todas tus pronunciaciones. La consistencia mejora la mantenibilidad.
  </Accordion>

  <Accordion title="Solo Palabras Individuales" icon="letter-a">
    Recuerda que las palabras objetivo no pueden contener espacios - solo funcionan palabras individuales completas. Usa `Nike` o `API`, no frases como `servicio al cliente`.
  </Accordion>
</AccordionGroup>
