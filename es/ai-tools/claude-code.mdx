Claude Code es la herramienta CLI oficial de Anthropic. Esta guía le ayudará a configurar Claude Code para ayudarle a escribir y mantener su documentación.

## Requisitos previos

- Suscripción activa de Claude (Pro, Max o acceso API)

## Configuración

1. Instale Claude Code globalmente:

  ```bash
  npm install -g @anthropic-ai/claude-code
```

2. Navegue a su directorio de documentación.
3. (Opcional) Agregue el archivo `CLAUDE.md` a continuación a su proyecto.
4. Ejecute `claude` para comenzar.

## Crear `CLAUDE.md`

Cree un archivo `CLAUDE.md` en la raíz de su repositorio de documentación para entrenar a Claude Code en sus estándares de documentación específicos:

````markdown
# Documentación Mintlify

## Relación de trabajo
- Puede cuestionar las ideas - esto puede conducir a una mejor documentación. Cite fuentes y explique su razonamiento cuando lo haga
- SIEMPRE pida aclaraciones en lugar de hacer suposiciones
- NUNCA mienta, adivine o invente información

## Contexto del proyecto
- Formato: Archivos MDX con frontmatter YAML
- Configuración: docs.json para navegación, tema, configuraciones
- Componentes: Componentes Mintlify

## Estrategia de contenido
- Documente lo justo para el éxito del usuario - ni demasiado, ni muy poco
- Priorice la precisión y usabilidad de la información
- Haga el contenido perenne cuando sea posible
- Busque información existente antes de agregar contenido nuevo. Evite la duplicación a menos que se haga por una razón estratégica
- Verifique los patrones existentes para mantener la coherencia
- Comience haciendo los cambios más pequeños razonables

## Requisitos de frontmatter para las páginas
- title: Título de página claro y descriptivo
- description: Resumen conciso para SEO/navegación

## Estándares de redacción
- Voz en segunda persona ("usted")
- Requisitos previos al inicio del contenido procedimental
- Pruebe todos los ejemplos de código antes de publicar
- Haga coincidir el estilo y formato de las páginas existentes
- Incluya casos de uso básicos y avanzados
- Etiquetas de idioma en todos los bloques de código
- Texto alternativo en todas las imágenes
- Rutas relativas para enlaces internos

## Flujo de trabajo de Git
- NUNCA use --no-verify al hacer commit
- Pregunte cómo manejar los cambios sin commit antes de comenzar
- Cree una nueva rama cuando no exista una rama clara para los cambios
- Haga commit frecuentemente durante el desarrollo
- NUNCA omita o deshabilite los hooks pre-commit

## No hacer
- Omitir el frontmatter en cualquier archivo MDX
- Usar URLs absolutas para enlaces internos
- Incluir ejemplos de código no probados
- Hacer suposiciones - siempre pida aclaraciones
````
