## Itens de Texto

### O que são Itens de Texto?

Itens de texto são conteúdo que você insere diretamente no editor da base de conhecimento do itellicoAI. Eles são o tipo de conteúdo mais direto e confiável.

### Quando Usar Itens de Texto

<AccordionGroup>
  <Accordion title="Escrevendo FAQs" icon="circle-question">
    Crie pares de perguntas e respostas diretamente no sistema.

    **Exemplo:**
    ```text wrap
    Título: Como redefinir minha senha?

    Conteúdo:
    Para redefinir sua senha:

    1. Vá para a página de login
    2. Clique em "Esqueci minha senha"
    3. Digite seu endereço de e-mail
    4. Verifique seu e-mail para um link de redefinição
    5. Clique no link e crie uma nova senha

    Requisitos de senha:
    - Mínimo de 8 caracteres
    - Pelo menos uma letra maiúscula
    - Pelo menos um número
    - Pelo menos um caractere especial

    Se você não receber o e-mail em 5 minutos, verifique sua pasta
    de spam ou entre em contato com suporte@empresa.com.
    ```
  </Accordion>

  <Accordion title="Criando resumos de políticas" icon="book">
    Escreva declarações de política claras e concisas.

    **Exemplo:**
    ```text wrap
    Título: Política de Devolução - Produtos Digitais

    Conteúdo:
    Produtos digitais podem ser reembolsados dentro de 30 dias da compra se:

    Elegível para reembolso:
    - Produto tem um defeito técnico que impede o uso
    - Descrição do produto foi materialmente imprecisa
    - Cliente não acessou ou baixou o produto

    Não elegível para reembolso:
    - Mudança de ideia após acessar o produto
    - Problemas de compatibilidade divulgados na descrição do produto
    - Erro do usuário ou mal-entendido dos recursos

    Para solicitar um reembolso:
    Envie e-mail para suporte@empresa.com com seu número de pedido e motivo
    da solicitação de reembolso.

    Tempo de processamento: 5-7 dias úteis
    Método de reembolso: Método de pagamento original
    ```
  </Accordion>

  <Accordion title="Documentando procedimentos" icon="list-check">
    Instruções passo a passo para processos.

    **Exemplo:**
    ```
    Título: Processo de Modificação de Pedido

    Conteúdo:
    Clientes podem modificar pedidos dentro de 24 horas após a realização.

    O que pode ser modificado:
    - Endereço de entrega
    - Velocidade de entrega
    - Quantidades de itens (se houver estoque disponível)

    O que não pode ser modificado:
    - Método de pagamento (deve cancelar e fazer novo pedido)
    - Itens após início do processamento
    - Pedidos feitos há mais de 24 horas

    Processo de modificação:
    1. Cliente entra em contato com o suporte via telefone ou e-mail
    2. Agente verifica se o pedido está dentro da janela de modificação
    3. Agente verifica estoque para as alterações solicitadas
    4. Agente atualiza pedido no sistema
    5. Cliente recebe e-mail de confirmação

    Se o pedido já foi enviado, o cliente deve usar o processo de devolução.
    ```
  </Accordion>

  <Accordion title="Informações de referência rápida" icon="clock">
    Informações breves e frequentemente referenciadas.

    **Exemplo:**
    ```
    Título: Horário Comercial e Informações de Contato

    Conteúdo:
    Suporte ao Cliente:
    - Telefone: 0800-555-0123
    - E-mail: suporte@empresa.com
    - Horário: Segunda a Sexta, 9h - 18h BRT
    - Após expediente: emergencia@empresa.com (apenas problemas urgentes)

    Vendas:
    - Telefone: 0800-555-0124
    - E-mail: vendas@empresa.com
    - Horário: Segunda a Sexta, 8h - 20h BRT

    Faturamento:
    - Telefone: 0800-555-0125
    - E-mail: faturamento@empresa.com
    - Horário: Segunda a Sexta, 9h - 17h BRT
    ```
  </Accordion>
</AccordionGroup>

### Criando Itens de Texto

<Steps>
  <Step title="Navegue até a pasta">
    Abra a pasta onde você deseja adicionar o item.
  </Step>

  <Step title="Clique em 'Adicionar Item de Conhecimento'">
    Clique no botão para criar um novo item.
  </Step>

  <Step title="Selecione categoria 'TEXT'">
    Escolha TEXT das opções de tipo de conteúdo.
  </Step>

  <Step title="Digite o título">
    Dê ao seu item um título claro e descritivo.
  </Step>

  <Step title="Escreva o conteúdo">
    Digite seu conteúdo no editor. Use formatação para clareza:
    - Títulos para seções
    - Marcadores para listas
    - Números para passos
    - Negrito para ênfase
  </Step>

  <Step title="Clique em 'Criar'">
    Salve seu item de texto. O status mostrará imediatamente como COMPLETED.
  </Step>
</Steps>


## Itens de Upload de Arquivo

### O que são Itens de Upload de Arquivo?

Itens de upload de arquivo permitem que você faça upload de documentos existentes em vários formatos. O sistema extrai o conteúdo de texto e o disponibiliza para seus agentes.

### Quando Usar Upload de Arquivos

<AccordionGroup>
  <Accordion title="Documentação existente" icon="file">
    Você já tem conteúdo em formato de documento.

    **Exemplos:**
    - Manuais do usuário
    - Especificações de produto
    - Documentos legais
    - Materiais de treinamento
  </Accordion>

  <Accordion title="Documentos formatados" icon="file-pdf">
    Documentos com layouts específicos que são mais fáceis de manter como arquivos.

    **Exemplos:**
    - Diagramas técnicos
    - Tabelas e gráficos
    - Layouts de múltiplas colunas
    - Modelos de marca
  </Accordion>

  <Accordion title="Documentos de terceiros" icon="building">
    Documentação que você recebe de fornecedores ou parceiros.

    **Exemplos:**
    - Catálogos de fornecedores
    - Documentos de conformidade
    - Materiais de certificação
  </Accordion>
</AccordionGroup>

### Requisitos de Arquivo

**Especificações de arquivo:**
- **Formatos:** PDF, DOC, DOCX, TXT e outros formatos de documentos
- **Limite de tamanho:** Máximo de 10MB
- **Conteúdo:** Documentos baseados em texto e imagens digitalizadas (análise avançada lida com a maioria das digitalizações)
- **Proteção:** Sem proteção por senha

<Note>
O sistema usa análise avançada de documentos para extrair texto de imagens digitalizadas e PDFs. A maioria dos documentos digitalizados será processada corretamente, embora digitalizações de qualidade muito baixa possam exigir entrada manual de texto.
</Note>

### Criando Itens de Arquivo

<Steps>
  <Step title="Navegue até a pasta">
    Abra a pasta onde você deseja adicionar o arquivo.
  </Step>

  <Step title="Clique em 'Adicionar Item de Conhecimento'">
    Clique no botão para criar um novo item.
  </Step>

  <Step title="Selecione categoria 'FILE'">
    Escolha FILE das opções de tipo de conteúdo.
  </Step>

  <Step title="Digite o título">
    Dê ao seu arquivo um título descritivo (isso é separado do nome do arquivo).
  </Step>

  <Step title="Faça upload do arquivo">
    Clique em **Upload** e selecione seu documento do seu computador.
  </Step>

  <Step title="Clique em 'Criar'">
    O sistema fará upload e começará a processar seu arquivo.

    Progressão do status:
    - PENDING → Arquivo carregado, na fila para processamento
    - PROCESSING → Extraindo conteúdo de texto
    - COMPLETED → Pronto para uso
    - FAILED → Ocorreu erro de processamento
  </Step>
</Steps>

### Exemplo JSON de Item de Arquivo

```json wrap
{
  "category": "FILE",
  "title": "Manual do Usuário - Produto X",
  "file": "/uploads/knowledge/manual-usuario-produto-x.pdf",
  "status": "COMPLETED"
}
```

### Tempo de Processamento

O tempo de processamento de arquivo varia com base em:
- **Tamanho do arquivo:** Arquivos maiores demoram mais
- **Contagem de páginas:** Mais páginas = processamento mais longo
- **Complexidade:** Tabelas, imagens e layouts complexos retardam o processamento
- **Qualidade do texto:** Texto limpo e simples extrai mais rápido

**Tempos típicos de processamento:**
- Arquivos pequenos (< 1MB, 10 páginas): 10-30 segundos
- Arquivos médios (1-5MB, 10-50 páginas): 30-90 segundos
- Arquivos grandes (5-10MB, 50+ páginas): 2-5 minutos

### Problemas Comuns com Arquivos

<AccordionGroup>
  <Accordion title="Processamento falhou" icon="circle-xmark">
    **Causas:**
    - Arquivo excede 10MB
    - Arquivo está protegido por senha
    - Arquivo está corrompido
    - Imagens digitalizadas de qualidade muito baixa

    **Soluções:**
    - Comprima o arquivo ou divida em arquivos menores
    - Remova a proteção por senha
    - Reexporte o arquivo da fonte
    - Para digitalizações de qualidade muito baixa, copie o conteúdo em um item de texto
  </Accordion>

  <Accordion title="Conteúdo extraído incorretamente" icon="text-slash">
    **Causas:**
    - Layouts complexos (múltiplas colunas, tabelas)
    - Imagens digitalizadas de qualidade muito baixa
    - Fontes ou codificação especiais
    - Campos de formulário e elementos interativos

    **Soluções:**
    - Verifique o conteúdo extraído no modo de edição
    - Recrie como item de texto com formatação adequada
    - Simplifique o layout do documento antes do upload
    - Exporte como documento de texto simples
  </Accordion>

  <Accordion title="Processamento demora muito" icon="clock">
    **O que fazer:**
    - Aguarde 5-10 minutos antes de assumir falha
    - Verifique tamanho do arquivo e contagem de páginas
    - Para arquivos grandes, considere dividir em vários arquivos
    - Converta para texto e faça upload como itens TEXT
  </Accordion>
</AccordionGroup>

### Melhores Práticas de Upload de Arquivo

<CardGroup cols={2}>
  <Card title="Otimize antes do upload" icon="compress">
    - Comprima arquivos grandes
    - Remova imagens desnecessárias
    - Use documentos baseados em texto
    - Mantenha abaixo de 5MB quando possível
  </Card>
  <Card title="Teste a extração" icon="vial">
    - Revise o conteúdo extraído após processamento
    - Verifique problemas de formatação
    - Verifique se informações críticas estão precisas
    - Faça novo upload se a extração for ruim
  </Card>
</CardGroup>

## Itens de URL

### O que são Itens de URL?

Itens de URL fazem scraping de conteúdo de páginas web e os armazenam na sua base de conhecimento. Isso é útil para referenciar documentação online, centrais de ajuda ou postagens de blog.

### Quando Usar Itens de URL

<AccordionGroup>
  <Accordion title="Documentação pública" icon="globe">
    Referencie documentação externa que você não mantém.

    **Exemplos:**
    - Documentação de API (sua ou de terceiros)
    - Bases de conhecimento públicas
    - Artigos da central de ajuda
    - Páginas de produtos
  </Accordion>

  <Accordion title="Conteúdo frequentemente atualizado" icon="arrows-rotate">
    Conteúdo que muda regularmente e você quer manter atualizado fazendo novo scraping.

    **Exemplos:**
    - Páginas de preços
    - Disponibilidade de produtos
    - Promoções atuais
    - Páginas de status
  </Accordion>

  <Accordion title="Postagens de blog ou artigos" icon="newspaper">
    Conteúdo educacional ou anúncios.

    **Exemplos:**
    - Guias de instruções
    - Artigos de melhores práticas
    - Anúncios de produtos
    - Tutoriais de recursos
  </Accordion>
</AccordionGroup>

### Criando Itens de URL

<Steps>
  <Step title="Navegue até a pasta">
    Abra a pasta onde você deseja adicionar a URL.
  </Step>

  <Step title="Clique em 'Adicionar Item de Conhecimento'">
    Clique no botão para criar um novo item.
  </Step>

  <Step title="Selecione categoria 'URL'">
    Escolha URL das opções de tipo de conteúdo.
  </Step>

  <Step title="Digite o título">
    Dê ao conteúdo um título descritivo.
  </Step>

  <Step title="Digite a URL fonte">
    Cole a URL completa incluindo https://

    **Exemplo:**
    ```
    https://docs.empresa.com/api/autenticacao
    ```
  </Step>

  <Step title="Clique em 'Criar'">
    O sistema buscará e processará a página web.

    Progressão do status:
    - PENDING → URL na fila para scraping
    - PROCESSING → Buscando e extraindo conteúdo
    - COMPLETED → Conteúdo disponível
    - FAILED → Ocorreu erro de scraping
  </Step>
</Steps>

### Exemplo JSON de Item de URL

```json wrap
{
  "category": "URL",
  "title": "Documentação de Autenticação da API",
  "source_url": "https://docs.empresa.com/api/autenticacao",
  "status": "COMPLETED"
}
```

### Requisitos de URL

**URLs funcionais:**
- Publicamente acessível (sem login necessário)
- Páginas de conteúdo HTML simples
- Sites de documentação
- Postagens de blog e artigos
- Páginas de conteúdo estático

**URLs problemáticas:**
- Páginas que exigem autenticação
- Aplicações pesadas em JavaScript (SPAs)
- Conteúdo com paywall
- Conteúdo carregado dinamicamente
- Aplicações interativas

<Note>
O scraping de URL funciona melhor com páginas web simples baseadas em texto. Aplicações web complexas podem não fazer scraping com sucesso.
</Note>

### Problemas Comuns com URL

<AccordionGroup>
  <Accordion title="Scraping falhou" icon="circle-xmark">
    **Causas:**
    - Página requer login/autenticação
    - URL está incorreta ou quebrada
    - Conteúdo carrega via JavaScript
    - Site bloqueia scraping (robots.txt)
    - Página não existe (404)

    **Soluções:**
    - Verifique se a URL está publicamente acessível
    - Teste a URL em janela anônima do navegador
    - Verifique se a URL está completa e correta
    - Copie o conteúdo manualmente em um item de texto
    - Use exportação em PDF da página
  </Accordion>

  <Accordion title="Conteúdo incompleto ou errado" icon="puzzle-piece">
    **Causas:**
    - Conteúdo renderizado por JavaScript não capturado
    - Carregamento de conteúdo dinâmico
    - Múltiplas abas/seções na página
    - Comentários ou barra lateral em scraping em vez do conteúdo principal

    **Soluções:**
    - Inspecione o conteúdo em scraping no modo de edição
    - Use URL direta para conteúdo específico
    - Copie o conteúdo desejado em um item de texto
    - Exporte a página como PDF e faça upload
  </Accordion>

  <Accordion title="Conteúdo fica desatualizado" icon="calendar">
    **Solução:**
    O conteúdo da URL é extraído uma vez no momento da criação. Para atualizar:
    - Exclua e recrie o item de URL
    - Ou copie o conteúdo atual em um item de texto para atualizações manuais

    Para conteúdo que muda com frequência, considere:
    - Itens de texto manuais que você atualiza regularmente
    - Exportações em PDF que você atualiza periodicamente
  </Accordion>
</AccordionGroup>

### Melhores Práticas de URL

<CardGroup cols={2}>
  <Card title="Teste acessibilidade" icon="unlock">
    - Abra a URL em janela anônima
    - Verifique se não há login necessário
    - Verifique se o conteúdo está visível
    - Garanta que a página carregue rapidamente
  </Card>
  <Card title="Revise conteúdo em scraping" icon="eye">
    - Verifique o conteúdo após scraping
    - Verifique se o conteúdo correto foi capturado
    - Procure problemas de formatação
    - Confirme que não há conteúdo extra (anúncios, barras laterais)
  </Card>
</CardGroup>

## Fluxo de Status de Processamento

Os itens de conhecimento passam por dois pipelines de processamento separados:

1. **Processamento de Conteúdo** - Extraindo texto de arquivos/URLs
2. **Indexação Vetorial** - Preparando conteúdo para RAG (busca semântica)

### Status de Processamento de Conteúdo

Isso rastreia a extração de conteúdo de texto da sua fonte.

<Steps>
  <Step title="PENDING">
    **Significado:** Item criado, na fila para extração de conteúdo

    **O que está acontecendo:**
    - Item foi salvo no banco de dados
    - Aguardando worker de processamento
    - Geralmente muito breve (segundos)

    **Ação necessária:** Nenhuma - aguarde o início do processamento
  </Step>

  <Step title="PROCESSING">
    **Significado:** O conteúdo do item está sendo extraído agora

    **O que está acontecendo:**
    - Para ARQUIVOS: Extraindo texto de PDF, Word, etc.
    - Para URLs: Buscando e fazendo scraping da página web específica
    - Para TEXTO: N/A (pula diretamente para COMPLETED)

    **Ação necessária:** Aguarde a conclusão do processamento
  </Step>

  <Step title="COMPLETED">
    **Significado:** Extração de conteúdo finalizada com sucesso

    **O que está acontecendo:**
    - Conteúdo foi extraído e armazenado
    - Indexação vetorial começará automaticamente
    - Item estará disponível quando a indexação for concluída

    **Ação necessária:** Nenhuma - aguarde indexação vetorial
  </Step>

  <Step title="FAILED">
    **Significado:** Extração de conteúdo encontrou um erro

    **O que está acontecendo:**
    - Extração de arquivo falhou (PDF corrompido, formato não suportado)
    - Scraping de URL falhou (página não encontrada, bloqueada, requer autenticação)
    - Conteúdo não está disponível para agentes

    **Ação necessária:** Verifique mensagem de erro e corrija o problema
  </Step>
</Steps>

**Fluxo de Processamento de Conteúdo:**
CREATED → PENDING → PROCESSING → COMPLETED ou FAILED

---

### Status de Indexação Vetorial

Depois que o conteúdo é extraído, ele deve ser indexado para RAG (busca semântica). Isso permite que os agentes encontrem conhecimento relevante com base no significado, não apenas em palavras-chave.

<Steps>
  <Step title="PENDING">
    **Significado:** Aguardando início da indexação vetorial

    **O que está acontecendo:**
    - Processamento de conteúdo concluído com sucesso
    - Na fila para geração de embeddings
    - Geralmente breve (segundos a minutos)

    **Ação necessária:** Nenhuma
  </Step>

  <Step title="INDEXING">
    **Significado:** Criando embeddings vetoriais para RAG

    **O que está acontecendo:**
    - Conteúdo está sendo dividido em pedaços
    - Embeddings de IA sendo gerados para cada pedaço
    - Vetores sendo armazenados na base de conhecimento

    **Ação necessária:** Aguarde - isso pode levar vários minutos para documentos grandes
  </Step>

  <Step title="INDEXED">
    **Significado:** Item está totalmente pronto para recuperação RAG

    **O que está acontecendo:**
    - Embeddings vetoriais armazenados com sucesso
    - Item pode ser recuperado via busca semântica
    - Agentes agora podem usar esse conhecimento

    **Ação necessária:** Nenhuma - item está totalmente pronto
  </Step>

  <Step title="FAILED">
    **Significado:** Indexação vetorial falhou

    **O que está acontecendo:**
    - Geração de embedding encontrou um erro
    - Item não aparecerá nos resultados RAG
    - Pode estar disponível apenas para injeção de contexto

    **Ação necessária:** Tente reindexar ou entre em contato com o suporte
  </Step>

  <Step title="REMOVED">
    **Significado:** Vetores foram excluídos

    **O que está acontecendo:**
    - Item foi excluído ou explicitamente removido do índice
    - Não está mais disponível para RAG

    **Ação necessária:** Reindexe se necessário
  </Step>
</Steps>

**Fluxo de Indexação Vetorial:**
PENDING → INDEXING → INDEXED ou FAILED

<Note>
**Ambos os status devem ter sucesso para funcionalidade completa:**
- Status de Conteúdo: COMPLETED
- Status de Vetor: INDEXED

Só então o item pode ser usado tanto em injeção de contexto quanto em recuperação RAG.
</Note>

## Tratamento de Erros

### Quando Itens Falham

Se um item de conhecimento mostrar status FAILED:

<Steps>
  <Step title="Verifique a mensagem de erro">
    Clique no item com falha para ver detalhes do erro.
  </Step>

  <Step title="Identifique a causa">
    Causas comuns:
    - **Arquivos:** Arquivo muito grande, corrompido, protegido por senha, imagem digitalizada
    - **URL:** Autenticação necessária, link quebrado, conteúdo não acessível
  </Step>

  <Step title="Tente soluções">
    - **Para Arquivos:** Comprima, remova proteção, adicione camada de texto ou converta para texto
    - **Para URLs:** Verifique acessibilidade, tente URL diferente ou copie conteúdo para item de texto
  </Step>

  <Step title="Recrie ou converta">
    - Exclua item com falha
    - Crie novo item com correções aplicadas
    - Ou converta para item de texto copiando conteúdo manualmente
  </Step>
</Steps>

### Prevenindo Erros

<CardGroup cols={2}>
  <Card title="Prevenção de Arquivo" icon="file-pdf">
    - Mantenha arquivos abaixo de 5MB

## Monitorando Processamento

### Indicadores do Painel

No painel da sua base de conhecimento, você pode ver o status de processamento rapidamente para cada item em suas pastas:

**Indicadores de status:**
- Marca de seleção verde = COMPLETED
- Ampulheta = PROCESSING
- Símbolo de pausa = PENDING
- X vermelho = FAILED

**Exemplo de visualização de pasta:**
- Manual do Usuario.pdf - COMPLETED
- Guia de Inicio Rapido.pdf - PROCESSING
- Documentacao API - PENDING
- Manual Legado.pdf - FAILED

### Processamento em Lote

Ao fazer upload de vários itens:

1. Itens processam sequencialmente ou em paralelo (dependente do sistema)
2. Verifique depois de 5-10 minutos para lotes grandes
3. Revise o status de cada item
4. Corrija quaisquer falhas individualmente

<Tip>
Faça upload de um item de teste primeiro para verificar formato e processamento antes de fazer upload em lote.
</Tip>

## Escolhendo o Tipo de Conteúdo Certo

Use esta árvore de decisão para selecionar o melhor tipo de conteúdo:

**Você tem conteúdo existente?**
- Não → Use TEXT (escreva diretamente)
- Sim, é um documento →
  - Menos de 10MB → Use FILE
  - Mais de 10MB → Extraia texto, use TEXT
- Sim, é uma página web →
  - Publicamente acessível → Use URL (se scraping falhar, copie para TEXT)
  - Não acessível → Copie conteúdo para TEXT

### Recomendações Rápidas

| Sua Situação | Melhor Tipo de Conteúdo |
|---------------|-------------------|
| Escrevendo FAQs do zero | TEXT |
| Tem documentos Word/PDF existentes | Upload como FILE |
| Tem documentos abaixo de 10MB | FILE |
| Tem documentos acima de 10MB | Divida em arquivos menores ou extraia para TEXT |
| Documentação web pública | URL (com TEXT como backup) |
| Conteúdo privado/autenticado | Copie para TEXT |
| Precisa de disponibilidade imediata | TEXT (sem atraso de processamento) |
| Formatação complexa importa | FILE |
