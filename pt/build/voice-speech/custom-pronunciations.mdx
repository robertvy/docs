## Visão Geral

Pronúncias personalizadas permitem definir como palavras específicas devem soar, garantindo que seu agente sempre pronuncie corretamente nomes de marca, termos técnicos e acrônimos.

**Como funciona:**
1. Agente gera resposta: "Ligue para nós na itellicoAI para ajuda"
2. Sistema aplica regras de pronúncia: "Ligue para nós na eye-tell-ee-co A I para ajuda"
3. Sintetizador de voz fala o texto modificado
4. Cliente ouve pronúncia correta

<Note>
Substituições de pronúncia estão disponíveis em **Voz > Pronúncias** na configuração do seu agente.
</Note>

## Adicionar Pronúncias

### Substituição Única

1. Clique em **Adicionar Substituição**
2. Digite a **palavra alvo** (a palavra a substituir)
3. Digite a **pronúncia** (como deve soar)
4. Clique em **Adicionar**

**Exemplos:**
- **Alvo:** Nike → **Pronúncia:** Nai-key
- **Alvo:** API → **Pronúncia:** A-P-I
- **Alvo:** 24/7 → **Pronúncia:** vinte e quatro sete
- **Alvo:** EST → **Pronúncia:** Horário do Leste

<Warning>
**Palavras alvo devem ser apenas palavras únicas** - espaços não são permitidos. Use palavras completas como `Nike` ou `API`, não frases como `atendimento ao cliente`.
</Warning>

### Importação em Massa via CSV

Para importar muitas pronúncias de uma vez:

1. Clique em **Importar CSV**
2. Baixe o modelo se necessário
3. Prepare seu CSV com duas colunas: `target`, `replacement`
4. Arraste e solte seu arquivo CSV ou clique para selecionar
5. Revise a visualização e clique em **Importar**

**Formato CSV:**
```csv
target,replacement
Nike,Nai-key
EST,Horário do Leste
24/7,vinte e quatro sete
API,A-P-I
```

**Limites:**
- Máximo 500 pronúncias por agente
- Tamanho máximo do arquivo CSV: 5MB

## Dicas de Ortografia Fonética

### Ortografia Fonética Simples

Escreva substituições como texto fonético simples e legível:

**Bons exemplos:**
- Nike → Nai-key
- SQL → sequel (ou) S-Q-L
- Wi-Fi → why-figh
- iOS → eye-oh-ess

**Dicas:**
- Use hífens para separar sílabas: Nai-key, A-P-I
- Soletre como você explicaria a pronúncia para alguém
- Teste com sua voz para ouvir como soa

### Padrões Comuns

**Acrônimos (soletrar cada letra):**
- API → A-P-I
- URL → U-R-L
- HTTP → H-T-T-P
- AWS → A-W-S

**Nomes de marca (ortografia fonética):**
- Nike → Nai-key
- Adidas → ah-dee-dahs
- Chipotle → chih-poht-lay

**Números e hora:**
- 24/7 → vinte e quatro sete
- 9am → nove A-M
- EST → Horário do Leste
- 5pm → cinco P-M

**Termos técnicos:**
- SQL → sequel (ou) S-Q-L
- API → A-P-I
- URL → you-are-ell
- WiFi → why-figh

## Avançado: Fonemas IPA

Para controle preciso de pronúncia, você pode usar notação do Alfabeto Fonético Internacional (IPA) com o formato `<< >>`:

**Formato:** `<<fonema|fonema|fonema>>`

**Exemplo:**
```
espresso → <<ɛ|s|p|r|ɛ|s|oʊ>>
```

<Note>
Fonemas IPA fornecem controle preciso mas requerem conhecimento de notação fonética. Ortografia fonética simples (Nai-key) funciona bem para a maioria dos casos.
</Note>

## Gerenciar Pronúncias

### Editar Substituições Existentes

1. Clique em qualquer chip de substituição na lista
2. Modifique o alvo ou a pronúncia
3. Clique em **Salvar**

### Excluir Substituições

- Clique no **ícone de lixeira** em qualquer chip de substituição para excluí-lo
- Ou clique em **Limpar Tudo** para remover todas as pronúncias (requer confirmação)

### Exportar Pronúncias

1. Clique em **Exportar CSV**
2. Arquivo CSV é baixado com todas as suas pronúncias atuais
3. Formato do nome do arquivo: `{nome_agente}_pronunciations.csv`

Use arquivos CSV exportados para fazer backup de suas pronúncias ou copiá-las para outros agentes.

## Testar Pronúncias

Após adicionar pronúncias:

1. Faça uma chamada de teste ao seu agente
2. Acione respostas contendo as palavras alvo
3. Ouça para verificar se a pronúncia soa correta
4. Ajuste a ortografia fonética se necessário

**Dicas de teste:**
- Teste em contexto, não isoladamente
- Peça aos membros da equipe para ouvir
- Ajuste hifenização ou ortografia se não soar bem

## Melhores Práticas

<AccordionGroup>
  <Accordion title="Comece com Problemas Comuns" icon="bullseye">
    Adicione pronúncias para palavras que você sabe que são pronunciadas incorretamente em chamadas reais. Revise transcrições de chamadas para identificar problemas de pronúncia recorrentes.
  </Accordion>

  <Accordion title="Mantenha Simples" icon="sparkles">
    Use ortografia fonética (Nai-key) em vez de IPA complexo, a menos que necessário. Texto fonético simples é mais fácil de manter e funciona bem para a maioria dos casos.
  </Accordion>

  <Accordion title="Teste Antes de Implementar" icon="vial">
    Sempre teste pronúncias com chamadas reais antes de ir para produção. Faça uma chamada de teste e acione respostas contendo as palavras alvo para verificar se soam corretamente.
  </Accordion>

  <Accordion title="Documente suas Regras" icon="file-arrow-down">
    Exporte seu CSV regularmente como backup. Use o botão Exportar CSV para salvar suas pronúncias e mantê-las versionadas junto com a configuração do seu agente.
  </Accordion>

  <Accordion title="Use Ortografia Consistente" icon="check">
    Se você tem "API" soletrado como "A-P-I", use esse formato consistentemente em todas as suas pronúncias. Consistência melhora a manutenibilidade.
  </Accordion>

  <Accordion title="Apenas Palavras Únicas" icon="letter-a">
    Lembre-se que palavras alvo não podem conter espaços - apenas palavras únicas completas funcionam. Use `Nike` ou `API`, não frases como `atendimento ao cliente`.
  </Accordion>
</AccordionGroup>
