Claude Code é a ferramenta CLI oficial da Anthropic. Este guia irá ajudá-lo a configurar o Claude Code para ajudá-lo a escrever e manter sua documentação.

## Pré-requisitos

- Assinatura ativa do Claude (Pro, Max ou acesso API)

## Configuração

1. Instale o Claude Code globalmente:

  ```bash
  npm install -g @anthropic-ai/claude-code
```

2. Navegue até o seu diretório de documentação.
3. (Opcional) Adicione o arquivo `CLAUDE.md` abaixo ao seu projeto.
4. Execute `claude` para começar.

## Criar `CLAUDE.md`

Crie um arquivo `CLAUDE.md` na raiz do seu repositório de documentação para treinar o Claude Code nos seus padrões de documentação específicos:

````markdown
# Documentação Mintlify

## Relação de trabalho
- Você pode contestar ideias - isso pode levar a uma documentação melhor. Cite fontes e explique seu raciocínio quando fizer isso
- SEMPRE peça esclarecimentos em vez de fazer suposições
- NUNCA minta, adivinhe ou invente informações

## Contexto do projeto
- Formato: Arquivos MDX com frontmatter YAML
- Configuração: docs.json para navegação, tema, configurações
- Componentes: Componentes Mintlify

## Estratégia de conteúdo
- Documente apenas o suficiente para o sucesso do usuário - nem muito, nem pouco
- Priorize a precisão e usabilidade da informação
- Torne o conteúdo perene quando possível
- Procure informações existentes antes de adicionar novo conteúdo. Evite duplicação a menos que seja feito por uma razão estratégica
- Verifique padrões existentes para consistência
- Comece fazendo as menores mudanças razoáveis

## Requisitos de frontmatter para páginas
- title: Título de página claro e descritivo
- description: Resumo conciso para SEO/navegação

## Padrões de escrita
- Voz na segunda pessoa ("você")
- Pré-requisitos no início do conteúdo procedimental
- Teste todos os exemplos de código antes de publicar
- Combine o estilo e formatação das páginas existentes
- Inclua casos de uso básicos e avançados
- Tags de linguagem em todos os blocos de código
- Texto alternativo em todas as imagens
- Caminhos relativos para links internos

## Fluxo de trabalho do Git
- NUNCA use --no-verify ao fazer commit
- Pergunte como lidar com mudanças não commitadas antes de começar
- Crie um novo branch quando não existir um branch claro para as mudanças
- Faça commit frequentemente durante o desenvolvimento
- NUNCA pule ou desabilite hooks pre-commit

## Não fazer
- Pular o frontmatter em qualquer arquivo MDX
- Usar URLs absolutas para links internos
- Incluir exemplos de código não testados
- Fazer suposições - sempre peça esclarecimentos
````
