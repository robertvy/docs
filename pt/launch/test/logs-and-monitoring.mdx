## Visão Geral

Cada chamada de teste gera logs abrangentes que ajudam você a validar o comportamento do agente, depurar problemas e rastrear métricas de qualidade. Aprenda a analisar logs efetivamente para garantir que seu agente esteja pronto para produção.

## Acessando Logs de Conversa

<Steps>
  <Step title="Navegar até logs">
    Vá para **Conversas → Logs** no painel. Você verá uma lista de todas as conversas recentes (teste e produção).
  </Step>

  <Step title="Filtrar chamadas de teste">
    Use filtros para encontrar suas chamadas de teste:
    - **Agent**: Selecione agente específico
    - **Date Range**: Última hora, últimas 24 horas, intervalo personalizado
    - **Status**: Sucesso, falha, em progresso
    - **Tags**: Filtrar por tag "test"
    - **Direction**: Entrada vs saída
  </Step>

  <Step title="Abrir detalhes da conversa">
    Clique em qualquer conversa para ver detalhes completos:
    - Reprodução de áudio
    - Transcrição completa
    - Logs de execução de ações
    - Recuperações da base de conhecimento
    - Resultados de objetivos
    - Métricas de qualidade
  </Step>
</Steps>

## Entendendo Componentes do Log

### Visualização de Transcrição

**O Que Você Vê**:
```
[00:03] Agent: Bem-vindo à itellicoAI. Como posso ajudá-lo hoje?
[00:05] Customer: Qual é o horário de funcionamento?
[00:07] Agent: Estamos abertos de segunda a sexta, das 9h às 17h, horário do leste.
[00:10] Customer: Obrigado.
[00:11] Agent: De nada! Há mais alguma coisa em que posso ajudar?
[00:13] Customer: Não, é só isso.
[00:14] Agent: Tenha um ótimo dia!
[Chamada encerrada - Duração: 00:11]
```

**Elementos-Chave**:
- **Timestamps**: Rastreie latência de resposta (tempo entre turnos)
- **Rótulos de Falante**: Agente vs Cliente claramente marcados
- **Pontuações de Confiança**: Passe o mouse sobre a transcrição para ver confiança da transcrição
- **Opção de Edição**: Corrija manualmente erros de transcrição para retreinamento

<Accordion title="Analisando tempo de resposta">
  **Latência Alvo**:
  - Resposta do agente: &lt;1 segundo (excelente), &lt;2 segundos (aceitável)
  - Chamadas de API: &lt;3 segundos no total (incluindo processamento do agente)

  **Sinais de Alerta**:
  - Intervalos >3 segundos indicam problemas de latência
  - Verifique aba **Qualidade** para métricas de rede
  - Revise seleção do modelo (GPT-4 mais lento que GPT-3.5)
</Accordion>

### Aba de Conhecimento

**Ver Recuperação do KB**:
```json
{
  "query": "qual é o horário de funcionamento",
  "chunks_retrieved": 3,
  "results": [
    {
      "content": "Nossa equipe de suporte está disponível de segunda a sexta, das 9h às 17h EST...",
      "similarity_score": 0.92,
      "source": "kb_article_123",
      "chunk_id": "chunk_456"
    },
    {
      "content": "Horários de feriados variam. Fechamos em feriados nacionais dos EUA...",
      "similarity_score": 0.78,
      "source": "kb_article_789"
    }
  ],
  "retrieval_time_ms": 123
}
```

**Análise**:
- **Alta Similaridade (>0.85)**: Boa correspondência, agente deve usar este conteúdo
- **Similaridade Média (0.7-0.85)**: Relevante mas pode precisar de melhor conteúdo ou query
- **Baixa Similaridade (&lt;0.7)**: Correspondência ruim, agente pode não ter resposta no KB

**Se Agente Diz "Não Sei"**:
- Verifique se conteúdo relevante foi recuperado (similaridade >0.8)
- Se sim → Problema de instrução do agente (não usou conteúdo recuperado)
- Se não → Conteúdo do KB ausente ou query incompatível

### Aba de Objetivos

**Rastreamento de Objetivos da Conversa**:
<Tabs>
  <Tab title="Resultados de Objetivos">
    ```json
    {
      "goals": [
        {
          "name": "appointment_scheduled",
          "achieved": true,
          "confidence": 0.94,
          "entities": {
            "date": "2024-01-15",
            "time": "14:00",
            "service": "consultation"
          }
        },
        {
          "name": "contact_info_collected",
          "achieved": true,
          "confidence": 0.88,
          "entities": {
            "email": "customer@example.com",
            "phone": "+12125551234"
          }
        }
      ]
    }
    ```

    **Interpretando Confiança**:
    - >0.9: Muito confiante, confiável
    - 0.7-0.9: Confiante, provavelmente preciso
    - &lt;0.7: Baixa confiança, revisar transcrição
  </Tab>

  <Tab title="Depuração de Objetivos">
    **Por Que Objetivos Não São Alcançados**:
    1. **Definição de Objetivo Muito Rígida**: Ajuste frases de disparo
    2. **Conversa Interrompida**: Cliente desligou antes da conclusão
    3. **Agente Não Seguiu Fluxo**: Problema de instrução
    4. **Entidades Não Extraídas**: Necessita respostas mais claras do cliente

    **Exemplo de Correção**:
    ```markdown
    # Antes (muito rígido)
    Objetivo: "appointment_scheduled"
    Disparo: Cliente diz "sim, marque para mim às 14h"

    # Depois (mais flexível)
    Objetivo: "appointment_scheduled"
    Disparo: Qualquer data/hora confirmada de reserva
    ```
  </Tab>
</Tabs>

### Aba de Qualidade

**Métricas de Qualidade da Chamada**:

| Métrica | Alvo | Indica |
|--------|--------|-----------|
| **MOS (Mean Opinion Score)** | >3.5 | Qualidade geral da chamada (escala 1-5) |
| **Jitter** | &lt;30ms | Estabilidade da rede (variação no tempo de pacotes) |
| **Perda de Pacotes** | &lt;1% | Confiabilidade da rede |
| **RTT (Round Trip Time)** | &lt;200ms | Latência da rede |
| **Bitrate** | 64kbps+ | Qualidade do áudio (maior = melhor) |

**Diagnóstico de Problemas de Áudio**:
- Alto jitter + perda de pacotes → Congestionamento de rede
- Alto RTT → Distância geográfica ou problema de roteamento
- Pontuação MOS baixa → Qualidade geral ruim, investigar múltiplos fatores

## Ferramentas de Monitoramento

### Monitor ao Vivo

**Monitoramento de Chamadas em Tempo Real**:
<Steps>
  <Step title="Abrir monitor ao vivo">
    Navegue para **Conversas → Monitor ao Vivo** para ver chamadas ativas em tempo real.
  </Step>

  <Step title="Assistir chamadas ativas">
    Ver lista de chamadas em progresso com:
    - ID do chamador
    - Agente atribuído
    - Duração da chamada (cronômetro ao vivo)
    - Transcrição atual (atualiza a cada 2-3 segundos)
    - Ações sendo executadas
  </Step>

  <Step title="Intervir se necessário">
    **Opções de Intervenção**:
    - **Whisper**: Falar com o agente sem o cliente ouvir (orientar agente)
    - **Barge In**: Entrar na chamada como participante de 3 vias
    - **Takeover**: Desconectar agente, assumir chamada você mesmo
    - **End Call**: Forçar término da chamada se necessário

    **Casos de Uso**:
    - Agente dando informação errada
    - Teste de caso extremo requer orientação
    - Situação de emergência
  </Step>
</Steps>

## Analisando Resultados de Teste

### Checklist de Qualidade

Após executar suite de testes, revise:

<Accordion title="Qualidade da transcrição">
  - [ ] Precisão da transcrição >90%
  - [ ] Sem palavras mal compreendidas importantes
  - [ ] Respostas do agente relevantes à entrada do cliente
  - [ ] Conversa flui naturalmente
  - [ ] Sem pausas ou atrasos desajeitados
</Accordion>

<Accordion title="Execução de ações">
  - [ ] Todas as ações disparadas nos momentos corretos
  - [ ] Ações concluídas com sucesso (sem erros)
  - [ ] Respostas da API usadas corretamente pelo agente
  - [ ] Latência de ação &lt;3 segundos
  - [ ] Tratamento de erros funciona (falhas tratadas graciosamente)
</Accordion>

<Accordion title="Precisão do conhecimento">
  - [ ] Queries do KB recuperam conteúdo relevante
  - [ ] Agente usa conteúdo recuperado nas respostas
  - [ ] Agente não alucina fatos não presentes no KB
  - [ ] Agente diz "Não sei" quando apropriado
  - [ ] Tempo de recuperação &lt;500ms
</Accordion>

<Accordion title="Qualidade da chamada">
  - [ ] Pontuação MOS >3.5
  - [ ] Jitter &lt;30ms
  - [ ] Perda de pacotes &lt;1%
  - [ ] Sem quedas ou distorções de áudio
  - [ ] Voz soa natural e clara
</Accordion>

### Comparando Execuções de Teste

**Rastrear Métricas ao Longo do Tempo**:
```markdown
| Data | Chamadas de Teste | Taxa de Sucesso | Duração Média | Alcance de Objetivos |
|------|-----------|--------------|--------------|------------------|
| 10 Jan | 25 | 96% | 3:45 | 68% |
| 11 Jan | 30 | 94% | 3:52 | 71% |
| 12 Jan | 28 | 98% | 3:38 | 75% |
```

**Análise de Tendência**:
- Taxa de sucesso melhorando → Bom progresso
- Duração diminuindo → Agente ficando mais eficiente
- Alcance de objetivos aumentando → Instruções otimizadas

**Detecção de Regressão**:
- Queda súbita na taxa de sucesso → Mudança recente quebrou algo
- Pico de duração → Novo problema de latência introduzido
- Queda no alcance de objetivos → Instruções ou KB mudaram negativamente

## Exportando Logs

### Para Revisão da Equipe

<Steps>
  <Step title="Exportar logs de conversa">
    De Conversas → Logs, selecione conversas e clique em **Exportar**:
    - Formato: JSON, CSV
    - Incluir: Transcrição, ações, objetivos, metadados
    - Baixar como arquivo ou enviar para email
  </Step>

  <Step title="Compartilhar com a equipe">
    **Casos de Uso**:
    - Equipe de QA revisa para garantia de qualidade
    - Equipe de produto analisa padrões de conversa
    - Engenharia depura problemas específicos
    - Equipe de compliance audita para requisitos regulatórios
  </Step>
</Steps>

### Acesso Programático

**Exportação via API**:
```bash
# Exportar todas as conversas de teste das últimas 24 horas
curl -H "Authorization: Bearer YOUR_API_KEY" \
     "https://api.itellico.ai/v1/conversations?start_date=2024-01-12&tag=test" \
     > test_conversations.json

# Analisar com jq
cat test_conversations.json | jq '.[] | select(.status == "failed")'
```

**Usar para**:
- Relatórios automatizados de teste
- Integração com frameworks de teste
- Validação de pipeline de CI/CD
- Painéis de análise personalizados
