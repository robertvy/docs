## Abilities Tab

Configure what your agent can do and how it behaves in conversations.

### Instructions (System Prompt)
Define your agent's personality, knowledge, and conversational flow.
- Jinja-aware editor with syntax highlighting
- Variable insertion and knowledge references
- Prompt templates and writing tips

[Learn more about Instructions →](/build/conversation/instructions)

### Greeting & Opening
Control how conversations start.
- User initiates, AI-generated, or fixed greeting
- Inbound vs outbound greeting overrides
- Interruptibility and delay settings

[Learn more about Greetings →](/build/conversation/greeting-messages)

### Knowledge Access
Connect knowledge bases, collections, and documents.
- Attach multiple knowledge sources
- Choose Context or RAG mode per source
- View remaining context tokens

[Learn more about Knowledge →](/build/knowledge/architecture)

### Actions & Integrations
Enable your agent to take real actions.
- Transfer calls (agent, phone, SIP)
- End conversations
- Smart booking (Cal.com)
- Custom API actions

[Learn more about Actions →](/build/actions/overview)

### DTMF Controls
Configure keypad input handling.
- Menu mode (single key) or Collector mode (multi-digit)
- Tone sending configuration
- Capture and emit settings

[Learn more about DTMF →](/build/advanced/dtmf-controls)

### MCP Servers
Attach Model Context Protocol integrations.
- Connect account-level MCP servers
- Manage server connections
- Access external tools mid-conversation

[Learn more about MCP →](/build/advanced/mcp-servers)

## Agent Architecture Types

### Pipeline Agents (Default)
Traditional multi-model architecture with separate components.

**Components:**
- Speech-to-Text (Transcriber)
- Language Model (LLM)
- Text-to-Speech (Voice)

**Advantages:**
- More provider options
- Granular control over each component
- Custom pronunciations support
- Generally more cost-effective

**Best For:**
- Most use cases
- Complex knowledge integration
- Custom actions and workflows

### Multimodal Agents
End-to-end models that handle speech natively.

**Components:**
- Single unified model handles speech and reasoning

**Advantages:**
- Lower latency
- More natural interruptions
- Better voice-specific understanding

**Best For:**
- Ultra-low latency requirements
- Highly conversational interactions
- Real-time responsiveness critical

<Note>
Most users should start with Pipeline agents as they offer more flexibility and control. Multimodal agents are available for specific use cases requiring ultra-low latency.
</Note>

## Navigation Tips

### Using the Configuration Panel

<Steps>
  <Step title="Start with Abilities">
    Configure instructions, greetings, and knowledge first
  </Step>
  <Step title="Configure Models">
    Select AI model, voice, and transcriber
  </Step>
  <Step title="Fine-tune Operations">
    Adjust VAD and timeout settings after testing
  </Step>
  <Step title="Set Up Analytics">
    Define goals for measurement
  </Step>
</Steps>

### Save Behavior

<Warning>
Changes to most settings are applied immediately when you click Save. Some settings (like model selection) take effect on the next conversation.
</Warning>

**Auto-save:** Not available - always click Save
**Unsaved changes:** Yellow bar appears warning about unsaved changes
**Validation:** Real-time syntax checking in instruction editor

### Quick Actions

- **Test Call:** Top-right button for quick testing
- **Duplicate Agent:** Create copy with all settings
- **Export Config:** Download agent configuration (API)
- **View History:** See configuration change log (coming soon)

## Configuration Workflow

### Initial Setup (First Time)
1. Set agent identity and type
2. Write basic instructions
3. Configure greeting
4. Select model, voice, transcriber
5. Test thoroughly

### Adding Capabilities
1. Add knowledge bases
2. Configure actions
3. Set up goals
4. Test new features

### Optimization (Ongoing)
1. Review analytics
2. Refine instructions
3. Tune VAD and timeouts
4. Update knowledge
5. Test and iterate
