## Overview

Your agent's identity includes the basic information that helps you and your team identify, organize, and manage agents in your dashboard. Some of these details (name and avatar) are also visible when you share demo links with others.

## Agent Name

The agent name is how you identify this agent in your dashboard, campaigns, and reports. It's also visible when you share demo links with others.

<Note>
When sharing demo links, the agent name is displayed to visitors, so choose a professional name that represents your brand.
</Note>

### Best Practices

**Be Descriptive**
```
✅ Good: "Customer Support - Product Questions"
✅ Good: "Outbound Sales - Lead Qualification"
✅ Good: "Appointment Booking - Dental Clinic"

❌ Avoid: "Agent 1"
❌ Avoid: "Test"
❌ Avoid: "New Agent"
```

**Include Purpose**
Names should indicate what the agent does, especially when managing multiple agents.

**Use Consistent Naming**
Develop a naming convention:
- `[Function] - [Specialty]`
- `[Department] - [Agent Type]`
- `[Use Case] - [Language]`

## Avatar

Upload an avatar image to visually identify your agent in the dashboard and in demo links you share with others.

<Note>
The avatar is visible when you share demo links, so choose an image that represents your brand professionally.
</Note>

### Avatar Guidelines

**Image Specifications**
- **Format:** PNG, JPG, or SVG
- **Size:** Recommended 200x200px minimum
- **Aspect Ratio:** Square (1:1)
- **File Size:** Under 2MB

**Visual Best Practices**
- Use icons or illustrations that represent the agent's function
- Maintain consistent visual style across all agents
- Use colors that differentiate agent types
- Avoid photos of real people (unless it's your brand mascot)

## Tags

Tags help organize and filter agents in your dashboard, especially useful when managing many agents.

### Tagging Strategies

**By Function**
```
support, sales, booking, billing, technical
```

**By Language**
```
english, spanish, french, german, multilingual
```

**By Department**
```
customer-service, sales-team, operations, finance
```

**By Status**
```
production, testing, archived, beta
```

**By Region**
```
us, eu, apac, latam
```

### Using Tags Effectively

<Steps>
  <Step title="Create a tagging system">
    Define categories and conventions before creating many agents
  </Step>
  <Step title="Apply consistently">
    Use the same tags across similar agents
  </Step>
  <Step title="Filter and search">
    Use tags to quickly find specific agent groups in the dashboard
  </Step>
  <Step title="Review periodically">
    Clean up unused tags and update as needs change
  </Step>
</Steps>

## Organization Best Practices

### For Small Teams (1-5 agents)

- Use simple, descriptive names
- Minimal tagging needed
- Focus on clear descriptions

### For Medium Teams (5-20 agents)

- Develop naming conventions
- Use tags for function and status
- Maintain regular notes on changes

### For Large Teams (20+ agents)

- Strict naming conventions mandatory
- Multi-level tagging (function, region, language, status)
- Detailed change logs in notes
- Consider creating agent documentation outside the platform

## Quick Reference

| Field | Purpose | Visible To |
|-------|---------|------------|
| **Name** | Identify agent in dashboard | Internal only |
| **Description** | Brief purpose & notes | Internal only |
| **Avatar** | Visual identification | Internal only |
| **Tags** | Organization & filtering | Internal only |
| **Notes** | Detailed documentation | Internal only |

<Warning>
None of these identity fields are visible to customers during calls. Customer-facing information is configured in [Greeting Messages](/build/conversation/greeting-messages) and [Instructions](/build/conversation/instructions).
