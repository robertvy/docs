## Overview

The Instructions field (also called the system prompt) is where you define your agent's personality, conversational style, objectives, and behavioral rules. This is the most important configuration for your agent's success.

Instructions are a text block sent to the AI model with every conversation turn. They tell the model:
- **Who it is** (role and identity)
- **How to behave** (tone, style, guardrails)
- **What to achieve** (objectives and goals)
- **When to use tools** (actions, knowledge, transfers)

<Note>
The platform automatically provides conversation history, contact data, and knowledge base content. Focus your instructions on rules and guidance only you can provide.
</Note>

## Using Prompt Templates

The instruction editor includes a **Prompt Library** with ready-to-use templates:

- **Lead Qualification** - B2B sales with BANT framework
- **Appointment Booking** - Service scheduling with confirmations
- **Customer Feedback** - NPS surveys with recovery workflows
- **Information Gathering** - Research and data collection
- **Outreach** - Cold calling with value-first approach
- **Industry-Specific** - Healthcare, real estate, home services, and more

**To use a template:**
1. Click "Browse Templates" in the instruction editor
2. Select a template that matches your use case
3. Customize the placeholders (`company_name`, `service_type`, etc.)
4. Test and refine based on your specific needs

<Tip>
Templates follow a proven structure: Role → Goal → Conversation Flow → Quick Reference. Start with a template and adapt it rather than starting from scratch.
</Tip>

## Basic Instruction Structure

Here's a simple template to get started:

```
You are [NAME/ROLE] for [COMPANY NAME].

Your role:
- [Primary responsibility]
- [Secondary responsibility]
- [Additional tasks]

Tone: [Describe communication style]

Important guidelines:
- [Key rule 1]
- [Key rule 2]
- [Key rule 3]

When [SITUATION], you should [ACTION].
```

### Example: Customer Support Agent

```
You are Alex, a friendly customer support agent for Acme Software.

Your role:
- Answer customer questions about our products and services
- Troubleshoot common technical issues
- Provide account information when requested
- Escalate complex issues to human support

Tone: Professional but warm, patient, and helpful. Keep responses concise and clear.

Important guidelines:
- Always greet customers warmly and ask how you can help
- Never make promises about features or pricing you're unsure of
- If you don't know something, admit it honestly and offer to transfer
- Confirm understanding before moving to solutions
- End by asking if there's anything else you can help with

When a customer mentions billing issues, provide basic information but always offer to transfer to the billing department for detailed help.
```

## What the Agent Already Knows

The platform automatically provides the following context to your agent with every conversation turn. **You don't need to include these in your instructions:**

<AccordionGroup>
  <Accordion title="Conversation Transcript" icon="messages">
    The complete conversation history between the agent and the contact, including all messages and tool calls.
  </Accordion>

  <Accordion title="Contact Information" icon="user">
    Contact data like name, email, phone number, and any custom fields you've added. Access via variables like `{{ contact.first_name }}`.
  </Accordion>

  <Accordion title="Knowledge Base Content" icon="book">
    When relevant, the platform automatically retrieves and includes content from your knowledge bases. You just need to tell the agent when and how to use it.
  </Accordion>

  <Accordion title="Available Actions" icon="bolt">
    The platform tells the agent what actions are available (transfer, book appointment, custom API actions). You just need to explain when to use them.
  </Accordion>

  <Accordion title="Current Date/Time" icon="clock">
    The agent automatically knows the current date, time, and timezone.
  </Accordion>
</AccordionGroup>

<Warning>
**Don't repeat system-provided information in your instructions.** Focus on behavioral rules, conversation flow, and decision-making guidance instead.
</Warning>

## Common Mistakes

Avoid these common pitfalls when writing instructions:

<Warning>
**Instructions too long** - Keep instructions under 1000 words. Long instructions confuse the model and slow response times.
</Warning>

<Warning>
**Too vague** - "Be helpful" doesn't provide actionable guidance. Instead: "Answer questions in under 30 seconds, ask one clarifying question at a time, and confirm understanding before providing solutions."
</Warning>

<Warning>
**Contradictory rules** - Conflicting instructions confuse the agent. Review carefully to ensure guidelines don't contradict each other.
</Warning>

<Warning>
**Hardcoded facts in instructions** - Facts, product details, and policies belong in knowledge bases, not instructions. Instructions should contain behavioral rules and conversation guidance.
</Warning>

<Warning>
**Overly complex logic** - Deeply nested conditionals and complex Jinja logic often fail. Keep logic simple and clear.
</Warning>

<Warning>
**Incorrect action names** - Always reference exact action names from the Actions section. Mismatched names prevent the agent from using actions correctly.
</Warning>

### Quick Fixes

| Issue | Fix |
|-------|-----|
| Agent too verbose | Add: "Keep responses under 25 seconds" |
| Doesn't use actions | Add explicit triggers: "When [situation], use [action name]" |
| Ignores knowledge | Reference collection names: "Check the Product FAQs collection for..." |
| Wrong tone | Add specific tone examples and constraints |
| Interrupts customers | Add: "Let customers finish speaking before responding" |
