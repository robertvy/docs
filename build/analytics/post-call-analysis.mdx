## Overview

Post-call analysis uses AI to automatically extract structured information from conversations. Define custom questions that analyze every call and generate consistent, structured data for reporting and automation.

**Access:** Agent editor ’ **Analytics ’ Call Analysis**

**Use cases:** Quality scoring, sentiment tracking, issue categorization, outcome measurement, compliance verification, custom business metrics

---

## How It Works

1. **Define analysis questions** - Create questions the AI should answer after each call
2. **Choose question type** - Yes/No, Open-ended, or Rating (1-5 stars)
3. **AI analyzes transcript** - After each call, AI reads transcript and answers questions
4. **View results** - Structured data appears in conversation details and analytics

<Info>
Analysis runs automatically after each conversation. Results are available immediately in the conversation view and aggregate in your analytics dashboard.
</Info>

## Setup

<Steps>
  <Step title="Access Call Analysis">
    Agent editor ’ **Analytics ’ Call Analysis** ’ **Add Analysis Question**
  </Step>

  <Step title="Configure question">
    **Name:** Internal identifier (e.g., "Customer Satisfaction")

    **Description:** The question AI will answer (be specific):
    ```
    Example: "On a scale of 1-5 stars, how satisfied was the customer
    with the resolution provided during this call?"
    ```
  </Step>

  <Step title="Choose question type">
    **Yes/No** - Binary outcome
    ```
    Example: "Was the customer's issue fully resolved?"
    ’ Answer: Yes or No
    ```

    **Open** - Text response
    ```
    Example: "What was the primary reason for the customer's call?"
    ’ Answer: "Billing inquiry about overcharge"
    ```

    **Rating** - 1-5 stars
    ```
    Example: "How professional was the agent during the call?"
    ’ Answer: 1-5 stars
    ```
  </Step>

  <Step title="Set category (optional)">
    Group related analyses:
    - Quality
    - Compliance
    - Outcomes
    - Sentiment
    - Custom
  </Step>

  <Step title="Save and enable">
    Toggle **Active** to start analyzing future calls
  </Step>
</Steps>

## Question Types

### Yes/No Questions

Binary outcomes for clear metrics.

**Best for:**
- Goal achievement tracking
- Compliance verification
- Feature usage detection

**Examples:**
```
"Did the customer agree to the upsell offer?"
"Was payment information collected?"
"Did the agent follow the script?"
"Was the caller transferred to another department?"
```

### Open Questions

Free-text responses for detailed insights.

**Best for:**
- Issue categorization
- Root cause analysis
- Conversation summaries
- Custom data extraction

**Examples:**
```
"What was the main topic discussed during the call?"
"What objections did the customer raise?"
"Summarize the customer's technical issue in one sentence"
"Which product was the customer inquiring about?"
```

### Rating Questions

1-5 star scale for qualitative assessment.

**Best for:**
- Quality scoring
- Sentiment analysis
- Performance evaluation
- Satisfaction measurement

**Examples:**
```
"On a scale of 1-5, rate the customer's sentiment during the call"
"How well did the agent handle objections? (1-5 stars)"
"Rate the clarity of the information provided (1-5)"
"Overall call quality score (1-5)"
```

## Best Practices

<AccordionGroup>
  <Accordion title="Write Specific Questions" icon="bullseye">
    L Vague: "Was the call good?"

     Specific: "Did the agent provide accurate information, address all customer questions, and maintain a professional tone throughout the call?"
  </Accordion>

  <Accordion title="Provide Context for Ratings" icon="star-half-stroke">
    For rating questions, define what each star level means:

    ```
    Rate agent professionalism (1-5):
    5 = Exceptionally professional
    4 = Very professional
    3 = Professional
    2 = Somewhat unprofessional
    1 = Unprofessional
    ```
  </Accordion>

  <Accordion title="Use Categories to Organize" icon="folder">
    Group related analyses for easier management:
    - **Quality** - Agent performance, call quality
    - **Compliance** - Policy adherence, legal requirements
    - **Outcomes** - Resolution, goals achieved
    - **Sentiment** - Customer emotions, satisfaction
  </Accordion>

  <Accordion title="Start with Key Metrics" icon="chart-line">
    Don't create too many analyses at once. Start with 3-5 critical questions:
    1. Issue resolved (Yes/No)
    2. Customer satisfaction (Rating)
    3. Call topic (Open)
    4. Sentiment (Rating)
    5. Compliance check (Yes/No)

    Add more as needed.
  </Accordion>

  <Accordion title="Test and Refine" icon="flask">
    After adding analysis questions:
    1. Make a few test calls
    2. Review AI responses in conversation details
    3. Adjust question wording if answers aren't accurate
    4. Provide more context in descriptions if needed
  </Accordion>
</AccordionGroup>

## Viewing Results

### Individual Conversations

View analysis results for each call:

1. Go to **Conversations** in your dashboard
2. Click on a conversation
3. Scroll to **Call Analysis** section
4. See all analysis questions with AI-generated answers

### Analytics Dashboard

Aggregate analysis across all conversations:

1. Go to **Analytics Dashboard**
2. Filter by date range, agent, or tags
3. View charts and metrics for each analysis question
4. Export data for reporting

[Learn more about Analytics Dashboard ’](/optimize/analytics-dashboard)

## Troubleshooting

<AccordionGroup>
  <Accordion title="Inaccurate answers" icon="triangle-exclamation">
    **Possible causes:**
    - Question too vague or ambiguous
    - Multiple interpretations possible
    - Context missing from description

    **Solutions:**
    - Make question more specific and detailed
    - Provide examples of good/bad answers in description
    - Add context about what to look for
    - For ratings, define each star level explicitly
  </Accordion>

  <Accordion title="Inconsistent ratings" icon="chart-mixed">
    **Possible causes:**
    - Rating criteria not clearly defined
    - Subjective interpretation by AI

    **Solutions:**
    - Define exact criteria for each rating level
    - Provide specific examples for each star rating
    - Use more objective Yes/No questions instead
  </Accordion>

  <Accordion title="Analysis not showing" icon="eye-slash">
    **Possible causes:**
    - Analysis question is disabled
    - Created after calls took place
    - Transcription not enabled

    **Solutions:**
    - Verify question is Active (toggle on)
    - Analysis only runs on new calls after creation
    - Ensure transcription is enabled for agent
  </Accordion>
</AccordionGroup>
