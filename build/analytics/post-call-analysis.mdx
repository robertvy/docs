## Overview

Post-call analysis uses AI to extract structured information from conversation transcripts. Define what you want the AI to analyze (quality scores, sentiment, categories, extracted data) and get consistent, actionable insights from every call.

**Access:** Agent settings ‚Üí **Call Analysis**

<Warning>
**Analysis vs Goals:** Analysis extracts information and judges quality (e.g., "How satisfied was the customer?"). Goals track conversions (e.g., "Was appointment booked?"). Use both together for complete reporting.
</Warning>

## Analysis Language

Set the language for AI analysis evaluation:

**Access:** Agent settings ‚Üí **Analytics ‚Üí Language**

Select the language in which goals and analytics will be evaluated. This determines what language the AI uses when analyzing transcripts and answering analysis questions.

**Example:** If you set analysis language to Spanish, the AI will evaluate conversations in Spanish, even if your agent speaks English to customers.

<Tip>
This setting applies to both Call Analysis and Goals evaluation.
</Tip>

## Standard vs Custom Analysis

**Standard Analyses** - Pre-configured by itellicoAI, always available:
- Cannot be edited or archived
- Toggle on/off with Active switch
- Optimized for common use cases

**Custom Analyses** - Create your own:
- Full control over name, description, and type
- Can edit, archive, and restore
- Tailored to your specific business needs

<Info>
Standard analyses provide a starting point. Create custom analyses for your unique requirements.
</Info>

## Setup

<Steps>
  <Step title="Access Call Analysis">
    Agent settings ‚Üí **Call Analysis** ‚Üí **Add Analysis**
  </Step>

  <Step title="Configure question">
    **Name:** Internal identifier

    ```
    Example: "Customer Satisfaction Score"
    ```

    **Description:** The question AI will answer (be as specific as possible)

    ```
    Example: "On a scale of 1-5 stars, rate the customer's overall
    satisfaction based on their tone, language, and whether their
    issue was fully resolved during the call."
    ```

    <Tip>
    More specific descriptions = more accurate AI analysis
    </Tip>
  </Step>

  <Step title="Choose question type">
    **Yes/No** - Binary answer
    ```
    Example: "Was the customer's issue fully resolved?"
    ```

    **Open** - Text response
    ```
    Example: "What was the main topic discussed?"
    ```

    **Rating** - 1-5 stars
    ```
    Example: "How professional was the agent?"
    ```
  </Step>

  <Step title="Save">
    Click **Add Analysis** to save
  </Step>
</Steps>

## Question Types

<Tabs>
  <Tab title="Yes/No">
    **Binary answer questions** for clear metrics.

    **Best for:**
    - Compliance checks
    - Quality verification
    - Feature usage detection

    **Examples:**
    - ‚úÖ "Was payment information collected?"
    - ‚úÖ "Did the agent follow the script?"
    - ‚úÖ "Was the caller transferred?"
    - ‚úÖ "Did customer agree to the offer?"
  </Tab>

  <Tab title="Open">
    **Text response questions** for detailed insights.

    **Best for:**
    - Issue categorization
    - Information extraction
    - Root cause analysis
    - Conversation summaries

    **Examples:**
    - üìù "What was the primary topic discussed?"
    - üìù "What objections did the customer raise?"
    - üìù "Which product was discussed?"
    - üìù "Summarize the customer's technical issue"
  </Tab>

  <Tab title="Rating">
    **1-5 star scale** for qualitative assessment.

    **Standard scale:**
    - **5** = Exceptional performance
    - **4** = Above average performance
    - **3** = Average/satisfactory performance
    - **2** = Below average performance
    - **1** = Minimal/poor performance

    **Best for:**
    - Quality scoring
    - Sentiment analysis
    - Performance evaluation
    - Satisfaction measurement

    **Examples:**
    - ‚≠ê "Rate customer sentiment (1=very negative, 5=very positive)"
    - ‚≠ê "How well did agent handle objections?"
    - ‚≠ê "Rate call quality (1=poor, 5=excellent)"
    - ‚≠ê "Customer satisfaction score"

    <Tip>
    Always define what each rating level means in your description for more accurate analysis
    </Tip>
  </Tab>
</Tabs>

## Analysis vs Goals

Use both together for complete conversation tracking:

| Feature | Goals | Analysis |
|---------|-------|----------|
| **Purpose** | Track conversions | Extract insights & judge quality |
| **Example** | "Appointment booked" | "How satisfied was customer?" |
| **Answer Type** | Achieved/Not achieved | Yes/No, Text, or 1-5 rating |
| **Use For** | Success metrics | Quality scoring, categorization |
| **Best For** | Business outcomes | Detailed reporting |

**Example combination:**
- **Goal:** "Appointment booked" (Did we achieve the conversion?)
- **Analysis:** "Customer Satisfaction" (How well did we do it?)
- **Analysis:** "Primary Call Topic" (What did they want?)

## Best Practices

<AccordionGroup>
  <Accordion title="Write Specific Questions" icon="pen">
    The AI uses your description to answer. Be detailed.

    ‚ùå Vague: "Was the call good?"

    ‚úÖ Specific: "Did the agent provide accurate information, address all customer questions, and maintain a professional tone throughout the call?"
  </Accordion>

  <Accordion title="Define Rating Scales" icon="star">
    For rating questions, always define what each level means.

    ```
    Rate agent professionalism (1-5):
    5 = Exceptionally professional
    4 = Very professional
    3 = Professional
    2 = Somewhat unprofessional
    1 = Unprofessional
    ```
  </Accordion>

  <Accordion title="Start with Key Questions" icon="list">
    Don't create too many analyses at once. Start with 3-5 critical questions:

    1. Issue resolved (Yes/No)
    2. Customer satisfaction (Rating)
    3. Call topic (Open)
    4. Sentiment (Rating)

    Add more as needed.
  </Accordion>

  <Accordion title="Archive Old Analyses" icon="folder">
    Archive outdated analysis questions to keep them out of the way while preserving historical data.

    **To archive:** Click archive icon next to analysis

    **To restore:** Click "View Archived" ‚Üí Restore
  </Accordion>
</AccordionGroup>

## Tips

- **Combine with goals** - Use goals for conversions, analysis for quality insights
- **Test and refine** - Review AI answers on a few calls and adjust descriptions if needed
- **Be consistent** - Use same analysis questions across similar agents for comparable data
- **Enable/disable as needed** - Toggle analysis questions on/off with the Active switch

## Troubleshooting

<AccordionGroup>
  <Accordion title="Inaccurate answers" icon="triangle-exclamation">
    **Causes:**
    - Question description too vague
    - Multiple interpretations possible
    - Missing context in description

    **Solutions:**
    - Make description more specific and detailed
    - Provide examples in description
    - For ratings, define each level explicitly
    - Test with sample calls and refine
  </Accordion>

  <Accordion title="Inconsistent ratings" icon="arrows-left-right">
    **Causes:**
    - Rating criteria not clearly defined
    - Subjective interpretation by AI

    **Solutions:**
    - Define exact criteria for each rating level
    - Provide specific examples for each star rating
    - Use Yes/No questions for more objective answers
  </Accordion>

  <Accordion title="Analysis not showing" icon="eye-slash">
    **Causes:**
    - Analysis question disabled
    - Created after calls took place
    - Transcription not enabled

    **Solutions:**
    - Verify analysis is Active (toggle on)
    - Analysis only runs on new calls after creation
    - Ensure transcription is enabled for agent
  </Accordion>
</AccordionGroup>
