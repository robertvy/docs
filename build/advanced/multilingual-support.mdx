## Overview

Enable your AI agents to communicate with customers across the globe by configuring comprehensive multilingual capabilities. Expand your business reach, provide professional customer service in multiple languages, and deliver culturally appropriate responses with automatic language detection and region-specific customization.

Multilingual support transforms your agent from a single-language assistant into a global communication platform capable of understanding and responding in dozens of languages while maintaining natural conversation flow and cultural awareness.

<Note>
Language configuration is primarily done through Speech-to-Text settings in the Models tab. This guide covers multilingual strategy and configuration options.
</Note>

### What Multilingual Agents Can Do

**Real-time capabilities:**
- Automatically detect the customer's spoken language within the first few seconds
- Respond fluently in the detected language without manual switching
- Access knowledge bases and provide accurate information across languages
- Maintain consistent personality and brand voice regardless of language
- Switch languages mid-conversation if the customer changes language

**Use cases:**
- **Global customer support** - Serve international customers without language barriers
- **Multilingual sales** - Conduct sales conversations in prospect's native language
- **Regional expansion** - Launch in new markets without hiring multilingual staff
- **Tourist services** - Assist visitors in their preferred language
- **Healthcare access** - Provide medical information in patient's native language

## Language Architecture

Your agent's language configuration is set through the Speech-to-Text provider:

**Audio Processing Flow:**
```text wrap
Audio → Transcriber → LLM → Voice Synthesizer → Audio
       [Language]   [Any]   [Language]
```

**Configuration:**
- Language set on Speech-to-Text provider (Deepgram, AssemblyAI, etc.)
- Voice provider must match language
- Each component configured independently for optimal control

## Setting Up Languages

### Step 1: Choose Primary Language

The primary language serves as your agent's default:

<Steps>
  <Step title="Access Language Configuration">
    Navigate to Models → Transcriber (Speech-to-Text)
  </Step>
  <Step title="Select Primary Language">
    Choose the language you expect to use for the majority of conversations

    **Common primary languages:**
    - **en-US** - English (United States)
    - **en-GB** - English (United Kingdom)
    - **es-ES** - Spanish (Spain)
    - **es-MX** - Spanish (Mexico)
    - **fr-FR** - French (France)
    - **de-DE** - German (Germany)
    - **pt-BR** - Portuguese (Brazil)
    - **zh-CN** - Chinese (Simplified)
    - **ja-JP** - Japanese
    - **ko-KR** - Korean
  </Step>
  <Step title="Match Voice to Language">
    Navigate to Voice & Speech → Select Voice

    Choose a voice that speaks your primary language natively

    Filter voices by language to see compatible options
  </Step>
</Steps>

### Step 2: Add Secondary Languages

Enable additional languages your agent should support:

1. Open **Models → Transcriber** section
2. Select a transcriber that supports multilingual capabilities
3. Configure primary and secondary languages as supported by your chosen provider
4. Recommended: 3-5 languages maximum for optimal accuracy
5. Save configuration

**Recommended limits:**
- **2-3 languages:** Optimal accuracy and performance
- **4-5 languages:** Good accuracy with minor overhead
- **6+ languages:** Increased detection complexity, test thoroughly

<Warning>
Each additional language increases auto-detection processing time slightly. Only enable languages you expect to use regularly based on customer demographics.
</Warning>

### Step 3: Configure Voice Selection per Language

Ensure authentic pronunciation by selecting native voices:

**Best practices:**
1. Navigate to Voice & Speech → Select Voice
2. For each supported language, choose a native voice
3. Test voice quality in that language
4. Consider regional differences (Mexican vs. Spain Spanish)

**Voice matching strategy:**
```text wrap
Primary Language: en-US → Voice: "Sarah" (US English)
Secondary: es-ES → Voice: "Elena" (Spain Spanish)
Secondary: fr-FR → Voice: "Pierre" (French)
Secondary: de-DE → Voice: "Hans" (German)
```

Some voice providers offer multilingual voices that can speak multiple languages with appropriate accents - these work well for agents supporting many languages.

## Language Detection

### How Auto-Detection Works

**Detection process:**
1. Customer speaks first utterance
2. Agent analyzes audio for language markers (accent, vocabulary, phonetics)
3. Language detected within 500ms-2 seconds
4. Agent responds in detected language
5. Language persists for conversation duration

**Detection confidence levels:**
- **High confidence:** Agent proceeds immediately in detected language
- **Medium confidence:** Agent may confirm: "I can help you in Spanish, correct?"
- **Low confidence:** Falls back to primary language or asks for clarification

### Factors Affecting Detection Accuracy

<AccordionGroup>
  <Accordion title="Utterance Length" icon="message">
    **Impact:** Longer initial utterance = better detection

    **Good:** "Hello, I'm calling about my account and I need help with a billing issue" (14 words)

    **Poor:** "Hi" (1 word - insufficient data)

    **Solution:** Configure greeting to encourage longer responses:
    - "Please tell me how I can help you today"
    - "Describe the reason for your call"

    Avoid yes/no questions in greeting.
  </Accordion>

  <Accordion title="Language Similarity" icon="scale-balanced">
    **Easy to distinguish:**
    - English vs. Spanish (very different phonetics)
    - Japanese vs. French (distinct sound patterns)
    - German vs. Italian (different structures)

    **Harder to distinguish:**
    - Spanish (Spain) vs. Spanish (Mexico) - same language, regional differences
    - Norwegian vs. Swedish - very similar languages
    - Portuguese vs. Spanish - similar vocabulary and structure

    **Solution:** Limit to one variant per language family when possible, or use explicit language selection.
  </Accordion>

  <Accordion title="Accent Strength" icon="user-voice">
    **Impact:** Strong regional accents may confuse detection

    **Native speaker:** Usually detected correctly (99% accuracy)

    **Strong accent:** May be misidentified (90-95% accuracy)

    **Non-native speaker:** Lower accuracy (85-90%) - might detect accent language instead of spoken language

    **Solution:** Add clarification logic to instructions:
    ```
    If you're uncertain about language detection, politely ask:
    "Just to confirm, would you prefer to continue in English or Spanish?"
    ```
  </Accordion>

  <Accordion title="Number of Enabled Languages" icon="list">
    **Accuracy by language count:**
    - **2-3 languages:** 97-99% accuracy
    - **4-6 languages:** 93-97% accuracy
    - **7-10 languages:** 88-93% accuracy
    - **10+ languages (allow all):** 80-88% accuracy

    **Principle:** Fewer choices = higher confidence

    **Solution:** Only enable languages you actually serve. If you primarily serve English and Spanish customers, don't enable French, German, Italian "just in case."
  </Accordion>
</AccordionGroup>

### Improving Detection Accuracy

**Strategy 1: Use explicit language selection**
```text wrap
For phone agents (IVR):
"Press 1 for English, Press 2 for Spanish, Press 3 for French"

For web agents:
Provide language selector dropdown before starting call
```

**Strategy 2: Optimize greeting for longer responses**
```text wrap
Instructions example:
Start the conversation by saying: "Thank you for calling Acme Corp.
Please tell me in a few words how I can help you today."

This encourages 5-10 word responses vs. "Hi" which aids detection.
```

**Strategy 3: Add confirmation for ambiguous detection**
```text wrap
Instructions example:
If language detection confidence is below 90%, confirm with customer:

In English: "I can help you in English or Spanish. Which do you prefer?"
In Spanish: "Puedo ayudarle en español o inglés. ¿Cuál prefiere?"

Then switch to confirmed language.
```

## Language-Specific Configuration

### Instructions Per Language

Customize agent behavior based on detected language:

**Approach 1: Language-agnostic instructions (recommended for most cases)**
```text wrap
You are a helpful customer service agent for Acme Corp.

Your role:
- Answer customer questions professionally
- Resolve issues efficiently
- Provide accurate information
- Maintain a friendly, helpful tone

[Model automatically adapts tone and formality to detected language]
```

**Approach 2: Language-specific instructions (for cultural customization)**
```text wrap
You are a helpful customer service agent for Acme Corp.

{% if language == "es" or language == "es-ES" or language == "es-MX" %}
Cultural guidelines for Spanish:
- Use formal "usted" form unless customer initiates "tú"
- Take time for rapport building before addressing business
- Be warm and personable, not overly efficient
- Use appropriate regional expressions when detected

{% elif language == "de" or language == "de-DE" %}
Cultural guidelines for German:
- Be direct and efficient in responses
- Use formal addressing (Sie form)
- Focus on precision and facts
- Minimize small talk unless customer initiates

{% elif language == "ja" or language == "ja-JP" %}
Cultural guidelines for Japanese:
- Use appropriate honorifics and polite forms
- Be extremely respectful and formal
- Avoid direct contradiction or saying "no" bluntly
- Express empathy and understanding frequently

{% else %}
Default to English conversational style
{% endif %}

[Universal rules apply to all languages]
```

### Knowledge Bases for Multilingual Agents

**Option 1: Separate knowledge bases per language**
```text wrap
Structure:
- "Product FAQ (English)" collection
- "Product FAQ (Spanish)" collection
- "Product FAQ (French)" collection

Instructions:
{% if language == "es" %}
Use "Product FAQ (Spanish)" knowledge base for answers
{% elif language == "fr" %}
Use "Product FAQ (French)" knowledge base for answers
{% else %}
Use "Product FAQ (English)" knowledge base for answers
{% endif %}
```

**Pros:** Native content, culturally appropriate, no translation errors

**Cons:** Duplication, high maintenance overhead, version drift risk

**Option 2: English knowledge with AI translation (recommended)**
```text wrap
Structure:
- Single "Product FAQ" collection in English

Instructions:
Access the Product FAQ knowledge base and translate responses
into the customer's language naturally. Ensure technical terms
remain accurate and culturally adapt examples when needed.
```

**Pros:** Single source of truth, easy maintenance, works well with modern AI

**Cons:** May miss cultural nuances, technical translation requires validation

**Option 3: Multilingual knowledge base**
```text wrap
Structure:
- Single collection with documents tagged by language
- Each article has language metadata

Instructions:
Retrieve knowledge matching both the topic AND the customer's language.
If no match in customer's language, translate English version.
```

**Pros:** Flexible, supports partial translation, single collection

**Cons:** Complex metadata management, requires language-aware retrieval

<Tip>
For most use cases, Option 2 (English knowledge with AI translation) provides the best balance of maintenance efficiency and quality. Modern AI models excel at accurate cross-lingual knowledge retrieval and response generation.
</Tip>

## Regional and Cultural Customization

### Locale-Specific Settings

Beyond language, customize for regional expectations:

**Date and time formats:**
```text wrap
{% if language == "en-US" %}
Today's date: {{ current_date | date("MM/DD/YYYY") }}
Appointment time: {{ appointment_time | date("h:mm A") }}
{% elif language == "en-GB" or language == "de-DE" %}
Today's date: {{ current_date | date("DD/MM/YYYY") }}
Appointment time: {{ appointment_time | date("HH:mm") }}
{% else %}
Use ISO 8601 format: {{ current_date | date("YYYY-MM-DD") }}
{% endif %}
```

**Currency and number formatting:**
```text wrap
{% if language == "en-US" %}
Total: $1,234.56 (US Dollars)
{% elif language == "de-DE" %}
Total: 1.234,56 € (Euro)
{% elif language == "ja-JP" %}
Total: ¥123,456 (Japanese Yen)
{% endif %}
```

**Cultural context:**
```text wrap
{% if language == "ja-JP" %}
When addressing customers, always use respectful honorifics.
Be extremely apologetic for any inconvenience.
Avoid direct refusals - offer alternatives.

{% elif language == "de-DE" %}
Be direct and factual. German customers appreciate efficiency.
Provide detailed technical information when requested.
Formal titles are important (Herr, Frau).

{% elif language == "es-MX" %}
Build rapport before business. Mexican customers value personal connection.
Use warm, friendly language. Avoid being overly formal or cold.
{% endif %}
```

## Testing Multilingual Agents

### Comprehensive Test Plan

<Steps>
  <Step title="Single Language Testing">
    Test each supported language individually:

    1. Start conversation in target language
    2. Run through common scenarios (FAQs, issue resolution, booking)
    3. Verify response quality and accuracy
    4. Check that agent stays in chosen language throughout
    5. Test with native speaker if possible

    **Test script example (Spanish):**
    - "Hola, necesito ayuda con mi cuenta"
    - "¿Cuál es el horario de atención?"
    - "Quiero hacer una reserva para mañana"
  </Step>

  <Step title="Language Detection Testing">
    Test auto-detection without pre-selection:

    1. Start calls in different languages randomly
    2. Use varying utterance lengths (short, medium, long)
    3. Test with different accents (native, non-native, regional)
    4. Verify correct language detected within 2 seconds
    5. Check fallback behavior when detection uncertain

    **Detection test matrix:**
    | Language | Short ("Hi") | Medium (5 words) | Long (15 words) |
    |----------|-------------|------------------|-----------------|
    | English  | Test        | Test             | Test            |
    | Spanish  | Test        | Test             | Test            |
    | French   | Test        | Test             | Test            |
  </Step>

  <Step title="Knowledge Retrieval Testing">
    Verify knowledge works cross-linguistically:

    1. Ask same question in different languages
    2. Confirm accurate retrieval and response
    3. Check answer consistency across languages
    4. Verify culturally appropriate responses
    5. Test technical term translation accuracy

    **Example:**
    - English: "What are your business hours?"
    - Spanish: "¿Cuál es su horario de atención?"
    - French: "Quelles sont vos heures d'ouverture?"

    All should retrieve same information and provide equivalent answers.
  </Step>

  <Step title="Edge Case Testing">
    Test unusual scenarios:

    - **Very short utterances:** "Hi", "Hello", "Help"
    - **Code-switching:** Mixing languages in one sentence
    - **Strong regional accents:** Test with various native speakers
    - **Background noise:** Verify detection works in noisy environments
    - **Similar languages:** es-ES vs. es-MX, en-US vs. en-GB
    - **Mid-conversation switching:** Change language halfway through
  </Step>

  <Step title="Native Speaker Validation">
    Have native speakers evaluate each language:

    **Evaluation criteria:**
    - Naturalness and fluency (1-10 rating)
    - Pronunciation quality (voice selection)
    - Cultural appropriateness (tone, formality)
    - Accuracy of information
    - Overall customer experience

    **Minimum quality threshold:** 7/10 in all categories before launch
  </Step>
</Steps>

### Test Scenarios by Language

Create identical test scripts for each language:

**Customer support scenario:**
1. Customer greets agent
2. Customer explains problem
3. Agent asks clarifying questions
4. Agent provides solution
5. Customer asks follow-up question
6. Agent confirms resolution
7. Call ends

Run this exact flow in English, Spanish, French, German, etc. to ensure consistency.

## Best Practices

<AccordionGroup>
  <Accordion title="Start with Primary Language Only" icon="flag">
    **Before adding multilingual support:**
    1. Perfect agent in primary language first
    2. Optimize instructions, knowledge, and actions
    3. Achieve target quality metrics (>80% satisfaction)
    4. Train team on agent capabilities
    5. Only then add secondary languages

    **Why:** Building multilingual from day one spreads resources thin. Get one language excellent first, then expand.
  </Accordion>

  <Accordion title="Add Languages Based on Demand" icon="chart-line">
    **Don't enable all possible languages speculatively**

    **Data-driven approach:**
    1. Review customer language demographics
    2. Identify languages used by >5-10% of customers
    3. Prioritize by volume and business impact
    4. Add one language at a time
    5. Monitor performance before adding next

    **Example:**
    - 70% English speakers → Primary language
    - 20% Spanish speakers → Add as secondary
    - 5% French speakers → Add as third language
    - 3% German, 2% Other → Don't add yet (< 5% threshold)
  </Accordion>

  <Accordion title="Test with Native Speakers" icon="users">
    **Never deploy a language you haven't tested with a native speaker**

    **Why:** AI translation sounds fluent but may be:
    - Culturally inappropriate
    - Using wrong formality level
    - Missing regional context
    - Technically accurate but unnaturally phrased

    **Testing strategy:**
    - Hire freelance native speakers for testing (Upwork, Fiverr)
    - Recruit bilingual team members
    - Partner with language-specific customer service teams
    - Budget $50-200 per language for professional review
  </Accordion>

  <Accordion title="Keep Instructions Language-Agnostic When Possible" icon="globe">
    **AI models adapt automatically**

    **Recommended approach:**
    - Write clear, universal guidelines
    - Avoid culture-specific references unless necessary
    - Let model handle language-appropriate tone
    - Add language-specific rules only when testing reveals issues

    **Over-specification risks:**
    - Reduces model's natural language adaptation
    - Creates maintenance overhead
    - May introduce contradictions
    - Limits flexibility for new languages
  </Accordion>

  <Accordion title="Monitor Language-Specific Metrics" icon="chart-bar">
    **Track performance separately by language**

    **Key metrics per language:**
    - Customer satisfaction scores
    - Goal completion rates
    - Average call duration
    - Transfer rates to human agents
    - Common failure points
    - Detection accuracy

    **Action plan:**
    - If one language underperforms, investigate root cause
    - May need better voice selection
    - May need language-specific instructions
    - May need native knowledge content
    - Consider separate agent per language if gap is large
  </Accordion>

  <Accordion title="Provide Language Selection Option" icon="list-ol">
    **Even with auto-detection, offer explicit selection**

    **Phone agents (IVR):**
    ```
    "Thank you for calling Acme Corp.
    For English, press 1.
    Para español, presione 2.
    Pour le français, appuyez sur 3."
    ```

    **Web widget:**
    - Language dropdown before starting call
    - Flags for visual language recognition
    - Remember user's previous language choice

    **Benefits:**
    - Eliminates detection errors
    - Faster conversation start (no detection delay)
    - Customer feels in control
    - Works for all accents and utterance lengths
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Wrong Language Detected" icon="triangle-exclamation">
    **Symptoms:** Agent responds in incorrect language

    **Possible causes:**
    1. Too many enabled languages (reduced accuracy)
    2. Very short initial utterance (insufficient data)
    3. Strong accent confusing detector
    4. Similar languages hard to distinguish

    **Solutions:**
    - **Reduce language count:** Disable unused languages
    - **Encourage longer utterances:** Update greeting to ask open-ended question
    - **Add explicit selection:** Implement IVR language menu
    - **Add confirmation:** Include language clarification in instructions

    **Instruction fix:**
    ```
    If you detect the customer speaking Spanish but are not 100% certain,
    politely confirm in both languages:

    "I can help you in English or Spanish. Which do you prefer?"
    "Puedo ayudarle en inglés o español. ¿Cuál prefiere?"
    ```
  </Accordion>

  <Accordion title="Agent Switches Languages Unexpectedly" icon="shuffle">
    **Symptoms:** Agent starts in one language, switches to another mid-conversation

    **Possible causes:**
    1. Customer code-switched (mixed languages)
    2. Detection re-ran mid-conversation
    3. Instructions causing confusion

    **Solutions:**
    - **Lock language after detection:** Add instruction:
      ```
      Once you detect and respond in a language, continue in that
      language for the entire conversation. Do not switch languages
      unless the customer explicitly requests it.
      ```

    - **Handle code-switching:** Add instruction:
      ```
      If customer mixes languages (code-switching), respond in the
      primary language they used in their opening utterance.
      ```
  </Accordion>

  <Accordion title="Response Quality Poor in Secondary Language" icon="star-half">
    **Symptoms:** Responses sound unnatural, awkward, or incorrect in non-primary language

    **Possible causes:**
    1. Knowledge base only in primary language (translation issues)
    2. Instructions optimized for primary language only
    3. Voice doesn't match language (accent issues)
    4. Cultural context not adapted

    **Solutions:**
    - **Create native knowledge:** Build language-specific knowledge bases
    - **Adapt instructions:** Add language-specific cultural guidelines
    - **Change voice:** Select native voice for that language
    - **Test with natives:** Get feedback from native speakers
    - **Consider separate agents:** For critical languages, create dedicated agents
  </Accordion>

  <Accordion title="Detection Too Slow" icon="clock">
    **Symptoms:** Long pause before agent responds in detected language

    **Possible causes:**
    1. Too many enabled languages (processing overhead)
    2. Model-specific latency
    3. Network latency

    **Solutions:**
    - **Reduce languages:** Only enable most common 2-3 languages
    - **Use explicit selection:** IVR menu eliminates detection time
    - **Check model:** Some models detect faster than others
    - **Test network:** Ensure low-latency connection to AI provider
  </Accordion>
</AccordionGroup>
