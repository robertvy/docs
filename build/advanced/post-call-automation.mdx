## Overview

Post-call emails send automatically after conversations based on outcomes, goals, sentiment, or custom conditions you define.

**Access:** Agent editor → **Analytics → Post-Call Emails**

**Use cases:** Appointment confirmations, callback requests, complaint escalations, feedback collection, sales follow-ups, unresolved issue tracking

---

## Setup

<Steps>
  <Step title="Access Post-Call Emails">
    Agent editor → **Analytics → Post-Call Emails** → **Create Email**
  </Step>

  <Step title="Choose starting point (optional)">
    **Templates** provide pre-configured triggers and content for common use cases:

    - **Callback Requested** - Customer asks for follow-up
    - **Appointment Booked** - Meeting/consultation scheduled
    - **Customer Complaint** - Escalates dissatisfaction
    - **Unanswered Question** - Agent couldn't answer
    - **Technical Issue** - Routes bugs/errors to tech teams
    - **Billing Inquiry** - Payment/invoice questions
    - **Escalation Needed** - Requires management attention
    - **Order Placed** - Purchase confirmation
    - **Demo Request** - Product demonstration interest
    - **Positive Feedback** - Share compliments with team

    <Info>
    Templates are fully customizable starting points. You can also start from scratch.
    </Info>
  </Step>

  <Step title="Configure notification name">
    **Name:** Internal identifier (e.g., "Callback Request Alert")
  </Step>

  <Step title="Set trigger condition">
    Describe when to send (AI analyzes this against call transcript):

    ```
    Example: "Did the customer request a callback, ask to be
    contacted later, or want someone to follow up with them?"
    ```

    <Tip>
    Be specific. The more detailed your trigger description, the more accurate the AI matching.
    </Tip>

    **Or:** Toggle "Always send after every call" to bypass condition checking
  </Step>

  <Step title="Configure recipients">
    **Fixed Recipients:** Enter specific email addresses (supports multiple)

    ```
    Example: support@company.com, manager@company.com
    ```

    **Dynamic Recipients (AI-powered):** AI determines recipient based on call content

    Use for:
    - Extracting email addresses mentioned during the call
    - Smart routing based on issue type/severity

    ```
    Example 1 (Extract from call):
    "Extract the email address if the customer mentioned where to send
    confirmation. Only use emails explicitly stated during the call."

    Example 2 (Smart routing):
    "Route based on severity:
    - If legal threat → legal@company.com
    - If angry/canceling → retention@company.com
    - Otherwise → support@company.com"
    ```
  </Step>

  <Step title="Design email template">
    **Subject:** `Callback requested`

    **Body:** Use rich text editor with variables

    - System variables: `{{customer_name}}`, `{{agent_name}}`, `{{conversation_summary}}`
    - Dynamic variables: `{{dyn_appointment_date}}`, `{{dyn_issue_summary}}`
  </Step>

  <Step title="Save and enable">
    Save notification → Toggle **Enabled** switch
  </Step>
</Steps>

## Template Variables

### System Variables

Always available from call data:

```
{{customer_name}}, {{customer_email}}, {{customer_number}}
{{conversation_date}}, {{conversation_time}}, {{conversation_duration}}
{{conversation_summary}}, {{conversation_transcript}}, {{conversation_url}}
{{agent_name}}, {{agent_number}}, {{account_name}}
```

### Dynamic Variables

AI extracts from transcript using `dyn_` prefix:

```
{{dyn_appointment_date}} - AI finds mentioned date
{{dyn_issue_summary}} - AI extracts problem description
{{dyn_ticket_number}} - AI locates ticket reference
{{dyn_action_items}} - AI lists next steps discussed
```

<Warning>
Variable names must be descriptive. Use `{{dyn_shipping_address}}` not `{{dyn_address}}`.
</Warning>

## Example

**Appointment Confirmation**

**Trigger:**
```
Did the caller schedule, book, or confirm an appointment or meeting?
```

**Subject:**
```
Appointment confirmed - {{dyn_appointment_date}}
```

**Body:**
```
Hi {{customer_first_name}},

Your appointment is confirmed:

📅 {{dyn_appointment_date}} at {{dyn_appointment_time}}
📍 {{dyn_location}}
🔗 {{dyn_meeting_link}}

Need to reschedule? Call {{agent_number}}

Best,
{{agent_name}}
```

## Recipient Routing

**Fixed Recipients**
```
Send to: support@company.com, manager@company.com
```

**Dynamic Routing (AI-powered)**
```
Routing rules:
- If legal threat → legal@company.com
- If angry/canceling → retention@company.com
- If product issue → quality@company.com
- Otherwise → support@company.com
```

AI analyzes call and routes accordingly.

## Tips

<AccordionGroup>
  <Accordion title="Write Specific Triggers" icon="bullseye">
    ❌ Bad: "Customer unhappy"

    ✅ Good: "Did the customer express dissatisfaction, frustration, mention being unhappy, or threaten to cancel/leave negative review?"
  </Accordion>

  <Accordion title="Test with Real Calls" icon="vial">
    Enable the notification and test with real or test calls to verify triggering and content.
  </Accordion>

  <Accordion title="Use Descriptive Dynamic Variables" icon="tag">
    ❌ `{{dyn_info}}`, `{{dyn_data}}`, `{{dyn_thing}}`

    ✅ `{{dyn_shipping_address}}`, `{{dyn_next_appointment_date}}`, `{{dyn_issue_category}}`
  </Accordion>

  <Accordion title="Keep Emails Short" icon="align-left">
    - 3-5 paragraphs max
    - One clear purpose
    - Single call-to-action
    - Bullet points for multiple items
  </Accordion>
</AccordionGroup>

## Troubleshooting

**Email not sending?**
- Verify notification is enabled (toggle On)
- Check trigger condition matches call content
- Ensure customer has valid email

**Variables not populating?**
- Use exact variable names (check variable picker)
- Make dynamic variables more specific
- Test with real calls to verify extraction

**Wrong routing?**
- Review routing rules description
- Make criteria more explicit
- Test with sample calls
